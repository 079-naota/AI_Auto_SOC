## 1.サーバーの初期設定
新しく導入した３台のさくらVPSの初期設定を行った。

手順を以下に示す。
- 1.事前に渡されたクラウドサーバーのサイトにログインし、サーバーの情報（IPv4)を取得した。
- 2.手元のLaptopからサーバーへ公開鍵をコピーし、正しくコピーされたかを確認した。
- 3.サーバーにログイン後、tailscaleをインストール・実行した。
- 4.ログイン方法を制限するために、設定ファイルを書き換えた。具体的には、公開鍵でのログインを有効にし、パスワードでのログインを無効化した。また、tailscaleのために5022ポートを開けておく。
- 5.sshサービスを再起動し、Laptopから公開鍵でログインできるかを確認した。

## 2.各サーバーのハニーポットの構築
新しいサーバーにおいてDockerを使い、Web、DBそしてIoTのハニーポットを構築した。


まず、サーバーにログイン後Dockerをインストールした。
その後、各ハニーポットようにymlファイルを作成した。
ymlファイルは以下のようなフォーマットで作成した。

``` docker-compose.yml
version: '3.8' # Docker Composeのファイルの書き方のバージョンを指定（書き方が古いが互換性あり）

services: # ここから下に、起動したいコンテナ（サービス）の定義を書く
  dionaea: # サービス名
    image: dinotools/dionaea:latest # Docker Hubから「Dionaeaの最新版公式イメージ」をダウンロードして使う
    container_name: dionaea-web # 起動したコンテナに付ける名前（今回はWebおとり用）
    restart: always # 万が一VPSが再起動したり、プログラムがクラッシュしたりしても、自動でコンテナを再起動する設定

    ports: # 公開する罠の入り口の指定
      - "80:80"   # 左側が「本物のグローバルポート」、右側が「コンテナ内のDionaeaのポート」
      - "443:443" # 外部からのWebアクセス（80/443）を、コンテナ内のDionaeaに直結させる

    volumes: # データを手元のサーバーに残す設定（マウント）
      # 左側が「VPS（ホスト）側のパス」、右側が「コンテナ内のパス」
      - ./dionaea/logs:/opt/dionaea/var/log
      # ↑ これにより、コンテナ内でDionaeaが記録した「攻撃ログ」が、VPSの ~/honeypot/dionaea/logs にリアルタイムで保存される
      
      - ./dionaea/binaries:/opt/dionaea/var/dionaea/binaries
      # ↑ これにより、Dionaeaが自動捕獲した「本物のマルウェア（検体）」が、VPSの ~/honeypot/dionaea/binaries に安全に隔離・保存される
```
