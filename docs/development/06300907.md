# WazuhManager（VPS1）とAgent（VPS2)のログ転送トラブル対応
VPS２のDionaeaハニーポットログをVPS１のWazuhManagerへ転送するための確認と修正を行った。

## 目的
VPS2上で動作しているWazuh Agentから、VPS1上のWazuh Managerへログを送信し、Manager側の archives.json にDionaeaログが反映される状態を目指した。


構成を以下に示す。
- VPS1: Wazuh Manager
- VPS2: Dionaea Honeypot+WazuhAgent
- VPN: Tailscale
- Manager IP: 100.xx.xx.xx
- Wazuh Agent通信ポート: xxxx/tcp

## 発生した問題
最初に、VPS1側で agent_control -l を確認したところ、Agentは登録されていたが Never connected の状態だった。　

```
ID: 004, Name: wazuh_web, IP: any, Never connected 
ID: 006, Name: server-707001-012, IP: any, Never connected
```

これは、VPS2のAgentがManagerへ新規登録を試みたが、Manager側に同じAgent名が既に存在していたため拒否されていたことを示している。
さらに、その後キー取得には成功したものの、次のエラーが発生した。

```ossec.log
CRITICAL: Could not open file 'queue/rids/008' due to [(13)-(Permission denied)].
```

この原因は、VPS2側の /var/ossec/queue/rids 周辺の所有者・権限が、現在のWazuh Agent実行ユーザーと合っていなかったためである。

確認すると、以下のように所有者が混在していた。
```
/var/ossec/queue root:wazuh 
/var/ossec/queue/rids root:ossec 
/var/ossec/etc/client.keys wazuh:wazuh
```

この環境ではWazuh Agentが wazuh 系のユーザー・グループで動作するため、queue/rids が root:ossec になっているとAgentが必要なファイルへアクセスできず、Permission deniedになっていた。


## 解決策
まず、VPS1側でWazuh ManagerがAgent通信ポートを待ち受けているか確認した。

```Bash
sudo ss -lntp | grep##-E '22|1514'
```

結果として、wazuh-remoted が 0.0.0.0:1514 で待ち受けていることを確認できたため、Manager側の受信ポートは正常だった。
次に、VPS2側のAgent認証情報と権限を確認した。

```
sudo awk '{print NR, $1, $2, $3}' /var/ossec/etc/client.keys
```

その結果、VPS2は最終的に ID: 009 / Name: server-707001-012 としてManagerに登録された。

その後、queue/rids の所有者と権限をWazuh Agentが読み書きできる形に修正した。

```Bash
sudo systemctl stop wazuh-agent 

sudo chown root:wazuh /var/ossec/queue 
sudo chmod 750 /var/ossec/queue 

sudo chown -R wazuh:wazuh /var/ossec/queue/rids 
sudo chmod 750 /var/ossec/queue/rids 
sudo chmod 660 /var/ossec/queue/rids/008 
sudo chown wazuh:wazuh /var/ossec/etc/client.keys 

sudo chmod 640 /var/ossec/etc/client.keys 
sudo systemctl restart wazuh-agent
```

この修正により、queue/rids のPermission deniedが解消され、Manager側でAgentがActiveになった。

```
ID: 009, Name: server-707001-012, IP: any, Active
```

これにより、VPS2 AgentとVPS1 Manager間の接続は成功した。

## archives.jsonの確認
その後、VPS1側で受信ログを確認するために archives.json を確認した。

最初は以下のエラーが出た。

```
tail: cannot open '/var/ossec/logs/archives/archives.json' for reading: No such file or directory
```

これは、Manager側でarchives出力がまだ生成されていなかった、または logall_json の設定が反映されていなかったためと考えられる。
その後、archives ディレクトリ内に archives.json が作成されていることを確認した。
最終的にVPS2側でDionaeaログへテスト文字列を追記し、VPS1側の archives.json に反映されることを確認できた。

```
echo "DIONAEA_TEST from VPS2 agent009 to Wazuh Manager" | sudo tee -a /home/ubuntu/honeypot/dionaea/logs/dionaea/dionaea.log
```

これにより、以下のログ転送経路が正常に動作していることを確認できた。

## 原因まとめ
今回ログ転送ができなかった主な理由は3つある。

1つ目は、VPS2のAgent登録情報がManager側と整理されておらず、同じAgent名で重複登録しようとしていたことである。そのため、Manager側で Duplicate agent name が発生していた。

2つ目は、VPS2側の queue/rids の所有者・権限が不適切だったことである。root:ossec と wazuh:wazuh が混在しており、Agentが queue/rids/008 を開けずに停止していた。

3つ目は、Manager側の archives.json が最初は存在しておらず、ログ受信確認の段階で確認対象ファイルがまだ生成されていなかったことである。
