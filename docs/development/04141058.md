# 4月14日11時までの報告

## 成果

### 1.Wazuhの有効化
WazuhManagerをインストールど追加でwazuh-agentをインストール・起動したところ、`wazuh-agent.service does not exist`エラーが発生した。

このエラーの原因はWazuhManagerインストール時点でOSを監視するAgent(ID:000)が自動的に組み込まれていたので、競合によるものだった。

`sudo /var/ossec/bin/agent_control -l`を実行し、ID:000がActiveか確かめた。

Wazuhの有効なのかを確かめるために、VPSのパブリックIPに対して、存在しないユーザーでSSH接続を試し、Wazuhへのログ反映をテストした。

テスト後WazuhのDashboardを確認したところアラートは表示されなかった。

1つ目の原因はSSHの接続をtailscale0のみを許可していたのでパブリックIPからのログインに対するポート22が閉じられていたからである。


これには、パスワード認証を無効化した後でポート22を開けることで解決した。

2つ目の原因は、Ubuntu24.04LTS以降では、標準で`rsyslog`がインストールされてないのでWazuhが監視している`/var/log/auth.logファイルが存在しないことである。


これには、rsyslogをインストール・起動した。この際、Wazuhはファイルの改ざんを検知した。このことから、Wazuhが正常に動いていることも確認できた。その後、もう一度パブリックIPへのSSH認証を試した。WazuhのDashboardを確認したところ正常に検知されていた。また、`cat /var/log/auth.log | hacker`を実行しログファイルも確認したところ、攻撃を発見できた。

### 2.攻撃検知の成果
SSHの認証テストで検知したものは以下の通りであった。
- Rule5710 (sshd:Attempt to login using a non-existent user)
- Rule5503 (PAM:User login failed)
