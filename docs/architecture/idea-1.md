# 卒研案１

## LLMによる高対話型ハニーポットからのIoC自律抽出と、セカンドペイロードの自動捕獲システムの構築

### 1.目的
高度化するサイバー攻撃に対し、シグネチャで検知するには困難だった「難読化攻撃の意図の読解」と「IoCの抽出」をLLMを用いて自動化する。

これにより、SOCアナリストのアラート疲労の解消、ローカルLLMへの移行によって機密性確保と運用コストゼロの両立するSOCアーキテクチャを目指す。

### 2.手順
#### 2.1 高対話型ハニーポットの構築とプロンプト・エンジニアリング(4~6月)
攻撃者を誘い込み、AIに正しく解読させるための目と脳をシステムに実装する。
- 環境構築：VPSのSSHポートを分離し、port22にCowrieを構築。Wazuhと連携し、侵入後の難読化コマンドを網羅的に収集する体制を作る。

- 収集間隔の最適化：攻撃者の「一連のセッション（キルチェーン）」をAIが文脈として理解しやすいよう、Wazuhからログを抽出する時間間隔（例：10分ごと、セッション切断時など）を実験し、最適なチャンクサイズを決定する。

- プロンプトの工夫：GeminiAPIに対し、「単なるログ要約」ではなく「難読化の解除、IoCの抽出、攻撃意図の解説」を強制する高度なプロンプトを設計・テストする。

#### 2.2　LLMの評価とデータセットの構造化（7月~9月）
　LLMの出力するレポートを採点し、ローカルLLMの教師データを抽出、蓄積する。
- 精度評価とチューニング：Wazuhの生ログを自身で解析した結果とLLMが出力したレポートを比較する。過検知やハルシネーションの発生率を数値化し、プロンプトをより洗練させる。

- 高品質なデータセットの抽出：評価したログとレポートのペアのみを抽出して、ローカルLLMの学習に直接投入できるような形式に変換して保存する流れを完成させる。

#### 2.3  ローカルLLMの構築と追加学習（10月~12月）
- ローカル環境の構築：Llama3などのオープンソースLLMを導入する。VRAMに収まるように軽量化し、実用的な推論速度が出る基盤を整える。

- QLoRAによる追加学習：2.1,2.2で収集・選別した高品質のデータセットを用いて追加学習を実行し、一般的なLLMをセキュリティ特化のものにカスタマイズする。

#### 2.4　LLMとローカルLLMの比較評価(1月~3月）
LLMとローカルLLMを比較し、１年間の成果をまとめる。

- 比較評価：新しい未知の攻撃ログを用意し、LLMとローカルLLMの両方に入力する。IoCの抽出精度、難読化の解析能力、そして推論にかかる処理速度を定量的に比較する。

- システムの有用性実証と総括：ローカルLLMへの移行により、API利用のコストと情報漏洩リスクをゼロにしつつ、実用レベルの自動トリアージが可能になったかを実証し、まとめる。






### 語句
- IoC:Indicator of Compromiseの略である。意味は、サイバー攻撃やシステム侵害がすでに発生した、あるいは進行中であることを示す痕跡や証拠のデータ。 
具体的にいうと、IPアドレス、不正ファイルハッシュまた、異常なログイン記録などが該当する。
