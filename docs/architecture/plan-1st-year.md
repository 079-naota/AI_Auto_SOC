# 高専５年次の卒業研究スケジュール

## 分野別のハニーポットの監視とプロンプト設計(4-6月)
SSH、Web、DB、そしてIoTを狙う攻撃を網羅的に収集する統合インフラを構築し、LLMに分析を依頼する流れを自動化するシステムの構築をする。


サーバーは4つ用意し各監視対象は以下のような構成を考える。

SSHの監視にについては、VPSのポートを分離し、port22にCowrieを構築する。Wazuhと連携し進入後の難読化コマンドとセカンドペイロードを収集する。
Webの監視については、Snare＆Tannerを導入(potr 80/443)し、悪意のあるリクエストをWazuhへ送信する。
DBの監視については、RedisHoneypotを導入（port 6379)し、データ破壊や脅迫文をWazuhへ送信する。
IoTの監視については、Dionaeaを導入(port 445/21)し、ボットネット活動をWazuhへ送信数する。

そして、4つから集まる複数のログに対し、「難読化の解除、IoCの抽出、攻撃意図の解説」を的確に行うようLLMのプロンプトを設計し、テストを行う。


## LLMの評価と統合データセットの構造化(7-9月)
まず、Wazuhに集約された各種の生ログを自身で解析し、その結果とLLMの出力するレポートを比較する。
そこで、過検知やLLM特有の嘘の発生率を数値化し、プロンプトに工夫を加えていく。

次に、評価したログとレポートのペアのみを選び、ローカルLLMの学習用に形式を整え保存する流れを自動化する。

## ローカルLLMの構築と追加学習(10-12月)
オープンソースLLMを導入し、VRAMに収まるように軽量化し実用的に使えるレベルの推論速度を確保する。
そして、収集した高品質なデータセットを用いて追加学習を実行し、セキュリティに特化したLLMへカスタマイズする。

## システムの比較(1-3月)
API型のLLMと追加学習したローカルLLMに同じ攻撃を仕掛けIoCの抽出精度、難読化の解析力、処理速度を定量に的に比較する。
