## SOC分析レポート

**日付**: 2026-05-01 (ログ最終更新時刻に基づく)
**攻撃元IP**: 124.18.182.99

### 1. 攻撃の概要と目的

攻撃元IP `124.18.182.99`から、SSHサービスに対して大規模な総当たり攻撃（ブルートフォースアタック）が複数回試行されました。攻撃者は一般的なシステムアカウント（例: `ubuntu`, `admin`, `postgres`, `mysql`）やアプリケーションアカウントと思われるユーザー名を、多数のパスワードと組み合わせて自動的に試行しています。

この攻撃は以下の段階で進行しました。

1.  **初期偵察・認証情報窃取**: 最初に広範囲なユーザー名とパスワードの組み合わせを試行し、正規の認証情報を特定しようとしました。このフェーズでは、`test`, `ubuntu`, `dockeruser`, `admin`, `postgres`, `mysql`などの多様なユーザー名が試行されています。
2.  **認証突破**: 攻撃者は複数の異なるパスワードで`root`ユーザーとしてログインに成功しています。
    *   `root` / `admin12#` (11:25:04Z)
    *   `root` / `3245gs5662d34` (11:25:05Z)
    *   `root` / `server@2021` (11:26:00Z)
    *   `root` / `3245gs5662d34` (11:26:01Z)
    これらの成功したログインはほぼ同時期に発生しており、自動化された攻撃スクリプトが多様なパスワードを試行し続けていたことを示唆しています。
3.  **永続化メカニズムの確立**: ログイン成功後、攻撃者はシステムへの継続的なアクセスを確保するため、自身のSSH公開鍵をターゲットの`authorized_keys`ファイルに追加しようと試みています。具体的には、既存の`.ssh`ディレクトリを削除し、新しいディレクトリを作成した上で、攻撃者の公開鍵（コメント: `mdrfckr`）を`~/.ssh/authorized_keys`に書き込み、適切なパーミッションを設定しています。これにより、以降パスワードなしでSSHアクセスが可能になります。
4.  **不審なコマンドの試行**: ログイン後、`chattr -ia .ssh`と`lockr -ia .ssh`というコマンドも試行されています。`chattr`はファイル属性を変更するコマンドであり、`.ssh`ディレクトリの改変を容易にする意図があったと推測されます。`lockr`は一般的なLinuxコマンドではなく、ログでは失敗しています。

攻撃の主な目的は、SSHサービスを介してシステムへの不正アクセスを確立し、`root`権限を奪取した上で、永続的なバックドアを設置することにあると推測されます。

### 2. 推測される手法・使用ツール

*   **攻撃手法**:
    *   **ブルートフォースアタック / パスワードスプレー攻撃**: 短い時間間隔で多数のユーザー名とパスワードの組み合わせを試行していることから、自動化された総当たり攻撃またはパスワードスプレー攻撃が実行されたと考えられます。
    *   **永続化**: SSHの`authorized_keys`メカニズムを悪用し、攻撃者の公開鍵を追加することで、システムへのバックドアを確立しようと試みています。
*   **使用ツール**:
    *   **Cowrieハニーポット**: このログはCowrieハニーポットによって記録されたものです。CowrieはSSHセッションを模倣し、攻撃者の行動を記録・分析するために使用されます。
    *   **自動化された攻撃スクリプト/ツール**: 短期間での多数のログイン試行と、ログイン成功後に一連のコマンドを自動的に実行していることから、Hydra、Medusa、Nmapのスクリプト、またはカスタムのボットネットスクリプトのような自動化されたブルートフォースツールが使用された可能性が高いです。
    *   **SSHクライアント**: 実際のSSH接続には標準的なSSHクライアントが使用されています。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

*   **root権限でのログイン成功**: 複数の異なる認証情報で`root`ユーザーとしてログインに成功している点は、最も深刻な脅威です。`root`権限はシステムに対する完全な制御を意味し、攻撃者は任意のコマンド実行、機密データの窃取、システム設定の改ざん、サービス停止、さらには他のシステムへの横展開などを自由に行うことが可能になります。
*   **永続化の試み**: 攻撃者が`authorized_keys`に自身の公開鍵を追加しようと試みたことは、長期的なアクセスを確立し、将来的にいつでもシステムに侵入できる状態を築こうとしている明確な意図を示しています。これは、ハニーポットでなければ、システムが完全に攻撃者の支配下に入る危険性がありました。
*   **自動化された攻撃**: 短期間に多数のログイン試行と一連のコマンド実行を行っていることから、この攻撃が特定のターゲットを狙ったものではなく、インターネット上に公開されたSSHサービスを無差別にスキャンして攻撃している可能性が高いです。これにより、今後も同様の攻撃が継続されるリスクがあります。
*   **攻撃者の悪意**: 公開鍵のコメントに`mdrfckr`という不適切な文字列が含まれていることは、攻撃者の悪意を明確に示唆しています。

このログはハニーポットのものであるため、実際のシステム侵害には至っていませんが、もし実稼働環境のサーバーに対して同様の攻撃が発生した場合、システムが完全に侵害され、重大な機密情報漏洩、サービス停止、他のシステムへのマルウェア拡散などに繋がる可能性が極めて高い、非常に危険な状況でした。

### 4. 推奨アクション

1.  **攻撃元IPのブロック**:
    *   即座にファイアウォール、IPS/IDS、またはセキュリティグループで攻撃元IP `124.18.182.99`からのSSH（TCP/22）アクセスを恒久的にブロックします。
2.  **既存システムの認証強化**:
    *   SSHサービスを公開している全てのサーバーにおいて、**パスワード認証を無効化し、鍵認証のみに移行する**ことを最優先で検討・実施します。
    *   `root`ユーザーのSSH直接ログインを禁止し、一般ユーザーでログイン後、`sudo`を利用した権限昇格を必須とします。
    *   SSHサービスのデフォルトポート（22番）を変更し、一般的なポートスキャンからの検出リスクを低減します（この対策は単独ではなく、他のセキュリティ対策と組み合わせるべきです）。
    *   強固なパスワードポリシーを適用し、ブルートフォース攻撃で推測されにくい複雑なパスワード（長さ、文字種混合）の使用を義務付け、定期的な変更を促します。
    *   多要素認証 (MFA) をSSHログインに導入することを検討し、認証強度を大幅に向上させます。
3.  **侵入検知・防止システムの強化**:
    *   SSHログイン試行失敗が一定回数以上続いた場合、自動的にIPアドレスをブロックする仕組み（例: Fail2ban）を導入・設定強化します。
    *   SIEM (Security Information and Event Management) システムを導入または活用し、SSHログインログをリアルタイムで監視し、異常なログイン試行やログイン後の不審なコマンド実行を検出するルールを強化します。
    *   今回のログで成功したパスワード（`admin12#`, `3245gs5662d34`, `server@2021`など）が、実稼働環境のシステムで現在使用されていないか緊急で確認し、もし使用されている場合は即座にパスワードを変更します。
4.  **脆弱性管理とパッチ適用**:
    *   SSHサーバーソフトウェアを含む、公開しているすべてのサービスの脆弱性スキャンを定期的に実施し、常に最新のセキュリティパッチを適用します。
5.  **ハニーポットの活用と脅威インテリジェンス**:
    *   今回のCowrieハニーポットのログは貴重な脅威インテリジェンスを提供しています。今後も継続的に監視し、新たな攻撃手法、使用される認証情報、および攻撃者の行動パターンを収集・分析することで、防御策の改善に役立てます。収集された脅威インテリジェンスは、IDS/IPSのシグネチャ更新やSIEMの検知ルール強化に活用します。
