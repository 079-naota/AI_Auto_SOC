SOC分析レポート

**1. 攻撃の概要と目的**

攻撃元IPアドレス `131.161.219.137` から、ターゲットシステムへのSSHサービスに対するブルートフォース攻撃が複数回成功し、`root`ユーザーの認証情報が奪取されました。攻撃者は、ログイン成功後に一貫したパターンで以下の行動を実行しました。

1.  **永続的なアクセス経路の確立**: `~/.ssh/authorized_keys`ファイルに自身のSSH公開鍵を設置し、パスワードなしでの再ログインが可能なバックドアを作成しました。
2.  **マルウェアペイロードの展開**: 設置したバックドアを通じて、2種類の異なるSHA256ハッシュ値を持つペイロードのダウンロードを試みました。これはシステムの不正利用を目的としていると推測されます。
3.  **システム情報の偵察**: CPU情報、OSバージョン、ディスク使用量、稼働プロセスなどの詳細なシステム情報を収集しました。
4.  **アカウント情報の改変**: `root`ユーザーのパスワードを`TDXIh8sgnqCM`に変更しようと試み、他の管理者や競合する攻撃者のアクセスを妨害しようとしました。
5.  **痕跡消去および競合対策**: `/tmp`ディレクトリ内の特定のスクリプトファイルや関連プロセスを削除し、`/etc/hosts.deny`を空にすることで、自身の活動の隠蔽や他のマルウェアの排除を試みました。

これらの行動は、ターゲットシステムをボットネットの一部として利用したり、クリプトマイニングを行うなど、システムの資源を不正に悪用することを主な目的としていると判断されます。

**2. 推測される手法・使用ツール**

攻撃は、MITRE ATT&CKフレームワークの複数のタクティクスとテクニックに該当します。

*   **Initial Access (初期アクセス)**
    *   **T1133 External Remote Services**: SSHサービスを標的とした攻撃。
    *   **T1078 Valid Accounts**: ブルートフォース攻撃により`root`アカウントの認証情報を取得。
*   **Credential Access (認証情報へのアクセス)**
    *   **T1110.003 Brute Force: Password Spraying / T1110.004 Brute Force: Credential Stuffing**: 複数の一般的なパスワードを試行して`root`アカウントのログインに成功。
*   **Persistence (永続化)**
    *   **T1098.004 Account Manipulation: SSH Authorized Keys**: `~/.ssh/authorized_keys`に攻撃者のSSH公開鍵を追加し、バックドアを設置。
    *   **T1098 Account Manipulation**: `root`ユーザーのパスワードを変更しようと試みた。
*   **Defense Evasion (防御回避)**
    *   **T1070.004 Indicator Removal: File Deletion**: `/tmp`内のスクリプトファイルを削除し、関連プロセスを終了させた。
    *   **T1562.001 Disable or Modify Tools**: `chattr -ia .ssh`で`.ssh`ディレクトリの保護属性を無効化しようとした。`/etc/hosts.deny`を空にすることで、自身のアクセスを許可しようとした。
*   **Discovery (探索)**
    *   **T1082 System Information Discovery**: `uname -a`, `lscpu`, `df -h`などでシステム情報を収集。
    *   **T1083 File and Directory Discovery**: `cat /proc/cpuinfo`などでCPU情報を収集。
*   **Execution (実行)**
    *   **T1059.004 Command and Scripting Interpreter: Unix Shell**: 各種シェルコマンド（`rm`, `mkdir`, `echo`, `chmod`, `chattr`, `chpasswd`, `pkill`, `cat`, `grep`, `wc`, `top`, `uname`, `lscpu`, `df`, `awk`）を実行。
*   **Impact (影響)**
    *   **T1496 Resource Hijacking**: CPU情報収集コマンドの頻繁な実行やペイロードの性質から、クリプトマイニングマルウェアによるリソースハイジャックを目的としている可能性が高い。

**3. 脅威レベルとその理由**

*   **脅威レベル**: **高 (High)**

*   **理由**:
    *   **root権限の完全奪取**: システムに対する最高の権限である`root`アカウントへのログインが複数回成功しており、システムのあらゆる操作が攻撃者の管理下に置かれる状態です。
    *   **永続的なアクセス経路の確立**: SSH公開鍵を利用したバックドアの設置により、攻撃者はパスワードが変更されたとしても、いつでもシステムに再侵入できる状態を確立しています。
    *   **複数ペイロードの展開**: 異なるSHA256ハッシュ値を持つ2種類のペイロードがダウンロードされており、システムの悪用（例：クリプトマイニング、ボットネットへの組み込み、さらなるマルウェアの展開）を目的としている可能性が非常に高いです。
    *   **システム構成の改変**: `root`パスワードの変更を試み、`/etc/hosts.deny`の改変を行うことで、他のユーザーやセキュリティツールからのアクセスを妨害し、自身の支配を強固にしようとしています。
    *   **広範な偵察**: システムリソースやOS情報の詳細な収集は、攻撃者がシステムの能力を評価し、その後の攻撃を最適化しようとしていることを明確に示唆しています。

これらの複合的な行動は、機密情報の窃取、システムの停止、サービス妨害、組織の他のリソースへの横展開、および経済的利益のためのリソース悪用（クリプトマイニング）など、ビジネスに深刻な影響を及ぼす可能性があります。

**5. 推奨アクション**

**a. 緊急対応 (Immediate Actions)**

1.  **攻撃元IPのブロック**: 攻撃元IPアドレス `131.161.219.137` からのすべての通信をネットワーク境界ファイアウォール（またはセキュリティグループ、ACLなど）で直ちにブロックしてください。
2.  **システムのネットワーク隔離**: 影響を受けた可能性のあるシステムを、組織のネットワークから直ちに隔離してください。これにより、これ以上のマルウェア拡散やデータ流出を防ぎます。
3.  **パスワードの緊急変更**: `root`アカウントおよびログに記載されたパスワードが使用されている可能性のある全てのシステムアカウント（特にSSHで利用されるもの）のパスワードを、直ちに複雑で推測困難なものへ変更してください。また、これらのパスワードが他のシステムで使い回されていないかを確認し、該当する場合は速やかに変更してください。
4.  **不正なSSH公開鍵の削除**: 検出された不正なSSH公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`)を、システム上の全てのユーザーの`~/.ssh/authorized_keys`ファイルからだけでなく、他の可能性のある場所からも削除してください。
5.  **マルウェアのスキャンと削除**: 抽出されたハッシュ値 (`a8460f446be540410004b1a8db4083773fa46f7fe76fa84219c93daa1669f8f2`, `01ba4719c80b6fe911b091a7c05124b64eeece964e09c058ef8f9805daca546b`) を用いて、システム全体をマルウェアスキャンし、検出されたファイルを安全に隔離または削除してください。特に、これらのファイルが自動実行されるよう設定されていないか確認してください。
6.  **フォレンジック調査の開始**: 攻撃の全容解明のため、システムの完全なディスクイメージやメモリダンプを取得し、詳細なフォレンジック調査を実施してください。これにより、攻撃者の活動履歴、導入されたマルウェアの機能、影響範囲、その他の永続化メカニズムなどを特定します。
7.  **ログの保全**: 現在のログを含む全ての関連ログ（SSHログ、システムログなど）を保全し、改ざんや削除を防いでください。

**b. 予防策 (Preventative Measures)**

1.  **SSHセキュリティの強化**:
    *   パスワード認証を無効にし、鍵認証のみを強制してください。
    *   強力なパスフレーズ付きSSH鍵を使用し、鍵の管理を徹底してください。
    *   `root`ユーザーのSSH直接ログインを禁止し、非特権ユーザーでのログイン後、`sudo`を利用する運用に変更してください。
    *   Fail2banやSSHGuardなどのブルートフォース対策ツールを導入し、不正なログイン試行を自動的に検知・ブロックするよう設定してください。
    *   SSHサービスで多要素認証 (MFA) の導入を検討してください。
2.  **強固なパスワードポリシーの適用**: 全てのシステムアカウントに対し、強力で複雑なパスワードポリシー（文字種、長さ、有効期限など）を適用し、定期的な変更を義務付けてください。
3.  **システム監視の強化**: ログ監視システム（SIEMなど）を導入し、不審なログイン試行、コマンド実行、ファイル変更、プロセス起動、ネットワーク通信などをリアルタイムで検知・アラートする体制を構築してください。
4.  **セキュリティパッチの迅速な適用**: OSおよび全てのアプリケーションに対し、最新のセキュリティパッチを迅速に適用し、既知の脆弱性を悪用されるリスクを低減してください。
5.  **EDR/IDS/IPSの導入**: エンドポイントでの挙動監視 (EDR) や、ネットワーク侵入検知・防御システム (IDS/IPS) を導入し、高度な脅威に対する検知・対応能力を強化してください。
6.  **定期的なセキュリティ監査**: 定期的にシステムの脆弱性スキャン、侵入テスト、設定レビューを実施し、セキュリティ体制の弱点を特定し改善してください。

**セカンドペイロードの意図推測**

ログから、攻撃者がログイン成功後に毎回、ほぼ同じタイミングでハッシュ値 `a8460f446be540410004b1a8db4083773fa46f7fe76fa84219c93daa1669f8f2` のファイルをダウンロードしていることが確認されます。その後、システム偵察コマンド (`cat /proc/cpuinfo`でCPUコア数をカウント、`top`でプロセス監視、`df -h`でディスク使用量確認など) が実行され、さらに異なるハッシュ値 `01ba4719c80b6fe911b091a7c05124b64eeece964e09c058ef8f9805daca546b` のファイルがダウンロードされています。

これらの行動と、特にCPU情報を複数回にわたって詳細に収集している点から、ダウンロードされたペイロードは**クリプトマイニングマルウェア**である可能性が極めて高いと推測されます。攻撃者は、永続的なアクセス経路を確保した後、システムの計算リソースを不正に利用して暗号通貨を採掘し、経済的利益を得ることを目的としていると考えられます。

また、`rm -rf /tmp/secure.sh; rm -rf /tmp/auth.sh; pkill -9 secure.sh; pkill -9 auth.sh; echo > /etc/hosts.deny; pkill -9 sleep;`といったコマンドは、自身が以前に展開した可能性のあるスクリプトや、競合する他のマイニングマルウェアを駆除し、自身のマイニング活動を最適化・排他的に行うための準備である可能性が考えられます。2種類のペイロードは、それぞれダウンローダー、マイナー本体、または異なる種類のマイナー（例：CPUマイナーとGPUマイナー、あるいは異なる種類の暗号通貨マイナー）である可能性や、システムに合わせた最適化されたバージョンである可能性も考えられます。
