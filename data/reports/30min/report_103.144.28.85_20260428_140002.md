## SOCアナリストレポート

### 1. 攻撃の概要と目的

このログは、攻撃元IP `103.144.28.85` からSSHサービスに対する侵入試行とその成功、およびその後の悪意ある活動を示しています。攻撃者はrootアカウントへのログインに成功し、システムに対する完全な制御権を獲得しました。

攻撃の主な目的は以下の通りと推測されます。

*   **永続的なアクセス経路の確立:** 攻撃者はrootアカウントの `.ssh/authorized_keys` ファイルに自身の公開鍵を追加することで、パスワードなしで将来的にシステムへ再アクセスできるバックドアを設置しました。
*   **セカンダリペイロードの展開:** 永続化を確保した後、未知のセカンドペイロードをシステムにダウンロードしました。これは、感染したシステムをボットネットの一部として利用する、仮想通貨マイニングを実施する、あるいはさらなる偵察や攻撃ツールを展開するなどの、より広範な悪意ある活動を行うための準備と考えられます。

短時間に異なるパスワードで2回rootログインに成功していることから、攻撃者は自動化された強力なブルートフォースまたはパスワードリスト攻撃ツールを使用している可能性が高いです。

### 2. 推測される手法・使用ツール

*   **侵入手法:**
    *   **SSH ブルートフォース/パスワードリスト攻撃:** rootアカウントに対する複数の異なるパスワードでのログイン成功履歴から、攻撃者は自動化されたパスワードリスト攻撃またはブルートフォースツールを用いて、脆弱なパスワードを特定し侵入したと推測されます。
*   **永続化手法:**
    *   **SSH公開鍵認証バックドア:** 攻撃者は `rm -rf .ssh && mkdir .ssh && echo "..." >> .ssh/authorized_keys` コマンドを実行し、既存のSSH設定を削除して、自身の公開鍵をシステムに登録しました。これにより、パスワード認証を経ずに鍵認証でシステムにアクセスし続けることが可能となります。
*   **ペイロード展開手法:**
    *   **SFTP/SCP、またはシェル以外のプロトコル:** コマンド履歴に `wget` や `curl` のようなダウンロードコマンドが記録されていないにもかかわらず、ファイルダウンロードイベント (`cowrie.session.file_download`) が検知されています。このことから、攻撃者はSFTP (SSH File Transfer Protocol) やSCP (Secure Copy Protocol) など、SSHチャネル上で動作するファイル転送プロトコルを利用してペイロードを転送した可能性が高いです。

*   **使用ツール（推測）:**
    *   **自動化されたSSHアタックツール:** Hydra, Medusa, Nmapスクリプト、またはカスタムスクリプトなど。
    *   **標準的なLinuxコマンド:** `cd`, `rm`, `mkdir`, `echo`, `chmod` など。
    *   `lockr` コマンドは標準的なLinuxコマンドではないため、攻撃者の入力ミス、または特定のカスタムツールの一部である可能性が考えられますが、一般的には実行に失敗するか、何も起こらないと予想されます。

### 3. 脅威レベルとその理由

**脅威レベル: 深刻 (CRITICAL)**

**理由:**

1.  **rootアカウントの完全な侵害:** システム最高権限であるrootアカウントへのログインが成功しているため、攻撃者はシステムに対する完全な制御権限を有しています。これにより、システムの改ざん、データの窃取、他のシステムへの横展開、サービス停止など、あらゆる種類の悪意ある活動が可能になります。
2.  **永続的なアクセス経路の確立:** 攻撃者のSSH公開鍵が `authorized_keys` に追加されており、これはパスワード変更後も攻撃者が永続的にシステムへアクセスできるバックドアを意味します。システムの復旧を非常に困難にします。
3.  **セカンドペイロードの展開:** 永続化確保の直後に不明なファイルがダウンロードされており、これはシステムがマルウェアに感染し、ボットネット、仮想通貨マイナー、またはその他の悪意ある活動に利用される可能性が極めて高いことを示唆しています。
4.  **自動化された攻撃:** 複数の異なるパスワードでのログイン成功や、一連の迅速な行動から、これは自動化された攻撃スクリプトによるものであり、同様の攻撃が他のシステムにも試みられている可能性が高いことを意味します。

### 5. 推奨アクション

この深刻な侵害に対応するため、以下の緊急かつ段階的なアクションを推奨します。

1.  **即時封じ込め:**
    *   **侵害されたシステムのネットワーク隔離:** 該当するシステムを直ちにネットワークから隔離し、攻撃者のさらなるアクセス、内部ネットワークへの横展開、および外部への攻撃活動を防ぎます。
    *   **SSHサービスの一時停止/アクセス制限:** 全ての外部からのSSHアクセスを一時的に停止するか、厳格にホワイトリスト方式で許可されたIPアドレスからのみアクセス可能とするようファイアウォールルールを変更します。
    *   **ルートパスワードの緊急変更:** 侵害された可能性のある全てのrootアカウントのパスワードを、複雑で強力なものに直ちに変更します。
    *   **悪意のあるSSH鍵の削除:** 全てのユーザーの `~/.ssh/authorized_keys` ファイルを確認し、上記IoCに記載された公開鍵（特にコメント `mdrfckr` のもの）およびその他身に覚えのない鍵を削除します。

2.  **根絶と復旧:**
    *   **マルウェアの特定と駆除:** 隔離されたシステム上で、ダウンロードされたセカンドペイロード（SHA256: `a8460f446be540410004b1a8db4083773fa46f7fe76fa84219c93daa1669f8f2`）の分析を行います。これがボットネットエージェント、マイニングスクリプト、または別のツールかを特定し、システム全体をスキャンして全てのマルウェアや不正なファイルを削除します。
    *   **システム再構築の検討:** root権限での侵入と永続化が確認されているため、システムの整合性が完全に失われている可能性が高いです。クリーンなバックアップからの復元、またはOSの再インストールを強く推奨します。再インストールが不可能な場合は、全てのシステムファイル、設定、ログ、実行中のプロセスを徹底的に監査し、不正な変更がないことを確認する必要があります。
    *   **ログの収集と詳細分析:** `/var/log/auth.log`、`/var/log/secure`、コマンド履歴、その他のシステムログを収集し、攻撃者の活動範囲、他のアカウントの侵害有無、データ窃取の有無、横展開の試みなどを詳細に調査します。
    *   **サービスアカウントの監査とリセット:** 他の重要なサービスアカウントやシステムアカウントについて、パスワードが侵害されていないか確認し、全て強力なパスワードにリセットします。

3.  **事後対策と予防:**
    *   **ファイアウォールルールの強化:** 攻撃元IP (`103.144.28.85`) からのアクセスを永久にブロックするファイアウォールルールを追加します。
    *   **SSH設定の強化:**
        *   パスワード認証を無効にし、SSH鍵認証のみを許可します。
        *   `PermitRootLogin no` を設定し、rootアカウントの直接ログインを禁止します。
        *   可能な場合は、多要素認証 (MFA) を導入します。
    *   **強固なパスワードポリシーの適用:** 全てのシステムアカウントに対して、複雑性、長さ、有効期限などを含む厳格なパスワードポリシーを導入し、定期的なパスワード変更を強制します。
    *   **セキュリティパッチの適用:** OSおよび全てのインストールされているソフトウェアに対して、常に最新のセキュリティパッチが適用されていることを確認します。

    *   **脆弱性診断とペネトレーションテスト:** 定期的にシステムの脆弱性診断やペネトレーションテストを実施し、セキュリティ上の弱点を特定し改善します。
    *   **セキュリティ教育:** ユーザーおよびシステム管理者にセキュリティ意識向上トレーニングを実施し、セキュリティベストプラクティスを徹底させます。
