## SOCアナリストレポート

### 攻撃概要

#### 1. 攻撃の概要と目的

2026年4月27日、攻撃元IPアドレス `213.154.77.61` からSSHサービスに対する攻撃が観測されました。この攻撃者は、`root` ユーザーのパスワード認証に成功し、不正なSSH公開鍵を `authorized_keys` に追加することで永続的なバックドアを確立しました。その後、不明なファイルをダウンロードしており、これはシステムの完全な侵害と二次マルウェアの展開を目的としたものと推測されます。

この一連の行動は、ターゲットシステムに対する完全な支配を確立し、ボットネットへの組み込み、仮想通貨マイニング、またはさらなるネットワーク内偵察と横展開の足がかりとすることを目的としていると考えられます。

#### 2. 推測される手法・使用ツール

**手法 (MITRE ATT&CK TTPs)**:
*   **初期アクセス (Initial Access)**: T1078.004 Valid Accounts: SSH Brute Force/Dictionary Attack (rootユーザーのパスワード認証成功)
*   **永続化 (Persistence)**: T1098.004 Account Manipulation: SSH Authorized Keys (不正なSSH公開鍵の追加)
*   **防御回避 (Defense Evasion)**: T1222.001 File and Directory Permissions Modification: Linux/Unix (chattrコマンドの使用試行)
*   **影響 (Impact)**: T1498.001 Network Denial of Service: Internet Denied (潜在的なDDoS攻撃への利用), T1496 Resource Hijacking (仮想通貨マイニングなど)

**使用ツール**:
*   標準的なLinuxコマンド (`cd`, `rm`, `mkdir`, `echo`, `chmod`, `chattr`)
*   不正なSSHクライアント
*   ダウンロードされたペイロード（機能不明のマルウェア）

#### 3. 脅威レベルとその理由

**脅威レベル: 非常に高い (Critical)**

**理由**:
*   **root権限での侵害**: 攻撃者がシステムの最高権限である`root`アカウントでログインに成功しています。これにより、システムに対するあらゆる操作が可能となります。
*   **永続的なアクセス確立**: SSH `authorized_keys`に不正な公開鍵が追加されており、パスワードを知らなくても攻撃者がいつでもSSH経由でシステムに再接続できる状態になっています。これは非常に危険なバックドアです。
*   **不明なファイルのダウンロード**: 永続化を確立した直後に不明なファイルがダウンロードされており、これはシステムに二次マルウェア（例: ボットネットエージェント、仮想通貨マイナー、ルートキットなど）をインストールするためのペイロードである可能性が極めて高いです。
*   **システム乗っ取りの可能性**: これらの行動から、攻撃者がシステムを完全に乗っ取り、悪意のある目的に利用する準備が整ったと判断されます。

### ファイルダウンロード検知の詳細分析

攻撃者は、`root`でのログインに成功し、自身のSSH公開鍵を`authorized_keys`に追加して永続化を図った直後に、ファイルダウンロードを実行しています。このタイムラインから、ダウンロードされたファイルは**侵害後の活動（Post-Exploitation）における主要なペイロード**であると強く推測されます。

具体的な取得元URLは不明ですが、この種の攻撃パターンにおいてダウンロードされるペイロードは、以下のいずれかである可能性が高いです。

*   **仮想通貨マイナー (Cryptominer)**: 攻撃者が感染したマシンの計算リソース（CPU/GPU）を利用して仮想通貨を不正にマイニングするスクリプトやバイナリ。これは、無差別なSSHブルートフォース攻撃で最もよく見られる目的の一つです。
*   **ボットネットエージェント (Botnet Agent)**: 感染したマシンを、DDoS攻撃、スパム送信、他のシステムへの攻撃などのためのボットネットの一部として利用するためのエージェントプログラム。
*   **セカンダリバックドア/ルートキット (Secondary Backdoor/Rootkit)**: SSH鍵以外の方法でもアクセスを維持するためのより強固なバックドアや、システムに深く潜伏するためのルートキット。
*   **スキャンツール/横展開ツール (Scanning/Lateral Movement Tools)**: 感染したシステムを起点として、同一ネットワーク内の他の脆弱なシステムを探索し、感染を広げるためのツール。

このログだけでは正確な種類を断定できませんが、攻撃者の目的が「侵害したシステムから利益を得る」または「他のシステムへの攻撃基盤とする」ことから、上記ペイロードのいずれかである可能性が非常に高いです。

### 5. 推奨アクション

このシステムは既に完全に侵害された可能性が高いため、迅速かつ包括的な対応が必要です。

#### 緊急対応 (Immediate Actions):
1.  **ネットワーク隔離**: 攻撃を受けたシステムをネットワークから即座に隔離し、さらなる被害拡大や外部への攻撃活動を防ぎます。
2.  **IPアドレスのブロック**: ファイアウォール、IDS/IPS等で攻撃元IPアドレス `213.154.77.61` からの通信をブロックリストに追加します。
3.  **パスワードリセット**: `root`ユーザーを含む、すべてのシステムユーザーのパスワードを強制的にリセットし、強力で推測されにくいものに変更します。
4.  **不正なSSH鍵の削除**: 侵害されたシステムの `/root/.ssh/authorized_keys` ファイルから、上記IoCにある不正なSSH公開鍵を直ちに削除します。他のユーザーの`.ssh`ディレクトリも確認が必要です。
5.  **不審なプロセスの停止**: システム上で実行中の不審なプロセスがないか確認し、発見した場合は直ちに停止します。

#### フォレンジック調査 (Forensic Investigation):
1.  **システムイメージの取得**: 侵害されたシステムの完全なディスクイメージを取得し、詳細なフォレンジック分析を行います。
2.  **ログ分析の強化**: `auth.log`, `syslog`, `history`ファイルなど、あらゆるシステムログを詳細に分析し、攻撃者の活動範囲、実行されたコマンド、作成されたファイル、横展開の有無などを特定します。
3.  **マルウェア分析**: ダウンロードされたファイルのSHA256ハッシュ `a8460f446be540410004b1a8db4083773fa46f7fe76fa84219c93daa1669f8f2` をVirusTotal等のマルウェアデータベースで検索し、その種類と機能を特定します。可能であれば、サンドボックス環境で動的解析を行い、詳細な挙動を把握します。
4.  **バックドアの徹底的な探索**: SSH鍵以外にも、crontab、サービス、スタートアップスクリプト、ルートキットなどで新たなバックドアが仕掛けられていないか、徹底的に調査します。
5.  **横展開の有無の確認**: 侵害されたシステムを足がかりに、同一ネットワーク内の他のシステムへの横展開が試みられていないか、ネットワークトラフィックと他のシステムのログを精査します。

#### 再発防止策 (Prevention Measures):
1.  **パスワードポリシーの強化**: 強力なパスワードポリシーを導入し、定期的なパスワード変更を義務付けます。
2.  **多要素認証 (MFA) の導入**: SSHログインに多要素認証を導入し、セキュリティを強化します。
3.  **SSH鍵認証の強制**: パスワード認証を無効化し、鍵認証のみに限定します。
4.  **アクセス制御の強化**: SSHサービスへのアクセス元IPアドレスを制限します（例: 特定の管理用IPアドレスのみ許可）。
5.  **不要なサービスの停止**: 不要なネットワークサービス（特にSSHのような公開サービス）は停止するか、アクセス制限をかけます。
6.  **セキュリティパッチの適用**: OSおよびアプリケーションのセキュリティパッチを常に最新の状態に保ちます。
7.  **侵入検知/防御システム (IDS/IPS) の導入**: 不審なトラフィックや攻撃パターンを検知・防御するシステムを導入します。
8.  **SIEMの導入と監視強化**: ログの一元管理と相関分析を行うSIEMを導入し、異常検知能力を向上させます。
9.  **定期的な脆弱性診断とペネトレーションテスト**: システムの脆弱性を定期的に診断し、改善します。
10. **セキュリティ教育**: 従業員に対するセキュリティ意識向上トレーニングを実施します。
