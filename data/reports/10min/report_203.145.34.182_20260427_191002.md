## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃元IP: `203.145.34.182` から、SSHサービスに対する一連の攻撃が観測されました。
この攻撃は、主に以下の目的を達成しようとしたものと推測されます。

*   **初期アクセスと権限昇格**: SSHのブルートフォース攻撃、または辞書攻撃によって、`root`ユーザーでのログインを試み、成功しています。
*   **永続化の確立**: 成功したログイン後、自身のSSH公開鍵をターゲットシステムの `root` ユーザーの `authorized_keys` ファイルに追加することで、パスワードなしでの再ログインを可能にし、永続的なバックドアを確立しようとしました。
*   **ペイロードの展開**: 永続化の確立後、外部から不正なファイルをダウンロードしようとしました。これは、システムの完全な乗っ取り、ボットネットへの組み込み、暗号通貨マイニング、またはデータ窃取などを目的としたマルウェアである可能性が高いです。

一連の行動は自動化されたスクリプトによって実行されている可能性が高く、非常に短時間で完結しています。

### 2. 推測される手法・使用ツール

*   **初期侵入手法**: SSHブルートフォース攻撃または辞書攻撃。一般的なユーザー名 (`root`) と弱いパスワードリスト (`Abc@1234567890`, `3245gs5662d34`) を使用してログインを試みています。
*   **権限昇格**: `root` ユーザーの認証情報を窃取し、最高権限でのアクセスを獲得しています。
*   **永続化手法**: `cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa ..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~` という一連のコマンドを実行し、自身のSSH公開鍵を追加することで、将来的なアクセス経路を確保しています。これにより、以降はパスワード認証なしでアクセスできるようになります。
*   **検知されたコマンド**: `chattr -ia .ssh; lockr -ia .ssh` は、`.ssh` ディレクトリの属性を変更し、書き込み可能にすることで、`authorized_keys` ファイルを改変しようとした意図が見られます。`lockr` はタイポか、特定のマルウェアが使用する非標準コマンドである可能性があります。
*   **ペイロード配布**: `cowrie.session.file_download` イベントにより、攻撃者が外部からファイルをダウンロードしようとしたことが示されています。ダウンロード元URLは不明ですが、ハッシュ値が記録されています。これは、先の目的で述べたような二次的なマルウェアのダウンロードを意図したものです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **root権限での不正ログインの成功**: 攻撃者がシステムに対する最高レベルの権限 (`root`) を奪取したため、システムへの完全な制御権を獲得したとみなされます。
*   **永続化の確立**: SSH公開鍵を追加することで、攻撃者は検知されにくい形でシステムの永続的なアクセス経路を確立しようとしています。これは長期的なシステム悪用を意図していることを示唆します。
*   **二次ペイロードの展開**: 不明なファイルのダウンロードは、システムをさらに悪用するためのマルウェア（例: ボットネットのダウンローダー、暗号通貨マイナー、高度なバックドアなど）である可能性が高く、システムに甚大な被害をもたらす可能性があります。
*   **自動化された攻撃**: 非常に短時間での一連の操作は、自動化された攻撃ツールによって行われたことを示唆しており、広範なスキャン活動の一部である可能性も考えられます。

**【ファイルダウンロード検知の意図推測】**
攻撃者は、`root` ユーザーとしてログインし、さらにSSH公開鍵による永続的なアクセス経路を確保しています。この段階でファイルをダウンロードする意図は、システムをさらに深く侵害し、その後の具体的な悪用を実行するための「二次ペイロード」を展開することにあります。

考えられるペイロードの種類と意図は以下の通りです。

*   **ボットネットのダウンローダー**: 最も一般的なシナリオの一つです。ダウンロードされたファイルは、さらに別のマルウェア（DDoS攻撃の実行、スパム送信、プロキシとしての利用などを目的としたボットネットエージェント）をダウンロード・実行するためのスクリプトまたはバイナリである可能性が高いです。これにより、このシステムが攻撃者のボットネットの一部として利用されます。
*   **暗号通貨マイナー**: 侵入したシステムのCPUやGPUリソースを不正に利用して、攻撃者のための暗号通貨（例: Monero）をマイニングするスクリプトやバイナリである可能性も非常に高いです。これはLinuxサーバーへの攻撃で頻繁に見られます。
*   **持続的なバックドア/リモートアクセスツール**: 標準的なSSH以外の通信チャネルや、よりステルス性の高いリモート操作を可能にするためのツールである可能性もあります。これにより、攻撃者はシステムの完全な制御を維持しつつ、検出を回避しようとします。
*   **情報窃取マルウェア**: システム設定、データベースの内容、ファイルシステム上の機密情報などを収集し、攻撃者のC2サーバーへ送信するためのマルウェアである可能性も否定できません。

攻撃者がすでにroot権限と永続化経路を確立していることから、このファイルダウンロードはシステムの完全な乗っ取りと、それによる経済的利益（マイニング、ボットネット利用）またはさらなる攻撃のための基盤構築を目的としていると強く推測されます。

### 5. 推奨アクション

このログはハニーポットからのものですが、もしこれが実システムで発生した場合を想定し、以下の推奨アクションを提案します。

**A. 緊急対応 (Initial Response)**

1.  **ネットワーク隔離**: 攻撃を受けた疑いのあるシステムを直ちにネットワークから隔離し、被害の拡大を防止します。
2.  **認証情報の変更**: `root` ユーザーおよび他の全ての管理ユーザーのパスワードを直ちに、強力でユニークなものに変更します。また、パスワードは定期的に変更すべきです。
3.  **不正なSSH鍵の削除**: 隔離されたシステム上で、`/root/.ssh/authorized_keys` ファイルを徹底的に確認し、不正に追加されたSSH公開鍵 (`mdrfckr` というコメントの鍵など) を直ちに削除します。
4.  **ダウンロードされたファイルの隔離と解析**: 記録されたハッシュ値 (`a8460f446be540410004b1a8db4083773fa46f7fe76fa84219c93daa1669f8f2`) を持つファイルを特定し、実行されていないことを確認した後、安全な環境 (サンドボックス) で詳細に解析し、その挙動とC2サーバーを特定します。
5.  **システムイメージの取得**: フォレンジック調査のため、現在のシステム状態の完全なイメージを取得します。

**B. 恒久対策 (Long-term Remediation & Prevention)**

1.  **SSHセキュリティ強化**:
    *   `root` ユーザーのSSH直接ログインを禁止する (`PermitRootLogin no`)。
    *   パスワード認証を無効化し、強力な鍵認証のみに限定する (`PasswordAuthentication no`)。
    *   MFA (多要素認証) をSSHログインに導入する。
    *   SSHサービスを標準ポート (22番) 以外の高位ポートに変更する (ポートスキャン対策)。
2.  **パスワードポリシーの強化**: 全てのユーザーアカウントに対し、複雑性、長さ、有効期限など、より厳格なパスワードポリシーを適用し、定期的な変更を義務付けます。
3.  **IDS/IPSの導入と監視強化**:
    *   不正なSSHログイン試行 (ブルートフォース/辞書攻撃) を検知し、自動的にブロックするIDS/IPSルールを導入・強化します。
    *   不明なIPからのファイルダウンロードや、異常なアウトバウンド通信を検知するルールを設定します。
4.  **脆弱性管理**:
    *   OS、アプリケーション、ミドルウェアのセキュリティパッチを常に最新の状態に保ち、既知の脆弱性を悪用した攻撃を防ぎます。
    *   定期的な脆弱性診断を実施し、設定ミスやゼロデイ以外の脆弱性を早期に発見し対処します。
5.  **ログ監視の強化**:
    *   SIEM (Security Information and Event Management) を導入し、SSHログ、システムログ、ネットワークログなどを集中管理し、リアルタイムで異常を検知できる体制を構築します。
    *   特にログイン失敗回数の閾値設定、不審なコマンド実行、異常なプロセス起動などを監視します。
6.  **インシデントレスポンス計画の更新**: 今回の事例を教訓に、インシデントレスポンス計画を見直し、同様の攻撃が発生した場合の対応フローを改善します。
7.  **セキュリティ意識向上トレーニング**: 従業員に対し、セキュリティに関する定期的なトレーニングを実施し、フィッシング、ソーシャルエンジニアリング、パスワード管理などの重要性を周知します。

この報告書は、観測されたログに基づいた分析と推測です。実際の攻撃はさらに広範囲にわたる可能性があります。
