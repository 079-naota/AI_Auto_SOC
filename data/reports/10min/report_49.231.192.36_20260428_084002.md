## SOC分析レポート

### 1. 攻撃の概要と目的

本レポートは、攻撃元IPアドレス `49.231.192.36` からのSSHサービスに対する不審な活動を記録したCowrieハニーポットのログに基づいています。

攻撃は、SSHサービスへのブルートフォース/辞書攻撃から始まり、`root:administrator` という一般的な認証情報を用いて初期アクセスに成功しました。アクセス確立後、攻撃者はシステムへの永続的なアクセスを確立するために、自身のSSH公開鍵をターゲットシステムの `~/.ssh/authorized_keys` ファイルに書き込みました。これにより、パスワード認証なしでの再接続を可能にし、将来的にパスワードが変更されてもアクセスを維持できる状態を作り出すことを目的としています。

永続化の試行に続いて、システム上で不審なファイルダウンロードが検知されました。このダウンロードは、攻撃者が初期アクセスと永続化の足がかりを確立した直後に行われているため、さらなる悪意のある活動（例：マルウェアの展開、暗号通貨マイニングスクリプトの実行、DDoSボットのインストールなど）のためのペイロードであると推測されます。

その後、攻撃者は異なるパスワード (`root:3245gs5662d34`) で再度 `root` ユーザーとしてログインに成功しており、これは自動化された攻撃ツールが複数の認証情報を試しているか、あるいは侵害されたシステムへの継続的な制御を試みていることを示唆しています。

総合的に見て、攻撃の目的は、ターゲットシステムを乗っ取り、永続的なアクセスを確保した上で、そのリソースを不正に利用することであると判断されます。

### 2. 推測される手法・使用ツール

*   **初期アクセス (Initial Access)**:
    *   **ブルートフォース/辞書攻撃 (T1110)**: 複数の認証情報 (`root:administrator`, `root:3245gs5662d34`) を用いたSSHログイン試行が成功していることから、`Hydra` や `Medusa`、`Nmap` のNSEスクリプトのような自動化されたブルートフォース/辞書攻撃ツールが使用された可能性が高いです。
*   **永続化 (Persistence)**:
    *   **SSH Authorized Keys (T1098.004)**: `rm -rf .ssh && mkdir .ssh && echo "ssh-rsa ..."` コマンドを実行し、攻撃者の公開鍵を `~/.ssh/authorized_keys` に追加しています。これにより、パスワード認証を回避し、鍵認証による永続的なアクセス経路を確立しようとしました。
    *   **特権昇格の試行 (T1068)**: `chattr -ia .ssh` の実行は、`.ssh` ディレクトリの不変属性を解除し、その内容を変更可能にすることを目的としています。これは、永続化メカニズムを確実に設定するための前処理と考えられます。`lockr` は一般的なLinuxコマンドではないため、誤入力か、特定のマルウェアが使用するカスタムコマンドの可能性がありますが、ログでは失敗しています。
*   **機能確立/実行 (Execution / Command and Control)**:
    *   **ファイルダウンロード (T1105)**: 不明なファイルのダウンロードが検知されています。具体的なコマンドは記録されていませんが、`wget` や `curl`、`scp` といったツール、またはシェルスクリプトを通じて、悪意のあるペイロードが取得された可能性が高いです。
    *   **シェルコマンド (T1059.004)**: `cd`, `rm`, `mkdir`, `echo`, `chmod` といった標準的なLinuxシェルコマンドを使用して、ディレクトリの操作やファイルへの書き込みを行っています。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

1.  **root権限の奪取**: 攻撃者はシステムの最高権限である `root` ユーザーとしてログインに成功しており、システム上のあらゆる操作（ファイルの読み書き、実行、設定変更、ユーザー作成など）が可能です。これにより、完全なシステム侵害に繋がる深刻な影響を及ぼす可能性があります。
2.  **永続化の確立**: SSH公開鍵の設置により、攻撃者はパスワードが変更された後でも、いつでもシステムへ再アクセスできる永続的なバックドアを作成しています。これは検出後の復旧を非常に困難にします。
3.  **悪性ペイロードのダウンロード**: 永続化確立直後のファイルダウンロードは、システムの計算リソースの不正利用（暗号通貨マイニング）、DDoS攻撃の実行、他のシステムへの横展開、または情報の窃取など、更なる悪意のある活動を実行するためのマルウェアやツールを導入しようとする明確な意図を示しています。
4.  **自動化された攻撃**: 複数のログイン試行と計画的な一連のコマンド実行から、攻撃が自動化されたスクリプトやボットによって行われている可能性が高く、同様の攻撃が他のシステムにも仕掛けられているリスクがあります。

ハニーポットのログであるため、実際のシステムが直接的な被害を受けたわけではありませんが、この活動が実際のサーバーで発生した場合、極めて深刻なセキュリティインシデントに発展する可能性が高いです。

### 5. 推奨アクション

1.  **即時ブロックと隔離**:
    *   攻撃元IPアドレス `49.231.192.36` からのすべての通信を、ファイアウォールやネットワークIDS/IPSで即座にブロックしてください。
    *   もし同様のログが実際のサーバーで確認された場合、当該サーバーをネットワークから即座に隔離し、他のシステムへの感染拡大を防いでください。
2.  **認証情報の変更と強化**:
    *   すべてのSSHユーザー（特に `root` アカウント）のパスワードを、強力でユニークなものに即座に変更してください。
    *   SSHサービスにおけるパスワード認証を可能な限り無効化し、多要素認証（MFA）を伴う鍵認証のみの使用を検討してください。
    *   既存のSSH鍵を確認し、不明な公開鍵 (`authorized_keys`) が存在しないか確認・削除してください。
3.  **SSH設定のセキュリティ強化**:
    *   `/etc/ssh/sshd_config` にて `PermitRootLogin no` を設定し、rootユーザーの直接ログインを禁止してください。
    *   SSHサービスがデフォルトポート `22` で動作している場合は、可能な範囲でポート番号を変更することを検討してください。
    *   `MaxAuthAttempts` や `LoginGraceTime` を適切に設定し、ブルートフォース攻撃に対する耐性を向上させてください。
4.  **IoCのスキャンと調査**:
    *   ダウンロードされたペイロードのSHA256ハッシュ値 (`a8460f446be540410004b1a8db4083773fa46f7fe76fa84219c93daa1669f8f2`) をVirusTotalなどのマルウェア解析サービスで検索し、既知の脅威と関連付けられるか確認してください。
    *   もしこのファイルが実際のシステムにダウンロードされていた場合、感染の有無を詳細に調査し、マルウェアの駆除またはシステムの再構築を検討してください。
5.  **ログ監視とアラートの強化**:
    *   SSHログイン試行、特に失敗ログ (`Failed password for invalid user` や `Failed password for root`) や異常なコマンド実行に対するログ監視を強化し、SIEMや集中ログ管理システムで異常を検知した際に即座にアラートを発するよう設定してください。
    *   ファイル整合性監視 (FIM) ツールを導入し、`.ssh` ディレクトリやその他の重要なシステムファイルへの不正な変更を監視してください。
6.  **脆弱性管理とパッチ適用**:
    *   OSやSSHサービスを含むすべてのソフトウェアに対して、最新のセキュリティパッチが適用されていることを確認してください。定期的な脆弱性スキャンを実施し、潜在的な脆弱性を特定・修正してください。
7.  **セキュリティ教育**:
    *   システム管理者やユーザーに対し、強力なパスワードの使用、不審な活動の報告、セキュリティベストプラクティスに関する定期的な教育を実施してください。
