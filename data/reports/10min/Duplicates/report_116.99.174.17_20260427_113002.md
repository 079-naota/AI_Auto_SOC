# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 11:33:01

---

## SOC分析レポート

### 1. 攻撃の概要と目的

提供されたログは、IPアドレス `116.99.174.17` からのSSHサービスに対するブルートフォース攻撃または辞書攻撃を示しています。攻撃者は様々なユーザー名とパスワードの組み合わせを試行し、最終的に複数の異なるパスワードで `root` アカウントを含むシステムへのログインに成功しています。

攻撃の主な目的は、標的システムへの不正な初期アクセスを確立することであると推測されます。ログイン成功後には、`cowrie.direct-tcpip.request` および `cowrie.direct-tcpip.data` のイベントが複数回記録されており、これは攻撃者が確立したSSHセッションを通じて、ポートフォワーディングやデータの転送など、さらなる悪意のある活動（例: 追加のマルウェアダウンロード、C2サーバーへの接続、他のシステムへの横展開準備）を試みていた可能性が非常に高いことを示唆しています。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃/辞書攻撃**: 攻撃者は「admin」「user」「root」といった一般的なユーザー名や、過去に漏洩した可能性のあるパスワードリストを用いて、SSHサービスに対する認証情報の総当たり攻撃を行っています。短時間に多数の異なるユーザー名とパスワードの組み合わせが試行されていることから、自動化されたスクリプトやツールが使用された可能性が高いです。
    *   **TCPポートフォワーディング/データ転送**: ログイン成功後、攻撃者はSSHの `direct-tcpip` チャネルを介して通信を確立しようとしています。これは、プロキシとして利用したり、内部ネットワークへのアクセスを試みたり、あるいは特定のデータを外部と送受信したりする目的で行われることがあります。

*   **使用ツール**:
    ログからは特定のツール名は特定できませんが、SSHブルートフォース攻撃に一般的に使用される「Hydra」や「Medusa」、Nmapのスクリプト（`ssh-brute`など）、あるいはカスタムスクリプトのような自動化されたツールが使用されたと推測されます。`direct-tcpip` の利用は、攻撃者が手動で操作した可能性もありますが、自動化されたマルウェアの挙動の一部である可能性も否定できません。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
1.  **不正アクセス成功**: 攻撃者が `root` ユーザーを含む複数のアカウントに対してログインに成功している記録があります。これは、もし本ログが実システムのものであれば、攻撃者がシステムへの管理者権限での初期アクセスを確立したことを意味し、システム全体が掌握される危険性があります。
2.  **`root` ユーザーの侵害**: 最も高い権限を持つ `root` ユーザーでのログインが複数回成功しており、これは攻撃者がシステムに対して完全な制御を奪取できる可能性があったことを示します。
3.  **継続的な攻撃活動**: 攻撃は短時間のうちに複数回繰り返され、異なるユーザー名とパスワードの組み合わせを試しています。これは、執拗な攻撃意図があることを示唆しています。
4.  **ログイン後の不審な活動**: ログイン成功直後に `cowrie.direct-tcpip.request` と `cowrie.direct-tcpip.data` が記録されていることから、攻撃者はログイン後、単なる確認に留まらず、何らかのペイロードの実行、追加ツールのダウンロード、あるいはC2サーバーとの通信確立といった、さらに悪意のある後続の活動を試みていた可能性が極めて高いです。

### 4. 抽出されたIoC（Indicators of Compromise）

*   **攻撃元IPアドレス**: `116.99.174.17`
*   **成功したログイン情報（ユーザー名/パスワード）**:
    *   `root` / `admin`
    *   `root` / `@`
    *   `root` / `root123`
    *   `root` / `12345`
    *   `root` / `abcd1234`
    *   `root` / `ipscan`
    *   `root` / `alpine`
*   **試行された主なユーザー名**: `admin`, `user`, `ubnt`, `config`, `squid`, `support`, `guest`, `1234`, `nikita`, `ftpuser`, `root`
*   **ダウンロード元URL、ファイルのハッシュ値**: ログには直接的なファイルダウンロードイベントやそのハッシュ値、ダウンロード元URLは含まれていません。

**ファイルダウンロード検知（推測される意図を持つペイロード）**:
ログに直接的なファイルダウンロードの記録はありませんが、ログイン成功後の `cowrie.direct-tcpip.request` および `cowrie.direct-tcpip.data` の発生は、攻撃者がSSHセッションを介してデータ転送を試みたことを強く示唆しています。これは、以下のような意図を持つペイロードの転送を試みていた可能性があります。

*   **ボットネットのダウンローダー**: 感染したシステムをボットネットの一部として利用するため、他のマルウェア（DDoS攻撃ツール、スパム送信モジュールなど）をダウンロード・実行しようとした可能性があります。
*   **暗号通貨マイニングスクリプト**: システムのリソースを不正に利用して暗号通貨を採掘するマイニングソフトウェアをダウンロードし、実行しようとした可能性があります。
*   **バックドアまたはC2クライアント**: 持続的なアクセスを確保するためのバックドア（例: SSHキーの登録、Netcatリスナーのセットアップ）や、攻撃者のコマンド＆コントロール（C2）サーバーと通信するためのクライアントをインストールしようとした可能性があります。
*   **スキャンツール/横展開ツール**: 内部ネットワークの他のホストを探索するためのスキャンツールや、脆弱性を悪用して横展開を図るためのツールをダウンロードしようとした可能性があります。
*   **プロキシサービスの設定**: 侵害したホストを、他の攻撃活動のためのプロキシサーバーとして利用する設定ファイルやツールを転送しようとした可能性があります。

これらの活動は、攻撃者が単なるアクセス確立に留まらず、侵害したシステムを足がかりにさらなる悪事を働くことを目的としていることを示しています。

### 5. 推奨アクション

このログはCowrieハニーポットのものであるため、実際のシステムが侵害されたわけではありませんが、実システムで同様の攻撃が発生した場合を想定し、以下の推奨アクションを提示します。

**A. 即時対応 (Immediate Action)**

1.  **攻撃元IPアドレスのブロック**: `116.99.174.17` からのすべての通信をファイアウォールまたはIPS/IDSで即座にブロックしてください。
2.  **成功した認証情報の変更と確認**:
    *   ログに記録されたログイン成功に使用されたすべてのユーザー名とパスワード (`root` / `admin`, `root` / `@`, `root` / `root123` など) を即座に複雑なものへ変更してください。
    *   これらの認証情報が他のシステムやサービスで使い回されていないかを確認し、もし使い回されている場合はそれらのパスワードも変更してください。
3.  **システム侵害の確認（実システムの場合）**:
    *   このログが実システムのものであった場合、ログイン成功後のシステムに対する不正な活動（ファイル作成、コマンド実行、不審なプロセスの起動、外部への通信、設定変更など）がないか、詳細なフォレンジック調査を実施してください。
    *   特に `direct-tcpip` イベントはポートフォワーディングやデータ転送を示唆するため、SSHデーモンのログ、ネットワークフローログ（NetFlow/IPFIX）、システムコールログなどを分析し、どのような通信が試みられたか、どのようなデータが転送されたかを特定してください。

**B. 予防措置 (Preventive Measures)**

1.  **SSHパスワード認証の無効化**: 可能な限り、SSHのパスワード認証を無効化し、よりセキュアな公開鍵認証（MFAと併用推奨）への移行を強く推奨します。
2.  **SSHポートの変更**: 標準的なSSHポート（22番）を別の高位ポートに変更することで、自動化された攻撃ツールのターゲットになる頻度を減らすことができます。
3.  **アクセス元IPアドレスの制限**: SSHへのアクセスを、信頼できる特定のIPアドレス範囲に限定する（ホワイトリスト方式）ことで、不正アクセスリスクを大幅に低減できます。
4.  **多要素認証 (MFA) の導入**: SSHログインにMFAを導入することで、認証情報が漏洩した場合でも不正ログインを防ぐことができます。
5.  **強力なパスワードポリシーの適用**:
    *   パスワードの長さ、複雑性、有効期限に関する厳格なポリシーを適用してください。
    *   一般的なパスワードや辞書に載っているパスワードの使用を禁止する機能を導入してください。
6.  **アカウントの管理**:
    *   「admin」「root」「user」といったデフォルトや推測されやすいユーザー名を避け、ユニークなユーザー名を使用してください。
    *   使用されていないアカウントや不要なアカウントは削除または無効化してください。
7.  **ブルートフォース攻撃対策ツールの導入**: Fail2banのようなツールを導入し、一定回数以上のログイン試行失敗があったIPアドレスを自動的にブロックする設定を行ってください。
8.  **ログ監視の強化**: SSHログイン試行、認証失敗、およびログイン後の異常な活動をリアルタイムで監視し、異常を検知した際にはアラートを発する体制を強化してください。SIEM（Security Information and Event Management）システムとの連携も検討してください。
9.  **脅威インテリジェンスの活用**: 攻撃元IPアドレス `116.99.174.17` を脅威インテリジェンスデータベースに照会し、既知の悪性IPアドレスであるかを確認し、定期的にIPレピュテーションリストを更新してください。
10. **セキュリティトレーニング**: 従業員に対し、安全なパスワードの利用やフィッシング詐欺に対する意識向上のための定期的なセキュリティトレーニングを実施してください。

以上の分析と推奨アクションにより、今回の攻撃パターンに対する防御力を向上させ、将来的なリスクを低減できると考えられます。
