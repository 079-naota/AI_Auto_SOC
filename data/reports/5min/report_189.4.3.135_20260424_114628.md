### 1. 攻撃の概要と目的

攻撃元IP `189.4.3.135`からの攻撃者は、SSHサービスに対するブルートフォース攻撃を複数回試行し、最終的に「root」ユーザーとして複数回ログインに成功しています。ログイン後、攻撃者は以下の活動を試みています。

*   **永続化メカニズムの確立:** SSH公開鍵を`~/.ssh/authorized_keys`に追加し、パスワード認証なしで将来的にアクセスできるよう試みています。また、rootユーザーのパスワード変更も試みています。
*   **システム情報の収集:** CPU情報、メモリ使用量、OS情報（カーネル、アーキテクチャ）、ディスク使用量、現在のログインユーザー、cronジョブリストなど、広範なシステム情報を収集するコマンドを実行しています。
*   **痕跡削除/防御回避:** 一時ファイルの削除、実行中のプロセスの終了、`/etc/hosts.deny`のクリアなどを試みています。

これらの行動から、攻撃の主な目的は、標的システムへの**永続的なアクセス経路を確立し、システム情報を収集した上で、将来的なマルウェアの展開、ボットネットへの組み込み、またはさらなる標的型攻撃の足がかりとする**ことであると推測されます。

### 2. 推測される手法・使用ツール

**手法 (MITRE ATT&CK Tacticに準拠):**

*   **初期アクセス (Initial Access - T1133: External Remote Services, T1078: Valid Accounts):**
    *   SSHサービスに対するブルートフォース攻撃または辞書攻撃。一般的なユーザー名（ali）や、`root`ユーザーに対して複数の推測しやすいパスワード（`Qwert123456`, `3245gs5662d34`, `qwertyuiop!@`, `a123456m`, `admin321`）を試行しています。
*   **永続化 (Persistence - T1098: Account Manipulation: SSH Authorized Keys, T1098.003: Account Manipulation: Local Account):**
    *   `~/.ssh/authorized_keys`に攻撃者のSSH公開鍵を追加することで、パスワードなしで再ログインできるバックドアを作成しようとしています。鍵のコメント「mdrfckr」は特定のマルウェアキャンペーンやボットネット活動でよく見られます。
    *   `echo "root:POBQ00x0XKIu"|chpasswd|bash` コマンドにより、rootユーザーのパスワードを強制的に変更し、別の永続化手段を確立しようとしています。
*   **防御回避 (Defense Evasion - T1070.004: Indicator Removal: File Deletion):**
    *   `.ssh`ディレクトリに対する`chattr -ia`や`lockr -ia`（いずれもCowrie上では失敗）は、属性変更を試み、痕跡を隠蔽しようとする意図が見受けられます。
    *   `rm -rf /tmp/secure.sh; rm -rf /tmp/auth.sh; pkill -9 secure.sh; pkill -9 auth.sh; echo > /etc/hosts.deny; pkill -9 sleep;` などのコマンドで、自身が使用したスクリプトや一時ファイル、実行中のプロセスを削除し、システムログの追跡を困難にしようと試みています。特に`/etc/hosts.deny`のクリアは、接続制限を無効化する意図があります。
*   **検出 (Discovery - T1082: System Information Discovery, T1049: System Network Connections Discovery, T1033: System Owner/User Discovery, T1016: System Network Configuration Discovery):**
    *   `cat /proc/cpuinfo`, `free -m`, `uname -a`, `lscpu`, `df -h` などのコマンドで、CPU、メモリ、ディスク、OSバージョン、システムアーキテクチャなどの詳細なシステム情報を収集しています。
    *   `crontab -l` でスケジュールされたタスクを調査し、永続化の機会や既存の防御メカニズムを探っている可能性があります。
    *   `w`, `whoami` で現在ログインしているユーザーや自身のユーザー情報を確認しています。

**使用ツール:**

*   **自動化されたSSHクライアント/ブルートフォースツール:** 複数のユーザー名とパスワードを高速に試行していることから、手動ではなく自動化されたツールが使用されていると考えられます。
*   **標準的なLinuxコマンドラインツール:** `cd`, `rm`, `mkdir`, `echo`, `cat`, `grep`, `awk`, `which`, `ls`, `crontab`, `w`, `uname`, `whoami`, `lscpu`, `df`, `chpasswd`, `pkill` など、システムに標準で存在するコマンドを悪用しています。
*   **シェルスクリプト:** 一連のコマンドが迅速かつ連続して実行されていることから、攻撃者はシェルスクリプトを用いてこれらの動作を自動化している可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **ログイン成功と永続化の試み:** 攻撃者は実際にrootアカウントでのログインに成功し、複数の永続化メカニズム（SSH鍵の追加、rootパスワードの変更）を確立しようと試みています。これは、実際のシステムであれば深刻な侵害に直結する活動です。
*   **広範な情報収集:** システムの環境、リソース、設定に関する詳細な情報を収集しており、これは攻撃者が次のステップ（例: 特定の脆弱性の悪用、マルウェアの選定、標的型攻撃の計画）に進むための偵察フェーズを示唆しています。
*   **防御回避の試み:** 自身の痕跡を消去し、防御メカニズムを無効化しようとする行動は、攻撃者が発見を避け、長期的なアクセスを維持しようとしていることを示しています。
*   **自動化された攻撃:** 複数のパスワード試行と一連のコマンド実行の迅速さから、この攻撃は自動化されたボットネットまたはスキャニング活動の一部である可能性が高いです。

このログはHoneypot（Cowrie）のものであるため、実際のシステムへの被害は発生していませんが、もしこれが実際のプロダクション環境であれば、すでに完全なシステム侵害に至っていた可能性が極めて高いと評価します。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPのブロック:** ファイアウォール、IDS/IPSなどのセキュリティデバイスにて、攻撃元IP `189.4.3.135`からのSSHポート（TCP 22）へのアクセスを即座にブロックしてください。
2.  **SSH設定の緊急確認:**

    *   パスワードポリシーが十分に強力であることを確認し、定期的なパスワード変更を促してください。
    *   公開されているSSHサーバーで、上記ログに記載されたパスワード（`Qwert123456`, `3245gs5662d34`, `qwertyuiop!@`, `a123456m`, `admin321`）を使用しているアカウントがないか緊急で監査し、もしあれば即座に変更してください。

#### 中期・長期対応 (Mid-to-Long Term Actions):

1.  **多要素認証 (MFA) の導入:** SSH接続にMFAを導入し、パスワードと公開鍵認証の組み合わせだけでなく、さらなるセキュリティ層を追加してください。

    *   `Fail2Ban`などのブルートフォース対策ツールを導入し、不正なログイン試行を自動的に検出しブロックする仕組みを構築してください。
    *   不要なポートは閉じ、最小権限の原則を適用してください。
3.  **セキュリティ監視の強化:**

4.  **定期的なセキュリティ監査と脆弱性診断:** 定期的にシステムに対するセキュリティ監査と脆弱性診断を実施し、設定不備や既知の脆弱性を特定し修正してください。
5.  **パッチ管理の徹底:** OSやアプリケーションのセキュリティパッチを常に最新の状態に保ってください。
