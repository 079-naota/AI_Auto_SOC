## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `116.99.174.17` から、当社のSSHサービス（本件ではCowrieハニーポット）に対するブルートフォース攻撃または辞書攻撃が観測されました。攻撃は短期間に集中して行われ、多数のユーザー名とパスワードの組み合わせが試されました。

**主な活動:**
*   **攻撃元IP**: 116.99.174.17
*   **期間**: 2026-04-27T02:13:34Z から 2026-04-27T02:31:53Z の約18分間
*   **標的サービス**: SSH
*   **試行回数**: 複数のSSHセッションが開始され、合計22回のログイン試行が記録されています。
*   **ログイン成功**: 以下の組み合わせで合計7回のログイン成功が記録されています（ハニーポット上）。
    *   `root` / `admin`
    *   `root` / `@`
    *   `root` / `root123`
    *   `root` / `12345`
    *   `root` / `abcd1234`
    *   `root` / `ipscan`
    *   `root` / `alpine`
*   **成功後の活動**: ログイン成功後、攻撃者は `cowrie.direct-tcpip.request` および関連イベントを発生させています。これはSSHトンネリングやポートフォワーディングの試行であり、内部ネットワークへのアクセスや、別のサービスへの接続を試みている可能性を示唆します。

**攻撃者の目的（推測）**:
攻撃者は、SSHサービスへの不正アクセスを確立し、その後に内部ネットワークの偵察、横展開、またはさらなる悪意のある活動（例: マルウェアの展開、データ窃取など）を目的としていると推測されます。特に `root` ユーザーでのログイン成功に執着していることから、システムに対する完全な制御権の獲得を目指していると考えられます。

### 2. 推測される手法・使用ツール

*   **ブルートフォース攻撃/辞書攻撃**: 短時間で多数の異なるユーザー名とパスワードの組み合わせを試行していることから、自動化されたブルートフォースツールや辞書攻撃ツールが使用されている可能性が非常に高いです。一般的なツールとしては、Hydra, NmapのSSH関連スクリプト、またはカスタムスクリプトなどが考えられます。
*   **SSHトンネリング/ポートフォワーディング**: ログイン成功後に `direct-tcpip` リクエストが観測されているため、攻撃者はSSHの機能を利用して、侵害したホストを介して内部ネットワーク上の他のターゲットにアクセスしようと試みています。これは、セキュリティ対策を迂回し、検出を回避するための一般的な手法です。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**
1.  **ログイン成功の頻度**: ハニーポットであるため実際のシステム侵害は発生していませんが、もしこの環境が本番システムであった場合、攻撃者は`root`アカウントで複数回ログインに成功しており、これはシステムの完全な制御を意味します。
2.  **`root`アカウントの標的**: 最上位権限である`root`アカウントの認証情報を執拗に狙い、実際に成功している点は、深刻な脅威です。
3.  **`direct-tcpip`の利用**: ログイン成功後に直ちにSSHトンネリング/ポートフォワーディングを試みていることから、攻撃者は単なるアクセスだけでなく、その後の横展開や本格的な侵入活動を計画していることが伺えます。これは、初期アクセスから特権昇格、横展開へと進む典型的な攻撃チェーンの一部である可能性があります。
4.  **試行パスワードの多様性**: 辞書的なパスワードから、やや複雑なパスワード（`root123`, `abcd1234`, `ipscan`, `alpine`など）まで幅広く試していることから、一般的な弱いパスワードだけでなく、実際の運用で使われがちなパスワードパターンも網羅しようとしていることが示唆されます。

本件はハニーポットのログであるため、直接的な被害は発生していませんが、実際の環境に同様の脆弱なSSHサービスが存在した場合、重大なセキュリティインシデントに発展する可能性が極めて高いです。

### 4. 推奨アクション

以下の対策を早急に実施し、類似の攻撃に対する防御力を強化することを強く推奨します。

1.  **攻撃元IPのブロック**: 攻撃元IPアドレス `116.99.174.17` からのアクセスをファイアウォールやWAF等でブロックリストに追加し、今後のアクセスを遮断してください。
2.  **SSHセキュリティ強化**:
    *   **パスワードポリシーの強化**: SSHを含む全てのシステムにおいて、十分な長さと複雑性を持つパスワードの使用を義務付け、定期的な変更を促してください。
    *   **多要素認証 (MFA) の導入**: SSHログインにMFAを必須とすることで、パスワードが漏洩した場合でも不正アクセスを防ぐことができます。
    *   **公開鍵認証の徹底**: パスワード認証を無効化し、公開鍵認証のみを許可することを強く推奨します。鍵の適切な管理とアクセス制御を行ってください。
    *   **SSHポートの変更**: デフォルトの22番ポートではなく、利用者の少ない高位ポートに変更することで、自動化されたスキャンからの露出を減らすことができます。
    *   **IPアドレス制限**: SSHアクセス元を信頼できる特定のIPアドレス範囲に限定してください。
3.  **既知の脆弱な認証情報の確認**: ログで確認されたログイン成功のユーザー名とパスワードの組み合わせ（`root/admin`, `root/@`, `root/root123`, `root/12345`, `root/abcd1234`, `root/ipscan`, `root/alpine`など）が、**実際の運用環境で利用されていないか**、緊急で棚卸し・確認し、使用されている場合は即座に変更してください。
4.  **ログ監視の強化**: SSHログインの試行（特に失敗回数が多い場合）や成功後の異常な活動（例: `direct-tcpip`リクエスト、シェルコマンドの実行履歴）を検知するためのログ監視体制を強化し、リアルタイムでのアラート通知を設定してください。
5.  **IDS/IPSの導入・強化**: ネットワークIDS/IPSを導入または既存システムのルールを強化し、SSHブルートフォース攻撃や不正なトンネリングの試行を検知・阻止できるようにしてください。
6.  **従業員へのセキュリティ教育**: 強いパスワードの重要性、不審な活動の報告方法など、セキュリティ意識向上トレーニングを定期的に実施してください。
