# SOC自動分析レポート: 190.244.39.232

**生成日時:** 2026-04-26 11:02:57

---

## SOC分析レポート

**件名: SSHサービスへのブルートフォース攻撃と不正アクセス試行の検出**

### 1. 攻撃の概要と目的

攻撃元IPアドレス `190.244.39.232` から、当社のSSHサービス（Cowrieハニーポットにより監視）に対し、継続的なブルートフォース攻撃が確認されました。攻撃者は主に `root` ユーザーの認証情報を狙い、複数のパスワードを試行しています。

ログインに成功した際には、以下の行動を試みていました。

1.  **永続化の確立**: 自身のSSH公開鍵をターゲットシステムの `~/.ssh/authorized_keys` ファイルに登録し、パスワードなしでの将来的なアクセス経路を確保しようとしていました。登録しようとした公開鍵のコメントは `mdrfckr` です。
2.  **防御機構の無効化試行**: `.ssh` ディレクトリのファイル属性を変更し、恒久的な変更や削除を防ぐためのセキュリティ設定（`chattr -ia`）を解除しようと試みていました。
3.  **マルウェア導入の可能性**: `cowrie.session.file_download` イベントが複数回発生しており、ログイン後に何らかの悪意のあるファイルをシステムにダウンロードしようとしていた可能性が高いです。

これらの行動から、攻撃の主な目的は、当社のシステムへの不正な長期アクセスを確立し、システムを乗っ取るか、マルウェアを展開してリソースを悪用すること（例: 暗号通貨マイニング、ボットネットへの組み込み、他のシステムへの攻撃の踏み台など）であると推測されます。

### 2. 推測される手法・使用ツール

*   **認証情報の不正取得**:
    *   **ブルートフォース攻撃 / 認証情報スタッフィング**: 複数の異なるユーザー名（`odoo`, `user0`, `shiv`, `oracle`, `ovpn`, `windows`, および不明な文字列）と多数のパスワードを組み合わせてSSHログインを試行しています。特に `root` ユーザーに対する攻撃が執拗に行われ、実際にハニーポット上で複数のパスワード（`135792468`, `3245gs5662d34`, `abcdefgh`, `Aa112211.`, `Gp123456`, `nPSpP4PBW0` など）でログインに成功しています。これは、一般的なパスワードリストや過去に漏洩した認証情報リストを使用している可能性を示唆します。
*   **永続化**:
    *   ログイン成功後、`cd ~ && rm -rf .ssh && mkdir .ssh && echo "..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~` という一連のコマンドを実行し、自身のSSH公開鍵を登録することで、永続的なバックドアを仕掛けようとしています。
    *   `chattr -ia .ssh` コマンドで、`.ssh` ディレクトリの不変属性（immutable attribute）を解除しようとしていますが、これはシステムが攻撃者の設定変更を妨げないようにする試みです。
    *   `lockr -ia .ssh` というコマンドも試みられていますが、これは標準的なLinuxコマンドではなく、特定のマルウェアやツールの一部であるか、攻撃者のタイプミスである可能性があります。
*   **追加のペイロード展開**:
    *   `cowrie.session.file_download` イベントは、攻撃者がSSHセッション内で外部からファイルを取得しようと試みたことを示しており、さらなるマルウェア（ボット、バックドア、マイナーなど）を導入する意図があったと考えられます。
*   **使用ツール**:
    *   自動化されたSSHクライアント（恐らくPythonスクリプトやGo言語で書かれたマルウェアの一部）や、Hydraなどの一般的なブルートフォースツールを使用している可能性が高いです。

### 3. 脅威レベルとその理由

*   **脅威レベル: 高 (High)**

**理由**:

1.  **rootアカウントへのログイン成功**: 攻撃者はハニーポットの `root` アカウントに対し、複数のパスワードでログインに成功しています。もしこれが実システムであった場合、最高権限を奪取されたことになり、システム全体が完全に制御下に入り、極めて深刻な被害が発生する可能性があります。
2.  **永続的なアクセス経路の確立試行**: ログイン成功後の行動から、攻撃者が自身のSSH公開鍵を登録し、永続的なバックドアを確立しようとしていたことが明らかです。これにより、たとえパスワードが変更されても、攻撃者は継続的にシステムにアクセスできてしまいます。
3.  **マルウェア導入の意図**: ファイルダウンロードの試行は、システムにさらなる悪意のあるペイロード（例: ボット、暗号通貨マイナー、データ窃取マルウェア）を導入し、システムの侵害を拡大しようとする明確な意図を示しています。
4.  **自動化された広範囲な攻撃**: 攻撃のパターン（多数のユーザー/パスワード試行、定型的なログイン後のコマンド実行）から、これは特定の標的を狙ったものではなく、インターネット上の脆弱なSSHサービスを自動的にスキャンし、侵害しようとするボットネットやワームの一種である可能性が高いです。これにより、当社の他のシステムも同様の攻撃に晒されている可能性があります。

### 4. 推奨アクション

このログはハニーポットからのものであるため、実際のシステムへの直接的な被害はありませんが、この攻撃が実システムに対して行われたと仮定し、以下の推奨アクションを講じるべきです。

#### 4.1. 即時対応 (Immediate Actions)

1.  **攻撃元IPアドレスのブロック**: ファイアウォールやIDS/IPSにおいて、攻撃元IPアドレス `190.244.39.232` からの全てのインバウンド接続を直ちにブロックリストに追加します。
2.  **アカウントのパスワード変更とSSH鍵の確認**:
    *   `root` アカウントを含む、ログに記載された成功したログイン試行で使用されたすべてのユーザーアカウントのパスワードを直ちに、かつ強力なものに変更します。
    *   全ユーザーの `~/.ssh/authorized_keys` ファイルを確認し、不審な公開鍵（特にログにあった `mdrfckr` の公開鍵）がないか確認し、発見した場合は削除します。
3.  **システム健全性チェック**:
    *   システム全体に対して、rootkitスキャナ（`rkhunter`、`chkrootkit`など）や最新のウイルス対策ソフトウェアを使用して、マルウェアやバックドアが仕掛けられていないか広範にスキャンします。
    *   不審なプロセス、ネットワーク接続、異常なファイルやディレクトリ（特に隠しファイルやシステムディレクトリ配下）がないか確認します。
4.  **詳細ログ分析**:
    *   SSHデーモンのログ（`auth.log`、`secure`など）、システムログ、Webサーバーログ、ネットワーク機器のログなど、他の関連するログを攻撃元のIPアドレスや時間帯で横断的に検索し、他の異常な活動や侵害の兆候がないか詳細に分析します。

#### 4.2. 中期対応 (Medium-Term Actions)

1.  **SSHセキュリティ強化**:
    *   **パスワード認証の無効化**: 可能であれば、SSHパスワード認証を無効にし、公開鍵認証のみを許可します。
    *   **rootログインの禁止**: `PermitRootLogin no` を設定し、`root` ユーザーでの直接SSHログインを禁止し、`sudo` を経由したアクセスを強制します。
    *   **SSHポートの変更**: 標準のポート22番を避けて、他のポートに変更します（完全な防御にはなりませんが、自動スキャンによるノイズを減らせます）。
    *   **`MaxAuthTries` の制限**: SSHデーモンの設定で、認証試行回数を少なく制限します。
    *   **Fail2Banなどの導入**: 不正なログイン試行を自動的に検出し、一定回数失敗したIPアドレスを一時的または永続的にブロックするツール（例: Fail2Ban）を導入します。
2.  **強力なパスワードポリシーの徹底**:
    *   すべてのシステムアカウントに対して、複雑性、長さの要件を満たす強力なパスワードポリシーを適用し、定期的なパスワード変更を義務付けます。
    *   多要素認証（MFA）の導入を積極的に検討し、セキュリティを強化します。
3.  **不要なアカウントとサービスの停止**:
    *   使用されていないユーザーアカウントや、外部に公開する必要のないサービスは無効化または削除します。
4.  **OS/ソフトウェアの定期的な更新**:
    *   OS、SSHデーモンを含むすべてのソフトウェアを最新の状態に保ち、既知の脆弱性へのパッチを速やかに適用します。
5.  **ネットワークアクセス制御の強化**:
    *   SSHサービスへのアクセスを、特定の信頼されたIPアドレス範囲に限定するようファイアウォールルールを見直します。可能であれば、VPNを介したアクセスに限定します。

#### 4.3. 長期対応 (Long-Term Actions)

1.  **セキュリティ情報およびイベント管理 (SIEM) の導入/強化**:
    *   SIEMシステムを導入または強化し、SSHログイン成功/失敗、rootユーザーによる特権コマンド実行、ファイル変更などのイベントをリアルタイムで監視し、異常なパターンを自動的に検出し、アラートを発する体制を構築します。
2.  **インシデントレスポンス計画の見直し**:
    *   今回のインシデントを踏まえ、セキュリティインシデント発生時の検出、封じ込め、根絶、復旧、事後分析の手順を含むインシデントレスポンス計画を定期的に見直し、訓練を実施します。
3.  **セキュリティ教育と啓発**:
    *   従業員に対し、安全なパスワード管理、ソーシャルエンジニアリングの脅威、フィッシング攻撃への注意など、セキュリティ意識向上トレーニングを定期的に実施します。
4.  **定期的な脆弱性診断とペネトレーションテスト**:
    *   定期的にシステムの脆弱性診断やペネトレーションテストを実施し、潜在的なセキュリティ上の弱点を事前に特定し、修正します。
