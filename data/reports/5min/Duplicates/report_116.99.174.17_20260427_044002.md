# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 04:41:41

## SOC分析レポート

### 1. 攻撃の概要と目的

このレポートは、攻撃元IPアドレス `116.99.174.17` からのSSHサービスに対する不審な活動に関する分析結果をまとめたものです。提供されたログはCowrieハニーポットのものであり、実際のシステム侵害ではなく、攻撃の試行が記録されたものと理解しています。

**概要**:
攻撃元IP `116.99.174.17` は、日本時間2026年4月27日02:13:34から02:31:53にかけて、当社のSSHサービス（ハニーポット）に対して継続的なログイン試行を行いました。この活動は、多様なユーザー名とパスワードの組み合わせを短時間に何度も試行するもので、典型的なブルートフォース攻撃または辞書攻撃のパターンを示しています。

ログには以下の通り、複数の認証成功が記録されています。

*   `root` / `admin`
*   `root` / `@`
*   `root` / `root123`
*   `root` / `12345`
*   `root` / `abcd1234`
*   `root` / `ipscan`
*   `root` / `alpine`

これらの認証成功後には、`cowrie.direct-tcpip.request` および `cowrie.direct-tcpip.data`/`ja4h` イベントが記録されており、攻撃者がログイン後に何らかのTCP転送（ポートフォワーディング、SCP/SFTPによるファイル転送、あるいはプロキシ利用など）を試みたことを示唆しています。

**目的**:
攻撃者の主な目的は、脆弱な認証情報を持つSSHサーバーへの不正アクセスを確立し、システムを乗っ取ることであると推測されます。不正アクセスに成功した場合、マルウェアの展開、C2（Command and Control）通信の確立、追加の探索、あるいはボットネットの一部として利用することなどを企図している可能性があります。試行されたパスワードのリストには、IoTデバイスのデフォルト認証情報（例: `ubnt`/`ubnt`, `root`/`alpine`）も含まれており、広範な種類のシステムをターゲットにしていることが伺えます。

### 2. 推測される手法・使用ツール

**推測される手法**:

*   **ブルートフォース攻撃 / 辞書攻撃**: 大量のユーザー名とパスワードの組み合わせを自動的に試行していることから、この手法が用いられている可能性が極めて高いです。試行されたユーザー名には `root`, `admin`, `user` など一般的・デフォルトのアカウントが多く、パスワードも `admin`, `1234`, `12345`, `password`, `default` など推測しやすいものが含まれています。
*   **Credential Stuffing**: 広範囲にわたる認証情報の試行から、過去に漏洩した認証情報のリストを試すCredential Stuffing攻撃の可能性も排除できません。
*   **初期アクセス**: 脆弱な認証情報を持つSSHサービスが攻撃者の初期アクセスベクトルとして狙われています。

**推測される使用ツール**:

*   **自動化された攻撃スクリプト/ツール**: 多数の異なるユーザー名とパスワードの組み合わせを短時間に効率よく試行していることから、`Hydra`、`Medusa`、`Nmap`のSSH関連スクリプト (`ssh-brute`)、またはカスタムのブルートフォーススクリプトのような自動化ツールが使用されていると考えられます。
*   **汎用SSHクライアント**: 認証成功後に記録された `direct-tcpip` イベントは、攻撃者がSSHセッション内でポートフォワーディングやファイル転送（SCP/SFTP）を実行しようとしたことを示唆しています。これは、標準的なSSHクライアントツール、あるいは特定のマルウェアに組み込まれた機能によって行われた可能性があります。

### 3. 脅威レベルとその理由

**脅威レベル**: **高 (High)**

**理由**:

1.  **継続的かつ自動化された攻撃**: 単一のIPアドレスから約20分間にわたり、多数の認証試行が繰り返し行われています。これは手動ではなく、自動化されたツールによる組織的・継続的な攻撃であることを示しており、もしこれが実際のシステムであれば、システムリソースの消費やサービス停止（DoS）につながる可能性も考えられます。
2.  **複数の認証成功**: ハニーポット上で複数の異なるユーザー名とパスワードの組み合わせ（特に`root`アカウント）でログインに成功しています。これは、もし実際のシステムであれば、攻撃者が完全なシステムアクセス権限を取得し、任意の操作を実行できたことを意味します。
3.  **認証後の活動 (`direct-tcpip`)**: ログイン成功後には `direct-tcpip` イベントが記録されており、攻撃者が単なるログイン試行に留まらず、システム内部でのさらなる活動（ポートフォワーディング、ファイル転送、コマンド実行など）を積極的に試みていることを示しています。これは、攻撃の意図がより深く、システム乗っ取り後の悪用を強く企図していることを示唆します。
4.  **広範なターゲットへの試行**: 一般的なアカウント名やパスワードだけでなく、IoTデバイスなどで使われるデフォルト認証情報も試されており、特定のターゲットだけでなく、脆弱なSSHサービスを持つ広範なシステムを無差別に狙っていることが分かります。

これらの理由から、当社の実際のシステムが同様の攻撃に晒された場合、重大な侵害につながるリスクが高いと判断し、「高」の脅威レベルと評価します。

### 4. 推奨アクション

この攻撃の深刻度を鑑み、以下の緊急および予防的アクションを推奨します。

**緊急対応**:

*   **攻撃元IPのブロック**: 攻撃元IPアドレス `116.99.174.17` を社内のファイアウォール、IDS/IPS、およびSSHサーバーの設定で即座にブロックし、さらなるアクセスを遮断してください。
*   **全SSHサービスのログ確認**: 組織内のすべてのSSHサービス（本番環境、開発環境、検証環境など）について、同様のブルートフォース攻撃を受けていないか、また不審なログイン成功がないか、直近のアクセスログを詳細に確認してください。
*   **認証情報の緊急監査と強化**: 脆弱な認証情報（デフォルトパスワード、短すぎるパスワード、推測しやすいパスワード）が使用されていないか緊急監査を実施し、すべてのSSHユーザーに対して複雑で長いパスワードの使用を強制する措置を講じてください。特に `root` アカウントのパスワードを直ちに確認・変更し、可能であればパスワード認証を無効化してください。

**予防的対応**:

*   **公開鍵認証の導入**: パスワード認証を無効化し、公開鍵認証方式を必須とすることを強く推奨します。これにより、ブルートフォース攻撃のリスクを大幅に低減できます。
*   **Fail2Banなどの導入**: 認証失敗回数を監視し、一定回数を超えたIPアドレスを自動的にブロックするツール（例: Fail2Ban）をすべてのSSHサーバーに導入してください。
*   **多要素認証 (MFA) の導入**: 特に管理者アカウントなど、重要なシステムへのSSHアクセスには、多要素認証を導入し、セキュリティを強化してください。
*   **SSHポートの変更**: 可能であれば、SSHのデフォルトポート (22) を標準的でないポート番号に変更し、一般的なポートスキャンからの検出を困難にしてください。
*   **アクセス制御の強化**: SSHアクセスを許可するIPアドレスを、管理用途に限定された信頼できるIPアドレスのみに限定する（IPホワイトリスト方式）ことを検討してください。
*   **監査ログの強化と監視**: SSH認証ログ、コマンド実行ログなどを詳細に記録する設定を強化し、SIEM（Security Information and Event Management）システムなどを用いて、不審な活動をリアルタイムで監視・分析する体制を構築してください。
*   **脆弱性スキャンとパッチ適用**: 定期的にシステム脆弱性スキャンを実施し、OSやSSHサーバーソフトウェアの脆弱性を特定し、速やかにパッチを適用してください。
*   **セキュリティ啓発**: ユーザーに対して、強固なパスワードの重要性や、不審な連絡への注意喚起などのセキュリティ啓発を継続的に実施してください。

**情報共有**:
