## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `182.70.243.207` から、当社のSSHハニーポットシステムに対し、約17分間にわたり断続的な攻撃が観測されました。
攻撃は主にSSHブルートフォース攻撃と、ログイン成功後のシステムへの永続的なアクセス確保を目的とした活動で構成されています。

攻撃者は、まず多数のユーザー名とパスワードの組み合わせ（"steam", "tunnel", "cloud", "gitlab"など）でログインを試行し、特に"root"ユーザーに焦点を当てています。
複数回のログイン試行の後、ハニーポット上で"root"ユーザーとして以下のパスワードでログインに成功しています（ログ記録上）。
*   `QAZwsx`
*   `Root1234567@@`
*   `1q2w3e..`
*   `root123123!!`
*   `@a123456`
*   `Root11111111@#`

ログイン成功後、攻撃者は以下の定型コマンドを実行しようとしました。
1.  `cd ~; chattr -ia .ssh; lockr -ia .ssh`: `.ssh` ディレクトリの不変属性を解除し、何らかのロックを解除しようとする試み。`lockr` コマンドはシステムに存在しないため失敗しています。
2.  `cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr">>.ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`: 既存の `.ssh` ディレクトリを削除し、新しいディレクトリを作成した上で、攻撃者のSSH公開鍵（コメント: `mdrfckr`）を `authorized_keys` ファイルに追加しようとしています。これはシステムへの永続的なアクセス経路を確立することが目的です。

SSH公開鍵の追加後、攻撃者は `345gs5662d34` および `3245gs5662d34` といったユーザー名で、SSH鍵認証による再ログインを試みていると推測されますが、ハニーポットの挙動により一部はパスワード認証成功として記録されています。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **SSH ブルートフォース攻撃/辞書攻撃**: 複数の異なるユーザー名とパスワードの組み合わせを短時間に大量に試行しています。これは自動化されたスクリプトやツールによる辞書攻撃またはパスワードリスト攻撃の典型的なパターンです。
    *   **永続化 (Persistence)**: ログイン成功後、SSH公開鍵を `authorized_keys` ファイルに追加することで、パスワード認証なしで将来的にシステムにアクセスできるバックドアを作成しようとしています。これはシステムへの長期的なアクセスを目的とした一般的な手法です。
    *   **既存セキュリティ対策の回避**: `chattr -ia .ssh` コマンドは、ファイルやディレクトリの属性を操作して、変更を防ぐための保護（i属性など）を無効にしようとするものです。

*   **使用ツール**:
    *   **ブルートフォースツール**: Hydra、Nmapの`ssh-brute`スクリプト、またはカスタムスクリプトなど、SSHの認証情報を自動で試行するツールが使用されている可能性が高いです。
    *   **標準的なSSHクライアント**: ログインやコマンド実行のために使用されています。
    *   **シェルスクリプト**: ログイン成功後のコマンドシーケンスが定型であることから、予め用意されたシェルスクリプトまたは自動化されたプログラムによって実行されていると考えられます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
*   **root権限での侵入意図**: 攻撃者はシステム上で最も高い権限を持つ `root` ユーザーの認証情報を執拗に狙い、ハニーポット上では複数回ログインに成功しています。実システムであれば、これは完全にシステムを掌握される事態に直結します。
*   **永続化の試み**: ログイン後、自身のSSH公開鍵を `authorized_keys` に追加しようとする明確な意図が見られます。これは一時的なアクセスだけでなく、恒久的なバックドアを仕掛けることで、いつでも再侵入できる状態を確立しようとするものです。このSSH鍵のコメント「mdrfckr」は、攻撃者の悪意を明確に示唆しています。
*   **自動化された攻撃**: 短期間に多数の認証情報を試行し、ログイン後のコマンドも定型的であることから、攻撃は自動化されたツールやボットネットの一部である可能性が高く、広範な標的を狙っていると考えられます。
*   **頻繁な攻撃**: 約17分間という短い時間で複数回のログイン成功と永続化の試みを繰り返しており、攻撃者の執拗さとスキルレベルを示しています。

### 4. 推奨アクション

この攻撃はハニーポットに対するものであり、実システムへの直接的な被害は発生していませんが、同様の手法が実環境で成功するリスクがあるため、以下の対策を速やかに実施することを推奨します。

1.  **攻撃元IPアドレスのブロック**:
    *   攻撃元IPアドレス `182.70.243.207` を、ファイアウォール、IDS/IPS、またはSSHサーバーのアクセス制御リスト（ACL）でブロックします。

2.  **SSHセキュリティの強化**:
    *   **パスワード認証の無効化**: 可能な限りSSH公開鍵認証を強制し、パスワード認証を無効化します (`PasswordAuthentication no`)。
    *   **rootログインの禁止**: `PermitRootLogin no` を設定し、`root` ユーザーでの直接ログインを禁止します。必要な場合は、一般ユーザーでログイン後、`su` または `sudo` を使用するように運用を徹底します。
    *   **強力なパスワードポリシーの徹底**: パスワード認証を有効にする必要がある場合は、複雑性、長さ、定期的な変更を義務付ける強力なパスワードポリシーを適用します。
    *   **多要素認証 (MFA) の導入**: SSHアクセスにMFAを導入し、認証の強度を高めます。
    *   **レートリミット/ブルートフォース対策**: Fail2Banなどのツールを導入し、短時間での複数回ログイン失敗を検知・自動ブロックする仕組みを導入します。
    *   **SSHポートの変更 (推奨されないが検討の余地あり)**: 標準の22番ポートではなく、別のポートを使用することで、自動化されたスキャン攻撃の一部を回避できる可能性があります（根本的な解決策ではありません）。

3.  **既存システムへの影響評価と監査**:
    *   **認証情報の確認**: 攻撃ログに記録されたログイン成功パスワード（`QAZwsx`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`, `3245gs5662d34`）が、当社の他のシステムやサービスで利用されていないか直ちに確認します。もし利用されている場合は、該当する認証情報を即座に変更し、関連するシステムへの影響がないか調査します。
    *   **`.ssh/authorized_keys` の監査**: 全てのLinux/Unixサーバーにおいて、各ユーザーの `.ssh/authorized_keys` ファイルに、今回の攻撃で用いられた公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`）を含む不審なエントリが追加されていないか確認し、発見した場合は即座に削除します。
    *   **ログ監視の強化**: SSHログ、認証ログ、システムログ（`chattr` などのファイル属性変更ログ、ファイル作成/削除ログ）に対する監視を強化し、不審なアクティビティを迅速に検知できる体制を確立します。

4.  **セキュリティ教育**:
    *   従業員に対し、安全なパスワードの利用、多要素認証の重要性、不審な接続試行への注意喚起など、セキュリティ意識向上のための教育を継続的に実施します。

本件はハニーポットで捕捉されたものでありますが、実環境に起こり得る脅威として認識し、上記の推奨アクションを迅速に実施することで、将来的な攻撃からのシステム保護を強化してください。
