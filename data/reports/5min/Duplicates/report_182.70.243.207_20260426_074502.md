## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `182.70.243.207` から、短時間にわたるSSHサービスへのブルートフォース攻撃が複数回観測されました。攻撃者は主に `root` ユーザーアカウントを標的とし、異なる複数のパスワードを試行して複数回認証に成功しています。認証成功後、攻撃者はシステムへの永続的なアクセス経路を確立するため、SSH公開鍵を設置しようと試みました。

**主な目的**:
*   SSHサービスへの不正ログイン（特にrootアカウントの乗っ取り）
*   `root` ユーザーの `authorized_keys` ファイルに自身の公開鍵を追加し、パスワードなしでの再ログインを可能にするバックドアの設置
*   長期的なシステムへのアクセス権の確保

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃 / 辞書攻撃**: 攻撃者は「steam」のような一般的なユーザー名から、「root」のような特権ユーザーまで、様々なユーザー名とパスワードの組み合わせを試行しています。特に `root` ユーザーに対しては、一般的なパスワードリストに含まれると思われる複数のパスワードで認証に成功しており、自動化されたブルートフォースツールや辞書攻撃が使用されている可能性が高いです。
    *   **永続化 (Persistence)**: 認証に成功した後、攻撃者は以下のコマンドを実行して、自身のSSH公開鍵をターゲットシステムの `root` ユーザーの `~/.ssh/authorized_keys` ファイルに書き込もうとしました。
        ```bash
        cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr">>.ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~
        ```
        この行為により、攻撃者はパスワード認証なしで任意のタイミングでシステムに再アクセスできるようになり、永続的なバックドアを確立しようとしています。また、`chattr -ia .ssh` や `lockr -ia .ssh` といったコマンドも試行されており、`authorized_keys` ファイルへの書き込みを阻害するファイル属性（immutable/append-only）を解除しようとする意図が見受けられます（ただし`lockr`は失敗）。
*   **使用ツール**:
    *   ログから直接的なツール名は特定できませんが、一連の自動化されたログイン試行とコマンド実行のパターンから、Hydra、Medusa、またはカスタムスクリプトなどのSSHブルートフォースツールが利用されていると推測されます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
1.  **特権ユーザーの認証成功**: 攻撃者が `root` ユーザーとして複数回認証に成功しています。これは、もしこれが実際のプロダクション環境であった場合、攻撃者がシステムに対する完全な制御権を獲得し、任意の悪意ある活動（データ窃取、マルウェア配布、システム破壊など）を実行できる状態になったことを意味します。
2.  **永続化の試み**: 攻撃者は認証成功後、SSH公開鍵を設置することで永続的なバックドアを確立しようとしています。これは、仮にパスワードが変更されたとしても、攻撃者がSSH鍵認証を利用して再侵入できる可能性が高く、システムのセキュリティ侵害が継続するリスクを示唆します。
3.  **複数パスワードでの成功**: 攻撃者は短時間のうちに異なる複数のパスワードで `root` ユーザーのログインに成功しており、これは攻撃者の持つ辞書が広範であること、または複数の攻撃ベクトルを試行していることを示唆します。
4.  **ハニーポットへの攻撃**: 今回のログはCowrieハニーポットのものであり、実際のシステムへの直接的な被害は発生していません。しかし、この攻撃者が同様の手法を他の実システムに対しても行っている可能性は非常に高く、現実の環境であれば重大な侵害に発展していたことを明確に示しています。

### 4. 推奨アクション

この攻撃はハニーポットに対するものであり、実際のシステム侵害には至っていませんが、同様の攻撃から実システムを保護するために以下の対策を強く推奨します。

**緊急対応 (Immediate Actions)**:
*   **攻撃元IPのブロック**: 攻撃元IPアドレス `182.70.243.207` からのすべてのSSH接続をファイアウォールやIPS/IDSでブロックリストに追加します。
*   **rootユーザーのSSH直接ログインの無効化**: すべてのLinux/Unixシステムにおいて、`root`ユーザーのSSH直接ログインを無効にし、一般ユーザーでログイン後に`sudo`コマンドを利用する運用に切り替えます。
*   **パスワード認証の無効化と公開鍵認証の強制**: SSHサービスにおいてパスワード認証を無効にし、よりセキュアな公開鍵認証の使用を強制します。
*   **`authorized_keys`ファイルの確認**: 既存のシステム上で、特に`root`ユーザーの`~/.ssh/authorized_keys`ファイルに、今回の攻撃で使用された公開鍵（またはその他の不明な公開鍵）が登録されていないか緊急で確認し、存在する場合は直ちに削除します。
    *   攻撃者が追加しようとした公開鍵: `ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`
*   **認証情報の変更**: ログに記録された、`root`ユーザーでログインに成功したパスワード（例: `QAZwsx`, `3245gs5662d34`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`）が、もし実際のシステムで使用されている場合は、直ちに複雑で推測されにくいパスワードに変更します。

**長期対策 (Long-term Measures)**:
*   **SSHサービス強化**:
    *   SSHサービスのリスニングポートを標準の22番から高ポート番号に変更します（Port Knockingの導入も検討）。
    *   SSHサービスへの接続に対して、レートリミットを適用し、ブルートフォース攻撃の試行回数を制限します。
    *   Fail2BanやCrowdSecなどの侵入防止システムを導入し、複数回のログイン失敗があったIPアドレスを自動的にブロックするように設定します。
*   **パスワードポリシーの強化**:
    *   すべてのユーザーアカウントに対して、複雑で推測困難なパスワードの使用を義務付け、定期的なパスワード変更を促すポリシーを適用します。
    *   共通の、または過去に漏洩したことのあるパスワードの使用を禁止する仕組みを導入します。
*   **多要素認証 (MFA) の導入**:
    *   SSHログインを含む重要な認証プロセスに多要素認証を導入し、セキュリティを強化します。
*   **ログ監視とアラート**:
    *   SSHログインの試行（成功/失敗）およびシステムへのコマンド実行ログを中央ログ管理システムに集約し、異常なログイン試行や不審なコマンド実行に対してリアルタイムでアラートを生成する体制を構築します。
*   **セキュリティ教育と訓練**:
    *   システム管理者およびユーザーに対して、SSHの安全な利用方法（rootログインの無効化、公開鍵認証の利用、強力なパスワードの使用、パスワードの使い回しの禁止など）に関する定期的なセキュリティ教育と訓練を実施します。
*   **脅威インテリジェンスの活用**:
    *   今回の攻撃元IPアドレスや公開鍵の情報（mdrfckr）を脅威インテリジェンスプラットフォームと照合し、既知のマルウェアキャンペーンや攻撃グループに関連する情報がないか確認します。
*   **ハニーポットの継続運用**:
    *   Cowrieのようなハニーポットを継続的に運用し、攻撃の傾向や新たな攻撃手法を早期に検知するための情報収集源として活用します。得られた情報は、セキュリティ対策の継続的な改善に役立てます。
