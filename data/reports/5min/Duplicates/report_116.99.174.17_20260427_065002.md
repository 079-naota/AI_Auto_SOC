# SOC自動分析レポート: 116.99.174.17

## SOCアナリストレポート：SSHブルートフォース攻撃分析

### 1. 攻撃の概要と目的

このレポートは、SSHハニーポット `Cowrie` のログに基づき、攻撃元IPアドレス `116.99.174.17` からの活動を分析したものです。

**攻撃概要:**
攻撃者は、2026年4月27日02時13分から02時31分にかけて、SSHサービスに対して継続的なブルートフォース/辞書攻撃を試行しました。複数の異なるユーザー名とパスワードの組み合わせを短い間隔で大量に試行しており、自動化されたツールが使用されている可能性が高いと推測されます。

**攻撃の目的:**
攻撃者の主な目的は、SSHサービスへの不正ログイン認証情報の取得です。ハニーポットのログでは、以下のユーザー名とパスワードの組み合わせで「ログイン成功」と記録されています。
*   `root` / `admin`
*   `root` / `@`
*   `root` / `root123`
*   `root` / `12345`
*   `root` / `abcd1234`
*   `root` / `ipscan`
*   `root` / `alpine`

ログイン成功後には、`cowrie.direct-tcpip.request` などのイベントが記録されており、これは攻撃者がSSHのポートフォワーディング機能を利用して、ハニーポットを踏み台に内部ネットワークの探索や他のサービスへのアクセスを試みようとしたことを示唆しています。これは、認証情報を取得した後の、さらなるシステム侵害や偵察活動を意図しているものと考えられます。

### 2. 推測される手法・使用ツール

*   **認証情報攻撃 (Credential Stuffing / Brute-Force Attack)**:
    *   多数の一般的なユーザー名（admin, user, root, ubnt, config, squid, support, guest, ftpuser, nikita）とパスワード（admin, user, root123, 1234, 123456, default, password, alpineなど）を組み合わせて試行しており、これは典型的なブルートフォース攻撃や辞書攻撃の手法です。
    *   一部のパスワードは非常に単純（例: `admin`, `12345`, `alpine`）であり、IoTデバイスや古いシステムでよく使われるデフォルトパスワードを狙っている可能性があります。
*   **自動化ツール**: 多数の試行が短時間に集中していることから、Hydra、Medusa、NmapのNSEスクリプト（例: `ssh-brute`）などの自動化されたSSHブルートフォースツールが使用されていると推測されます。
*   **ポートフォワーディング試行**: ログイン成功後に `cowrie.direct-tcpip.*` イベントが記録されていることから、攻撃者はSSH接続を介したTCP転送（ポートフォワーディング）を試みています。これは、侵害したサーバーをプロキシとして利用し、そのサーバーから到達可能な他の内部システムへの接続や、インターネット上のC2サーバーへの通信を確立しようとする意図を示します。

### 3. 脅威レベルとその理由

**脅威レベル：高 (High)**

**理由:**

1.  **継続的なブルートフォース攻撃**: 長時間にわたり多数の認証情報を試行しており、執拗な攻撃意図がうかがえます。
2.  **`root` ユーザーでのログイン成功**: 複数のケースでシステム上の最高権限を持つ`root`ユーザーでのログインが成功（ハニーポット上での擬似的成功）していることは、攻撃者がターゲットシステムへの完全な制御を狙っていることを示します。本番環境であれば、これは即座にシステム全体が侵害されるレベルのインシデントに発展します。
3.  **ログイン後の活動**: ログイン成功後に `direct-tcpip.request` が記録されていることから、攻撃者はハニーポットを足がかりに、さらに内部ネットワークの探索や他の攻撃を展開しようとしています。これは単なる認証情報窃取に留まらない、より深い侵害を企図している証拠です。
4.  **自動化された広範な攻撃**: 様々なユーザー名とパスワードを試す自動化された攻撃であるため、このIPアドレスからの攻撃は特定のシステムだけでなく、広範なSSHサービスを標的としている可能性が高いです。

本件はハニーポットのログであるため、実際のシステムへの直接的な被害は発生していませんが、もしこれが本番環境のシステムであれば、重大な情報漏洩やシステム破壊に繋がりかねない極めて危険な攻撃です。

### 4. 推奨アクション

以下の対策を速やかに実施することを推奨します。

1.  **攻撃元IPアドレスのブロック**:
    *   攻撃元IPアドレス `116.99.174.17` をファイアウォールやIDS/IPSで即座にブロックし、今後のアクセスを遮断します。
2.  **SSHサービス設定の強化**:
    *   **パスワード認証の無効化**: 可能な限り、パスワード認証を無効化し、公開鍵認証のみに移行します。
    *   **MFA (多要素認証) の導入**: 公開鍵認証と併せてMFAを導入し、認証の堅牢性を高めます。
    *   **`root`ユーザーの直接ログイン禁止**: `/etc/ssh/sshd_config` で `PermitRootLogin no` を設定し、`root`ユーザーでの直接ログインを禁止します。通常のユーザーでログイン後、`sudo` を利用して昇格するようにします。
    *   **強力なパスワードポリシーの強制**: パスワードの最小長、複雑性（大文字、小文字、数字、記号の組み合わせ）、有効期限などを厳しく設定します。
    *   **SSHポートの変更**: 22番ポートではなく、一般的なポートではないポート番号を使用することも、自動化されたスキャン攻撃を減少させるのに役立ちます（ただし、根本的な対策ではありません）。
3.  **侵入検知・防止システムの導入・強化**:
    *   **Fail2Banなどの導入**: 認証失敗が頻繁に発生するIPアドレスを自動的にブロックするツール（Fail2Banなど）を導入または設定強化します。
    *   **IDS/IPSによる監視強化**: SSHブルートフォース攻撃のパターンを検知できるよう、IDS/IPSのルールを最新化し、監視体制を強化します。
4.  **ログ監視の強化とアラート設定**:
    *   SSHログイン成功/失敗のログ、特に`root`ユーザーや特権ユーザーのログイン試行に関するログに対して、リアルタイムアラートを設定し、異常なアクセスを早期に検知できるようにします。
5.  **ハニーポットログの詳細分析**:
    *   `direct-tcpip`イベントで具体的にどのポートへの転送を試みたのか、ハニーポットの完全なログがあれば詳細を分析し、攻撃者の具体的な目的や次に狙う可能性のあるサービスを特定します。
6.  **インシデントレスポンスプランの再確認**:
    *   本番環境で同様の事象が発生した場合のインシデントレスポンス手順を再確認し、迅速かつ適切に対応できる体制を維持します。
