# SOC自動分析レポート: 171.104.143.176

## SOC分析レポート

### 1. 攻撃の概要と目的

提供されたCowrieハニーポットのログを分析した結果、攻撃元IP `171.104.143.176` からSSHサービスに対するブルートフォース攻撃と、それに続くシステム乗っ取りを試みる活動が確認されました。

**攻撃の概要**:
攻撃者は複数回のSSHログイン試行を行い、そのうち2回のセッションで異なる認証情報（`root:1234ASDF` および `root:QWEasd123456`）でログインに成功しています。ログイン成功後、攻撃者は以下の悪意のある活動を試みました。

1.  **永続化の試み（バックドアの設置）**: SSH公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) を `root` ユーザーの `~/.ssh/authorized_keys` に追加することで、パスワードなしでの再ログインを可能にするバックドアを設置しようとしました。
2.  **権限昇格/永続化の試み**: `root` ユーザーのパスワードを2回にわたり、異なる強力な文字列（`dTTVY8OaDRX7` および `UfvraTsg18FW`）に変更しようとしました。
3.  **情報収集（偵察）**: システムのCPU情報、メモリ、ディスク使用量、OSバージョン、カーネル情報、実行中のプロセス、cronジョブ、ログインユーザーなど、多岐にわたるシステム情報を収集するコマンドを多数実行しました。
4.  **競合排除と痕跡消去の試み**: `/tmp` ディレクトリ内のシェルスクリプト（`secure.sh`, `auth.sh`）の削除とそれらのプロセス終了、`/etc/hosts.deny` のクリア、`sleep` プロセスの終了など、他のマルウェアや競合する攻撃者の活動を排除し、自身の活動を隠蔽しようとする動きが見られました。また、`chattr` および `lockr` コマンドを使用して `.ssh` ディレクトリの属性を変更し、バックドアの永続化を図ったと考えられますが、これらのコマンドはハニーポット環境では失敗しています。

**攻撃の目的**:
本攻撃の主な目的は、SSH経由で不正にシステムへ侵入し、永続的なアクセス経路（バックドア）を確立した上で、システムを完全に制御下に置くことであると推測されます。情報収集は、その後の攻撃フェーズ（例: さらなるマルウェアの展開、リソースの悪用、内部ネットワークへの横展開など）の準備と判断できます。

### 2. 推測される手法・使用ツール

**推測される手法**:

*   **ブルートフォース/辞書攻撃**: SSHサービスに対して、一般的なユーザー名とパスワードの組み合わせを試行し、正規の認証情報を特定しようとしました。ログから`deploy:admin`での失敗、`root:1234ASDF`および`root:QWEasd123456`での成功が確認できます。
*   **初期アクセス**: 脆弱な認証情報を持つSSHサービスへのログイン。
*   **永続化 (Persistence)**:
    *   `authorized_keys`ファイルへの公開鍵登録によるSSHバックドアの設置。
    *   `root`ユーザーのパスワード変更によるアカウント乗っ取り。
*   **偵察 (Discovery / Reconnaissance)**:
    *   `cat /proc/cpuinfo`, `free -m`, `df -h`, `uname -a`, `lscpu`, `crontab -l`, `w`, `top` などの標準的なLinuxコマンドを多用し、システムの環境や状態を詳細に把握しようとしました。
*   **防御回避 (Defense Evasion)**:
    *   `rm -rf /tmp/secure.sh; rm -rf /tmp/auth.sh; pkill -9 secure.sh; pkill -9 auth.sh; echo > /etc/hosts.deny; pkill -9 sleep;` などのコマンドは、既存のマルウェアやセキュリティ対策を無効化し、攻撃者自身の活動が妨げられないようにする目的があると見られます。
    *   `chattr`コマンドの使用は、ファイル属性を変更して`.ssh`ディレクトリの削除を防ぐ意図があった可能性があります。
*   **システムへの影響 (Impact)**:
    *   ハニーポットのログであるため、実際のシステムへの直接的な被害はありません。しかし、実際のシステムでこれらの攻撃が成功した場合、完全なシステム乗っ取りにつながり、深刻な情報漏洩、マルウェア感染、リソースの悪用（例: マイニング、DDoS攻撃の踏み台化）などの被害が発生する可能性がありました。

**使用ツール**:

*   特定のマルウェアや高度なツール名は明示されていませんが、一般的なSSHクライアント（OpenSSHクライアントなど）と標準的なLinuxコマンド（`cd`, `rm`, `mkdir`, `echo`, `chmod`, `chpasswd`, `pkill`, `cat`, `grep`, `awk`, `head`, `wc`, `free`, `crontab`, `w`, `uname`, `top`, `whoami`, `lscpu`, `df`）を組み合わせて実行されています。
*   攻撃の自動性やコマンドのパターンから、事前に準備されたシェルスクリプトや自動化ツールが利用されている可能性が高いです。特に複数のログイン試行と、ログイン後の定型的な情報収集・永続化コマンドの実行は、ボットネットの一部として機能していることを示唆しています。
*   `lockr`というコマンドが実行されていますが、これは一般的なLinuxコマンドではないため、攻撃者が特定のカスタムツールを使用しているか、または誤って入力した可能性があります。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

1.  **認証突破の成功**: 攻撃者が`root`アカウントで2回もログインに成功している点は非常に深刻です。これは、システムが脆弱な認証情報を使用している、または認証情報が既に漏洩している可能性を示唆しています。`root`権限はシステムへの完全な制御を意味します。
2.  **永続化の明確な意図**: SSH公開鍵の登録と`root`パスワードの変更は、攻撃者が一度侵入したシステムへの恒久的なアクセス手段を確保しようとする明確な意図を示しています。これにより、システムの完全な乗っ取りと持続的な悪用が可能となります。
3.  **広範な偵察活動**: CPU、メモリ、ディスク、OS、カーネル、プロセス、cronジョブなどの詳細なシステム情報の収集は、攻撃者がシステムの環境を徹底的に分析し、次の段階の攻撃（マルウェアの展開、脆弱性の悪用、内部ネットワークへの横展開など）の準備をしていることを示します。
4.  **競合排除の試み**: 他のマルウェアや攻撃者の活動を排除しようとする行動は、攻撃者がシステムを排他的に制御しようとしていることを示しており、その悪意の度合いが高いことを裏付けています。
5.  **自動化された攻撃の可能性**: ログイン試行のパターンや、ログイン後の定型的なコマンド実行から、これは人間による手動操作ではなく、自動化されたスクリプトやボットネットの一部である可能性が高いです。このような自動化された攻撃は、迅速かつ大規模に拡散する可能性があります。
6.  **ハニーポットログの性質**: これらはハニーポットのログであるため、実際のシステムへの被害は発生していませんが、この攻撃手法が本番環境で実行された場合、システムが完全に侵害され、機密情報の漏洩、サービス停止、他のシステムへの攻撃基盤となるなど、極めて重大な影響を引き起こす危険性があります。

### 4. 推奨アクション

この攻撃活動は高い脅威レベルと評価されるため、以下の緊急および長期的なアクションを推奨します。

**緊急アクション (Immediate Actions)**:

1.  **攻撃元IPのブロック**: 攻撃元IPアドレス `171.104.143.176` を即座にファイアウォール、IDS/IPS、WAFなどでブロックリストに追加し、今後のアクセスを遮断します。
2.  **SSH認証情報の確認と変更**:
    *   ログに記録されたログイン成功パスワード (`1234ASDF`, `QWEasd123456`) および変更を試みられたパスワード (`dTTVY8OaDRX7`, `UfvraTsg18FW`) が、組織内の他のシステムやサービスで利用されていないか、緊急で全システムを調査します。
    *   もし利用されている場合は、該当するすべてのアカウントのパスワードを直ちに、かつ強力なものに変更します。
    *   特に`root`アカウントのパスワードは直ちに変更し、二度と同じパスワードが使われないようにパスワード履歴を確認します。
3.  **SSH設定の強化**:
    *   **パスワード認証の無効化**: 可能な限りSSHパスワード認証を無効にし、公開鍵認証のみに限定します。
    *   **root直接ログインの禁止**: `PermitRootLogin no` を設定し、`root`ユーザーでの直接SSHログインを禁止します。代わりに一般ユーザーでログイン後、`sudo`を使用して必要な権限を取得するようにします。
    *   **不正ログイン対策**: Fail2banやSSHGuardなどのツールを導入または設定を強化し、短時間での複数回ログイン失敗を検知・ブロックするよう設定します。
    *   **ポート変更**: 標準のSSHポート (22番) を変更し、一般的なスキャンからの検出率を下げます（これはセキュリティの強化というよりは、攻撃対象を絞る目的です）。
4.  **SSH公開鍵の確認**: 組織内のすべてのシステムで、`~/.ssh/authorized_keys` ファイルに攻撃者が設置しようとした公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) が登録されていないか確認し、もし存在する場合は直ちに削除します。

**調査アクション (Investigation Actions)**:

1.  **攻撃元IPの追加調査**: 攻撃元IPアドレス `171.104.143.176` のレピュテーション情報（VirusTotal, AbuseIPDB, Talos Intelligenceなど）を調査し、既知の脅威アクターやマルウェアキャンペーンとの関連性を確認します。
2.  **ログの広範な分析**: 組織内の他のシステムログ（SSH認証ログ、システムログ、アプリケーションログなど）を過去に遡って分析し、同様のブルートフォース攻撃、ログイン試行、または不審なコマンド実行の痕跡がないか確認します。
3.  **脆弱性スキャン**: 外部に公開されているSSHサービスに対し、脆弱性スキャンを実施し、既知の脆弱性がないか確認します。
4.  **ハニーポットログの改善**: Cowrieハニーポットのログ設定を見直し、より詳細な情報（例: 実行コマンドの出力、ファイルのダウンロード/アップロードの内容など）が記録されるように調整を検討します。

**長期対策 (Long-term Measures)**:

1.  **強力なパスワードポリシーの導入**: 複雑性、長さ、有効期限など、強力なパスワードポリシーを組織全体で強制します。
2.  **多要素認証 (MFA) の導入**: SSHアクセスを含む重要なサービスには多要素認証の導入を検討し、認証の堅牢性を高めます。
3.  **定期的な脆弱性管理**: OSやミドルウェアのセキュリティパッチを常に最新の状態に保ち、定期的な脆弱性スキャンとペネトレーションテストを実施します。
4.  **ネットワーク分離とアクセス制御**: 公開されているSSHポートへのアクセスを最小限に制限し、特定のIPアドレスからのアクセスのみを許可するか、VPN経由でのアクセスを強制するなどのネットワーク分離戦略を検討します。
5.  **セキュリティ監視体制の強化**: SIEM (Security Information and Event Management) などのログ管理・監視システムを導入し、不正なログイン試行、不審なコマンド実行、異常なネットワーク通信などをリアルタイムで検知し、アラートを発報できる体制を構築します。
6.  **インシデント対応計画の見直し**: 今回の攻撃パターンを考慮に入れ、インシデント発生時の対応計画を定期的に見直し、従業員へのセキュリティ教育を強化します。
