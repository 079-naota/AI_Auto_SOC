## SOCアナリストレポート

### 1. 攻撃の概要と目的

このレポートは、攻撃元IPアドレス `103.172.205.139` から観測されたSSHサービスへの不正アクセス試行および潜在的な侵入後の活動について分析したものです。

攻撃者はSSHサービスに対し、複数のユーザー名とパスワードの組み合わせを用いたブルートフォース/辞書攻撃を繰り返し実施しました。初期の試行ではログインに失敗していますが、最終的に複数の`root`ユーザーの認証情報（`root`/`abcd1234.`, `root`/`3245gs5662d34`, `root`/`Password.123`, `root`/`MoeClub.org`, `root`/`a12348765`, `root`/`abc@123456`など）を特定し、ログインに成功しています。

ログイン成功後、攻撃者はシステムへの永続的なアクセス経路を確立することを目的とし、以下の悪意のあるコマンドを実行しようとしました。

*   `cd ~; chattr -ia .ssh; lockr -ia .ssh`
*   `cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK_rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr">>.ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`

これらのコマンドは、SSH公開鍵認証によるバックドアを設置し、攻撃者がパスワードなしで将来的にシステムに再接続できるようにすることを目的としています。特に、提供されたSSH公開鍵にはコメントとして`mdrfckr`が含まれており、これは特定の攻撃キャンペーンやマルウェアに関連付けられる可能性があります。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **SSHブルートフォース/辞書攻撃**: 多数のユーザー名とパスワードの組み合わせを自動的に試行し、有効な認証情報を特定しています。ターゲットユーザーとしては、`root`の他に`user7`, `deployer`, `frappe`, `ubuntu`, `user`, `mary`, `admin`, `zhaoxj`などが試行されました。
    *   **永続化 (Persistence)**: ログイン成功後、`~/.ssh/authorized_keys`ファイルに攻撃者自身のSSH公開鍵を追加することで、将来的な不正アクセスのためのバックドアを設置しようとしました。
    *   **防御回避 (Defense Evasion)**: `chattr -ia .ssh` コマンドは、ファイルの不可変属性や追加のみ属性を解除しようとするもので、セキュリティ設定を無効化し、自身の操作を可能にしようとする意図が見られます。`lockr`コマンドは標準的なLinuxコマンドではないため、特定のマルウェアの一部であるか、誤入力の可能性があります。

*   **使用ツール**:
    *   **自動化されたSSHクライアント**: 大量の認証情報を高速に試行していることから、Hydra、Medusa、NmapのNSEスクリプト、またはカスタムスクリプトのようなブルートフォースツールが使用されている可能性が高いです。
    *   **特定マルウェア/ボットネット**: `mdrfckr`というSSH鍵のコメントや、不明な`lockr`コマンドの使用試行から、特定のマルウェアやボットネットの一部として、自動化された攻撃が行われている可能性も考慮されます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

*   **認証情報の特定**: Honeypotログではありますが、攻撃者が複数の`root`ユーザーの有効な認証情報（パスワード）を特定できたことが確認されています。これは、もし本番環境であった場合、攻撃者が管理者権限でシステムに侵入したことを意味します。
*   **永続化の試み**: ログイン成功後、攻撃者は直ちに`authorized_keys`ファイルに自身のSSH公開鍵を追加し、永続的なアクセス経路を確立しようとしています。これは、単なる偵察ではなく、明確な侵入と制御維持の意図を示しています。
*   **管理者権限の標的**: 最も特権の高い`root`アカウントを標的とし、そのパスワードを複数特定していることは、システム全体への深刻な影響をもたらす可能性があります。
*   **自動化された攻撃**: 攻撃は断続的かつ多数のクレデンシャルを試しており、自動化されたスクリプトまたはツールによるものと推測されます。このような攻撃は、発見が遅れると広範囲のシステムに影響を及ぼす可能性があります。

### 4. 推奨アクション

このログはHoneypot上での挙動ですが、本番環境で同様の状況が発生した場合に備え、以下の緊急対応および長期対策を推奨します。

#### 4.1. 緊急対応 (Immediate Actions)

1.  **攻撃元IPアドレスのブロック**:
    *   攻撃元IPアドレス `103.172.205.139` からのSSH接続を、ファイアウォール、IPS、またはその他のネットワークセキュリティデバイスで直ちにブロックリストに追加してください。
2.  **認証情報の確認と変更**:
    *   ログに記録されたログイン成功パスワード (`abcd1234.`, `3245gs5662d34`, `Password.123`, `MoeClub.org`, `a12348765`, `abc@123456`) が、組織内の本番システムで`root`ユーザーまたは他の重要なアカウントに使用されていないか、緊急で確認してください。
    *   もし使用されている場合は、直ちに強力でユニークなパスワードに変更してください。
3.  **SSH公開鍵の調査**:
    *   攻撃者が設置しようとしたSSH公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK_rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) のフィンガープリント（`SHA256:fM4V2X...` など）を算出し、組織内のすべてのサーバーの`authorized_keys`ファイルに存在しないか検索してください。
    *   もし存在した場合は、該当する鍵を削除し、そのサーバーが侵害された可能性を前提に詳細なフォレンジック調査と復旧プロセスを開始してください。

#### 4.2. 長期対策 (Long-term Measures)

1.  **SSHパスワード認証の無効化**:
    *   可能であれば、すべての本番SSHサーバーでパスワード認証を無効にし、より安全な鍵認証（公開鍵認証）のみに制限することを強く推奨します。
2.  **強力なパスワードポリシーの適用**:
    *   全てのユーザーアカウントに対し、複雑性、長さ、定期的な変更を強制する強力なパスワードポリシーを導入・強化してください。
3.  **多要素認証 (MFA) の導入**:
    *   重要なシステムや管理者アカウントへのSSHアクセスに、多要素認証（MFA）を義務付けることを検討してください。
4.  **IPアドレス制限 (ホワイトリスト)**:
    *   SSHアクセスを、管理者が接続する特定の信頼できるIPアドレス範囲に限定するよう、ファイアウォールルールやSSHデーモン設定 (`AllowUsers`, `AllowGroups`, `sshd_config`の`ListenAddress`) を見直してください。
5.  **ブルートフォース対策の強化**:
    *   `fail2ban`などのツールを導入し、ログイン失敗回数が閾値を超えた場合に自動的にIPアドレスをブロックする設定を強化してください。
6.  **監視とアラートの強化**:
    *   SIEM（Security Information and Event Management）システムやIDS/IPSに、SSHブルートフォース攻撃や不正な`authorized_keys`ファイルへの書き込み、`chattr`などの特権コマンドの実行を検知するルールを追加し、アラートを発報するように設定してください。
7.  **セキュリティ監査と脆弱性診断**:
    *   定期的にシステムのセキュリティ監査と脆弱性診断を実施し、未パッチの脆弱性や設定ミスがないか確認してください。

このレポートは、Honeypotログに基づいた分析です。本番環境での同様の攻撃は、重大なセキュリティインシデントに発展する可能性が高いため、上記の推奨アクションを速やかに実行することを推奨します。
