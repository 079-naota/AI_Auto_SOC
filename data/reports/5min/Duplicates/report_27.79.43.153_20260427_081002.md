## SOCアナリストレポート：SSHブルートフォース攻撃分析

### 1. 攻撃の概要と目的

攻撃元IPアドレス `27.79.43.153` から、当社のハニーポットに対して継続的なSSHログイン試行が行われました。これは、一般的なユーザー名とパスワードの組み合わせを試すブルートフォース攻撃（辞書攻撃）と判断されます。

ログからは、以下のユーザー名とパスワードの組み合わせでログインが試行されたことが確認できます。

**ログイン失敗例:**
*   `sshd` / `sshd`
*   `oracle` / `oracle`
*   `btf` / `321start`
*   `guest1` / `guest1`
*   `plex` / `plex`
*   `admin` / `abc123`
*   `psybnc` / `psybnc`
*   `helpdesk` / `helpdesk`
*   `test` / `admin`
*   `admin` / `1111`
*   `software` / `software`
*   `anton` / `anton`

**ログイン成功例（全てユーザー名 `root`）:**
*   `root` / `abcdefg`
*   `root` / `admin123`
*   `root` / `temp`
*   `root` / `libreelec`
*   `root` / `welc0me`
*   `root` / `explorer`

攻撃者は、特権アカウントである `root` でのログイン成功後、`cowrie.direct-tcpip.request` イベントを発生させています。これは、SSHのポートフォワーディング機能を利用して、ハニーポットを足がかりに他のシステムへの接続やネットワークスキャンを試みようとした可能性を示唆します。

攻撃の主な目的は、SSHアクセス権を不正に取得し、さらにそのアクセスを利用して内部ネットワークへの侵入、情報窃取、あるいはマルウェアの展開など、さらなる悪意のある活動を行うことであると推測されます。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース攻撃（辞書攻撃）:** 大量のユーザー名とパスワードの組み合わせを試行することで、認証情報を突破しようとしています。特に、`root` ユーザーと一般的によく使われるパスワードの組み合わせを重点的に試しています。
    *   **ポートフォワーディング/プロキシ利用の試み:** ログイン成功後、`direct-tcpip.request` が発生していることから、SSHを介したポートフォワーディングやプロキシとして利用し、内部ネットワークに対する偵察活動や他のシステムへのアクセスを試みようとした可能性が高いです。

*   **使用ツール:**
    *   複数のログイン試行が短時間で繰り返されていることから、攻撃は手動ではなく、自動化されたスクリプトやボットネットの一部であるSSHクライアントツールが使用されていると推測されます。特定のツールを特定できる直接的なログ情報はありませんが、HydraやMedusaといった一般的なパスワードクラッキングツール、またはカスタムスクリプトが考えられます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **複数回のログイン成功:** ハニーポットへの攻撃であったため実害はありませんが、実際に運用中のシステムであった場合、攻撃者は複数回にわたってSSHアクセスに成功していました。これはシステムの深刻な侵害を意味します。
*   **特権アカウント（`root`）の標的化:** 攻撃者はシステムの最も強力な権限を持つ `root` アカウントを狙い、そのパスワードを複数見破っています。`root` アクセスはシステム全体に対する完全な制御を可能にするため、非常に危険です。
*   **自動化された継続的な攻撃:** 短時間で多数のログイン試行を行っており、自動化された攻撃であることから、今後も同様の攻撃が継続する可能性が高いです。
*   **追加の活動の試み:** ログイン成功後に `direct-tcpip.request` が発生していることから、単にログインするだけでなく、そのアクセスを足がかりとして、さらなる悪意のある活動（例: 内部偵察、横展開、データ窃取など）を計画していた可能性が高いです。

### 4. 推奨アクション

この攻撃を受けて、以下の緊急対応および恒久対策を推奨します。

**A. 緊急対応**

1.  **攻撃元IPのブロック:** 攻撃元IPアドレス `27.79.43.153` からのすべての接続（特にSSHポート）を、ファイアウォールやIPS/IDSにて直ちにブロックリストに追加してください。
2.  **既存の監視アラートの確認:** SSHログイン失敗回数、特に `root` ユーザーへのログイン試行に対する既存のアラート設定が適切に機能しているか確認し、必要に応じてしきい値を調整してください。
3.  **情報共有:** 攻撃元IPアドレスおよび使用されたクレデンシャル情報を、関係するセキュリティチームや脅威インテリジェンスプラットフォームと共有し、将来の脅威への備えとしてください。

**B. 恒久対策**

1.  **SSH認証の強化:**
    *   **公開鍵認証の導入と強制:** パスワード認証のみではなく、より安全な公開鍵認証を必須とし、パスワード認証を無効化することを強く推奨します。
    *   **パスワードポリシーの強化:** 推測されにくい、複雑で長いパスワードの使用を強制するポリシーを導入してください。今回成功したパスワード (`abcdefg`, `admin123` など) をブラックリスト化し、使用できないように設定することも検討してください。
    *   **多要素認証 (MFA) の導入:** 可能であれば、SSHログインにMFAを導入し、セキュリティを大幅に向上させてください。
2.  **SSH設定のセキュリティ強化:**
    *   **`root` ログインの禁止:** `PermitRootLogin no` を設定し、`root` ユーザーによる直接ログインを禁止してください。特権操作は通常のユーザーでログイン後、`sudo` コマンドを使用するように徹底してください。
    *   **SSHポートの変更:** 標準ポート22番以外のポートを使用するよう変更することで、自動化されたスキャンからの露出を減らすことができます（ただし、これはセキュリティバイオブスキュリティであり、根本的な対策ではありません）。
    *   **レートリミットとアカウントロックアウト:** ログイン失敗回数に応じて一時的にアカウントをロックしたり、特定IPからの接続を制限したりする仕組み（例: `fail2ban`）を導入してください。
3.  **ログ監視とアラートの強化:**
    *   SSHログイン成功/失敗ログの監視を強化し、不審なログイン試行（特に特定のIPからの連続した試行や特権アカウントへの試行）に対しては即座にアラートが上がるように設定してください。
    *   ログイン成功後の異常な挙動（例: ポートフォワーディングの試み、未知のコマンド実行、異常なファイルアクセスなど）を検知するルールを導入してください。
4.  **システムアカウントの監査:**
    *   今回の攻撃で試されたユーザー名とパスワードの組み合わせ（特にログインに成功した `root` のパスワード）が、当社の他のシステムやサービスで利用されていないか、緊急で監査を実施してください。

以上の対策を実施することで、同様のSSHブルートフォース攻撃に対する耐性を向上させることができます。
