# SOC自動分析レポート: 27.79.5.166

**生成日時:** 2026-04-27 06:28:06

---

## セキュリティインシデント分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `27.79.5.166` から、当組織のCowrieハニーポット（SSHサービス）に対し、ブルートフォース攻撃および辞書攻撃が試行されました。

攻撃者は短時間で多数の異なるユーザー名とパスワードの組み合わせを試しており、特に一般的な管理者アカウント（`root`, `admin`）をターゲットにしています。ログによると、ハニーポット上では以下の2回のログイン成功が記録されています。

*   `root` / `password` (02:33:25Z)
*   `root` / `toor` (02:44:15Z)

ログイン成功後、攻撃者はSSHの `direct-tcpip` 機能（ポートフォワーディング）を介した追加の活動を試みています。これは、認証成功後に内部ネットワークへの偵察、他のサービスへのアクセス、またはコマンド＆コントロール（C2）チャネルの確立を意図している可能性が高いです。

攻撃の主な目的は、SSHサービスの認証情報を窃取し、それを通じてシステムへの不正アクセスを確立することにあります。その後の活動としては、情報窃取、マルウェアの配布、システムのリソース悪用（例: ボットネットへの組み込み、クリプトマイニング）、または他のシステムへの横展開などが考えられます。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース攻撃 / 辞書攻撃:** 大量のユーザー名とパスワードの組み合わせを試行していることから、自動化されたパスワード推測攻撃が実行されたと推測されます。試行されたパスワードの中には `password`, `12345678` のような脆弱なものや、ユーザー名と同じパスワード（`rebecca:rebecca`, `bin:bin`など）が多く含まれており、一般的な辞書攻撃のパターンに合致します。
    *   **ポートフォワーディング（Direct-TCP/IP）の悪用:** ログイン成功後に `direct-tcpip.request` が記録されていることから、攻撃者はSSHトンネルを確立し、ハニーポットの背後にあると想定される内部ネットワークや他のサービスへのアクセスを試みたと推測されます。

*   **使用ツール:**
    上記のような高速かつ多様なログイン試行および成功後の自動的な活動から、攻撃には自動化されたツールが使用されたと強く推測されます。具体的なツールのフィンガープリントはログに明示されていませんが、一般的には以下のようなツールが考えられます。
    *   **パスワードクラッキングツール:** Hydra, Medusa, Nmap (Nmap Scripting Engine), Metasploitなどのツールや、Pythonスクリプトなどによるカスタムツール。
    *   **SSHクライアント:** 成功後の `direct-tcpip` の試行は、一般的なSSHクライアントまたはそれに準ずる機能を持つツールによって行われた可能性があります。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **自動化された、執拗な攻撃:** 短時間で多くの異なる認証情報を試行しており、攻撃が自動化されていることを明確に示しています。これは単なる偶発的な試行ではなく、システム侵害を目的とした意図的な攻撃です。
*   **特権アカウントへの集中:** `root` や `admin` といった特権ユーザーアカウントを重点的に狙っており、これらのアカウントが侵害された場合の潜在的な被害は甚大です。
*   **認証成功後の追加活動:** ログイン成功後に `direct-tcpip` を試みていることは、攻撃者が単なる認証情報の確認だけでなく、システム内部への侵入、ポートスキャン、情報窃取、またはC2通信の確立など、より広範な悪意ある活動を意図していることを示唆しています。
*   **潜在的なシステム侵害リスク:** 今回の攻撃はハニーポットで検知されたため、実際のシステムへの直接的な被害は発生していません。しかし、もし実際のSSHサービスが同様の脆弱な認証情報を使用していた場合、容易にシステムが侵害され、深刻なセキュリティインシデントに発展していた可能性が高いです。このログは、実際のシステムが同様の脅威に晒されていることを示す警告と捉えるべきです。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPのブロック:** ファイアウォール、IPS/IDS、またはSSHアクセス制御リスト（ACL）にて、攻撃元IPアドレス `27.79.5.166` を直ちにブロックしてください。
2.  **SSHログの緊急監査:** インターネットに公開されている全てのSSHサービスについて、過去のログインログを緊急で確認し、攻撃元IP `27.79.5.166` からの不正なアクセス、特に `root:password` や `root:toor` などの脆弱な認証情報でのログイン成功がないか徹底的に調査してください。
3.  **脆弱な認証情報の変更:** もし `root:password` や `root:toor`、またはログに記録されたその他の脆弱なユーザー名/パスワードの組み合わせが実際のシステムで使用されている場合、直ちに強力でユニークなパスワードに変更してください。

#### 長期的な対策 (Long-term Measures):

1.  **認証強化:**
    *   **公開鍵認証の導入:** パスワード認証を無効化し、公開鍵認証に完全に移行することを強く推奨します。これにより、ブルートフォース攻撃のリスクを大幅に軽減できます。
    *   **多要素認証 (MFA) の導入:** 可能であれば、SSHログインにMFAを導入し、セキュリティを一層強化してください。
    *   **パスワードポリシーの強化:** パスワード認証を維持する場合、定期的なパスワード変更、複雑性要件（大文字、小文字、数字、記号の組み合わせ、十分な長さ）、および履歴制限を義務付ける強力なパスワードポリシーを設定してください。
2.  **アクセス制御の強化:**
    *   **IPアドレス制限:** SSHへのアクセスを、業務上必要とされる特定の信頼できるIPアドレス範囲（オフィスIPなど）に限定してください。
    *   **ポート変更:** 標準のSSHポート22番を非標準ポートに変更することで、一般的な自動スキャンツールによる攻撃の対象から外れる可能性を高めます（ただし、これはセキュリティバイオブスキュリティであり、根本的な対策ではありません）。
    *   **特権ユーザーの直接ログイン禁止:** `PermitRootLogin no` を設定し、`root` ユーザーのSSH直接ログインを禁止してください。必要に応じて一般ユーザーでログイン後、`sudo` を利用する運用に切り替えてください。
    *   **不要なユーザーアカウントの削除/無効化:** 使用されていないユーザーアカウントは削除または無効化し、攻撃対象を減らしてください。
3.  **監視とアラートの強化:**
    *   **ログインログの継続的な監視:** SSHログイン試行ログをSIEM（Security Information and Event Management）システム等で集中管理し、短時間での多数のログイン失敗や、特定のユーザー名への集中攻撃、非通常時間帯のアクセスなどの異常を検知するアラートを設定してください。
    *   **IPS/IDSの活用:** ブルートフォース攻撃やその他の不正なSSHアクティビティを自動で検知し、ブロックできるIDS/IPSソリューションを導入・最適化してください。
4.  **システムとソフトウェアの管理:**
    *   **パッチ管理:** OSやSSHサーバソフトウェアを含む全てのシステムコンポーネントを常に最新の状態に保ち、既知の脆弱性を悪用されるリスクを低減してください。
    *   **脆弱性診断:** 定期的に外部からの脆弱性診断を実施し、潜在的なセキュリティホールを特定して対処してください。
5.  **脅威インテリジェンスの活用:**
    *   今回検知された攻撃元IPアドレスを、社内の脅威インテリジェンスデータベースや共有プラットフォームに追加し、他のシステムでの検知やブロックに役立ててください。
