## SOCレポート：SSHブルートフォースおよびバックドア設置試行の分析

**レポートID:** SOC-20260427-001
**作成日時:** 2026-04-27 04:00:00 JST

### 1. 攻撃の概要と目的

攻撃元IP `43.155.21.198` から、当組織のSSHサービスに対する不正アクセス試行が複数回確認されました。この攻撃は、`cowrie` ハニーポットによって検知・記録されたものです。

攻撃者はまず、`root` ユーザーに対するブルートフォース攻撃（または辞書攻撃）を試み、複数の弱いパスワードを用いてログインに成功しています。ログイン成功後、攻撃者は以下の2段階の行動を試みています。

1.  既存のSSH認証設定（`.ssh` ディレクトリ）への変更を試みるコマンド（`chattr`, `lockr`）を実行。
2.  攻撃者自身のSSH公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAA... mdrfckr`）を `root` ユーザーの `authorized_keys` ファイルに追記し、持続的なアクセス経路（バックドア）を確立しようと試みました。

この一連の行動から、攻撃の主な目的は、SSH経由での初期アクセス権限の獲得、およびその後の永続的なアクセス経路の確保（持続性）であると推測されます。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース/辞書攻撃**: 標準的なSSHポート（TCP/22）に対して、`root` ユーザーに対する複数のパスワード試行が短時間のうちに繰り返し行われました。ログからは以下の認証情報の成功が確認されています:
        *   `root:6yhnji90-`
        *   `root:Pan5202614`
        *   `root:zhaojia`
        *   `root:P@$$w0rd12`
        *   `root:Login!@456`
        *   `root:mail@1234`
        *   `root:3245gs5662d34` (複数回)
    *   **永続化 (Persistence)**: ログイン成功後、攻撃者は以下のコマンドを実行し、自身の公開鍵をシステムに追加することで、パスワードなしでの再ログインを可能にしようとしました。
        *   `cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa [攻撃者の公開鍵] mdrfckr">>.ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`
        *   このコマンドは、既存のSSH設定を削除し、攻撃者の公開鍵のみを登録することで、自身のアクセスを確実にするものです。また、`chattr -ia .ssh` や `lockr -ia .ssh` といったコマンドも試みており、ファイル属性を変更することで検出を困難にする、あるいは既存の防御メカニズムを迂回しようとした可能性があります（ハニーポット上では失敗）。
*   **使用ツール**:
    *   攻撃の短時間での繰り返し、および定型的なコマンド実行から、自動化されたスクリプトまたはSSHブルートフォースツール（例：Hydra、Nmapの`ssh-brute`スクリプトなど）が使用された可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**
1.  **最高権限アカウントへの攻撃**: 攻撃者はシステム上で最も強力な権限を持つ `root` ユーザーの認証情報を標的としています。`root` ユーザーの侵害は、システム全体の制御権喪失に直結します。
2.  **永続的アクセス経路の確立試行**: ログイン成功後にSSH公開鍵バックドアを設置しようとする行動は、一度侵入に成功すると、パスワード変更後も攻撃者がシステムにアクセスし続けることを可能にするため、非常に危険です。
3.  **既知の弱いパスワードの利用**: ログに記録された成功パスワードの多くは、一般的な単語や数字の組み合わせ、または過去の漏洩パスワードリストに含まれるような弱いものです。これは、当組織内に同様の弱いパスワードを使用しているシステムが存在する場合、容易に侵害されるリスクがあることを示唆します。
4.  **自動化された広範な攻撃**: 短期間に異なるパスワードで繰り返しログインを試み、成功時に一貫したコマンドを実行していることから、この攻撃は特定のターゲットを狙ったものではなく、インターネット上の脆弱なSSHサービスを網羅的に探索する自動化されたキャンペーンの一部である可能性が高いです。
5.  **ハニーポットで検知された事実**: 今回の事象はハニーポット `cowrie` で記録されたため、実際のシステムへの侵入は免れましたが、この攻撃手法と成功した認証情報は現実の脅威であり、実環境であれば重大なセキュリティインシデントにつながった可能性が極めて高いです。

### 4. 推奨アクション

#### 4.1. 緊急対応 (Immediate Actions)

1.  **攻撃元IPアドレスのブロック**: 攻撃元IPアドレス `43.155.21.198` からのSSHポート (TCP/22) へのアクセスを、ファイアウォールまたはエッジルーター、IPS/IDS等で即座にブロックしてください。
2.  **認証情報の確認とリセット**: ログに記載された `root` ユーザーの成功パスワード（`6yhnji90-`, `Pan5202614`, `zhaojia`, `P@$$w0rd12`, `Login!@456`, `mail@1234`, `3245gs5662d34`）が、もし当組織内の実際のシステムで有効なパスワードである場合、**直ちにこれらのパスワードを複雑で推測困難なものに変更してください。** また、該当するユーザーアカウントの活動ログを精査し、不正な操作が行われていないか確認してください。
3.  **システム全体のセキュリティスキャン**: 当組織内の全サーバーやSSHサービスを提供する機器に対し、マルウェア感染、不正なユーザーアカウント、または他のバックドアの有無を確認するための包括的なセキュリティスキャンを実施してください。特に、`.ssh/authorized_keys` ファイルの内容を精査し、不審な公開鍵（特にログに記載された `mdrfckr` の公開鍵）が存在しないことを確認してください。

#### 4.2. 予防措置 (Preventive Measures)

1.  **SSH設定の強化**:
    *   **`root` ユーザーの直接ログイン禁止**: `sshd_config` にて `PermitRootLogin no` を設定し、`root` ユーザーの直接SSHログインを禁止してください。管理者権限が必要な場合は、一般ユーザーでログイン後、`su` または `sudo` を使用することを義務付けてください。
    *   **パスワード認証の無効化**: 可能であれば、SSHパスワード認証を全面的に無効化し、セキュリティ強度の高い公開鍵認証のみを許可する設定に変更してください。
    *   **強力なパスワードポリシーの強制**: 全てのSSHユーザーに対して、大文字・小文字・数字・記号を組み合わせた12文字以上の複雑なパスワードの使用を強制し、定期的なパスワード変更を義務付けてください。
    *   **SSHポートの変更**: デフォルトのSSHポート（22番）を非標準のポートに変更することを検討してください。これにより、自動化された一般的なスキャンからの攻撃試行数を減らすことができます。
2.  **不正ログイン対策ツールの導入**: `Fail2ban` や `sshd_block` のようなツールを導入し、一定回数以上のログイン失敗があったIPアドレスを自動的にブロックする仕組みを構築してください。
3.  **ログ監視の強化**: SSHログインログ (`/var/log/auth.log` や `journalctl`) を含め、システムログの監視体制を強化してください。特に、`root` ユーザーのログイン試行、認証成功、および `authorized_keys` ファイルへの変更に関するログアラートを設定してください。
4.  **定期的なセキュリティ監査**: `.ssh` ディレクトリ内のファイル（`authorized_keys` など）やその他の重要な設定ファイルに対して、定期的に整合性チェックやパーミッション監査を実施し、不正な変更がないか確認してください。
5.  **公開鍵の管理と監査**: 組織内で使用される全てのSSH公開鍵を厳密に管理し、不正な鍵が存在しないか定期的に監査してください。
6.  **セキュリティ意識向上トレーニング**: 従業員に対して、SSHのセキュリティベストプラクティス、強力なパスワードの重要性、不審な活動の報告方法などに関する定期的なセキュリティ意識向上トレーニングを実施してください。
7.  **ハニーポットの継続運用**: `cowrie` ハニーポットの運用を継続し、新たな攻撃手法や攻撃元に関するインテリジェンスの収集を継続してください。
