### 2. 推測される手法・使用ツール

*   **ブルートフォース/パスワードスプレー攻撃:**
    *   複数のユーザー名とパスワードを短時間で連続して試行していることから、Hydra、Medusa、John the Ripper、またはNmapのSSH関連スクリプト (e.g., `ssh-brute`) のような自動化されたツールが使用されている可能性が高いです。
*   **バックドア設置スクリプト:**
    *   ログイン成功後に実行された一連のコマンド (`rm -rf .ssh && mkdir .ssh && echo "ssh-rsa..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh`) は、攻撃者が事前に用意した自動スクリプトの一部であると考えられます。
*   **ファイル属性操作ツール:**
    *   `chattr` コマンドの使用は、Linuxファイルシステムの拡張属性を操作する意図を示しており、システムの防御メカニズムを迂回しようとするものです。`lockr` は標準的なLinuxコマンドではないため、誤入力か、特定のマルウェアやツールに付属する独自コマンドの可能性があります。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **root権限の奪取意図:** 攻撃者は明確に `root` ユーザーアカウントを標的とし、実際に複数のパスワードでログインに成功しています。`root` 権限はシステムに対する完全な制御を意味するため、最も深刻な脅威の一つです。
*   **持続的なアクセス経路の確立:** 不正なSSH公開鍵を `authorized_keys` に追加しようとした行為は、パスワードの変更やアカウントロックアウトなどの対策が講じられても、攻撃者が永続的にシステムへアクセスできる状態を作ろうとするものです。
*   **防御回避の試み:** `chattr` コマンドでファイル属性の保護を解除しようとしたことは、攻撃者がシステム管理者が設定する防御機構を理解し、それを無効化しようとする高度な意図を示しています。
*   **広範な影響の可能性:** もしこの攻撃が本番環境で成功していた場合、攻撃者はシステム内の機微な情報へのアクセス、データの改ざん・破壊、マルウェアの展開、C2（コマンド＆コントロール）通信の確立、さらには他のシステムへの横展開など、甚大な被害を引き起こす可能性があります。
*   **自動化された攻撃:** 攻撃は断続的に行われていますが、一連の行動は自動化されたツールまたはスクリプトによって行われている可能性が高く、特定のターゲットに限定されない広範なスキャン活動の一部である可能性も示唆されます。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPアドレスのブロック:** ファイアウォール、IPS/IDS、またはセキュリティグループ等で、攻撃元IPアドレス `182.70.243.207` からのすべての通信を即座にブロックしてください。
2.  **影響を受けたシステムの特定と隔離:**
    *   このログはハニーポットのものであるため実際の侵害は発生していませんが、もし本番環境で同様のログが確認された場合、攻撃者が `root` ユーザーとしてログインに成功したシステムを特定し、直ちにネットワークから隔離またはインターネットからのアクセスを遮断してください。
    *   隔離されたシステムに対し、徹底的なフォレンジック調査を実施し、さらなる侵害活動（データ漏洩、マルウェア感染、他のバックドアの設置など）がないか確認してください。
3.  **不正なSSH鍵の確認と削除:**
    *   全てのLinux/Unix系サーバーにおいて、`root`ユーザーおよびその他の特権ユーザーの `~/.ssh/authorized_keys` ファイルの内容を精査してください。
    *   ログに示された公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) が存在する場合は、直ちに削除し、ファイルパーミッションが適切であること (`chmod 600 ~/.ssh/authorized_keys`, `chmod 700 ~/.ssh`) を確認してください。
4.  **パスワードのリセット:** ログインに成功した可能性のある全ての `root` アカウントおよび関連するサービスアカウントのパスワードを、複雑で推測しにくいものへ即座にリセットしてください。

#### 予防策とセキュリティ強化 (Preventative Measures & Security Hardening):

1.  **SSHパスワード認証の無効化:** 可能であれば、SSHサービスでパスワード認証を完全に無効にし、公開鍵認証のみを必須としてください。これはブルートフォース攻撃に対する最も効果的な対策の一つです。
2.  **強力なパスワードポリシーの実施:** 全てのユーザーアカウントに対し、複雑性、長さ、有効期限などを含む強力なパスワードポリシーを適用し、定期的な変更を促してください。
3.  **多要素認証 (MFA) の導入:** SSH接続を含む重要なサービスへのアクセスにMFAを導入し、認証情報の漏洩時のリスクを軽減してください。
4.  **アカウントロックアウトポリシーの設定:** ログイン失敗回数に基づいたアカウントロックアウトポリシーを設定し、ブルートフォース攻撃を自動的に抑制するメカニズムを導入してください。
5.  **不要なアカウントの削除/無効化:** デフォルトアカウントや使用されていないアカウントを特定し、削除または無効化してください。
6.  **SSH監視とアラートの強化:**
    *   SSHログイン成功/失敗のログ、特に `root` ユーザーや他の特権ユーザーに対するログイン試行をリアルタイムで監視し、異常を検知した際にアラートが上がるように設定してください。
    *   ログイン後の不審なコマンド実行（特にファイルシステム変更やユーザー追加など）についても監視を強化してください。
7.  **侵入検知/防御システム (IDS/IPS) の導入/設定強化:** SSHブルートフォース攻撃パターンを検知し、自動的に攻撃元をブロックできるIDS/IPSソリューションを導入し、設定を最適化してください。
8.  **ファイル属性保護の徹底:** 重要な設定ファイルやディレクトリ (`/etc/passwd`, `/etc/shadow`, `~/.ssh` など) に対して `chattr +i` などのコマンドを使用して不変属性を設定し、不正な変更から保護することを検討してください。ただし、システム運用に影響がないか十分に評価が必要です。
9.  **定期的な脆弱性スキャンとパッチ適用:** システムの脆弱性スキャンを定期的に実施し、発見された脆弱性には速やかにパッチを適用してください。
10. **セキュリティ意識向上トレーニング:** ユーザーと管理者に対し、パスワード管理やフィッシング攻撃など、一般的なセキュリティ脅威に関する意識向上トレーニングを実施してください。
