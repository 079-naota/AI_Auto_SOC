## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IP `101.100.242.72` から、ターゲットシステムに対する複数のSSH接続試行が確認されました。これはSSHサービスへのブルートフォース攻撃、または辞書攻撃と判断されます。

攻撃者は、一般的なユーザー名とパスワードの組み合わせを繰り返し試行した後、最終的に`root`ユーザーで複数の異なるパスワードを用いて計9回ログインに成功しました。ログイン成功後、攻撃者は以下のコマンドを実行し、自身のSSH公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`）を`.ssh/authorized_keys`ファイルに追加しようと試みています。

**主要な攻撃コマンド:**
`cd ~ && rm -rf .ssh && mkdir .ssh && echo "..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`

この攻撃の目的は、`root`権限でのシステムへの不正侵入を試み、一度ログインに成功した後は、自身のSSH公開鍵を登録することで、パスワードなしで将来にわたりシステムにアクセスできる永続的なバックドアを確立することにあります。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース/辞書攻撃**: 多数のユーザー名とパスワードの組み合わせ（例: `frappe:frappeuser`, `deployer:password123`, `ubuntu:default`, `admin:Admin05`など）を試行していることから、自動化されたツールによるブルートフォース攻撃または辞書攻撃が実行されたと推測されます。
    *   **認証情報の永続化 (Persistence)**: `root`アカウントにSSH公開鍵を追加しようとする行為は、システムへの長期的なアクセス経路を確保するための一般的な手法です。これにより、パスワードが変更されても攻撃者はシステムにアクセスし続けることが可能になります。
*   **使用ツール**:
    *   **SSHクライアント**: 基本的なSSH接続ツールが使用されています。
    *   **ブルートフォースツール**: `hydra`、`nmap`の`ssh-brute`スクリプト、`medusa`などのSSHパスワードクラックツールが、ログイン試行の自動化に使用された可能性が高いです。ログからは特定のツール名は特定できませんが、連続した試行パターンから推測されます。

### 3. 脅威レベルとその理由

**脅威レベル: 非常に高い (CRITICAL / SEVERE)**

**理由:**

1.  **Root権限での複数回ログイン成功**: 攻撃者がシステムに対する最高権限である`root`アカウントでのログインに複数回成功している点は、最も深刻な脅威です。これにより、攻撃者はシステム上で任意のコマンドを実行し、システム設定の変更、データの窃取・改ざん・削除、マルウェアの展開など、あらゆる不正活動を行うことが可能になります。
2.  **永続化の試み**: 攻撃者は自身のSSH公開鍵を`root`ユーザーの`authorized_keys`に追加しようとしています。これは、システムへの永続的なアクセス経路（バックドア）を確立することを目的としており、たとえパスワードが変更されても攻撃者がシステムに再侵入できるリスクを生じさせます。
3.  **自動化された広範な攻撃**: 多数の一般的なユーザー名とパスワードを試行していることから、特定のターゲットを狙ったものではなく、インターネット上に公開されたSSHサービスへの無差別なスキャンおよび攻撃である可能性が高いです。ハニーポットログであるため、実際のシステムへの影響は出ていませんが、本物のシステムであれば大規模な侵害に繋がりかねません。

### 4. 推奨アクション

このログはハニーポットから得られたものですが、実際のシステムで同様の攻撃が発生した場合を想定し、以下の緊急対応と恒久的な対策を強く推奨します。

#### 緊急対応 (Immediate Actions):

*   **攻撃元IPのブロック**: 攻撃元IPアドレス `101.100.242.72` を、ファイアウォール、IDS/IPS、またはWebアプリケーションファイアウォール（WAF）等で即座にブロックし、これ以上のアクセスを遮断します。
*   **SSH認証ログの緊急確認**: 実際のSSHサーバの認証ログ（例: `/var/log/auth.log`）を徹底的に確認し、同様のログイン試行や、特に`root`ユーザーへのログイン成功、および不正なSSH公開鍵の追加がされていないかを確認します。
*   **侵害された可能性のある認証情報の確認とリセット**: ログに記録されたログイン成功パスワード（`qwer1234QWER!@#$`, `3245gs5662d34`, `Ni123456`, `ubuntu@123`, `Password.123`, `a12348765`）が実際のシステムで使用されていないか確認し、もし使用されている場合は、即座にこれら全てのアカウントのパスワードをリセットし、多要素認証（MFA）を適用します。
*   **不正なSSH鍵の確認と削除**: `root`ユーザーおよび他のユーザーの`.ssh/authorized_keys`ファイルを確認し、ログに記録された攻撃者の公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5p... mdrfckr`）が存在しないか確認します。発見された場合は、直ちに削除します。
*   **システム侵害の有無の調査**: `root`権限でのログイン成功が確認されたため、システムが侵害された可能性を考慮し、追加のマルウェア、不正なプロセス、不審な設定変更などがないか、フォレンジック調査を実施します。

#### 恒久的な対策 (Long-term Actions):

*   **SSHパスワードポリシーの強化**:
    *   推測されにくい、複雑なパスワード（長さ、大文字・小文字、数字、記号の組み合わせ）の使用を義務付けるパスワードポリシーを導入します。
    *   アカウントロックアウトポリシーを設定し、一定回数以上のログイン失敗でアカウントを一時的にロックします。
*   **多要素認証 (MFA) の導入**: SSHへのログインにMFAを必須とし、セキュリティを大幅に強化します。
*   **SSH鍵認証の推奨とパスワード認証の無効化**:
    *   可能であれば、より安全なSSH鍵認証を推奨し、パスワード認証を無効化します (`PasswordAuthentication no`)。
    *   鍵認証を用いる場合も、鍵ファイルの適切な管理とパスフレーズ設定を徹底します。
*   **Rootログインの禁止**: SSHD設定で`PermitRootLogin no`を設定し、`root`ユーザーでの直接ログインを禁止します。一般ユーザーでログイン後、`sudo`を使用して権限昇格する運用に切り替えます。
*   **不要なアカウントの削除**: デフォルトアカウント、テストアカウント、および現在使用されていないアカウントを全て削除または無効化します。
*   **SSHサービスへのアクセス制限**:
    *   ファイアウォールでSSHポート（通常22番）へのアクセスを、必要なソースIPアドレス（社内ネットワーク、VPNなど）のみに制限します。
    *   ポートノッキングやSSHポートの変更も検討します（ただし、根本的な解決策ではない）。
*   **侵入検知・防御システム (IDS/IPS) の導入・強化**: ブルートフォース攻撃や不正アクセスを自動的に検知し、防御できるIDS/IPSソリューションを導入し、最新のシグネチャを適用します。
*   **ログ監視とSIEMの導入**: SSH認証ログを含むシステムログをリアルタイムで監視し、異常なログイン試行、ファイル変更、コマンド実行などを早期に検知できる体制を構築します。SIEM（Security Information and Event Management）にログを統合し、相関分析を通じて脅威を可視化します。
*   **脆弱性管理とパッチ適用**: 定期的にシステムに対する脆弱性スキャンを実施し、発見された脆弱性には速やかにパッチを適用します。
