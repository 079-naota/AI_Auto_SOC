# SOC自動分析レポート: 103.120.227.88

**生成日時:** 2026-04-27 00:50:41

---

## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃は、攻撃元IP `103.120.227.88` からのSSH接続を通じて開始されました。初期段階で、`root` ユーザーの認証情報（ユーザー名: `root`, パスワード: `Qw1234`）を悪用してシステムにログインしました。これは、ブルートフォース攻撃、辞書攻撃、または既知の脆弱なデフォルトパスワードの悪用によって成功した可能性が高いです。

ログイン後、攻撃者は以下の主要な行動をとりました。

*   **永続化:** `root` ユーザーのホームディレクトリに存在する `.ssh` ディレクトリを操作し、自身のSSH公開鍵を `authorized_keys` ファイルに追加することで、パスワードなしでの再接続を可能にしました。さらに、`root` ユーザーのパスワードを `brm8bXcKjEOQ` に変更し、パスワード認証によるアクセス経路も確保しました。
*   **環境認識:** システムのCPU情報、メモリ使用状況、ディスク容量、OSバージョン、稼働中のプロセス、ログインユーザー、cronジョブなど、多岐にわたるシステム情報を収集しました。これは、システムの性能や構成を把握し、その後の悪意ある活動（マルウェアの展開、リソースの悪用など）に最適な環境であるか評価するためのものと推測されます。
*   **競合排除/防御回避:** `/tmp/secure.sh` や `/tmp/auth.sh` といった他のマルウェアの痕跡と思われるファイルを削除し、関連プロセスを強制終了しました。また、`/etc/hosts.deny` ファイルをクリアすることで、外部からのアクセス制限を緩和し、自身の活動の妨げとなる可能性のある設定を排除しようとしました。

これらの行動から、攻撃者の主な目的は、標的システムを完全に掌握し、永続的なアクセス経路を確立した上で、そのシステムリソースを悪用すること（例: 仮想通貨マイニング、ボットネットへの組み込み、DDoS攻撃の発信源、さらなる内部ネットワークへの侵入）であると推測されます。特に、CPUやメモリなどのリソース情報を詳細に確認していることから、仮想通貨マイニングのボットネットに組み込もうとしている可能性が非常に高いです。

### 2. 推測される手法・使用ツール

*   **初期アクセス:**
    *   **T1110.001 - Brute Force: Password Guessing:** `root` ユーザーに対する脆弱なパスワード（`Qw1234`）の推測または辞書攻撃。
*   **永続化:**
    *   **T1098.004 - Account Manipulation: SSH Authorized Keys:** 攻撃者のSSH公開鍵を `root` ユーザーの `authorized_keys` ファイルに追加。
    *   **T1098.002 - Account Manipulation: Local Account:** `chpasswd` コマンドを使用して `root` ユーザーのパスワードを変更。
*   **環境認識:**
    *   **T1082 - System Information Discovery:** `cat /proc/cpuinfo`, `uname -a`, `lscpu`, `free -m`, `df -h` などでOS、CPU、メモリ、ディスク情報を収集。
    *   **T1033 - System Owner/User Discovery:** `w`, `whoami` でログインユーザー情報を収集。
    *   **T1057 - Process Discovery:** `top` で実行中のプロセスとシステム負荷を監視。
    *   **T1053.003 - Scheduled Task/Job: Cron:** `crontab -l` で既存のスケジュールタスクを確認。
*   **防御回避 / 影響:**
    *   **T1070.004 - Indicator Removal on Host: File Deletion:** `rm -rf` コマンドで不審なファイル（競合マルウェアなど）を削除。
    *   **T1562.001 - Disable or Modify Tools: Service Stop:** `pkill -9` コマンドで不審なプロセスを強制終了。
    *   `/etc/hosts.deny` をクリアすることで、外部からのアクセス制限設定を緩和。
*   **使用ツール:**
    *   攻撃者は主に標準的なLinuxコマンドラインツール（`cd`, `rm`, `mkdir`, `echo`, `chmod`, `cat`, `grep`, `wc`, `awk`, `head`, `free`, `which`, `ls`, `crontab`, `w`, `uname`, `top`, `whoami`, `lscpu`, `df`, `chpasswd`, `pkill`）を利用しています。
    *   `lockr` コマンドは試行されましたが、ハニーポット環境では実行に失敗しています。これは、特定のマルウェアやスクリプトの一部である可能性があります。

### 3. 脅威レベルとその理由

*   **脅威レベル: 深刻 (Critical)**

*   **理由:**
    1.  **完全なシステムコントロールの確立:** 攻撃者は `root` 権限でシステムにログインし、自身のSSH公開鍵の登録と `root` パスワードの変更によって、システムの完全なコントロールを掌握し、永続的なアクセス経路を確立しました。これにより、システムに対するあらゆる操作が可能となります。
    2.  **持続的アクセス:** 複数かつ冗長な永続化メカニズム（公開鍵認証とパスワード認証）を設定しているため、システムが再起動されたり、初期の脆弱性が修正されたりしても、攻撃者は容易にシステムに再侵入できます。
    3.  **積極的な競合排除:** 他のマルウェアや攻撃者の活動を排除しようとする行動は、攻撃者がこのシステムを専有し、自身の悪意ある目的のために利用する強い意思を示しています。これは、システムがさらに深刻なマルウェア（例: 仮想通貨マイナー）に感染する準備段階であることを示唆します。
    4.  **広範な環境認識:** 攻撃者が収集した詳細なシステム情報は、後続の攻撃フェーズ（例: 特定のアーキテクチャに合わせたマルウェアの展開、C2通信の確立、データ窃取、大規模なボットネットへの組み込み）に向けた準備であり、高度な計画性を示しています。
    5.  **既知の攻撃パターンとの合致:** これらの行動は、SSHを悪用したサーバー侵入後の典型的な攻撃チェーン（永続化、環境認識、リソース悪用）と一致しており、実際にシステムが侵害されている状況では極めて危険な兆候です。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions)

1.  **システム隔離:**
    *   直ちに影響を受けたシステムをネットワークから隔離し、さらなる被害拡大や攻撃者による悪用を阻止してください。
2.  **認証情報の変更とバックドアの除去:**
    *   `root` ユーザーのパスワードを直ちに、かつ非常に強力なものに変更してください。
    *   `root` ユーザーのホームディレクトリ (`~/.ssh/`) にある `authorized_keys` ファイルを確認し、不審な公開鍵（特にコメントが `mdrfckr` のもの）を削除してください。
    *   `/etc/passwd` や `/etc/shadow` を確認し、他の不審なアカウントやパスワード変更がないか検証してください。
3.  **不審なファイルの削除とプロセスの終了:**
    *   ログに記載された `/tmp/secure.sh`, `/tmp/auth.sh` などのファイルをシステム全体から検索し、削除してください。
    *   現在稼働中のプロセスを監視し、不審なプロセス（CPUやメモリを異常に消費するものなど）を特定し、強制終了してください。
4.  **`hosts.deny` の復元:**
    *   `/etc/hosts.deny` ファイルがクリアされている場合は、以前の適切なアクセス制限ルールを復元するか、推奨されるセキュリティ設定で上書きしてください。

#### 調査と分析 (Investigation and Analysis)

1.  **ログの深掘り分析:**
    *   システムのSSHログ、認証ログ (`/var/log/auth.log` や `journalctl` など)、システムログ (`/var/log/syslog` など) を確認し、攻撃元IP `103.120.227.88` からの過去のアクセス履歴や、不審なログイン試行、他のシステム変更がないかを徹底的に調査してください。
    *   攻撃者が実行したコマンドの完全なログを確認し、未発見のバックドアやさらなる意図がないかを特定してください。
2.  **フォレンジックイメージ作成:**
    *   証拠保全のため、影響を受けたシステムのディスクイメージを取得し、詳細なフォレンジック分析を行ってください。
3.  **マルウェアスキャン:**
    *   システム全体を最新のアンチウイルス/アンチマルウェアソフトウェアでスキャンし、隠されたマルウェアや追加のバックドアがないか確認してください。
4.  **他の永続化メカニズムの確認:**
    *   cronジョブ、systemdサービス、スタートアップスクリプト、システム環境変数、カーネルモジュールなどに不審な設定がないか、広範に確認してください。

#### 予防と強化 (Prevention and Hardening)

1.  **SSHセキュリティ強化:**
    *   SSHのデフォルトポート (22番) を変更し、難読化を図ってください。
    *   `PermitRootLogin no` を設定し、`root` ユーザーによる直接SSHログインを禁止してください。
    *   パスワード認証を無効化し、強力なパスフレーズで保護された公開鍵認証のみを使用するように設定してください。
    *   `sshd_config` の設定を見直し、脆弱な暗号スイートの使用を制限し、強固な設定を適用してください。
    *   Fail2BanやSSHGuardなどのツールを導入し、ブルートフォース攻撃に対する防御を強化してください。
2.  **強力な認証情報の採用:**
    *   全てのシステムアカウントに対し、複雑で推測されにくいパスワードを強制し、定期的なパスワード変更ポリシーを導入してください。多要素認証 (MFA) の導入も検討してください。
3.  **パッチ管理とソフトウェアアップデート:**
    *   全てのOSとインストールされているアプリケーションを常に最新の状態に保ち、既知の脆弱性を解消してください。
4.  **最小権限の原則:**
    *   各ユーザーやサービスが必要最小限の権限のみを持つように設定し、`root` 権限の使用を厳しく制限してください。
5.  **ネットワークセグメンテーションとファイアウォール:**
    *   重要なシステムは適切にネットワークをセグメンテーションし、必要最小限の通信のみを許可するファイアウォールルールを設定してください。
6.  **侵入検知/防御システム (IDS/IPS) の導入・強化:**
    *   異常なログイン試行、不審なコマンド実行、ファイル変更などをリアルタイムで検知・防御できるシステムを導入・強化してください。
7.  **定期的なバックアップ:**
    *   定期的にシステムとデータのバックアップを取得し、インシデント発生時の迅速な復旧に備えてください。
8.  **セキュリティ監視の強化:**
    *   SIEM (Security Information and Event Management) を活用し、ログの一元管理と相関分析を行い、早期に異常を検知できる体制を構築してください。

これらのアクションを迅速かつ包括的に実施することで、現在の脅威に対処し、将来の攻撃に対するセキュリティ体制を大幅に強化することが可能です。
