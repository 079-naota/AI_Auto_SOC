# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 07:02:18

---

## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `116.99.174.17` から、ターゲットシステム（Cowrieハニーポット）のSSHサービスに対するブルートフォース攻撃が複数回実施されました。この攻撃の主要な目的は、不正な認証情報を繰り返し試行することで、システムへの不正アクセス権（特に特権ユーザーである `root` アカウント）を取得することであると推測されます。

ログからは、攻撃者が以下のユーザー名とパスワードの組み合わせを用いて、繰り返しログインを試み、最終的に複数の異なるパスワードで `root` アカウントへのログインに成功していることが確認できます。

**ログイン成功事例:**
*   `root` / `admin`
*   `root` / `@`
*   `root` / `root123`
*   `root` / `12345`
*   `root` / `abcd1234`
*   `root` / `ipscan`
*   `root` / `alpine`

ログイン成功後には `cowrie.direct-tcpip.request` などのイベントが記録されており、これは攻撃者がシステム内で何らかのコマンド実行、ポートフォワーディング、あるいは外部システムへの接続（C2サーバーへの通信確立など）を試みようとしていた可能性を示唆しています。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃 / 辞書攻撃**: 攻撃者は一般的なユーザー名（`admin`, `user`, `root`, `ubnt`など）と、辞書攻撃によく用いられるパスワード（`admin`, `123456`, `default`, `alpine`など）の組み合わせを広範囲にわたって試行しています。特に `root` ユーザーに対する執拗な試行が目立ちます。
    *   **自動化された攻撃**: 短期間に多数のログイン試行が繰り返されていることから、手動ではなく、HydraやNmapの`ssh-brute`スクリプトのような自動化されたブルートフォースツールが使用されている可能性が高いです。
    *   **初期アクセス後の偵察・C2通信試行**: ログイン成功後に`direct-tcpip`リクエストが記録されていることから、攻撃者は初期アクセス権の取得に成功した後、システム内部の偵察、追加のマルウェアのダウンロード、またはコマンド＆コントロール（C2）サーバーとの通信確立を試みていた可能性があります。

*   **使用ツール**:
    *   ログからは特定のツール名を直接特定できませんが、標準的なSSHクライアントおよび、上述のような自動化されたブルートフォースツールが組み合わせて使用されていると推測されます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

1.  **特権アカウントへの複数回ログイン成功**: `root`アカウントへのログインが複数回成功していることは、システムセキュリティに対する極めて深刻な侵害を示唆します。`root`権限はシステム上の最高権限であり、攻撃者はこの権限を用いてシステム設定の変更、データの窃取・改ざん、マルウェアの展開、バックドアの設置など、あらゆる悪意のある活動が可能となります。ハニーポット環境であるため実際のシステムへの被害はありませんが、本番環境であれば完全なシステム侵害を意味します。
2.  **執拗な攻撃とパスワードの多様性**: 攻撃者は短時間のうちに多数の異なるユーザー名とパスワードの組み合わせを試しており、広範な辞書攻撃のリストを使用していることが伺えます。これは特定のターゲットを狙ったものではなく、インターネットに公開された脆弱なSSHサービスを網羅的に探索する、いわゆる「ワーム型」または「広範なスキャン」の一部である可能性も高いですが、その実行能力は高いと言えます。
3.  **初期アクセス後の行動の兆候**: ログイン成功後の`direct-tcpip`リクエストは、攻撃者が単なるログイン試行に留まらず、アクセス後の活動（内部偵察やC2通信の確立など）へと移行しようとしていたことを示唆しており、これは侵害の深度と潜在的な影響を増大させます。

### 4. 推奨アクション

このログが実際のシステムで観測された場合を想定し、以下の緊急対応および予防的措置を推奨します。

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPアドレスのブロック**:
    *   攻撃元IPアドレス `116.99.174.17` からのすべてのネットワークトラフィック（特にSSHポート22番）を、ファイアウォールやIPS/IDSによって即座にブロックしてください。
2.  **侵害された認証情報の無効化とパスワード変更**:
    *   ログに登場したユーザー名 `root` および、ログイン試行されたその他の一般的なユーザー名（`admin`, `user`, `ubnt`, `config`, `squid`, `support`, `guest`, `1234`, `nikita`, `ftpuser`）について、システムおよび関連する全てのサービスのパスワードを、複雑で推測困難なものに即座に変更してください。特に、ログイン成功が確認されたパスワード（`admin`, `@`, `root123`, `12345`, `abcd1234`, `ipscan`, `alpine`）が他のシステムやサービスで使い回されていないか緊急で確認し、使用されている場合は直ちに全て変更してください。
3.  **システム侵害の有無と範囲の調査**:
    *   このログイン成功が発生した時間帯に、対象システム上で不審なプロセス、ファイル変更（特に`/tmp`, `/var`などの一時ディレクトリや設定ファイル）、追加されたユーザーアカウント、不審なネットワーク接続（外部へのTCP接続など）がないか、詳細なフォレンジック調査を実施してください。
    *   SSHログイン成功後のコマンド実行履歴（`~/.bash_history`など）も確認してください。
4.  **既存のバックドアやマルウェアの有無の確認**:
    *   システムが侵害されている可能性があるため、ルートキット検出ツール（例: rkhunter, chkrootkit）やマルウェアスキャンツールを用いて、バックドアやマルウェアが設置されていないか確認してください。

#### 予防的措置 (Preventive Measures):

1.  **SSH設定の強化**:
    *   **公開鍵認証の強制**: パスワード認証を無効化し、公開鍵認証のみを許可するように設定を変更してください。
    *   **`root`ユーザーの直接ログイン禁止**: `root`ユーザーがSSHで直接ログインすることを禁止し、一般ユーザーでログイン後に`su`または`sudo`コマンドを使用させるようにしてください。
    *   **SSHポートの変更**: 標準のSSHポート22番を、使用頻度の低い高位ポート番号に変更してください。
    *   **ログイン試行回数制限**: Fail2Banなどのツールを導入し、一定回数のログイン失敗があったIPアドレスを自動的にブロックする設定を導入してください。
    *   **多要素認証 (MFA)**: 可能な限り、SSHログインに多要素認証を導入してください。
2.  **強力なパスワードポリシーの徹底**:
    *   すべてのシステムユーザーに対し、複雑性、長さ、有効期限などを含む強力なパスワードポリシーを強制し、定期的なパスワード変更を義務付けてください。
    *   安易なパスワード（`admin`, `123456`など）の使用を禁止するブラックリストを導入してください。
3.  **システムとアプリケーションのアップデート**:
    *   すべてのオペレーティングシステム、アプリケーション、およびミドルウェアを最新の状態に保ち、既知の脆弱性を悪用されるリスクを低減してください。
4.  **ログ監視とアラート体制の強化**:
    *   SSH認証ログを含む、重要なシステムログを一元的に収集・監視し、不審なログイン試行や成功に対してリアルタイムでアラートを発するシステム（SIEMなど）を導入または強化してください。
5.  **脅威インテリジェンスの活用**:
    *   今回の攻撃元IPアドレスや試行された認証情報を脅威インテリジェンスプラットフォームと共有し、将来的な攻撃に対する防御策を強化してください。
