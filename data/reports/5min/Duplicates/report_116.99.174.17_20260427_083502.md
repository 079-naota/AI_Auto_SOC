# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 08:37:20

---

## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `116.99.174.17` から、当組織のシステム（Cowrieハニーポット）に対してSSHサービスへの継続的なブルートフォース攻撃が確認されました。攻撃者は多様なユーザー名とパスワードの組み合わせを自動的に試行しており、中には一般的なデフォルトクレデンシャルや単純なパスワードも含まれています。

この攻撃の主な目的は、SSHサービスへの不正ログインを試み、システムへのアクセス権を取得することであると推測されます。ログにはログイン成功後の `direct-tcpip` イベントが複数記録されており、これは攻撃者がポートフォワーディングやデータ転送といった、認証後のさらなる活動（内部ネットワークへの侵入、情報の窃取、マルウェアの展開など）を試みている可能性を示唆しています。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース攻撃 / 辞書攻撃:** 攻撃者は `admin`, `root`, `user` などの一般的なユーザー名や、`admin`, `123456`, `password`, `default` といった脆弱なパスワードを繰り返し試行しています。
    *   **デフォルトクレデンシャル攻撃:** `ubnt/ubnt`, `config/config`, `squid/squid`, `support/support`, `ftpuser/asteriskftp` など、特定のデバイスやアプリケーションで使われる可能性のあるデフォルトの認証情報を狙った攻撃も含まれます。
    *   **認証後の活動:** ログイン成功後には `direct-tcpip.request`, `direct-tcpip.ja4h`, `direct-tcpip.data` といったイベントが記録されており、これはSSHのポートフォワーディング機能を利用して、リモートからのコマンド実行、ファイル転送、または他のネットワークサービスへのアクセスを試みていることを強く示唆しています。
*   **使用ツール:**
    *   複数のログイン試行が短時間かつ連続して行われていることから、Hydra、Medusa、NmapのNSEスクリプト（`ssh-brute`）などの自動化されたSSHブルートフォースツール、またはカスタムスクリプトが使用されている可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **継続的な攻撃:** 短時間のうちに多数のログイン試行が確認されており、組織のSSHサービスに対する継続的な侵害活動が行われています。
*   **ログイン成功の記録:** ハニーポット上ではありますが、以下のユーザー名とパスワードの組み合わせでログインが「成功」しています。
    *   `root` / `admin`
    *   `root` / `@` (パスワードが `@` のみは非常に珍しいですが、ログに残っているため注意が必要です)
    *   `root` / `root123`
    *   `root` / `12345`
    *   `root` / `abcd1234`
    *   `root` / `ipscan`
    *   `root` / `alpine`
    これらのパスワードは一般的に推測されやすいものであり、もし実際のシステムでこれらの認証情報が使われていた場合、攻撃者はすでにシステムへのアクセス権を取得している状態にあります。特に `root` ユーザーでのログイン成功は、システムに対する完全な制御を許すことになり、極めて深刻な脅威となります。
*   **アクセス後の意図:** ログイン成功後に `direct-tcpip` イベントが頻繁に発生していることから、攻撃者は単なるアクセス取得だけでなく、さらなる内部システムへの侵入、情報窃取、マルウェア展開など、より広範囲な攻撃を目論んでいる可能性が高いです。

### 4. 推奨アクション

#### 4.1. 緊急対応 (Immediate Actions)

1.  **攻撃元IPアドレスのブロック:**
    *   ファイアウォール、IDS/IPS、またはルーターで、攻撃元IPアドレス `116.99.174.17` からのSSH接続を即座にブロックしてください。
2.  **既存ユーザーのパスワード緊急監査と変更:**
    *   ログでログイン成功が記録されたユーザー名 (`root` など) およびパスワード (`admin`, `@`, `root123`, `12345`, `abcd1234`, `ipscan`, `alpine` など) が、実稼働環境のどのシステムでも使用されていないか緊急で確認してください。
    *   もしこれらの認証情報が使用されている場合は、直ちに複雑で推測困難なパスワードへ変更するよう、対象ユーザーに指示し、変更完了を確認してください。特に `root` ユーザーのパスワードが脆弱でないことを最優先で確認してください。

#### 4.2. 予防的対応 (Preventive Actions)

1.  **SSHサービスへのアクセス制御強化:**
    *   SSHサービスへのアクセスを、信頼できる特定のIPアドレス範囲（IPホワイトリスト方式）のみに制限してください。
    *   VPN経由でのSSHアクセスを必須とし、インターネットからの直接アクセスを原則禁止してください。
2.  **二要素認証 (MFA/2FA) の導入:**
    *   SSHログインに二要素認証（例: TOTP、FIDO2など）を必須化し、パスワードのみでの認証を無効にしてください。
3.  **公開鍵認証の利用:**
    *   パスワード認証を無効にし、公開鍵認証のみを許可するようSSHサーバーの設定を変更してください。
4.  **ログイン試行回数制限の導入:**
    *   Fail2Banなどのツールを導入し、一定回数以上のログイン失敗があった場合に、当該IPアドレスを一時的または永続的にブロックするメカニズムを実装してください。
5.  **デフォルトユーザーアカウントの管理:**
    *   `admin`, `root`, `user` など一般的なユーザー名のアカウントは、可能な限り使用を避け、ユニークなユーザー名を設定してください。
    *   使用されていない、または不要なアカウントは無効化または削除してください。
6.  **SSHポートの変更 (セキュリティバイアスクリティ):**
    *   SSHの標準ポートである22番ポートの使用を避け、別のポートに変更することを検討してください。ただし、これは根本的な解決策ではなく、スキャンによる発見を遅らせる効果に限定されます。

#### 4.3. 長期的な対策 (Long-term Measures)

1.  **セキュリティログの継続的な監視と分析:**
    *   Cowrieのようなハニーポットや、実際のSSHサービスログをSIEMなどのツールで継続的に監視し、不審なログイン試行やパターンを早期に検知できる体制を維持・強化してください。
2.  **脅威インテリジェンスの活用:**
    *   攻撃元IPアドレス `116.99.174.17` を脅威インテリジェンスデータベースで確認し、過去の活動、関連するマルウェア情報、または関連する攻撃グループの有無などを調査してください。
3.  **従業員へのセキュリティ教育:**
    *   強力なパスワード設定の重要性、フィッシング詐欺への注意喚起など、従業員全体のセキュリティ意識向上に向けた教育を定期的に実施してください。
4.  **定期的な脆弱性診断・ペネトレーションテスト:**
    *   定期的にシステムの脆弱性診断やペネトレーションテストを実施し、潜在的な脆弱性を特定し、セキュリティ対策の有効性を確認してください。
