この攻撃は、主にSSHパスワードのブルートフォース（辞書攻撃）を目的としており、特に`root`ユーザーアカウントへの不正アクセスを試みています。複数の一般的なユーザー名とパスワードの組み合わせが試行された後、**`root`ユーザーに対して5回ものログイン成功が確認されています。**

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

*   **理由:**
    *   **複数の認証成功:** ハニーポットのログであるものの、`root`ユーザーに対して5回ものログイン成功が確認されています。これは現実のシステムであれば、即座にシステムへの完全な侵害（RCE: Remote Code Execution、特権昇格、データ窃取など）を許す状況に該当します。
    *   **特権ユーザー（root）の標的:** 攻撃者がシステムへの完全な制御を意図していることを明確に示しています。`root`権限の奪取は、システム全体のセキュリティを完全に損なうことにつながります。
    *   **永続化の試み:** SSH公開鍵の設置は、攻撃者が一度アクセスを得た後も、継続的にシステムへのアクセスを維持しようとする強い意図を示しています。これは、一時的な偵察や偶然のアクセスではなく、計画的かつ持続的な脅威であることを意味します。
    *   **自動化された攻撃:** 多数のログイン試行と、成功後の定型的なコマンド実行から、自動化されたマルウェアまたはボットネットの一部である可能性が高いです。このような攻撃は、脆弱なパスワードを持つシステムを広範囲にスキャンして狙うため、多数の標的を同時に危険に晒す可能性があります。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPアドレスのブロック:** ファイアウォール、IDS/IPS、またはSSHデーモン（`iptables`や`fail2ban`など）で、攻撃元IPアドレス `101.100.242.72` からの全てのインバウンド通信、特にSSH (`TCP/22`) を直ちにブロックしてください。
2.  **SSH認証ログの緊急監査:** 全てのプロダクションシステムおよび開発環境において、`auth.log`などのSSH認証ログを直ちに確認し、この攻撃元IPアドレスからの不正なログイン試行や認証成功がないかを徹底的に調査してください。
3.  **パスワードの強制変更:** 全てのSSH接続が可能なユーザーアカウント（特に`root`および特権アカウント）のパスワードを強力なものに強制変更し、パスワードポリシーを強化してください。
4.  **SSH公開鍵認証の確認:** 全てのユーザーの `~/.ssh/authorized_keys` ファイルの内容を確認し、不審な公開鍵が追加されていないか監査してください。もし不審な鍵が見つかった場合、直ちに削除し、関連するユーザーのパスワードを変更してください。

#### 予防的措置 (Preventative Measures):

1.  **多要素認証 (MFA) の導入:** SSHログインに対してMFAを必須化し、認証情報の漏洩だけでは不正アクセスができないようにセキュリティを強化してください。
2.  **SSHサービスへのアクセス制限:** SSHサービスへのアクセスを、特定の信頼できるIPアドレス範囲（オフィスIP、VPN経由のIPなど）のみに制限し、インターネットからの直接アクセスを可能な限り排除してください。
3.  **アカウントロックアウトポリシーの導入:** `fail2ban`などのツールを導入し、一定回数以上のログイン失敗があったIPアドレスを自動的にブロックする設定を有効にしてください。
4.  **SSHポートの変更:** デフォルトのSSHポート（22番）を非標準ポートに変更することで、一般的なスキャンからの可視性を低下させることが可能です（これはセキュリティの強化ではなく、ノイズの削減策です）。
5.  **不必要なアカウントの削除:** 不要なユーザーアカウントやサービスアカウントを特定し、削除または無効化してください。最小特権の原則を徹底してください。
6.  **定期的なログ監視とアラート設定:** SSH認証ログに対する異常検知（多数のログイン失敗、未知のIPからのログイン成功など）アラートを強化し、迅速な対応を可能にしてください。
7.  **システムパッチと脆弱性管理:** OSおよびアプリケーションのセキュリティパッチを常に最新の状態に保ち、定期的な脆弱性スキャンを実施してください。
8.  **ハニーポットの活用:** Cowrieのようなハニーポットを引き続き運用し、攻撃者の動向、使用されるユーザー名・パスワード、攻撃スクリプト、C2サーバーIPアドレスなどの脅威情報を継続的に収集・分析してください。
