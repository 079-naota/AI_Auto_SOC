## SOC分析レポート

**件名: SSHサービスに対するブルートフォース攻撃および永続化試行の検知**

**攻撃元IPアドレス**: 122.177.246.97
**検知日時**: 2026-04-25T08:21:35Z から 2026-04-25T08:55:36Z まで

### 1. 攻撃の概要と目的

攻撃者はIPアドレス `122.177.246.97` から、ターゲットシステム（Cowrieハニーポット）のSSHサービスに対して、継続的なブルートフォース攻撃を仕掛けていました。この攻撃は、複数のユーザー名（`deployer`, `root`, `deploy`, `vnc`, `345gs5662d34`）とパスワードの組み合わせを短時間に大量に試行するものでした。

特に、`root` ユーザーに対して以下のパスワードでログインを試み、ハニーポット上では複数回ログインに成功したと記録されています:
*   `ccQQ123`
*   `3245gs5662d34`
*   `AAaa112233`
*   `Master2025`

ログイン成功後、攻撃者はシステムの永続的なアクセス経路を確立しようとしました。具体的には、`root` ユーザーのホームディレクトリにある `.ssh` ディレクトリを削除し、再作成した後、攻撃者自身のSSH公開鍵（コメント: `mdrfckr`）を `authorized_keys` ファイルに書き込むことで、パスワードなしでの再ログインを可能にしようとしていました。また、`chattr` コマンドを使用して `.ssh` ディレクトリの属性変更も試みていますが、`lockr` コマンドが失敗しているログも確認できます。

### 2. 推測される手法・使用ツール

*   **ブルートフォース攻撃/辞書攻撃**: 短時間に異なるユーザー名とパスワードの組み合わせを大量に試行していることから、自動化されたブルートフォース攻撃、または辞書攻撃ツールを使用していると推測されます。
*   **アカウント乗っ取り (Account Takeover)**: ログインに成功した場合、永続的なアクセス経路を確立するためにSSH鍵を設置しようとしていることから、最終的な目的はシステムへの不正アクセスと権限の奪取であると考えられます。
*   **自動化されたスクリプト/ボットネット**: 攻撃が短時間で繰り返されており、定型的なコマンド実行が行われていることから、特定の脆弱性を狙った自動化されたスクリプト、またはボットネットの一部である可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
1.  **root権限でのログイン成功**: 攻撃者が最も機密性の高い `root` ユーザーとしてログインに成功している点が最も深刻です。もしこれが実環境であれば、システム全体が完全に制御され、データの破壊、窃取、マルウェアの埋め込みなど、あらゆる種類の被害が発生する可能性があります。
2.  **永続化の試み**: ログイン後にSSH公開鍵を設置しているのは、検出を回避しつつ、いつでもシステムに再アクセスできる裏口（バックドア）を確立しようとする明確な悪意ある行為です。これにより、たとえパスワードが変更されても、攻撃者は鍵認証を用いて継続的にアクセスを試みることができます。
3.  **複数のパスワードによる成功**: 複数の異なるパスワードで `root` ユーザーとしてログインに成功していることから、攻撃者が複数の有効なパスワードを所持しているか、ターゲットシステムが広く知られた、または推測されやすいパスワードを使用している可能性が高いことを示唆しています。
4.  **自動化された攻撃**: 攻撃のパターンから、単一の標的を狙った手動攻撃ではなく、広範囲に展開される自動化されたスキャンと侵害活動の一部であると考えられます。

### 4. 推奨アクション

以下の対策を速やかに実施することを推奨します。

1.  **攻撃元IPアドレスのブロック**:
    *   ファイアウォール、IDS/IPS、またはエッジルーターにて、攻撃元IPアドレス `122.177.246.97` からのすべての通信を即座にブロックしてください。

2.  **SSH認証の強化**:
    *   **パスワード認証の無効化**: 可能であれば、SSHパスワード認証を完全に無効化し、鍵認証のみを許可するように設定を変更してください。
    *   **rootユーザーの直接SSHログイン禁止**: `PermitRootLogin no` を設定し、`root` ユーザーでの直接SSHログインを禁止してください。管理者ユーザーは一般ユーザーでログイン後、`su` または `sudo` を利用するように徹底してください。
    *   **多要素認証 (MFA) の導入**: SSHログインにMFAを導入し、セキュリティを強化してください。
    *   **SSHポートの変更**: 標準の22番ポートから異なるポートへ変更することも検討してください（これはセキュリティバイオブスキュリティですが、自動スキャンの対象から外れる効果はあります）。

3.  **アカウントとパスワードの棚卸しと変更**:
    *   ログに記録されたログイン成功パスワード（`ccQQ123`, `3245gs5662d34`, `AAaa112233`, `Master2025`）が、実環境のいずれのシステムでも使用されていないか緊急で確認してください。もし使用されている場合は、即座に複雑で推測されにくいパスワードに変更してください。
    *   すべてのシステムで、複雑なパスワードポリシーを導入し、定期的なパスワード変更を強制してください。
    *   `deployer`, `deploy`, `vnc`, `345gs5662d34` など、ログに登場した他のユーザー名についても、システム内に不要なアカウントがないか確認し、あれば削除またはロックアウトしてください。

4.  **SSH公開鍵の確認と削除**:
    *   すべてのサーバーにおいて、`~/.ssh/authorized_keys` ファイルの内容を確認し、ログに記載された攻撃者の公開鍵（コメント: `mdrfckr` のもの）が存在しないことを確認してください。もし発見された場合は、即座に当該エントリを削除してください。
    *   不正な鍵が存在しないことを確認した後も、`authorized_keys` ファイルと `.ssh` ディレクトリのパーミッションが適切であるか確認してください（通常 `600` と `700`）。

5.  **ログ監視と侵入検知システムの強化**:
    *   SSHブルートフォース攻撃を検知し、自動的に攻撃元IPアドレスをブロックするようなIDS/IPSルールを強化してください。
    *   SSHログイン成功イベント、特に`root`ユーザーや管理者ユーザーのログインに対して、異常検知アラートを強化してください。
    *   ハニーポット（Cowrie）からのアラートを継続的に監視し、同様の攻撃パターンがないか警戒してください。

6.  **定期的な脆弱性診断**:
    *   定期的な外部/内部からの脆弱性診断を実施し、SSHサービスを含む公開サービスに脆弱性が存在しないか確認してください。

この攻撃は深刻な脅威を示唆しており、迅速かつ包括的な対応が必要です。
