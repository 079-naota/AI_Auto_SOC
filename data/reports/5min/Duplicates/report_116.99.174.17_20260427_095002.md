# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 09:52:39

---

## SOC分析レポート

### 1. 攻撃の概要と目的

このログは、攻撃元IPアドレス `116.99.174.17` から、当社のSSHハニーポット（Cowrie）に対するブルートフォース攻撃/辞書攻撃の記録です。攻撃者は、短期間に複数の異なるユーザー名とパスワードの組み合わせを使用して、SSHサービスへの不正ログインを試みていました。

攻撃の主な目的は、SSHサービスへの不正アクセス権の取得であると推測されます。ログには、`root` ユーザーに対して複数の異なるパスワード (`admin`, `@`, `root123`, `12345`, `abcd1234`, `ipscan`, `alpine`) でのログイン成功が繰り返し記録されています。ログイン成功後には、`direct-tcpip.request` および `direct-tcpip.data` のアクティビティが複数回確認されており、これは攻撃者がリモートポートフォワーディングを確立したり、ログイン後に何らかのマルウェアのダウンロード、スクリプトの実行、または別のホストへの接続を試みたりしている可能性を示唆しています。これは単なるログイン試行に留まらず、さらなるシステム侵害を意図した行動と考えられます。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース攻撃/辞書攻撃:** 大量の一般的なユーザー名（`admin`, `user`, `ubnt`, `root`, `config`, `squid`, `support`, `guest`, `ftpuser`, `nikita` など）と、よく使われるパスワード（`admin`, `user`, `ubnt`, `config`, `squid`, `support`, `guest`, `admin@123`, `password`, `1234`, `123456`, `default`, `12345`, `abcd1234`, `ipscan`, `alpine`, `asteriskftp`, `@`, `root123` など）の組み合わせを網羅的に試行しています。
    *   **自動化された攻撃:** 短時間で多数のログイン試行を行っており、手動ではなく、自動化されたスクリプトやツールが使用されていることが明らかです。
    *   **ログイン後の活動:** ログイン成功後に `direct-tcpip` のリクエストが発生していることから、攻撃者が認証突破後に、さらに他のシステムへの接続試行や、悪性ペイロードのダウンロード、コマンド&コントロール(C2)通信の確立など、次の段階の攻撃を実行しようとしている可能性が高いです。

*   **使用ツール:**
    *   Hydra、Medusa、NmapのSSHブルートフォーススクリプト（`ssh-brute`）などの汎用的な認証情報クラッキングツール、またはそれらをベースにしたカスタムスクリプトが使用されている可能性が高いです。
    *   `direct-tcpip` の活動から、ログイン後の自動スクリプトが組み込まれたマルウェア、またはボットネットクライアントの一部であることも考えられます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **複数のrootユーザーログイン成功:** 攻撃者は複数の異なる脆弱なパスワード（`admin`, `@`, `root123`, `12345`, `abcd1234`, `ipscan`, `alpine`）を使用して `root` ユーザーでのログインに成功しています。これは、実際のシステムに対してこれらのパスワードが使用されていた場合、システム管理者権限が完全に奪取される危険性があったことを意味します。
*   **ログイン後の活動:** ログイン成功後に `direct-tcpip` の通信を試みていることから、単なる偵察や認証情報収集に留まらず、システム内での足がかりの確立、マルウェアの展開、他のシステムへの横展開、またはC2チャネルの構築など、さらなる深刻な侵害を目的としている可能性が高いです。
*   **継続的な攻撃:** 短時間にわたって執拗にログイン試行を繰り返しており、攻撃者の意図の強さがうかがえます。
*   **一般的な脆弱性を悪用:** 攻撃で使用されているユーザー名とパスワードの組み合わせは、システム管理者によって変更されていないデフォルトの認証情報や、推測されやすい単純なパスワードを狙ったものであり、多くのシステムで存在する潜在的な脆弱性を悪用しています。

この攻撃はハニーポットに対するものであり、直接的なシステム侵害には至っていませんが、もし当社の運用環境に同様の脆弱なSSHサービスが存在した場合、深刻な被害につながる可能性が極めて高かったため、「高」と評価します。

### 4. 推奨アクション

以下の推奨アクションを実施し、類似の攻撃に対する防御を強化してください。

1.  **攻撃元IPのブロック:** 攻撃元IPアドレス `116.99.174.17` を、エッジファイアウォール、IDS/IPS、またはSSHサーバーのホストベースファイアウォール (e.g., `iptables`, `firewalld`) でブロックリストに追加し、今後の接続を拒否します。
2.  **SSH認証の強化:**
    *   **パスワードポリシーの厳格化:** 推測困難で複雑なパスワード（大文字、小文字、数字、記号を含む最低12文字以上）の使用を全ユーザーに義務付けます。
    *   **公開鍵認証の強制:** パスワード認証を無効化し、公開鍵認証のみを許可することを強く推奨します。これにより、ブルートフォース攻撃のリスクを大幅に軽減できます。
    *   **多要素認証 (MFA) の導入:** 可能であれば、SSHログインにMFAを導入し、セキュリティを一層強化します。
    *   **rootユーザーのSSHログイン禁止:** `root` ユーザーの直接的なSSHログインを禁止し、一般ユーザーでログイン後、`sudo` を利用して特権操作を行う運用に切り替えます。
3.  **SSHサービス設定の最適化:**
    *   **デフォルトポートの変更:** SSHの標準ポートである22番ポートを、認知されにくい別のポート番号に変更することを検討します（ポートスキャンには効果が薄いものの、自動化された攻撃スクリプトの一部を回避できる場合があります）。
    *   **アクセス元IPアドレスの制限:** 必要最小限のIPアドレスからの接続のみを許可するよう、ファイアウォールルールを設定します（ホワイトリスト方式）。
    *   **`sshd_config` の強化:** `PermitRootLogin no`, `PasswordAuthentication no`, `AllowUsers` 等の設定を適切に行い、脆弱な設定を排除します。
    *   **Fail2banなどの導入:** 連続したログイン失敗を検知し、一時的にIPアドレスをブロックするツール（Fail2banなど）を導入します。
4.  **既存システムに対する監査:**
    *   当社のネットワーク内に、今回の攻撃で成功したユーザー名とパスワードの組み合わせ（特に `root` + `admin`, `root` + `@`, `root` + `12345`, `root` + `alpine` など）を使用しているSSHサービスがないか、緊急で棚卸しと監査を実施します。該当する場合は直ちにパスワードを変更してください。
    *   一般的なユーザー名（`admin`, `user`, `ubnt` など）とデフォルトパスワードを使用しているシステムがないか、脆弱性診断を実施します。
5.  **ログ監視の継続:** Cowrieなどのハニーポットログを含む、すべてのSSH認証ログを継続的に監視し、異常なログイン試行や成功を早期に検知できる体制を維持します。
