### 1. 攻撃の概要と目的

攻撃元IPアドレス `190.244.39.232` から、約40分間にわたり当社のSSHサービス（Cowrieハニーポット）に対する組織的なブルートフォース攻撃が確認されました。この攻撃者は、複数の異なるユーザー名とパスワードの組み合わせを試行し、特に `root` ユーザーに対して複数回にわたりログインに成功しています。

ログイン成功後、攻撃者は以下の行動を試みています。

*   **永続化の確立:** `root` ユーザーのホームディレクトリ配下にある `.ssh/authorized_keys` ファイルに特定のSSH公開鍵を追加し、パスワード認証なしでの将来的なアクセス経路（バックドア）を確立しようとしました。この公開鍵は全ての成功セッションで同一のものです。
*   **防御回避の試み:** `.ssh` ディレクトリに対して、`chattr -ia` および `lockr -ia` コマンドを実行しようとしています。これは、システムのファイル属性を変更してセキュリティ対策を無効化するか、特定のロック機構を操作することで、自身の活動を隠蔽・永続化しようとする意図が伺えます。

攻撃の主な目的は、システムへの不正アクセス、最高権限の奪取、および将来的なアクセスを可能にする永続的なバックドアの設置であると推測されます。これにより、システムへの完全な制御権を取得し、その後のマルウェア展開、データ窃取、または他の攻撃活動の足がかりとすることを狙っていたと考えられます。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォースアタック / 辞書攻撃:** ログには、`odoo`、`root`、`345gs5662d34`、`user0`、`shiv`、`oracle`、`ovpn`、`windows` など、様々なユーザー名とパスワードの組み合わせでのログイン試行が記録されています。特に `root` ユーザーに対しては、`135792468`, `3245gs5662d34`, `abcdefgh`, `Aa112211.`, `Gp123456`, `nPSpP4PBW0` といった複数の異なるパスワードでのログイン成功が確認されており、自動化されたブルートフォースツールや辞書攻撃が用いられている可能性が高いです。
    *   **永続化 (Persistence):** ログイン成功後、攻撃者は `cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa [公開鍵] >> .ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~` という一連のコマンドを実行しています。これは、Linuxシステムにおいて、SSH公開鍵認証を利用した永続的なバックドアを設置する一般的な手法です。
    *   **防御回避 (Defense Evasion):** 攻撃者は `cd ~; chattr -ia .ssh; lockr -ia .ssh` というコマンドを複数回試行しています。`chattr -ia` はファイルの不変属性を解除する試みであり、システム管理者が設定した変更防止策を無効化する目的があったと考えられます。`lockr` は標準的なLinuxコマンドではなく、`command.failed` と記録されていることから、攻撃者が意図した処理（例えば、特定のマルウェアの一部や、未知のツール）が実行できなかったか、コマンド自体がシステムに存在しないかのいずれかであると考えられます。

*   **使用ツール (推測):**
    *   **SSHブルートフォースツール:** Hydra, Medusa, Nmapのssh-bruteスクリプトなど、広く利用されている自動化されたSSHパスワードクラッキングツールが使用された可能性があります。
    *   **カスタムスクリプトまたはボットネットの一部:** ログイン成功後のコマンド実行パターンが非常に定型的であることから、攻撃者は手動で操作しているのではなく、事前に準備されたスクリプトが自動的に実行されていると考えられます。この攻撃元IPアドレスは、より大規模なボットネットの一部として活動している可能性も排除できません。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

1.  **認証突破の成功:** Cowrieハニーポット上ではあるものの、`root` ユーザーとして複数回ログインに成功している記録があります。これは実際のシステムであれば、攻撃者が最高権限でのアクセスを完全に確立した状態を意味し、極めて深刻な侵害につながります。
2.  **永続化の試み:** 攻撃者はSSH公開鍵を設置することで、システムへの永続的なアクセス経路を確立しようとしています。これにより、パスワードが変更されたとしても、攻撃者は秘密鍵を用いて継続的にシステムにアクセスできる状態を作り出すことが可能になります。
3.  **最高権限の標的:** `root` ユーザーを直接的に標的とし、その権限での永続化を図っていることから、攻撃者の意図がシステムに対する完全な制御権の奪取にあることが明確です。
4.  **自動化された攻撃:** 複数の異なるパスワード試行と、ログイン成功後の定型的なコマンド実行から、この攻撃が自動化されたボットネットまたはマルウェアによるものである可能性が高いです。これは、特定の脆弱性を狙ったものではなく、インターネットに公開された広範なシステムを無差別にスキャンし、脆弱な認証情報を探索する一般的な攻撃パターンに合致します。
5.  **防御回避の意図:** `chattr` コマンドによる属性変更の試みは、システム管理者が設定した防御メカニズムを迂回しようとする明確な意図を示しています。

### 4. 推奨アクション

このログはハニーポットのものですが、もしこれが実際のプロダクションシステムで発生していた場合、深刻な侵害が発生していた可能性が高いため、以下の対策を速やかに実施することを強く推奨します。

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPアドレスのブロック:** 攻撃元IPアドレス `190.244.39.232` からの全てのSSH接続を、ファイアウォールまたはIPS/IDSで即座にブロックしてください。
2.  **パスワードの強制変更:** ログに記録されたすべての`root`ユーザーの成功パスワード（`135792468`, `3245gs5662d34`, `abcdefgh`, `Aa112211.`, `Gp123456`, `nPSpP4PBW0`）および他の推測が可能なパスワードは、即座に複雑で推測困難なものへ変更してください。また、他の特権ユーザーや管理者アカウントのパスワードも念のため変更を推奨します。

    *   SSH設定ファイル (`/etc/ssh/sshd_config`) の `AuthorizedKeysFile` 設定が意図しないパスを指していないか確認してください。
4.  **システムログの確認:** 実際のサーバー上のSSHログ (`/var/log/auth.log` や `journalctl` など) を確認し、ハニーポットログに記録された成功したログイン試行と同じタイムスタンプで、実際のシステムでも同様のログイン成功が記録されていないか確認してください。また、公開鍵の追加や`chattr`コマンド実行の痕跡がないか詳細に調査してください。
5.  **フォレンジック調査の実施:** もし実際にログイン成功の兆候や不正な活動の痕跡が見られる場合、即座に侵害された可能性のあるシステムのイメージを取得し、詳細なフォレンジック調査を実施して、さらなる活動（マルウェアの設置、データ流出、アカウント作成、設定変更など）がないかを確認してください。

#### 長期的な対策 (Long-term Measures):

1.  **多要素認証 (MFA) の導入:** SSH接続に多要素認証 (MFA) を必須とし、パスワード認証のみによるアクセスを制限することで、ブルートフォース攻撃に対する耐性を大幅に向上させます。
2.  **SSHパスワード認証の無効化と公開鍵認証の徹底:** 可能であれば、SSHのパスワード認証を無効化し、信頼できる公開鍵認証のみを許可する設定に変更します。公開鍵は厳重に管理される必要があります。
3.  **強力なパスワードポリシーの適用:** 辞書攻撃やブルートフォース攻撃に耐えうる、複雑で長いパスワード要件を強制します。定期的なパスワード変更も検討してください。
4.  **ログイン試行回数の制限:** Fail2Banなどのツールを導入し、一定回数以上のログイン失敗があった場合にIPアドレスを一時的または恒久的にブロックする設定を行います。
5.  **不要なポートの閉鎖とアクセス制御の強化:** SSHサービスが必要な場合を除き、外部からのアクセスを制限します。信頼できる特定のIPアドレス範囲からのみSSHアクセスを許可するなどのアクセス制御を強化します。
6.  **ログ監視の強化:** 不正なログイン試行、特に `root` ユーザーに対する試行や、認証成功後の異常なコマンド実行を検出するためのログ監視とアラートシステムを強化します。SIEM (Security Information and Event Management) システムを活用し、リアルタイムでの脅威検知能力を向上させます。
7.  **定期的なセキュリティ監査:** システム設定の脆弱性や不正な変更がないかを定期的に監査し、速やかに修正を行います。
8.  **セキュリティ教育:** ユーザーに対して、強力なパスワードの重要性、フィッシングへの注意喚起、疑わしい活動の報告など、基本的なセキュリティ意識向上トレーニングを継続的に実施します。
