# SOC自動分析レポート: 102.219.126.124

**生成日時:** 2026-04-26 11:36:24

---

## SOC分析レポート

### 1. 攻撃の概要と目的

2026年4月26日のUTC 07:33:56から07:57:09にかけて、攻撃元IPアドレス `102.219.126.124` からSSHサービスに対する攻撃が複数回観測されました。

この攻撃は、まずSSHポートへの接続試行から始まり、様々なユーザー名とパスワードの組み合わせを使用したブルートフォース攻撃（または辞書攻撃）によって `root` ユーザーの認証情報を特定しようとしました。最初のログイン失敗から約15分後、`root` ユーザーでログインに成功しています。

ログイン成功後、攻撃者は以下の行動を試みました。
1.  `.ssh` ディレクトリの属性変更（`chattr -ia .ssh; lockr -ia .ssh`）を試みましたが、これはハニーポット環境では失敗しました。
2.  `root` ユーザーのホームディレクトリにある既存の `.ssh` ディレクトリを削除し、新たに作成しました。
3.  攻撃者自身のSSH公開鍵（`ssh-rsa AAAAB3NzaC1yc2E... mdrfckr`）を `~/.ssh/authorized_keys` ファイルに追加し、パーミッションを適切に設定しました。
4.  その後も、異なるパスワードで `root` ユーザーとして繰り返しログインを試みたり、特定のユーザー名（`345gs5662d34`）でのログイン試行を行っています。

攻撃の主な目的は、SSHブルートフォース攻撃によりシステムへの不正ログインを成功させ、その後 `authorized_keys` ファイルに自身の公開鍵を追加することで、パスワード認証なしで永続的なアクセス経路（バックドア）を確立し、システムに対する完全な制御権を確保することであると推測されます。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **SSH ブルートフォース攻撃 / 辞書攻撃**: 複数の一般的なユーザー名（`drcom`, `ubuntu`, `user`）と、多数の異なるパスワード（`1qaz@WSX3edc$RFV!@`, `ali123456`, `Root8`, `Aa112211.`, `Config123`, `Aa123321` など）を `root` ユーザーに対して試行していることから、自動化されたブルートフォース攻撃または辞書攻撃が実施されたと考えられます。
    *   **永続化 (Persistence)**: ログインに成功した後、`~/.ssh/authorized_keys` ファイルに自身の公開鍵を追加することで、将来的にパスワードなしでシステムに再アクセスできる状態を構築しようとしています。これは、初期アクセス後にシステムへの永続的な足がかりを確保するための一般的な手法です。
*   **使用ツール**:
    *   **SSHクライアント**: SSHプロトコルで接続するための標準的なクライアント。
    *   **ブルートフォースツール**: `Hydra` や `Medusa`、あるいは`Nmap`のSSH関連スクリプトのような、SSH認証情報を自動的に試行するツールが使用された可能性が高いです。
    *   **自動化スクリプト / ボットネット**: 一連のログイン試行とコマンド実行がほぼ一定のパターンと間隔で繰り返されていることから、攻撃は自動化されたスクリプト、またはボットネットの一部として実行されている可能性が高いと推測されます。

### 3. 脅威レベルとその理由

*   **脅威レベル: 高 (High)**

*   **理由**:
    *   **`root` 権限でのログイン成功**: 攻撃者がシステムの最高権限である `root` アカウントで複数回ログインに成功しています。これにより、もしこれが本物のシステムであった場合、システムに対する完全な制御権を奪取される可能性が極めて高いことを意味します。
    *   **永続的アクセスの確立試行**: 攻撃者はログイン後、自身のSSH公開鍵を `authorized_keys` に追加することで、バックドアを設置しようとしました。これにより、パスワードが変更されたとしても、攻撃者は依然としてSSH経由でシステムにアクセスできる永続的な経路を確保できることになります。
    *   **自動化された攻撃パターン**: 攻撃が自動化されたプロセスによって行われていることから、これは単一のターゲットに限定されたものではなく、インターネット上の脆弱なSSHサービスを探索し、広く侵害しようとする大規模なキャンペーンの一部である可能性が高いです。
    *   **潜在的な影響の重大性**: 攻撃者が `root` 権限でシステムにアクセスできた場合、情報窃取、データの改ざん・破壊、マルウェアのインストール、他のシステムへの攻撃の踏み台化など、甚大な被害が発生するリスクがあります。

### 4. 推奨アクション

1.  **攻撃元IPアドレスのブロック**:
    *   直ちにファイアウォールやIDS/IPSで攻撃元IPアドレス `102.219.126.124` からの全ての通信をブロックします。
2.  **SSHパスワード認証の無効化と公開鍵認証への移行**:
    *   全てのSSHサーバにおいて、パスワード認証を無効化し、パスフレーズ付きの強力な公開鍵認証のみを許可するように設定を変更します。
    *   `PermitRootLogin no` を設定し、`root` ユーザーでの直接SSHログインを禁止します。通常のユーザーでログイン後、`sudo` を利用する運用に切り替えます。
3.  **既存のSSHキーの確認と不審な鍵の削除**:
    *   全てのユーザーアカウント、特に `root` ユーザーの `~/.ssh/authorized_keys` ファイルの内容を確認します。ログに記録された公開鍵（コメント `mdrfckr` の鍵 `ssh-rsa AAAAB3NzaC1yc2E...`）が存在しないか確認し、発見した場合は直ちに削除します。
4.  **強固なパスワードポリシーの適用とアカウントロックアウト設定**:
    *   全てのシステムユーザーに対し、複雑性要件（大文字、小文字、数字、記号の組み合わせ）を満たす、推測困難なパスワードの使用を強制します。
    *   複数回のログイン失敗でアカウントが一時的または永続的にロックされるようなアカウントロックアウトポリシーを設定し、ブルートフォース攻撃への耐性を高めます。
5.  **多要素認証 (MFA) の導入**:
    *   可能な限り、SSH接続に多要素認証 (MFA) を導入し、認証情報の漏洩や推測による不正アクセスリスクを低減します。
6.  **SSHログイン監査の強化**:
    *   SSHログインログを集中ログ管理システム（SIEMなど）に転送し、異常なログイン試行（例: 多数のログイン失敗、未知のIPからの接続）をリアルタイムで検知・アラートするルールを設定します。
7.  **システム全体の脆弱性スキャンとセキュリティパッチ適用**:
    *   OSやSSHサーバソフトウェア（OpenSSHなど）を含む全てのシステムコンポーネントを最新の状態に保ち、既知の脆弱性が悪用されないように定期的にセキュリティパッチを適用します。
    *   定期的な脆弱性スキャンを実施し、潜在的な弱点を特定・修正します。
8.  **脅威インテリジェンスの活用**:
    *   攻撃元IPアドレスやSSH公開鍵のフィンガープリントを脅威インテリジェンスプラットフォームで検索し、既知の脅威アクターやキャンペーンに関連しているか調査します。
