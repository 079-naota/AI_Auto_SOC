### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース/辞書攻撃:** 大量のユーザー名とパスワードの組み合わせが短時間で試行されており、自動化されたブルートフォース攻撃であると断定できます。特に`root`ユーザーをターゲットに、複数の一般的な、または推測されやすいパスワードが試されています。
    *   **永続化:** ログイン成功後、`~/.ssh/authorized_keys`ファイルに攻撃者のSSH公開鍵を埋め込むことで、パスワード認証なしでの継続的なアクセス（バックドア）を確立しようとしています。これは初期アクセス後の活動として非常に典型的です。
    *   **防御回避の試み:** `chattr -ia .ssh`コマンドは、Linuxの拡張ファイル属性（immutableやappend-onlyなど）を無効にし、`.ssh`ディレクトリの変更を可能にしようとするものです。これは、既存のファイルシステム保護メカニズムを迂回する試みと見られます。
*   **使用ツール:**
    *   **自動化されたSSHクライアント/ブルートフォースツール:** 複数のログイン試行が連続して行われていること、ログイン成功後に一連のコマンドが迅速に実行されていることから、Hydra, NmapのNSEスクリプト, Metasploitなどの既製のブルートフォースツール、または攻撃者が作成したカスタムスクリプトが使用されている可能性が高いです。
    *   **シェルスクリプト:** ログイン成功後に実行されるコマンド (`cd ~; chattr -ia .ssh; lockr -ia .ssh` および公開鍵の埋め込みコマンド) は、定型化されており、自動化されたスクリプトの一部であると考えられます。`lockr`コマンドが`cowrie.command.failed`となっていることから、攻撃者のスクリプトが標的の環境に完全に適合していない可能性、または特定のLinuxディストリビューション向けに作成されたものではない可能性が示唆されます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**
1.  **root権限での複数回ログイン成功:** ハニーポット上であるものの、攻撃者は最も高い権限を持つ`root`アカウントで複数回のログインに成功しています。実環境であれば、これはシステム全体の制御を完全に奪取された状態に相当します。
2.  **永続化メカニズムの確立試行:** ログイン成功後、すぐにSSH公開鍵を埋め込むことで、恒久的なバックドアを設置しようとしています。これにより、攻撃者はいつでもパスワードなしでシステムに再侵入できるようになります。
3.  **防御回避の試み:** `chattr`コマンドを使用してファイル属性の変更を試みるなど、攻撃者は検出を回避し、システムの防御を無力化しようとする積極的な意図が見られます。
4.  **自動化された攻撃:** 攻撃は自動化されており、特定のターゲットに限定されず、広範囲のシステムを無差別にスキャンし、脆弱なSSHサービスを持つホストを狙っている可能性が高いです。これにより、今後も同様の攻撃が継続するリスクがあります。
5.  **既知の攻撃パターンとの一致:** この攻撃パターン（ブルートフォース -> ログイン成功 -> SSHキー埋め込み）は、多くのマルウェア（特にボットネットのエージェント）や攻撃グループによって利用される一般的な手法であり、深刻な被害につながる可能性があります。

### 4. 推奨アクション

このログはハニーポットからのものであるため、実際のシステム被害は発生していませんが、もしこれが実運用環境のシステムであれば、以下の緊急対応および恒久的な対策が直ちに必要です。

**A. 緊急対応（Immediate Actions - If this were a real system）:**

1.  **システム隔離:** 影響を受けた可能性のあるシステムを、ネットワークから即座に隔離し、攻撃者の活動を遮断し、さらなる被害拡大を阻止します。
2.  **rootパスワードのリセット:** ログに記録されたすべての`root`アカウントのパスワード (`abcd1234.`, `Password.123`, `MoeClub.org`, `a12348765`, `abc@123456`, `3245gs5662d34`) および他のログイン成功した可能性のあるアカウントのパスワードを、予測困難な強力なものにリセットします。
3.  **SSH公開鍵の削除:** `~/.ssh/authorized_keys`ファイルに攻撃者によって埋め込まれた公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK_rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) を特定し、削除します。
4.  **システムイメージの取得:** 詳細なフォレンジック分析を行うため、システムのディスクイメージを取得します。
5.  **マルウェアスキャンと整合性チェック:** システム全体でマルウェアスキャンを実行し、不正なファイルやバックドアが導入されていないか徹底的に確認します。また、システムバイナリや設定ファイルの整合性を検証します。
6.  **ログの深掘り分析:** 他のシステムログ (認証ログ、プロセスログ、ネットワーク接続ログなど) を詳細に分析し、攻撃の全容、影響範囲、および他の不正な活動がないかを確認します。

**B. 恒久的な対策（Long-term Mitigations）:**

1.  **SSH設定の強化:**
    *   **多要素認証 (MFA) の導入:** SSH接続にMFAを必須とし、パスワード推測によるリスクを大幅に低減します。
    *   **パスワード認証の無効化:** 可能であれば、パスワード認証を無効にし、より安全なSSH公開鍵認証に移行します。この際、鍵のパーミッションが適切に設定されていることを確認します。
    *   **rootログインの禁止:** SSH経由での直接の`root`ログインを禁止し、一般ユーザーでログイン後に`sudo`を使用する運用に変更します。
    *   **PermitEmptyPasswords no:** 空のパスワードでのログインを禁止します。
    *   **AllowUsers / DenyUsers:** 許可されたユーザーのみSSHアクセスを許可する設定を行います。
    *   **SSHポートの変更:** 標準のSSHポート (22番) を変更し、一般的なスキャンからの可視性を低減します。
2.  **強力なパスワードポリシーの強制:** すべてのシステムユーザーに対して、複雑性、長さ、および定期的な変更を強制する強力なパスワードポリシーを導入します。
3.  **不要なアカウントの削除/無効化:** デフォルトアカウントや未使用のアカウントを特定し、削除または無効化します。
4.  **アクセス制御の強化:**
    *   **ファイアウォール/セキュリティグループ:** 信頼できるIPアドレス範囲からのSSHアクセスのみを許可するようにファイアウォールまたはセキュリティグループを設定します。
    *   **レートリミット/接続制限:** 一定時間内のSSHログイン試行回数を制限し、ブルートフォース攻撃の効果を低減します。
5.  **侵入検知システム (IDS/IPS) の導入と調整:** ブルートフォース攻撃や不正なアクセス試行を検知し、自動的にブロックするIDS/IPSを導入または既存システムの設定を最適化します。
6.  **脅威インテリジェンスの活用:** 攻撃元IPアドレス `103.172.205.139` を脅威インテリジェンスデータベースで確認し、ファイアウォールのブロックリストに追加することを検討します。また、攻撃者が使用したSSH公開鍵 (`mdrfckr`のもの) もセキュリティ対策でブラックリスト化を検討します。
7.  **ファイル整合性モニタリング (FIM):** `~/.ssh/authorized_keys`を含む重要な設定ファイルやシステムバイナリの変更を監視するため、FIMツールを導入します。
8.  **セキュリティ意識向上トレーニング:** システム管理者およびユーザーに対し、セキュリティベストプラクティス、フィッシング対策、不審な活動の報告方法などに関する定期的なトレーニングを実施します。
9.  **ハニーポットの継続的な運用:** Cowrieハニーポットは効果的に攻撃を捕捉しているため、引き続きデコイとして運用し、新たな攻撃トレンドや攻撃者の戦術・技術・手順 (TTPs) の監視を継続します。
