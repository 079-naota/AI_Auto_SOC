# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 03:42:04

## SOCアナリストレポート

### 攻撃ログ分析結果

#### 1. 攻撃の概要と目的

本レポートは、Cowrieハニーポットによって捕捉されたSSHサービスへの攻撃ログを分析したものです。このログは、攻撃者が実際にシステムに侵入したと認識している状況を示しており、もし実システムで発生した場合、深刻な侵害につながる可能性を秘めています。

*   **攻撃元IPアドレス**: `116.99.174.17`
*   **攻撃対象サービス**: SSH (ポート22)
*   **攻撃の種類**: 主にSSHブルートフォース攻撃 / パスワードスプレー攻撃
*   **攻撃期間**: 2026-04-27T02:13:34Z から 2026-04-27T02:31:53Z (約18分間)
*   **攻撃の目的**: 不正な認証情報を試行し、SSHサービスへの不正ログインを試みることで、システムへのアクセス権を確立すること。ログイン成功後には、SSHポートフォワーディング（`direct-tcpip`）を試みていることから、内部ネットワークへの横展開、マルウェアの展開、情報の窃取、あるいはシステムの悪用（例：DDoSボット、仮想通貨マイニング）を企図している可能性が高いです。

#### 2. 推測される手法・使用ツール

攻撃元IPアドレス `116.99.174.17` は、短期間に多数の認証情報を繰り返し試行しています。

*   **認証情報の試行パターン**:
    *   `admin`, `user`, `root`, `guest`といった一般的なユーザー名と、`admin`, `user`, `password`, `1234`, `123456`, `default`などの単純なパスワードの組み合わせを多用しています。
    *   `ubnt`, `config`, `squid`, `support`, `ftpuser`, `nikita`といった、特定のデバイスやサービスに関連する可能性のあるユーザー名も試行しており、ターゲットシステムのプロファイリングを行っている可能性も示唆されます。
    *   特に`root`ユーザーに対する試行が目立ち、複数回異なるパスワードでのログインが成功しています（例：`root/admin`, `root/@`, `root/root123`, `root/12345`, `root/abcd1234`, `root/ipscan`, `root/alpine`）。これは攻撃者が最も権限の高いアカウントを狙っていることを明確に示しています。
*   **ログイン成功後の挙動**:
    *   ログインに成功したセッション（`root`ユーザーでのログインが複数回）においては、直後に`cowrie.direct-tcpip.request`イベントが記録されています。これはSSHポートフォワーディングチャネルの確立を試みていることを示しており、攻撃者が認証突破後に、内部ネットワークのリソースへのアクセスや、外部のC2サーバーとの通信チャネル確立を試みている可能性が高いです。
*   **使用ツール**:
    *   攻撃の頻度、速度、および多様な認証情報の試行パターンから、手動ではなく自動化されたブルートフォースツール（例：Hydra, Nmapの`ssh-brute`スクリプト, Medusaなど）が使用されていると強く推測されます。

#### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

1.  **執拗なブルートフォース攻撃**: 短時間にわたる連続的かつ広範なユーザー名/パスワードの試行は、システムへの不正アクセスを強く意図した攻撃であり、一般的なインターネット上のノイズとは一線を画します。
2.  **複数回のログイン成功**: ログには`root`ユーザーに対する7回のログイン成功が記録されています。もしこれが実稼働中のシステムであった場合、最高権限での侵入を許したことになり、システム全体が完全に侵害された状態となります。
3.  **ログイン後のポートフォワーディング試行**: 認証成功後に`direct-tcpip`リクエストが頻繁に発生していることは、攻撃者が単なる認証情報の確認に留まらず、次のステップとして内部ネットワークへの侵入やデータの外部転送、あるいはマルウェアのインストールなどを計画していることを示唆しています。
4.  **自動化された攻撃**: 人手によるものではなく、自動化されたスクリプトやボットネットの一部による攻撃である可能性が高く、このような攻撃は広範なターゲットに対して行われ、成功すれば急速に被害が拡大する恐れがあります。

#### 4. 推奨アクション

この攻撃は、組織のSSHサービスに対する潜在的なリスクを浮き彫りにするものです。以下の推奨アクションを速やかに実施し、セキュリティ態勢を強化することを強く推奨します。

**A. 即時対応 (Immediate Actions)**

1.  **攻撃元IPアドレスのブロック**: ファイアウォールやIPS/IDSにて、攻撃元IPアドレス `116.99.174.17` からのSSH (ポート22) への接続を即座にブロックしてください。
2.  **実稼働中のSSHサーバログの確認**: 組織内のすべての実稼働中のSSHサーバの認証ログ（`auth.log`など）を確認し、同時刻帯にこのIPアドレスからのログイン試行、特にログイン成功が記録されていないかを緊急で調査してください。
3.  **認証情報の監査と変更**:
    *   ログに記録されたログイン成功クレデンシャル（例: `root/admin`, `root/@`, `root/root123`, `root/12345`, `root/abcd1234`, `root/ipscan`, `root/alpine`）が、もし実システムで利用されている場合、該当するユーザーアカウントのパスワードを直ちに、かつ強力なものに変更してください。
    *   これらのアカウントが不正利用された形跡がないか、システムログや監査ログを詳細に確認してください。
4.  **内部ネットワークの確認**: ログイン成功後の`direct-tcpip`試行のログを鑑み、もし実システムで侵害が発生していた場合、内部ネットワークでの不審な通信（特にSSHトンネル経由の通信）やマルウェアの活動がないか、ネットワークトラフィックおよびエンドポイントを詳細に調査してください。

**B. 予防的措置 (Preventive Measures)**

1.  **SSHセキュリティの強化**:
    *   **パスワード認証の無効化**: 可能な限り、SSH接続においては公開鍵認証のみを許可し、パスワード認証を無効にしてください。
    *   **多要素認証 (MFA) の導入**: パスワード認証を許可する必要がある場合は、すべてのアカウントに対してMFAを必須にしてください。
    *   **強力なパスワードポリシーの適用**: パスワードの複雑性（長さ、文字種、ランダム性）を強制し、定期的な変更を促すポリシーを徹底してください。
    *   **`root`ログインの制限**: `root`ユーザーでの直接ログインを禁止し、一般ユーザーでログイン後に`sudo`コマンドを使用する運用に切り替えてください。
    *   **SSHポートの変更**: 標準のポート22ではなく、あまり一般的でないポートを使用することで、スクリプトによる自動攻撃の一部を防ぐことができます（ただし、ポートスキャンによる発見の可能性は残ります）。
    *   **Fail2Banなどの導入**: 連続したログイン失敗を検知し、自動的にIPアドレスを一時的または永続的にブロックするシステムを導入してください。
    *   **アクセス元IPアドレスの制限**: 信頼できる特定のIPアドレス範囲からのみSSH接続を許可するよう、ファイアウォールルールを設定してください。
2.  **監視体制の強化**:
    *   SSH認証ログ（成功/失敗）をSIEM（Security Information and Event Management）システムなどのログ管理ソリューションで集約・監視し、短時間での複数回失敗や、地理的に異常な場所からのアクセスなど、不審なログイン試行パターンを検知できるようアラートを設定してください。
    *   SSHポートフォワーディング（`direct-tcpip`）やその他のシェルコマンド実行ログも詳細に記録・監視し、不審な活動を検知できる体制を構築してください。
3.  **脆弱性管理**: 定期的にシステムやアプリケーションの脆弱性スキャンを実施し、OSやSSHサーバソフトウェアを含むすべてのコンポーネントを最新の状態に保ち、既知の脆弱性を修正してください。
4.  **セキュリティ意識向上トレーニング**: 従業員に対して、フィッシング詐欺やソーシャルエンジニアリング攻撃に関するセキュリティ意識向上トレーニングを継続的に実施し、認証情報の漏洩リスクを低減してください。
