## SOCアナリストレポート：SSH不正アクセス試行および永続化の試みに関する分析

### 1. 攻撃の概要と目的

このログは、攻撃元IPアドレス `190.244.39.232` からのSSHサービスに対する連続的な攻撃活動を示しています。攻撃者はブルートフォース/辞書攻撃によりSSHへの不正ログインを試み、複数の異なるパスワードを試した後、`root`ユーザーとして認証に成功（ハニーポット上での模擬成功）したと認識しています。

ログイン成功後、攻撃者はシステムへの永続的なアクセス経路を確立することを目的とし、以下の行動を試みました。

*   既存のSSH設定をクリアし、自身のSSH公開鍵を`.ssh/authorized_keys`に追加。これにより、パスワードなしで将来的に再アクセスできるバックドアを設置しようとしました。
*   追加したファイルやディレクトリの属性を変更し、不正な変更から保護したり、削除を困難にしたりする目的で、`chattr`および`lockr`コマンドの使用を試みました（これらのコマンドはハニーポット環境では実行に失敗しています）。

攻撃の最終的な目的は、標的システムへの完全な制御を奪い、悪意のある活動（例：マルウェアの展開、C2通信、他のシステムへの踏み台利用など）を行うための準備と推測されます。

### 2. 推測される手法・使用ツール

*   **ブルートフォース/辞書攻撃**: 短期間に大量のユーザー名とパスワードの組み合わせを試行していることから、`hydra`や`medusa`などの自動化されたブルートフォースツール、またはカスタムスクリプトが使用された可能性が高いです。特に`root`ユーザーを標的としていることが顕著です。
*   **永続化 (Persistence)**: ログイン成功後、以下のコマンドによりSSH公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) を追加し、`.ssh`ディレクトリのパーミッションを変更しています。これは、パスワード認証を回避し、鍵認証によるアクセスを継続するための一般的なバックドア設置手法です。
    *   `cd ~ && rm -rf .ssh && mkdir .ssh && echo "..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`
*   **ファイル属性変更/改ざん防止の試み**: 以下のコマンドが試行されましたが、ハニーポット環境では失敗しています。
    *   `cd ~; chattr -ia .ssh; lockr -ia .ssh`
    *   `chattr`はLinuxの拡張ファイル属性を変更するコマンドで、`+i`属性を設定するとファイルやディレクトリがimmutable（変更不可）になります。`-i`はその解除です。攻撃者は永続化のために設置した`.ssh`ディレクトリが容易に削除されないよう、`+i`属性を設定しようとした可能性があります。
    *   `lockr`は標準的なLinuxコマンドではなく、特定のマルウェアやカスタムスクリプトの一部であるか、Typoの可能性が考えられます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **最高権限アカウントの侵害**: 攻撃者は`root`ユーザーでのログインに成功したと認識しており、これはLinuxシステムにおける最高権限アカウントの侵害を意味します。実システムでこれが成功した場合、システム全体が乗っ取られる深刻なリスクがあります。
*   **永続化の試み**: SSH公開鍵によるバックドアの設置は、攻撃者がシステムへのアクセスを長期的に維持し、防御側の対策を回避しようとする明確な意図を示しています。一度鍵が登録されると、パスワード変更などの対策では排除が困難になります。
*   **広範囲な辞書攻撃**: 攻撃者は`odoo`, `root`, `345gs5662d34`, `user0`, `shiv`, `oracle`, `ovpn`, `windows`など、様々なユーザー名とパスワードの組み合わせを試しており、インターネットに露出しているSSHサービスに対する組織的なスキャンおよび攻撃活動の一部である可能性が高いです。
*   **悪意あるコマンドの実行**: ログイン後に`rm -rf .ssh`で既存のSSH設定を破壊し、自身の公開鍵を設置する行為は、システムのセキュリティ構成を積極的に改ざんするものです。また、`chattr`による保護試みも悪質性が高いです。

### 4. 推奨アクション

このログはハニーポット（Cowrie）のものであり、実際のシステムへの被害は発生していませんが、同様の攻撃が実システムに対して行われる可能性は極めて高いと判断できます。以下の対策を速やかに実施することを強く推奨します。

1.  **攻撃元IPアドレスのブロック**:
    *   ファイアウォール、IDS/IPS、WAFなどのセキュリティデバイスで、攻撃元IPアドレス `190.244.39.232` からのSSH通信を即座にブロックしてください。
    *   必要に応じて、このIPアドレスが属する国やASからの不審な通信についても調査し、ブロックを検討してください。
2.  **SSHサービス設定の緊急強化**:
    *   **rootユーザーの直接ログイン禁止**: `/etc/ssh/sshd_config` ファイルで `PermitRootLogin no` を設定し、rootユーザーの直接SSHログインを禁止してください。特権操作は一般ユーザーでログイン後、`sudo`を使用して行う運用を徹底してください。
    *   **パスワード認証の無効化**: 可能な限り `PasswordAuthentication no` を設定し、鍵認証のみを許可してください。
    *   **強力なパスワードポリシーの適用**: パスワードの複雑性、長さ、定期的な変更を強制するポリシーを全アカウントに適用してください。
    *   **不要なアカウントの削除/無効化**: ログに試行された`odoo`, `user0`, `shiv`, `oracle`, `ovpn`, `windows`などのアカウントがシステムに存在し、かつ不要であれば直ちに削除または無効化してください。
    *   **SSHポートの変更**: 標準ポート22番以外のポートを使用することも、攻撃のスキャン対象から一時的に外れる効果があります（根本的な解決策ではありません）。
3.  **多要素認証 (MFA) の導入**:
    *   SSHログインに多要素認証を必須とすることで、仮にパスワードが漏洩した場合でも不正アクセスを防ぐことができます。
4.  **侵入検知・防止システムの強化**:
    *   `fail2ban`などのツールを導入し、一定回数以上の不正ログイン試行を行ったIPアドレスを自動的にブロックする設定を導入してください。
    *   SIEM/ログ管理システムにおいて、SSHログインの成功/失敗、特に`root`ユーザーでのログインや不審なユーザーからのログイン試行に対するアラートを強化してください。
    *   `authorized_keys`ファイルへの予期せぬ変更、`chattr`コマンドの使用、その他の異常なファイル操作を監視するルールを設定してください。
5.  **既知の悪意あるSSH公開鍵の監視**:
    *   攻撃者が使用した公開鍵（レポートに記載されている文字列）をブラックリストに登録し、システム内の全ての`authorized_keys`ファイルにこの鍵が存在しないか定期的にスキャンする仕組みを導入してください。
6.  **定期的な脆弱性診断とペネトレーションテスト**:
    *   外部に公開されているサービス（SSHなど）に対して定期的に脆弱性診断やペネトレーションテストを実施し、潜在的な弱点を特定・修正してください。
