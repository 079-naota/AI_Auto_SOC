# SOC自動分析レポート: 171.104.143.176

**生成日時:** 2026-04-25 17:49:00

## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `171.104.143.176` から、対象システムへのSSHサービスに対するブルートフォースアタックが確認されました。最初の試行では `deploy/admin` という一般的な認証情報を使用しましたがログインに失敗しています。

その後、攻撃者は **`root` ユーザーの認証情報（`root/1234ASDF` および `root/QWEasd123456`）を何らかの方法で入手し、複数回ログインに成功**しています。

ログイン成功後、攻撃者は以下の目的で一連のコマンドを実行しました。

*   **永続的なアクセス経路の確立（Persistence）:**
    *   自身のSSH公開鍵を `~/.ssh/authorized_keys` に追加することで、パスワードなしでの再アクセスを試みています。
    *   `root` ユーザーのパスワードを攻撃者自身の管理下にあるパスワード（`dTTVY8OaDRX7` および `UfvraTsg18FW`）に変更しようとしています。
*   **競合排除および防御回避（Defense Evasion / Resource Hijacking）:**
    *   `/tmp` ディレクトリ内の `secure.sh` および `auth.sh` というスクリプトファイルを削除し、関連プロセスを強制終了しようとしています。これは、他の攻撃者によって設置された可能性のあるマルウェアを排除し、システムを独占的に利用しようとする意図を示唆します。
    *   `/etc/hosts.deny` ファイルをクリアすることで、アクセス制限を解除しようとしています。
*   **情報収集（Discovery）:**
    *   システムのCPU情報、メモリ情報、OSバージョン、アーキテクチャ、稼働中のプロセス、ディスク使用量、cronジョブ、ログインユーザーなど、多岐にわたるシステム情報を詳細に収集しています。これは、さらなる攻撃（例: マルウェアの展開、C2通信の確立、仮想通貨マイニング、ラテラルムーブメント）のための事前偵察と推測されます。

この一連の行動は、ターゲットシステムに対する完全なコントロールを確立し、将来的に悪意のある活動を行うための準備段階であると考えられます。

### 2. 推測される手法・使用ツール

*   **初期アクセス:**
    *   **SSHブルートフォースアタック:** `deploy/admin` でのログイン失敗ログから、一般的な認証情報を試していることが伺えます。
    *   **資格情報の悪用:** `root` ユーザーでのログイン成功から、攻撃者は事前に何らかの方法で有効なrootパスワード（辞書攻撃、クレデンシャルスタッフィング、あるいは別の情報漏洩など）を取得していたと推測されます。

*   **防御回避・競合排除:**
    *   **ファイル属性操作:** `chattr -ia .ssh` や `lockr -ia .ssh` を試行し、`.ssh` ディレクトリへの変更を困難にすることで、自身の設置したバックドアの永続性を高めようとしています（`lockr` は通常環境では存在しないコマンドであり失敗）。

*   **情報収集:**
    *   `cat /proc/cpuinfo`, `free -m`, `uname -a`, `lscpu`, `df -h`, `crontab -l`, `top`, `w`, `whoami` といった標準的なLinuxコマンドを多数使用し、システムの詳細な情報を収集しています。
*   **使用ツール:**
    *   **SSHクライアント:** SSH接続のために使用。
    *   **スクリプト/自動化ツール:** ログイン成功後の迅速かつ連続的なコマンド実行のパターンから、手動操作ではなく、自動化されたスクリプトやツール（例: Shell script, Python script, Metasploitなどのフレームワーク）が使用された可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

1.  **最高権限アカウントの侵害:** `root` ユーザーでのログインに成功しており、システムに対する完全な制御権が確立された状態です。これにより、攻撃者は任意のコマンドを実行し、システムの構成を変更し、データにアクセスまたは破壊することが可能です。
2.  **永続化の確立:** SSH公開鍵の設置および `root` パスワードの変更は、攻撃者がシステムへの恒久的なアクセス経路を確保しようとしていることを明確に示しています。これにより、一度排除されても容易に再侵入されるリスクがあります。
3.  **広範な偵察活動:** 攻撃者は、システムの詳細な情報（ハードウェア、OS、リソース、他のプロセス、cronジョブなど）を収集しています。これは、さらなる悪意のある活動（例: ボットネットへの組み込み、仮想通貨マイニング、データの窃取、システム破壊、ラテラルムーブメント）を実行するための計画段階である可能性が極めて高いです。
4.  **競合排除の動き:** 他のマルウェアを削除しようとする行動は、攻撃者がこのシステムを「独占」し、自身の目的のためにリソースを最大限に利用しようとしていることを示唆します。

これらの活動は、MITRE ATT&CKフレームワークの「初期アクセス」「実行」「永続化」「特権昇格」「防御回避」「資格情報アクセス」「情報収集」といった複数の攻撃フェーズに該当し、極めて悪質かつ深刻な侵害の兆候です。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPのブロック:** 攻撃元IPアドレス `171.104.143.176` を直ちにファイアウォール、IDS/IPS、またはWAFでブロックし、以降のアクセスを遮断してください。
2.  **SSHサービスの一時停止/アクセス制限:** SSHサービスへのインターネットからのアクセスを一時的に停止するか、信頼できるネットワーク/IPアドレスからのアクセスのみに厳しく制限してください。
3.  **rootパスワードの即時変更:** もしこのハニーポットログが実際の環境からのものであった場合、直ちに `root` ユーザーのパスワードを非常に強力なものに変更し、他の特権ユーザーアカウントのパスワードも変更を推奨します。
4.  **Authorized Keysの確認と削除:** `root` ユーザーの `~/.ssh/authorized_keys` ファイルを確認し、身に覚えのない公開鍵（特に `mdrfckr` のコメントが付いているもの）が存在する場合は、直ちに削除してください。

#### インシデントレスポンス (Incident Response):

1.  **詳細なログ調査:** 攻撃元IPからの他の接続、同じ時間帯の他の認証ログ、システムログ、アプリケーションログを詳細に調査し、攻撃者の活動範囲と影響を特定してください。
2.  **フォレンジック調査の実施:** 実際のシステムが侵害された場合は、専門家によるフォレンジック調査を実施し、攻撃者がシステムに行った全ての変更、設置されたマルウェア、新たなバックドア、データの窃取や改ざんの有無を徹底的に特定してください。
3.  **マルウェアスキャンとクリーンアップ:** システム全体を最新のアンチウイルス/マルウェア対策ツールでスキャンし、検出された脅威を隔離・削除してください。特に `/tmp` ディレクトリの内容を確認してください。
4.  **脆弱性スキャン:** システム全体に対して脆弱性スキャンを実施し、他の潜在的な侵入経路がないか確認し、特定された脆弱性を修正してください。
5.  **アカウントの棚卸し:** 不明なユーザーアカウントや、通常の業務で使用されない特権アカウントが存在しないか確認し、不要なアカウントは削除または無効化してください。

#### 予防策 (Proactive Measures):

1.  **強力なパスワードポリシーの徹底:** すべてのシステムアカウントに対し、複雑性、長さ、有効期限などを含む強力なパスワードポリシーを強制し、定期的なパスワード変更を義務付けてください。
2.  **SSH鍵認証の推奨とMFA導入:** パスワード認証を無効にし、より安全なSSH鍵認証を必須とすることを検討してください。さらに、可能であればSSHへの多要素認証（MFA）を導入し、セキュリティを強化してください。
3.  **SSHサービスへのアクセス制限:** SSHサービスをインターネットに直接公開せず、VPN経由でのアクセスを強制するか、ファイアウォールで信頼できる特定のIPアドレスからのアクセスのみを許可してください。
4.  **侵入検知・防御システム (IDS/IPS) の導入とルール強化:** 不審なログイン試行や、本レポートで確認されたような悪意のあるコマンドパターンを検知・ブロックできるIDS/IPSソリューションを導入・強化してください。
5.  **定期的なログ監視とアラート設定:** Cowrieのようなハニーポットのログを含む、すべてのセキュリティログを継続的に監視し、ブルートフォースアタック、特権アカウントへのログイン成功、不審なコマンド実行などに対してアラートが発報されるよう設定してください。
6.  **システムのパッチ管理:** OSやインストールされているソフトウェアを常に最新の状態に保ち、既知の脆弱性を悪用されないようにしてください。
7.  **セキュリティ教育:** ユーザーおよびシステム管理者にセキュリティ意識向上トレーニングを実施し、フィッシングやソーシャルエンジニアリングによる認証情報窃取のリスクを周知してください。
