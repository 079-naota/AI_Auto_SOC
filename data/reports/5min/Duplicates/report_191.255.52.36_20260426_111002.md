# SOC自動分析レポート: 191.255.52.36

**生成日時:** 2026-04-26 11:13:25

## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `191.255.52.36` から、2026年4月26日 06:50Zから07:19Zにかけて、SSHサービスに対する継続的な攻撃が観測されました。この攻撃は、以下の段階で進行したと分析されます。

1.  **偵察と認証情報の推測（Initial Access）**: 複数の一般的なユーザー名（`ubuntu`, `wang`）と推測しやすいパスワード、および複数の疑わしいパスワード（`Hello@123`, `123123Aa`, `odoo123`, `Aa111111!`, `3245gs5662d34`）を用いたSSHブルートフォース/辞書攻撃が行われました。特に `root` ユーザーを標的としていました。
2.  **特権昇格と永続化（Persistence & Privilege Escalation）**: `root` アカウントへのログイン成功後、攻撃者はシステムへの永続的なアクセス経路を確立しようとしました。具体的には、自身のSSH公開鍵（コメント `mdrfckr` の鍵）を `root` ユーザーの `authorized_keys` ファイルに複数回追加しようとし、さらに `root` ユーザーのパスワードを `V7x7C26rMbb4` に変更しようとしました。
3.  **防御回避とシステム情報の収集（Defense Evasion & Discovery）**: 攻撃者は `/etc/hosts.deny` を空にすることでホストベースのアクセス制御を無効化しようとしました。また、`cat /proc/cpuinfo`, `free -m`, `crontab -l`, `w`, `uname -a`, `lscpu` などのコマンドを広範囲に実行し、システムのハードウェア、OS、実行中のプロセス、ユーザー、cronジョブなど、詳細なシステム情報を収集しました。
4.  **痕跡のクリーンアップ/競合排除**: `/tmp/secure.sh`, `/tmp/auth.sh` の削除および関連プロセスの強制終了を試みており、これは自身の活動痕跡を消すか、あるいは競合する他のマルウェアを排除する意図があった可能性があります。

**攻撃の目的**: 攻撃の最終的な目的は、`root` 権限を確立した上でシステムへの永続的なバックドアを設置し、その後のデータ窃取、さらなる横展開、クリプトマイナーの展開、またはボットネットへの組み込みなどの悪意ある活動を行うための足がかりを築くことであると推測されます。

### 2. 推測される手法・使用ツール

*   **SSHブルートフォース/辞書攻撃**: 既知のユーザー名とパスワードリストを用いた自動化されたログイン試行。HydraやMedusaのようなSSH攻撃ツールが使用された可能性があります。
*   **認証情報アクセス (T1078)**: 成功したログイン試行によってシステムへのアクセスを獲得。
*   **永続化 (T1132 - Modify Authentication Process, T1098 - Account Manipulation)**:
    *   **SSH Authorized Keys (T1098.004)**: `root` ユーザーの `~/.ssh/authorized_keys` ファイルに攻撃者の公開鍵を追加し、パスワードなしで将来的にアクセスできるよう試みました。
    *   **Account Manipulation: Local Account (T1098.003)**: `root` ユーザーのパスワードを `echo "root:V7x7C26rMbb4"|chpasswd|bash` コマンドで変更しようとしました。
*   **防御回避 (T1562 - Impair Defenses)**:
    *   **Disable or Modify Tools (T1562.001)**: `chattr -ia .ssh` を試み、`.ssh` ディレクトリの不変/追加専用属性を解除しようとしました。
    *   **Modify Host Firewall (T1562.004)**: `echo > /etc/hosts.deny` でIPベースのアクセス制限を無効化しようとしました。
*   **情報収集 (T1082 - System Information Discovery, T1083 - File and Directory Discovery)**:
    *   `cat /proc/cpuinfo` (CPU情報)
    *   `free -m` (メモリ情報)
    *   `uname -m`, `uname -a` (OS/カーネル情報、アーキテクチャ)
    *   `lscpu` (CPU詳細情報)
    *   `crontab -l` (永続化メカニズムの探索)
    *   `w` (現在ログイン中のユーザーとプロセス)
    *   `whoami` (現在のユーザー)
    *   `ls -lh $(which ls)` (特定のコマンドの実行パスと属性)
*   **影響 (Impact)**: 成功した場合、完全なシステム制御、データの機密性・完全性・可用性の侵害。
*   **使用ツール**: Cowrieのログのため直接的なツール名は不明ですが、標準的なLinuxコマンドラインツール（`cd`, `rm`, `mkdir`, `echo`, `cat`, `grep`, `awk`, `chattr`, `chpasswd`, `pkill`, `crontab`, `w`, `uname`, `top`, `lscpu` など）が悪用されています。

### 3. 脅威レベルとその理由

**脅威レベル: 非常に高い (Critical)**

**理由:**

1.  **`root` 権限の奪取に成功**: 攻撃者は複数回にわたり、システムに対する最高権限である `root` アカウントでのログインに成功しています。これはシステムのセキュリティ境界が完全に破られ、システム全体が侵害された状態を示します。
2.  **永続的なアクセス経路の確立試行**: 攻撃者はSSH公開鍵の設置と`root`パスワードの変更を試みており、これにより検出後もシステムへの継続的なアクセスを確保しようとしています。これは、単発的な攻撃ではなく、長期的なシステム利用を企図していることを示唆します。
3.  **広範な情報収集と防御回避**: 攻撃者はシステム環境の詳細を把握しようと広範な情報収集を行い、同時に `/etc/hosts.deny` の無効化など、防御メカニズムの回避を試みています。これは、その後のより深刻な攻撃（マルウェア展開、データ窃取など）のための準備段階と考えられます。
4.  **自動化された攻撃の可能性**: 同じSSH公開鍵の複数回挿入試行や、特定のユーザー名/パスワードの繰り返し使用は、攻撃が自動化されたスクリプトまたはボットネットによって実行されている可能性を示唆します。

これらの行動は、もしハニーポットでなければ、深刻なデータ漏洩、システム破壊、マルウェア感染、または組織インフラ全体への横展開に繋がる可能性が極めて高く、最も緊急性の高い対応が求められます。

### 4. 推奨アクション

このログが実際のシステムで発生したものと仮定し、以下の緊急対応、詳細調査、予防策を推奨します。

#### 4.1. 緊急対応 (Immediate Actions)

1.  **対象システムからのネットワーク分離**: 侵害された可能性のあるシステムを直ちにネットワークから物理的または論理的に隔離し、これ以上の被害拡大や横展開を阻止します。
2.  **rootパスワードの即時変更**: すべての管理者アカウント、特に `root` ユーザーのパスワードを、予測困難な強力なものに即座に変更します。他のシステムで同じパスワードを使用している場合は、それらも変更します。
3.  **SSH鍵認証設定のレビューと不正な鍵の削除**:
    *   `root` ユーザーを含む、すべてのユーザーの `~/.ssh/authorized_keys` ファイルを徹底的にレビューします。
    *   ログに示された `ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOKRvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr` の公開鍵、およびその他の不審な鍵を直ちに削除します。
    *   `.ssh` ディレクトリのパーミッションが適切であるか (`chmod 700 ~/.ssh`, `chmod 600 ~/.ssh/authorized_keys`) 確認します。
4.  **`/etc/hosts.deny` のレビューと修正**: `/etc/hosts.deny` ファイルの内容を確認し、不正にクリアされていないか、または他の不審なエントリがないか確認し、適切な状態に戻します。
5.  **すべてのSSHセッションの強制終了**: 現在アクティブなSSHセッションを強制的に切断します。

#### 4.2. 詳細調査 (Investigation)

1.  **ログの包括的な分析**: 隔離後のシステム（またはバックアップ）上で、SSHログ（`auth.log`）、システムログ、監査ログ、アプリケーションログなどを詳細に分析し、以下の情報を特定します。
    *   最初の侵入経路と時刻。
    *   攻撃者が実行したすべてのコマンド。
    *   アクセスされたファイルやデータ。
    *   他のシステムへの横展開の試みや通信履歴。
    *   不正なファイル（マルウェア、バックドアなど）の有無。
2.  **ファイルシステムインテグリティチェック**: AIDEやTripwireなどのツールを使用して、システムの重要なファイルや設定ファイルに不正な変更がないか確認します。特に `/tmp`, `/var/tmp`, `/usr/local/bin`, `/etc/rc.local`, `crontab` などを重点的に確認します。
3.  **マルウェアスキャン**: 最新の定義ファイルを持つアンチウイルス/EDRソリューションでシステム全体をスキャンし、マルウェア感染の有無を確認します。
4.  **メモリフォレンジック**: 可能であれば、システムのメモリダンプを取得し、不審なプロセスや隠蔽された活動がないか分析します。
5.  **ネットワークトラフィック分析**: 侵入前後の期間のファイアウォール、IDS/IPS、プロキシなどのネットワークログを分析し、攻撃元IPアドレス以外の不審な通信先や、データ流出の有無を確認します。

#### 4.3. 予防策・再発防止 (Preventive Measures & Remediation)

1.  **SSH設定の強化**:
    *   **パスワード認証の無効化**: 可能な限り、SSH接続におけるパスワード認証を無効化し、鍵認証のみを許可します。
    *   **Rootログインの禁止**: `PermitRootLogin no` を設定し、`root` ユーザーのSSH直接ログインを禁止します。必要な場合は一般ユーザーでログイン後、`sudo` を使用させます。
    *   **多要素認証 (MFA) の導入**: SSHログインにMFAを導入し、セキュリティを強化します。
    *   **ポート変更**: 標準のSSHポート（22番）を別の高番号ポートに変更し、自動化されたスキャンからの露出を減らします（根本的な解決策ではないため、他の対策と併用）。
    *   **接続試行回数の制限**: `/etc/ssh/sshd_config` で `MaxAuthTries` を設定し、同時に `Fail2Ban` などのツールを導入して、不正なログイン試行を検出し、IPアドレスを一時的にブロックします。
2.  **強力なパスワードポリシーの適用**: すべてのユーザーアカウントに対し、複雑性、長さ、定期的な変更を強制する強力なパスワードポリシーを導入し、定期的に監査します。
3.  **脆弱性管理**: OS、ミドルウェア、アプリケーションのセキュリティパッチを常に最新の状態に保ち、定期的な脆弱性スキャンを実施します。
4.  **侵入検知/防御システム (IDS/IPS) の導入/強化**: 不審なログイン試行、コマンド実行、ファイルの変更などを検知・ブロックできるIDS/IPSソリューションを導入または設定強化します。
5.  **エンドポイントセキュリティの強化**: EDR（Endpoint Detection and Response）ソリューションを導入し、不審な挙動の検知と対応能力を向上させます。
6.  **セキュリティ意識向上トレーニング**: ユーザーに対し、ソーシャルエンジニアリングやフィッシング攻撃に対するセキュリティ意識向上トレーニングを定期的に実施します。
7.  **定期的なバックアップと復旧計画**: 定期的にシステムの完全なバックアップを取得し、インシデント発生時の迅速な復旧計画を策定・テストしておきます。

これらの対策を講じることで、将来的な同様の攻撃に対する耐性を高め、システムのセキュリティ体制を強化することができます。
