# SOC自動分析レポート: 202.108.48.228

## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃は、攻撃元IP `202.108.48.228` からのSSH接続によって開始されました。初期段階で`root`ユーザーとしてログインに成功しており、これはブルートフォース、パスワード推測、または既知の漏洩認証情報の利用によるものと推測されます。ログイン後、攻撃者は以下の複数の目的を達成しようと、短時間（約34秒間）で集中的な活動を行いました。

*   **永続化 (Persistence) の確立**: 攻撃者は`root`ユーザーの`.ssh/authorized_keys`ファイルに自身のSSH公開鍵を追加しようとしました。これにより、パスワード認証を必要とせず、SSH鍵認証による恒久的なアクセス経路を確保する狙いがあります。また、`root`ユーザーのパスワードを`XioTFwXKpZ06`に変更する試みも行っており、これは別の永続化手段の確立、または既存のパスワード認証をハイジャックする意図を示しています。
*   **防御回避 (Defense Evasion) および競合排除**: `/etc/hosts.deny`ファイルを空にすることで、IPアドレスによるアクセス制限を無効化しようとしました。また、`/tmp/secure.sh`や`/tmp/auth.sh`といった不審なスクリプトを削除し、関連プロセスを強制終了するコマンドを実行しており、これは他のマルウェアや攻撃者の痕跡を消去するか、競合するマルウェアを排除しようとする意図があると考えられます。
*   **情報収集 (Discovery)**: 侵入したシステムの環境（CPU情報、メモリ使用量、ディスク容量、OSバージョン、システムアーキテクチャ、実行中のプロセス、cronジョブ設定など）を広範に調査し、その後の攻撃ステップ（例：マルウェアの展開、脆弱性の探索、権限昇格、データの窃取など）のための情報を収集しようとしました。

一連の行動は、ターゲットシステムを完全に掌握し、継続的に悪用するための典型的な攻撃チェーンを示しています。

### 2. 推測される手法・使用ツール

*   **初期アクセス (Initial Access)**:
    *   **SSHによるブルートフォース/パスワード推測**: ログには`root`ユーザーでのログイン成功が記録されており、これは一般的な辞書攻撃、デフォルトパスワード、または漏洩した認証情報の悪用によるSSH経由の初期アクセス手法です。
*   **永続化 (Persistence)**:
    *   **SSH公開鍵の追加**: 攻撃者の公開鍵（コメント `mdrfckr` 付き）を`.ssh/authorized_keys`に書き込むことで、SSH鍵認証によるバックドアを作成しました。
    *   **アカウントパスワード変更**: `echo "root:XioTFwXKpZ06"|chpasswd` コマンドにより、`root`ユーザーのパスワードを強制的に変更しようとしました。
*   **防御回避 (Defense Evasion)**:
    *   **ファイル属性変更**: `chattr -ia .ssh` (および非標準コマンド`lockr`) を用いて、`.ssh`ディレクトリのimmutable/append-only属性を削除し、ファイルシステムレベルの保護を解除しようとしました。
    *   **システムアクセス制限の解除**: `echo > /etc/hosts.deny` により、`/etc/hosts.deny`ファイルを空にすることで、IPベースのアクセス制限を無効化しようとしました。
    *   **ファイル削除とプロセス終了**: `rm -rf /tmp/secure.sh; rm -rf /tmp/auth.sh; pkill -9 secure.sh; pkill -9 auth.sh; pkill -9 sleep;` コマンドにより、他の（あるいは自身の以前の）マルウェアやプロセスの痕跡を消去し、セキュリティ監視や競合する活動を妨害しようとしました。
*   **情報収集 (Discovery)**:
    *   **システム情報コマンドの実行**: `cat /proc/cpuinfo` (CPU情報), `free -m` (メモリ情報), `df -h` (ディスク情報), `uname -a` (OS/カーネル情報), `w` (ログインユーザーと実行プロセス), `crontab -l` (cronジョブ), `top` (プロセス監視), `whoami` (現在のユーザー) などの標準的なLinuxコマンドを多数実行し、ターゲットシステムの詳細な情報を収集しました。
*   **使用ツール**: 特定のマルウェアのダウンロードや実行は確認されませんでしたが、一連のコマンドは非常に迅速かつ連続して実行されており、自動化されたシェルスクリプトやボットネットクライアントなどのツールによって操作されている可能性が高いです。特に、公開鍵のコメント `mdrfckr` は、特定の攻撃グループやマルウェアキャンペーンに特徴的な署名である可能性があり、脅威インテリジェンス調査の足がかりとなり得ます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

1.  **最高権限アカウントでの侵入**: `root`アカウントでのログインに成功しているため、攻撃者はシステムに対する完全な制御権を獲得しています。これにより、システムの構成変更、データ窃取、マルウェアのインストール、他のシステムへの攻撃拠点としての悪用など、あらゆる悪意ある活動が可能になります。
2.  **明確な永続化の試み**: SSH公開鍵の設置および`root`パスワードの変更は、初期アクセスが遮断された後もシステムへのアクセスを維持するための明確な意図を示しています。これは、検出や対応が遅れると、攻撃者がシステムへのアクセス権を回復し続ける危険性があることを意味します。
3.  **防御回避行動**: `/etc/hosts.deny`のクリアや他のプロセスの終了は、システム管理者が設定したセキュリティ対策を意図的に無効化しようとするものであり、システムのセキュリティ体制を弱体化させ、検出を困難にする悪質な行動です。
4.  **広範な情報収集**: 多岐にわたるシステム情報の収集は、攻撃者が次にどのような行動を起こすかを計画するための偵察活動です。これは、特定の脆弱性の探索、貴重なデータの特定、あるいはボットネットへの組み込みなど、より高度な攻撃に繋がる前段階であり、潜在的な被害の範囲が広いことを示唆します。
5.  **自動化された攻撃の可能性**: 短時間での連続したコマンド実行パターンは、手動操作よりも自動化されたスクリプトやツールによる攻撃である可能性が高く、攻撃者が複数のターゲットに対して同様の攻撃を仕掛けている可能性を物語ります。

これらの要素から、本攻撃はシステムの完全な乗っ取りを意図しており、放置すれば深刻なセキュリティインシデントに発展するリスクが極めて高いため、「高」と評価します。

### 4. 推奨アクション

このログはハニーポットの記録ですが、もし同様の攻撃が実システムに対して発生した場合を想定し、以下の対応を推奨します。

1.  **緊急対応 (インシデント発生時)**:
    *   **システム隔離**: 影響を受けた可能性のあるシステムをネットワークから即座に隔離し、感染拡大やデータ流出を防ぎます。
    *   **認証情報の変更**: `root`ユーザーのパスワードを直ちに非常に複雑なものに変更します。また、他の特権ユーザーアカウントやサービスアカウントについてもパスワードの変更を検討します。
    *   **SSH鍵の確認と削除**: `root`ユーザーのホームディレクトリ内（特に`.ssh/authorized_keys`）に攻撃者が追加した不正なSSH公開鍵が存在しないか確認し、発見した場合は直ちに削除します。
    *   **フォレンジック調査**: システムの完全なイメージを取得し、詳細なフォレンジック調査を実施します。
        *   システムログ (`auth.log`, `secure`など) を確認し、不審なログインやコマンド実行、ファイル変更の形跡がないか調査します。
        *   不審なファイルやディレクトリの有無、パーミッションの変更、cronジョブの追加などを確認します。
        *   `/etc/hosts.deny`ファイルの内容が書き換えられていないか確認し、必要に応じて元の状態に戻します。
        *   バックアップが改ざんされていないか確認し、信頼できるバックアップからの復旧を検討します。
    *   **攻撃元のブロック**: 攻撃元IPアドレス `202.108.48.228` からのアクセスをファイアウォールでブロックします。

2.  **予防策強化 (再発防止)**:
    *   **SSHセキュリティの強化**:
        *   **パスワード認証の無効化**: 可能な限りSSHではパスワード認証を無効にし、公開鍵認証のみを許可します。
        *   **`root`ログインの禁止**: `sshd_config`ファイルで`PermitRootLogin no`を設定し、`root`ユーザーの直接SSHログインを禁止します。特権作業は一般ユーザーでログイン後、`sudo`を使用して行います。
        *   **複雑なパスワードポリシー**: 全てのシステムユーザーに対し、強力で定期的に更新されるパスワードポリシーを強制します。
        *   **ポート変更**: SSHのデフォルトポート (22) を一般的なものから変更することで、自動スキャンによる攻撃の機会を減らします。
        *   **多要素認証 (MFA)**: 重要なシステムへのSSHアクセスにはMFAの導入を検討します。
    *   **IDS/IPSおよびWAFの導入**: 異常なトラフィックや攻撃パターンを検出・防御するための侵入検知/防御システム (IDS/IPS) やWeb Application Firewall (WAF) を導入・強化します。
    *   **ログ監視とアラートの強化**: SSHログイン成功/失敗、特権ユーザーのコマンド実行、重要なシステムファイルの変更などのイベントについて、リアルタイムで監視し、異常を検知した際には即座にアラートを発するシステムを構築します。
    *   **パッチ管理の徹底**: OSおよびインストールされている全てのソフトウェアのセキュリティパッチを常に最新の状態に保ち、既知の脆弱性を悪用されるリスクを低減します。
    *   **最小権限の原則**: 各ユーザーおよびサービスには、その業務遂行に必要最小限の権限のみを付与し、過剰な権限設定を避けます。
    *   **定期的なセキュリティ監査**: 定期的に脆弱性スキャン、ペネトレーションテスト、設定レビューなどを実施し、セキュリティ体制の弱点を継続的に特定・改善します。
    *   **脅威インテリジェンスの活用**: 攻撃者のSSH公開鍵（`mdrfckr`コメント付き）などの情報を脅威インテリジェンスプラットフォームと共有し、将来の攻撃検知に役立てます。
