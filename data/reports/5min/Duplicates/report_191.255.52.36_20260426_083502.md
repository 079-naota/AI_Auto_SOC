# SOC自動分析レポート: 191.255.52.36

## SOC分析レポート

### 1. 攻撃の概要と目的

提供されたログを分析した結果、攻撃元IP `191.255.52.36` から、SSHサービスを標的とした継続的かつ悪質な攻撃が確認されました。この攻撃の目的は、標的システムへの永続的なアクセス経路を確立し、システムを完全に掌握することにあると推測されます。

攻撃者はまず、`ubuntu`ユーザーに対するパスワード認証の試行から始まり、その後`root`ユーザーに対する複数のパスワード認証を試行し、最終的に複数の異なるパスワードで`root`としてログインに成功しています。ログイン後、攻撃者は以下の主要な行動を実行しています。

*   **永続化の確立**: `root`ユーザーのSSH `authorized_keys`ファイルに自身の公開鍵を追加し、パスワードを知らなくても今後のSSHアクセスを可能にするバックドアを設置。また、`root`ユーザーのパスワードを強制的に変更しようとしています。
*   **システム偵察**: CPU情報、メモリ使用量、OS情報、ログインユーザー、cronジョブなど、広範なシステム情報を収集しています。これは、次の攻撃フェーズ（マルウェア展開、データ窃取、横展開など）のための情報収集であると考えられます。
*   **防御回避・競合の排除**: `/tmp`ディレクトリ内の不審なスクリプト（`secure.sh`, `auth.sh`）を削除し、関連するプロセスを停止させようとしています。これは、他の攻撃者によるマルウェアやバックドアを排除し、システムを自身が完全に支配しようとする行動、あるいは自身のマルウェア展開のための準備である可能性があります。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (Critical)**

**理由**:

1.  **root権限の奪取**: 攻撃者は`root`ユーザーとして複数回ログインに成功しており、システムに対する最高レベルの制御権を得ています。これにより、システム上のあらゆる操作（ファイルシステムへのアクセス、設定変更、マルウェアのインストール、ユーザーアカウントの操作など）が可能になります。
2.  **永続化メカニズムの確立**: SSH公開鍵の埋め込みと`root`パスワードの変更により、攻撃者は防御側が既存の認証情報を変更しても、別の手段でシステムに再侵入できる恒久的なバックドアを設置しています。これは、防御側にとって非常に深刻な問題です。
3.  **広範な偵察活動**: システムに関する詳細な情報を収集していることから、攻撃者は次に実行する攻撃（例: 機密データの窃取、マルウェアの展開、C2（コマンド＆コントロール）サーバーとの通信確立、他のシステムへの横展開）の計画を立てている段階であると推測されます。
4.  **競合排除の動き**: 他のマルウェアや攻撃者の痕跡を消そうとする行動は、攻撃者がシステムを完全に掌握し、長期的な利用を意図していることを示しており、その悪意と目的の明確さを強調します。
5.  **複数の攻撃手法の組み合わせ**: ブルートフォースから永続化、偵察、防御回避と、攻撃ライフサイクルに沿った複数の手法が巧みに組み合わされており、組織的な攻撃である可能性が高いです。

これらの理由から、この攻撃は組織のセキュリティにとって極めて高い脅威をもたらすものと判断されます。

### 4. 推奨アクション

**緊急対応 (Immediate Actions)**:

1.  **システム隔離**: 攻撃を受けた可能性のあるシステム（またはハニーポットが模倣するシステム）をネットワークから直ちに隔離し、さらなる侵害や横展開を防ぎます。
2.  **SSHサービスの停止/制限**: SSHサービスを一時的に停止するか、信頼できる管理用IPアドレスのみからのアクセスに厳格に制限します。
3.  **攻撃元IPのブロック**: ファイアウォールやIPS/IDSにて、攻撃元IP `191.255.52.36` からのすべての通信をブロックリストに追加します。
4.  **不正なSSH公開鍵の確認と削除**: `root`ユーザーおよびその他の特権ユーザーのホームディレクトリ（`~/.ssh/authorized_keys`）に不正な公開鍵（特にコメントが`mdrfckr`のもの）が追加されていないか確認し、発見した場合は直ちに削除します。
5.  **パスワードのリセット**: 攻撃者がパスワード変更を試みた`root`ユーザーおよびその他すべてのシステムアカウントのパスワードを、複雑で推測されにくいものに直ちにリセットします。
6.  **不正なユーザーアカウントの確認**: 不審なユーザーアカウントが作成されていないか確認し、発見した場合は削除または無効化します。
7.  **不正プロセスの終了とファイルの削除**: `/tmp/secure.sh`, `/tmp/auth.sh`など、ログに記録された不審なファイルやプロセスが存在しないか確認し、発見した場合は強制終了および削除します。`/etc/hosts.deny`の内容も確認し、不審な変更がないか元に戻します。

**フォレンジック調査 (Forensic Investigation)**:

1.  **ログの詳細分析**: 今回のCowrieログ以外の全てのシステムログ（例: `auth.log`, `syslog`, `history`、ウェブサーバーログなど）を収集し、攻撃者の行動をさらに深く分析します。特に、攻撃者がシステムにアクセスした最初の時点や、他のファイル操作、ネットワーク接続、外部へのデータ転送などの痕跡を特定します。
2.  **侵害範囲の特定**: 攻撃者がアクセス、変更、作成した可能性のあるファイル、ディレクトリ、データベース、設定などを詳細に調査し、データの漏洩や改ざんがないかを確認します。
3.  **マルウェアスキャン**: システム全体を最新のマルウェア対策ソフトウェアでスキャンし、隠されたバックドア、ルートキット、その他の不正なプログラムがないか確認します。
4.  **脆弱性スキャンと侵入経路の特定**: システムのSSHサービスやその他の公開サービスに既知の脆弱性がないかスキャンし、今回の侵入経路となった弱点を特定します。

**予防策 (Preventive Measures)**:

1.  **強力なパスワードポリシーの適用**: 全てのシステムアカウントに対し、強力で複雑なパスワードの使用を義務付け、定期的な変更を促します。一般的な辞書に載っているパスワードや推測しやすいパスワードは許可しない設定とします。
2.  **SSHパスワード認証の無効化と公開鍵認証の強制**: 可能であれば、SSHサービスでのパスワード認証を無効化し、公開鍵認証のみを許可します。`root`ユーザーでの直接ログインは無効にし、一般ユーザーでログイン後、`sudo`を利用するように設定します。
3.  **二要素認証 (MFA) の導入**: SSHログインにMFAを導入し、認証セキュリティを強化します。
4.  **定期的なシステムパッチ適用**: OSやSSHサービスを含むすべてのソフトウェアのセキュリティパッチを常に最新の状態に保ち、既知の脆弱性を悪用されることを防ぎます。
5.  **侵入検知システム (IDS/IPS) の導入と強化**: 不審なSSHログイン試行や異常なコマンド実行をリアルタイムで検知・ブロックできるIDS/IPSを導入し、既存のルールセットを強化します。
6.  **セキュリティ監視の強化**: ログ監視体制を強化し、不審なログイン試行、特権昇格、異常なプロセス起動、ファイル変更などをリアルタイムで検知できるようアラートを設定します。SIEM (Security Information and Event Management) を活用し、ログの相関分析を行います。
7.  **ハニーポットの活用**: 今回のように、Cowrieなどのハニーポットをデプロイし、脅威インテリジェンス収集に活用することで、新たな攻撃手法や攻撃元の情報を早期に把握し、防御策を強化します。
