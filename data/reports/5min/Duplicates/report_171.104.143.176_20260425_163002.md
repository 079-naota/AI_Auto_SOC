# SOC自動分析レポート: 171.104.143.176

## SOC分析レポート

**件名: SSHに対するブルートフォース攻撃と不正アクセス、およびシステム偵察活動**

**報告日時**: 2026-04-25T07:05:00Z
**攻撃元IP**: 171.104.143.176
**対象システム**: Cowrieハニーポット（SSHサービス）

---

### 1. 攻撃の概要と目的

攻撃元IPアドレス `171.104.143.176` から、当社のハニーポットシステムに対し、継続的なSSHブルートフォース/辞書攻撃が観測されました。この攻撃により、`root`アカウントでの不正ログインが複数回成功しています。

ログイン成功後、攻撃者は以下の主要な行動をとっています。

1.  **永続化メカニズムの確立**:
    *   `root`ユーザーの `.ssh/authorized_keys` ファイルに攻撃者の公開鍵を追加し、パスワードなしでの再アクセス経路を確保。
    *   `root`ユーザーのパスワードを2回にわたり変更し、既存の認証情報を上書きすることで、自身のアクセス権限を強固に維持しようと試みています。
2.  **システム情報の偵察**:
    *   CPU情報、メモリ情報、OSバージョン、ディスク使用量、実行中のプロセス、cronジョブ、現在のログインユーザーなど、システムに関する広範な情報を収集するコマンドを多数実行しています。
3.  **防御回避と痕跡のクリーンアップ**:
    *   `/etc/hosts.deny` ファイルをクリアすることで、IPアドレスによるアクセス制限を解除しようとしています。
    *   `/tmp/secure.sh` や `/tmp/auth.sh` といったスクリプトや、`sleep` コマンドに関連するプロセスを停止・削除しようとしており、これは他の攻撃者によるマルウェアや自身の残骸をクリーンアップする意図があると推測されます。

これらの行動から、攻撃の主な目的は、標的システムへの完全な制御を確立し、永続的なアクセス経路を確保した上で、その後の悪意ある活動（例: ボットネットへの組み込み、クリプトマイニング、データの窃取、さらなる内部ネットワークへの侵入など）のための足がかりを構築することであると判断されます。

### 2. 推測される手法・使用ツール

*   **初期アクセス手法**: SSHブルートフォース/辞書攻撃。一般的なユーザー名（`deploy`、`root`）と推測しやすい、またはリストに含まれるパスワードを組み合わせて試行しています。
*   **永続化手法**:
    *   SSH公開鍵認証の悪用 (`authorized_keys`への鍵追加)。
    *   アカウントパスワードの変更 (`chpasswd`コマンド)。
*   **偵察手法**: 標準的なLinuxコマンド（`cat`, `grep`, `wc`, `awk`, `free`, `ls`, `crontab`, `w`, `uname`, `top`, `whoami`, `lscpu`, `df`）を組み合わせてシステム情報を収集しています。
*   **防御回避手法**: `/etc/hosts.deny`のクリアや、不審なプロセス/ファイルの削除。
*   **使用ツール**:
    *   SSHクライアント。
    *   SSHの認証情報を試行するブルートフォース/辞書攻撃ツール (例: Hydra, Nmapの`ssh-brute`スクリプトなど)。
    *   ログイン後のコマンド実行が非常に迅速かつ定型的であることから、一連の攻撃手順を自動化するスクリプトまたはフレームワーク（例: Bashスクリプト、Pythonスクリプトなど）が使用されている可能性が高いです。また、`lockr`コマンドの失敗は、攻撃者が特定の環境で利用可能なツールを期待しているか、タイプミスである可能性を示唆します。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
1.  **root権限でのログイン成功**: 攻撃者がシステムへの最高権限を取得していることは、システムのセキュリティ体制が著しく侵害されたことを意味します。これにより、システムの完全な制御を奪われ、任意の操作が可能になります。
2.  **永続化メカニズムの確立**: 攻撃者がSSH公開鍵の追加とrootパスワードの変更を試みていることから、一時的なアクセスに留まらず、恒久的なアクセス経路を確立しようとしている意図が明確です。これにより、将来にわたる攻撃活動の基盤が築かれる可能性があります。
3.  **広範な偵察活動**: 攻撃者がCPU、メモリ、OS、ディスク、プロセス、cronジョブなどの情報を詳細に収集していることは、ターゲットシステムを深く理解し、その後の攻撃（例：特定の脆弱性の悪用、マルウェアの最適化された展開、リソースの不正利用など）のための準備を進めていることを強く示唆します。
4.  **自動化された攻撃**: 一連の行動が短時間で効率的に実行されていることから、自動化されたスクリプトやツールが使用されていると推測されます。これは、この攻撃が単発的なものではなく、広範なキャンペーンの一部である可能性が高く、他のシステムも同様の標的になっている恐れがあります。

このログはハニーポット上での活動ですが、これらの手法がもし実際のプロダクション環境に適用された場合、深刻な情報漏洩、システム停止、不正利用などの被害につながる可能性が極めて高いです。

### 4. 推奨アクション

この攻撃はハニーポットで検知されたものではありますが、実際のシステムが同様の攻撃にさらされるリスクを考慮し、以下の対応を推奨します。

**A. 緊急対応（即時実施）**

1.  **攻撃元IPのブロック**:
    *   攻撃元IPアドレス `171.104.143.176` からの全てのSSH接続試行を、ネットワーク境界のファイアウォール（またはホストベースのファイアウォール）で直ちにブロックしてください。
2.  **パスワード変更とMFAの導入**:
    *   全ての公開SSHサーバーにおいて、`root`アカウントおよびその他の特権アカウントのパスワードを直ちに、かつ強力なものに変更してください。
    *   可能であれば、SSHログインに多要素認証（MFA）を強制的に適用してください。
3.  **SSH鍵認証の確認**:
    *   公開SSHサーバー上の全てのユーザーの `~/.ssh/authorized_keys` ファイルの内容を確認し、不正な公開鍵が追加されていないか徹底的に調査してください。発見された場合は直ちに削除してください。
4.  **サービスの設定変更**:
    *   SSHサービスにおけるrootユーザーの直接ログインを無効化し、鍵認証のみを許可する設定に変更してください。
5.  **ログの監視強化**:
    *   当社の実際のSSHサーバーの認証ログ（`/var/log/auth.log` など）を緊急でレビューし、`171.104.143.176` からの接続履歴、不正なログイン試行、特にログイン成功の有無を確認してください。

**B. 中期的な対策**

1.  **SSHポートの公開制限**:
    *   SSHポート（通常22番）へのインターネットからの直接アクセスを可能な限り制限し、VPN接続経由のみとするか、信頼できる特定のIPアドレスからのみアクセスを許可するようにしてください。
2.  **脆弱性スキャンとパッチ適用**:
    *   全てのインターネット公開システムに対し、定期的な脆弱性スキャンを実施し、OSやアプリケーションの既知の脆弱性を速やかに特定・修正してください。
    *   定期的なセキュリティパッチの適用を徹底してください。
3.  **侵入検知システム (IDS/IPS) の強化**:
    *   不正なログイン試行、異常なコマンド実行、不審なファイル変更などを検知・ブロックできるIDS/IPSソリューションの導入または設定強化を検討してください。
4.  **SIEM/ログ管理の強化**:
    *   SSH認証ログ、システムログ、ファイアウォールログを一元的に収集・分析するSIEM（Security Information and Event Management）システムを導入・強化し、リアルタイムでの脅威検知・アラート体制を構築してください。

**C. 長期的な対策**

1.  **セキュリティポリシーの見直し**:
    *   パスワードポリシー、アクセス制御ポリシー、インシデント対応計画など、関連するセキュリティポリシーを定期的に見直し、最新の脅威に対応できるように更新してください。
2.  **従業員へのセキュリティトレーニング**:
    *   フィッシング攻撃、ソーシャルエンジニアリング、強力なパスワードの重要性など、従業員へのセキュリティ意識向上トレーニングを継続的に実施してください。
3.  **バックアップとリカバリ計画**:
    *   重要なシステムとデータの定期的なバックアップを確立し、災害復旧・事業継続計画を策定・テストしてください。
