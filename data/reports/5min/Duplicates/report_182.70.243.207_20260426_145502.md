### 1. 攻撃の概要と目的

攻撃元IPアドレス 182.70.243.207 から、CowrieハニーポットがエミュレートするSSHサービスに対し、短時間のうちに多数のログイン試行が行われました。これは、主にブルートフォース/パスワードスプレー攻撃の手法を用いて、SSHサービスへの不正なアクセスを試みるものでした。

攻撃の主な目的は以下の2点であると推測されます。

1.  **不正なSSHログインの確立:** 特に `root` ユーザーに対する複数の異なるパスワードでのログイン試行が確認されており、管理者権限の奪取を狙っています。
2.  **永続的なアクセス経路の確保:** ログイン成功後、速やかに攻撃者のSSH公開鍵をターゲットシステムの `~/.ssh/authorized_keys` に登録しようとしています。これにより、パスワード認証が不要なSSHバックドアを設置し、将来的にパスワードが変更されてもアクセスを継続できる永続的な侵入経路を確保することを意図しています。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース/パスワードスプレー攻撃:** 複数のユーザー名（`steam`, `tunnel`, `cloud`, `gitlab`, `root`, `345gs5662d34`など）と、それに対応すると思われる様々なパスワード（`steamsteamsteam`, `123456`, `123`, `gitlab@2025`, `QAZwsx`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`など）を試行しており、自動化されたログイン試行であることは明らかです。
    *   **永続化メカニズムの確立:** ログインに成功した後、直ちに `rm -rf .ssh && mkdir .ssh && echo "..." >>.ssh/authorized_keys && chmod -R go= ~/.ssh` というコマンドを実行し、自身のSSH公開鍵を `authorized_keys` に追加して永続的なアクセス手段を確立しようとしています。これは典型的なバックドア設置の手法です。
    *   **`.ssh`ディレクトリ属性変更の試み:** `chattr -ia .ssh; lockr -ia .ssh` というコマンドもログイン後に試行されていますが、これらは `command.failed` と記録されており、ハニーポット環境では実行できなかったか、または`lockr`が不正なコマンドであるか、特定の環境に依存するコマンドであった可能性が考えられます。`chattr -ia` はimmutable/append-only属性の解除を試みるものであり、通常は変更できない重要ファイルを改変するための準備行為として使われることがあります。

*   **使用ツール:**
    *   **自動化されたSSHクライアント:** 短時間で多数のログイン試行とコマンド実行を行っていることから、Hydra、Medusa、Nmapのスクリプト、またはカスタムスクリプトなどのブルートフォースツールが使用されている可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

1.  **管理者権限 (`root`) への執拗な攻撃:** 攻撃は `root` ユーザーアカウントを主要なターゲットとしており、実際に複数のパスワードでログイン成功を収めています。これは、システムに対する最大の権限を奪取しようとする明確な意図を示しています。
2.  **永続化の試み:** ログイン成功後、すぐにSSH公開鍵を設置してバックドアを確立しようとしている点は、攻撃者が一時的なアクセスに留まらず、将来にわたってシステムを支配下に置こうとしていることを示します。このキーは `mdrfckr` というコメントが付与されており、攻撃者の識別子と考えられます。
3.  **自動化された明確な攻撃フロー:** 一連の行動は、ログイン試行 → 成功 → 永続化（SSHキー設置）という明確な攻撃チェーンを示しており、単なるランダムなスキャンではなく、具体的な目的を持った自動化された攻撃プログラムによるものである可能性が高いです。
4.  **実際のシステムへの影響 (想定):** このログはハニーポットのものであるため、実際のシステムへの直接的な被害はありませんが、もしこれが実環境であれば、システムは完全に侵害され、攻撃者に任意のコマンド実行、データ窃取、マルウェア配布などの活動を許すことになったでしょう。

### 4. 推奨アクション

このログはハニーポットのものであるため、実際の侵害は発生していませんが、同様の攻撃が実環境に及ぼされることを想定し、以下の推奨アクションを実施してください。

#### 4.1. 緊急対応 (Immediate Actions - If this were a real system breach)

1.  **攻撃元IPのブロック:** 直ちに攻撃元IPアドレス `182.70.243.207` をネットワーク境界のファイアウォール、IDS/IPS、またはSSHデーモンの設定（`/etc/hosts.deny`など）でブロックし、さらなるアクセスを遮断します。
2.  **侵害されたアカウントのパスワード変更とロックダウン:** `root`ユーザーを含む、ログインに成功した全てのユーザーアカウントのパスワードを直ちに変更し、二要素認証（MFA）を有効にします。また、一時的にアカウントをロックすることも検討します。
3.  **不正なSSHキーの削除:** ログイン成功したユーザーの `~/.ssh/authorized_keys` ファイルを緊急で確認し、ログに記載されている公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`）が存在しないか確認し、存在する場合は削除します。
4.  **侵害範囲の調査:** 攻撃者がシステム内で他にどのような活動を行ったか（データの窃取、追加のマルウェア設置、横展開など）を特定するため、システムログ、プロセスログ、ネットワーク通信ログなどを詳細に調査します。
5.  **システムイメージの取得:** フォレンジック分析のため、侵害されたシステムのディスクイメージを取得することを検討します。

#### 4.2. 予防策 (Preventive Measures)

1.  **SSH設定の強化:**
    *   **パスワード認証の無効化:** SSHアクセスは鍵認証のみに限定し、パスワード認証を無効にします (`PasswordAuthentication no` in `sshd_config`)。
    *   **`root`ユーザーの直接ログイン禁止:** `root`ユーザーでの直接SSHログインを禁止します (`PermitRootLogin no` in `sshd_config`)。必要な場合は一般ユーザーでログイン後、`sudo`を利用させます。
    *   **SSHポートの変更:** デフォルトのSSHポート（22番）を非標準のポートに変更し、一般的なスキャンからの検出を減らします（これはセキュリティバイオブスキュリティであり、主要な防御策ではありませんが、ノイズを減らす効果はあります）。
    *   **ログイン試行回数の制限:** `MaxAuthTries` を低く設定し、ブルートフォース攻撃の効率を下げます。
2.  **強力なパスワードポリシーの適用:** 全てのユーザーアカウントに対し、複雑性、長さ、有効期限などを含む強力なパスワードポリシーを強制します。
3.  **二要素認証 (MFA) の導入:** SSHを含む重要なサービスへのアクセスにMFAを導入し、認証情報の漏洩によるリスクを軽減します。
4.  **ログ監視とアラートの強化:**
    *   SSHのログイン試行、成功、失敗に関するログを継続的に監視し、異常なパターン（短時間の多数の失敗、不明なIPからのログイン試行など）を検知した際にアラートを生成する仕組みを導入します。
    *   集中ログ管理システム（SIEM）にログを転送し、相関分析を可能にします。
5.  **定期的な脆弱性診断とパッチ適用:** システムおよびSSHサービスが最新の状態に保たれているか、既知の脆弱性が存在しないか定期的に確認し、パッチを適用します。
6.  **ハニーポットの継続的な活用:** Cowrieのようなハニーポットを継続的に運用し、攻撃の傾向、使用されるツール、パスワードリストなどを収集・分析することで、新たな脅威に対する防御策の改善に役立てます。
