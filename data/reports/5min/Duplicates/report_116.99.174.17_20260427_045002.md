# SOC自動分析レポート: 116.99.174.17

## SOCアナリストレポート

### 攻撃ログ分析レポート

**日付**: 2026年4月27日
**分析対象**: SSHハニーポットログ
**攻撃元IP**: 116.99.174.17
**脅威レベル**: 高 (High)

---

### 1. 攻撃の概要と目的

攻撃元IPアドレス `116.99.174.17` から、当社のSSHサービス（ハニーポット）に対して、約18分間にわたり断続的な認証試行攻撃（ブルートフォース/辞書攻撃）が観測されました。

攻撃者は主に以下のユーザー名とパスワードの組み合わせを試行しました。

*   **一般的なユーザー名**: `admin`, `user`, `root`, `guest`
*   **特定のデバイスやサービスで使われがちなユーザー名**: `ubnt`, `config`, `squid`, `support`, `ftpuser`, `nikita`
*   **パスワード**: `admin`, `user`, `1234`, `12345`, `123456`, `password`, `default`, `root123`, `abcd1234`, `ipscan`, `alpine`, `asteriskftp` など、複数の一般的なおよび推測されやすいパスワード

この攻撃の特徴は、`root`ユーザーに対して特に執拗なログイン試行が行われ、ハニーポット環境上ではありますが、以下のパスワードで**複数回のログイン成功**が記録されている点です。

*   `root` / `admin`
*   `root` / `@` (非常に珍しいパスワード)
*   `root` / `root123`
*   `root` / `12345`
*   `root` / `abcd1234`
*   `root` / `ipscan`
*   `root` / `alpine`

ログイン成功後には、`cowrie.direct-tcpip.request` などのイベントが記録されており、これは攻撃者が認証後にポートフォワーディングやプロキシ接続を試み、さらにシステム内部へのアクセスや偵察、外部との通信チャネル確立を試みた可能性を示唆しています。

攻撃の主な目的は、SSHサービスへの不正アクセスを通じてシステムを侵害し、その後の偵察、マルウェアの展開、データ窃取、あるいは他の攻撃のための踏み台として利用することであると推測されます。

---

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃 / 辞書攻撃**: 大量のユーザー名とパスワードの組み合わせを自動的に試行していることから、この手法が用いられていることは明らかです。
    *   **自動化された攻撃 / ボットネット**: 短期間に多数の試行が繰り返されていることから、攻撃は手動ではなく、自動化されたスクリプトやボットネットの一部として実行されている可能性が極めて高いです。
*   **使用ツール**:
    *   具体的なツールの情報はログには含まれていませんが、SSHブルートフォース攻撃で一般的に利用されるツールとしては、**Hydra**、**Medusa**、**NmapのSSH関連スクリプト (例: `ssh-brute`)**、または独自のカスタムスクリプトなどが考えられます。
    *   `direct-tcpip`イベントは、認証後にプロキシ機能やポートフォワーディングを利用して、さらに内部ネットワークへのアクセスを試みたり、C2サーバーとの通信経路を確立しようとしたりする際に発生する可能性があります。これは、より高度なペイロードのデプロイや内部偵察を意図していることを示唆しています。

---

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

1.  **複数回の認証成功**: ハニーポット環境での記録ではありますが、`root`ユーザーを含む複数のアカウントでログインに成功しています。これは、もし実環境のシステムであれば、重大な侵害が発生していた可能性が高いことを意味します。
2.  **特権アカウントへの焦点**: `root`アカウントが繰り返しターゲットにされ、実際に複数のパスワードでログインが成功していることは、攻撃者がシステムに対する最高レベルの権限を狙っていることを示しており、侵害された場合の影響が極めて大きくなります。
3.  **攻撃の自動化と継続性**: 攻撃が一定時間継続され、自動化された辞書攻撃のパターンを示しています。このような攻撃は、広くインターネットに公開されているSSHサービスを持つシステムであれば、無差別に標的となるリスクが高いです。
4.  **認証後の活動の兆候**: 単なる認証試行に留まらず、ログイン成功後に `direct-tcpip` イベントが記録されていることから、攻撃者はシステムへの侵入後、さらに偵察、マルウェアのダウンロード、内部ネットワークへの横展開など、より深刻な活動を計画していた可能性が高いと推測されます。

### 4. 推奨アクション

このログはハニーポットからのものですが、これらの攻撃が実環境に影響を与えないよう、以下の対策を速やかに実施することを推奨します。

1.  **緊急対応**:
    *   **攻撃元IPのブロック**: 攻撃元IPアドレス `116.99.174.17` からのSSH接続を、ファイアウォール、IDS/IPS、またはセキュリティグループ等で直ちにブロックします。
    *   **ブルートフォース対策の強化**: SSHサービスへの短時間での多数のログイン試行を検知し、攻撃元IPを自動的にブロックするツール（例: Fail2Ban）が導入されていることを確認し、設定を最適化します。
2.  **認証方法の強化**:
    *   **パスワード認証の無効化とSSH鍵認証への移行**: 最も効果的な対策として、パスワード認証を無効にし、公開鍵認証のみを許可するようにSSH設定を変更します。
    *   **多要素認証 (MFA) の導入**: パスワード認証を残す場合でも、SSHログインにMFAを必須とすることで、セキュリティを大幅に向上させます。
    *   **rootログインの禁止**: SSH経由でのrootユーザーによる直接ログインを禁止し、必要に応じて一般ユーザーでログイン後に `sudo` を利用する運用に切り替えます。
3.  **アカウントとパスワードの管理**:
    *   **脆弱なパスワードの排除**: ログに記録されたような推測されやすいパスワードやデフォルトパスワードを使用しているアカウントがないか、全システムで監査し、すべて複雑でユニークなパスワードに強制変更します。
    *   **不要なアカウントの削除**: 長期間使用されていないアカウントや不要なアカウントを削除します。
4.  **アクセス制御の強化**:
    *   **IPアドレス制限**: SSHへのアクセスを、特定の信頼できるIPアドレス範囲またはVPN経由のみに制限します。
    *   **標準ポートの変更**: 標準のSSHポート (22/TCP) を非標準のポートに変更することで、無差別なポートスキャンやブルートフォース攻撃の対象になる頻度を減らすことができます（これはセキュリティの根本的な強化にはなりませんが、攻撃量を減らす効果が期待できます）。
5.  **監視と監査の強化**:
    *   **SSHログの継続的な監視**: すべてのSSHサーバーで認証ログを詳細に記録し、SIEMなどのログ管理システムで集中的に監視することで、異常なログイン試行や成功を迅速に検知できる体制を維持・強化します。
    *   **セキュリティアラートのチューニング**: ブルートフォース攻撃やログイン成功に関するセキュリティアラートが適切に設定され、即座に担当者に通知されることを確認します。
6.  **追加調査**:
    *   攻撃元IPアドレス `116.99.174.17` のレピュテーション情報を継続的に監視し、脅威インテリジェンスと照合します。
    *   ハニーポットの詳細なセッションログ（攻撃者が認証後に実行したコマンドや試みた活動）があれば、より詳細な分析を行い、攻撃者の具体的な目的や使用マルウェアの特定に努めます。
