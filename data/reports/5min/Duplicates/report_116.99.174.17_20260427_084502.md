# SOC自動分析レポート: 116.99.174.17

**生成日時:** 2026-04-27 08:47:25

---

## SOCアナリストレポート

### 攻撃概要レポート

**発生日時:** 2026-04-27T02:13:34Z から 2026-04-27T02:31:53Z の間
**攻撃元IPアドレス:** 116.99.174.17
**対象サービス:** SSH (ポート22番が一般的な想定)
**ログソース:** Cowrie Honeypot

---

### 1. 攻撃の概要と目的

攻撃元IPアドレス `116.99.174.17` から、短時間で複数回にわたりSSHサービスへのログイン試行が行われました。これは典型的なブルートフォース攻撃または辞書攻撃のパターンを示しています。

ログには計25回のSSH接続試行が記録されており、そのうち以下の**6回のログイン成功**が確認されました。

1.  `root` / `admin`
2.  `root` / `@`
3.  `root` / `root123`
4.  `root` / `12345`
5.  `root` / `abcd1234`
6.  `root` / `ipscan`
7.  `root` / `alpine`

（注：`root:12345`は成功していますが、その後に`admin:12345`が失敗しており、パスワードリスト攻撃の一部として認識されます。）

ログイン成功後には、`cowrie.direct-tcpip.request` および `cowrie.direct-tcpip.data`、`cowrie.direct-tcpip.ja4h` といったイベントが連続して発生しています。これは、攻撃者が認証成功後、ポートフォワーディングの確立や何らかのペイロード（追加のマルウェアやツールなど）の転送、あるいは内部ネットワークへのアクセスを試みていることを強く示唆しています。

攻撃の主な目的は、SSHサービスへの不正アクセスを通じて対象システムを侵害し、その後の悪意ある活動（例: マルウェアの展開、コマンド実行、他のシステムへの踏み台としての利用、データ窃取など）を行うことにあると推測されます。

---

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **ブルートフォース攻撃 / 辞書攻撃 (Brute-force / Dictionary Attack):** 大量の一般的なユーザー名とパスワードの組み合わせを繰り返し試行し、認証情報の突破を試みています。特に`root`アカウントに対し、よく使われるパスワードや推測されやすいパスワードが試されています。
    *   **Credential Stuffing (認証情報使い回し攻撃):** 試行されたパスワードには、他の情報漏洩で流出した可能性のある簡易なパスワード（例: `admin`, `1234`, `123456`, `password`など）が含まれていることから、既存の認証情報リストを利用している可能性も考えられます。
*   **使用ツール:**
    *   これらの自動化されたログイン試行は、`hydra`、`Medusa`、`ncrack`などの専用のブルートフォースツール、またはカスタムスクリプト（Python, Perl, Bashなど）が使用されている可能性が高いです。
    *   ログにはSSHクライアントのバージョン情報などが記録されていますが、具体的なツールを特定するに足る情報は含まれていません。

---

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **最高権限アカウントへの侵害:** `root`ユーザーとして複数回ログインに成功していることは、実システムであれば攻撃者がシステムに対する完全な制御権を獲得したことを意味します。これにより、システムの改ざん、データ窃取、マルウェア感染、永続的なバックドアの設置など、あらゆる種類の悪意ある活動が可能になります。
*   **「direct-tcpip」リクエストの観測:** ログイン成功後にポートフォワーディングなどの追加通信を確立しようとしていることは、単なる偵察ではなく、本格的なシステム侵害や内部ネットワークへの横展開を意図していることを示唆しています。
*   **継続的な攻撃の試み:** 短期間に多数の異なる認証情報を試していることから、標的型攻撃というよりも、インターネット上の脆弱なSSHサービスを無差別に狙う広範な攻撃キャンペーンの一部である可能性が高いです。しかし、これが実システムであれば、継続的な攻撃により最終的に突破されるリスクが高いことを示しています。
*   **ハニーポットログからの洞察:** 今回のログはハニーポットの記録であるため、実際のシステムが直接的な被害を受けたわけではありません。しかし、このハニーポットは、もし当社の実システムが同様の脆弱な認証情報を利用していた場合、同様の攻撃により確実に侵害されていたであろうことを明確に示唆しており、深刻な脅威と評価されます。

---

### 4. 推奨アクション

この分析に基づき、以下のセキュリティ強化策を速やかに実施することを推奨します。

1.  **攻撃元IPアドレスのブロック:**
    *   ファイアウォールまたはIPS/IDSにて、攻撃元IPアドレス `116.99.174.17` からのすべての通信を恒久的にブロックする。
2.  **パスワードポリシーの強化と認証情報の更新:**
    *   今回の攻撃で成功した認証情報（`root` / `admin`, `root` / `@`, `root` / `root123`, `root` / `12345`, `root` / `abcd1234`, `root` / `ipscan`, `root` / `alpine` など）が実システムで利用されていないか緊急で確認し、もし利用されている場合は直ちに強力なパスワードに変更する。
    *   すべてのシステムにおいて、推測されやすいユーザー名やパスワードの使用を禁止する厳格なパスワードポリシー（文字数、複雑性、辞書登録ワードの禁止など）を強制する。
    *   デフォルトパスワードの使用を廃止し、システム導入時に必ず初期パスワードを変更する運用を徹底する。
3.  **多要素認証 (MFA) の導入:**
    *   SSHアクセスを含むすべての重要な認証経路に多要素認証を導入し、認証情報の窃取・突破によるリスクを大幅に低減する。
4.  **公開鍵認証の利用とパスワード認証の無効化:**
    *   SSHサービスにおいて、よりセキュリティの高い公開鍵認証を必須とし、パスワード認証を無効化することを強く推奨する。
5.  **SSHアクセス制限の強化:**
    *   SSHサービスへのアクセス元IPアドレスを制限し、許可された特定のIPアドレスまたはサブネットからのみ接続を許可する。
    *   SSHポートを標準の22番以外に変更することを検討する（ただし、これは一時的な緩和策であり、本質的な対策ではない）。
    *   SSHサービスへのログイン試行回数にレートリミットを設定し、ブルートフォース攻撃を困難にする。
6.  **ログ監視とアラート機能の強化:**
    *   SSHログイン試行失敗が一定回数以上発生した場合に、即座にセキュリティチームへアラートが送信されるようSIEM/ログ管理システムの設定を強化する。
    *   SSHログイン成功後における不審なアクティビティ（例: `direct-tcpip`リクエスト、未知のコマンド実行、異常なプロセス起動など）を検出できるよう、ログ監視ルールを見直す。
7.  **定期的な脆弱性スキャンとペネトレーションテスト:**
    *   社内システムに対する定期的な脆弱性スキャンおよびペネトレーションテストを実施し、同様の認証情報の脆弱性や設定ミスがないか確認・改善する。
8.  **従業員のセキュリティ意識向上トレーニング:**
    *   強力なパスワードの重要性やフィッシング詐欺など、一般的なサイバー攻撃に関する従業員のセキュリティ意識向上トレーニングを定期的に実施する。

これらの対策を実施することで、同様のブルートフォース攻撃に対する耐性を向上させ、潜在的なセキュリティリスクを低減することができます。
