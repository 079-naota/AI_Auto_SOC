# SOC自動分析レポート: 176.65.132.254

## SOC分析レポート

**件名: SSHサービスに対するブルートフォース攻撃および偵察活動の検出**

### 1. 攻撃の概要と目的

*   **インシデント発生日時:** 2026-04-26T00:11:25Z から 2026-04-26T00:14:18Z
*   **攻撃元IPアドレス:** 176.65.132.254
*   **攻撃対象サービス:** SSH (ポート22)
*   **攻撃の種類:** ブルートフォース攻撃（辞書攻撃）、システム情報偵察

**概要:**
攻撃者は、約3分間という短時間に、単一のIPアドレス (176.65.132.254) からSSHサービスに対し、集中的なログイン試行を実施しました。多数の異なるユーザー名とパスワードの組み合わせが試行され、その中には一般的なデフォルトクレデンシャル (`sam/sam`, `odoo/odoo`, `plex/plex`, `david/david` など) が含まれていました。

ログはCowrieハニーポットによって記録されており、ハニーポット環境では複数回にわたって `root` ユーザーでのログインが成功したと記録されています（これはハニーポットの応答であり、実際のシステムへの侵入ではありません）。ログインが成功したとされる後には、一貫して `uname -s -v -n -r -m` コマンドが実行されています。

**攻撃の目的:**
攻撃の主要な目的は、SSHサービスに対する不正な認証情報を特定し、システムへのアクセス権を獲得することにあると推測されます。ログイン成功後に `uname` コマンドを実行していることから、攻撃者は侵入後の初期偵察として、標的システムのオペレーティングシステム、カーネルバージョン、ホスト名、CPUアーキテクチャなどの基本的なシステム情報を収集しようとしていたと考えられます。この情報は、後続の攻撃フェーズ（例: 特定のOSやバージョンに存在する脆弱性を悪用するエクスプロイトの選択、マルウェアの配布など）のための準備として利用される可能性があります。

### 2. 推測される手法・使用ツール

*   **攻撃手法:**
    *   **ブルートフォース攻撃 (辞書攻撃):** 短時間で多様なユーザー名とパスワードの組み合わせを試行していることから、攻撃者が辞書攻撃によって脆弱な認証情報を特定しようとしていることが強く示唆されます。特に `root` ユーザーに対して多数のパスワードを試行し、複数の異なるパスワードでのログインが記録されています。
    *   **システム情報の偵察 (Reconnaissance):** ログイン成功後に毎回 `uname -s -v -n -r -m` コマンドが実行されていることは、攻撃者がターゲットシステムの環境情報を収集し、その後の攻撃計画に役立てようとしている明確な証拠です。
*   **使用ツール (推測):**
    *   **自動化されたブルートフォースツール:** 連続的かつ高速なログイン試行パターンから、`Hydra`、`Nmap` のNSEスクリプト、`Metasploit` のモジュール、またはカスタムスクリプトなど、自動化されたSSHブルートフォースツールが使用されている可能性が高いです。
    *   **シェルコマンド:** `uname` コマンドの実行は、ログイン後のシェルセッションを通じて行われており、攻撃者が予め準備したコマンドリストやスクリプトの一部であると推測されます。

### 3. 脅威レベルとその理由

**脅威レベル: 中 (Medium)**

**理由:**

1.  **ハニーポットへの攻撃であるため直接的な被害はない:** 今回のログはハニーポット (Cowrie) 上で観測されたものであり、実際の運用システムが直接的な被害を受けたわけではありません。
2.  **ブルートフォースと偵察の複合的な性質:** しかし、この活動は、システムへの不正アクセスを試み、その後にシステムの情報を収集するという、典型的な攻撃チェーンの初期段階を示しています。これは、実際のシステムに同様の攻撃が行われた場合、重大なセキュリティ侵害に繋がる可能性が高いことを意味します。
3.  **持続的な攻撃意図:** 短時間での大量な認証試行と、ログイン成功後のシステム情報収集は、攻撃者が執拗にシステムへの侵入と支配を目指している強い意図を示しており、無差別なスキャンに留まらない悪意があります。
4.  **`root` ユーザーへの集中:** `root` ユーザーのクレデンシャルを執拗に狙っている点は、最も高い権限でのシステム掌握を意図しているため、特に危険です。

以上の理由から、現在のところ直接的な被害はないものの、実際のシステムであれば重大なセキュリティインシデントにつながる可能性があるため、脅威レベルを「中」と評価します。

### 4. 推奨アクション

1.  **攻撃元IPアドレスの緊急ブロック:**
    *   ファイアウォール、IDS/IPS、またはルーターにおいて、攻撃元IPアドレス `176.65.132.254` からのSSH (ポート22) へのアクセスを直ちにブロックします。
    *   必要に応じて、このIPアドレスが属する広範な地理的・ネットワーク的範囲からのアクセス制限も検討します。
2.  **SSHセキュリティ設定の強化 (一般的な推奨事項):**
    *   **パスワード認証の無効化:** 可能であれば、SSHパスワード認証を無効にし、公開鍵認証のみに移行します。
    *   **強力なパスワードポリシーの実施:** パスワード認証を維持する場合、十分な長さ、文字種混合を強制する複雑なパスワードポリシーを適用し、定期的なパスワード変更を義務付けます。
    *   **`root` ユーザーの直接ログイン禁止:** SSH経由での `root` ユーザーの直接ログインを禁止し、一般ユーザーでログイン後、`sudo` を利用するように設定します。
    *   **SSHポートの変更:** デフォルトの22番ポートから別の高位ポートに変更することを検討します（ただし、これは攻撃を完全に防ぐものではなく、自動化されたスキャンからの露出を減らす効果があります）。
    *   **認証試行回数の制限:** `sshd_config` で `MaxAuthTries` を低い値（例: 3回）に設定し、ログイン失敗回数を制限します。
3.  **ブルートフォース対策ツールの導入/設定強化:**
    *   `Fail2Ban` のようなツールを導入・設定し、連続したSSHログイン失敗を検知した場合、そのIPアドレスを自動的かつ一時的にブロックする仕組みを導入します。
4.  **ログ監視とアラートの強化:**
    *   SSHログインログ（特に `root` ユーザーへのログイン試行や、短時間での複数回ログイン失敗）の監視を強化し、異常が検知された際には速やかにセキュリティチームにアラートが上がるように設定します。
    *   ハニーポットのログも継続的に収集・分析し、新たな攻撃トレンドやクレデンシャルリストの更新を把握します。
5.  **システム脆弱性の定期的な評価とパッチ適用:**
    *   攻撃者がシステム情報を収集した目的は、特定の脆弱性を悪用するためである可能性が高いため、定期的な脆弱性スキャンを実施し、見つかった脆弱性には速やかにパッチを適用します。
6.  **インテリジェンスの共有:**
    *   今回観測された攻撃元IPアドレスや試行されたクレデンシャル情報（特に成功したとされるもの）を、関連するISACやCSIRT、セキュリティコミュニティと共有し、他組織への注意喚起と脅威インテリジェンスの向上に貢献します。
