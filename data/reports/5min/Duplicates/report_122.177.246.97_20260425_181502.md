# SOC自動分析レポート: 122.177.246.97

SOC分析レポート

**件名: SSHサービスに対するブルートフォース攻撃および不正アクセス、永続化の試み**

**攻撃元IP**: 122.177.246.97
**分析日時**: 2026-04-25T09:00:00Z (UTC)

---

### 1. 攻撃の概要と目的

攻撃者はIPアドレス `122.177.246.97` から、ターゲットシステムに対するSSHブルートフォース攻撃を実行しました。この攻撃では、複数のユーザー名（`deployer`, `root`, `deploy`, `vnc`, `345gs5662d34`）と複数のパスワードの組み合わせが試行されました。

攻撃は最終的に`root`ユーザーに対して複数のパスワード（`ccQQ123`, `3245gs5662d34`, `AAaa112233`, `Master2025`）でログインに成功しています。ログイン成功後、攻撃者は直ちにシステムへの永続的なアクセス経路を確立しようとしました。具体的には、ユーザーのホームディレクトリにある既存の`.ssh`ディレクトリを削除し、自身のSSH公開鍵（コメント: `mdrfckr`）を`~/.ssh/authorized_keys`に書き込むことで、パスワード認証なしで将来的に再侵入を試みるバックドアを設置しようとしました。

一連の行動は、ターゲットシステムへの不正アクセスと、その後の永続的な足がかりの確立を目的としています。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃 / パスワードスプレー攻撃**: 複数の一般的なユーザー名や推測されるユーザー名と、辞書攻撃や総当たり攻撃によるパスワードの組み合わせを自動的に試行し、認証情報を特定しています。特に`root`アカウントに対する集中的なパスワード試行が観測されています。
    *   **永続化**: ログイン成功後、`~/.ssh/authorized_keys`に攻撃者のSSH公開鍵を設置することで、パスワード認証を経ずにSSH鍵認証で再アクセスできるバックドアを確立しようとしています。これは、システムの永続的な制御を目的とした典型的な手法です。
    *   **ファイル属性変更試行**: `chattr -ia .ssh`および`lockr -ia .ssh`といったコマンドを試行していることから、`.ssh`ディレクトリに対する不変属性の解除や、ファイルの不正な属性付与を通じて、自身の操作を容易にしようとした可能性があります。（`lockr`コマンドはハニーポット環境では存在しないため失敗しています。）
*   **使用ツール**:
    *   **自動化されたSSHクライアント**: 大量のログイン試行および成功後のコマンド実行が短時間に繰り返されていることから、HydraやNmapのNSEスクリプト（`ssh-brute`など）、あるいはカスタムスクリプトのような自動化ツールが使用されていると推測されます。
    *   **SSH鍵ペア**: 永続化のために、攻撃者は自身のSSH鍵ペア（公開鍵のコメントは`mdrfckr`）を用意しています。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

1.  **最高権限アカウントの侵害**: `root`ユーザーとして複数のパスワードでログインに成功しており、これはシステム内で最高の権限を持つアカウントが侵害されたことを意味します。`root`権限は、システムの完全な制御権を攻撃者に与えるため、非常に深刻な事態です。
2.  **永続的なアクセス経路の確立**: 攻撃者がSSH公開鍵を設置し、パスワードなしでの再ログインを試みたことは、システムへの恒久的なアクセスポイントを確立しようとする明確な意図を示しています。これにより、一度ログインされてしまえば、パスワード変更後も攻撃者がシステムに再侵入できるリスクが生じます。
3.  **システム乗っ取りの危険性**: `root`権限での不正ログインと永続化の試みは、攻撃者がシステムのデータを窃取したり、マルウェアをインストールしたり、さらにはシステムをC2（Command and Control）サーバーの一部として悪用したりする可能性が高いことを示唆しています。
4.  **自動化された攻撃の可能性**: 攻撃のパターンから、自動化されたスクリプトやツールが使用されている可能性が高く、これはより広範な攻撃キャンペーンの一部である可能性も示唆しています。

### 4. 推奨アクション

#### 4.1. 緊急対応（本番環境で発生した場合を想定）

1.  **対象システムの緊急隔離**: 攻撃を受けたシステムをネットワークから直ちに隔離し、さらなる被害拡大や情報漏洩を阻止します。
2.  **侵害された認証情報の無効化と再設定**:
    *   ログに記録されたすべての成功したパスワード（`ccQQ123`, `3245gs5662d34`, `AAaa112233`, `Master2025`）を含む、`root`ユーザーのパスワードを直ちに非常に複雑なものに変更します。
    *   これらのパスワードが他のシステムやサービスで使い回されていないか緊急で確認し、使い回されている場合はすべて変更します。
    *   その他のログイン失敗ログに登場したユーザー名（`deployer`, `deploy`, `vnc`, `345gs5662d34`）についても、当該ユーザーの存在確認とパスワードリセットを検討します。
3.  **不正なSSH公開鍵の削除**:
    *   `root`ユーザーのホームディレクトリ内（`~/.ssh/authorized_keys`）に攻撃者によって追加されたSSH公開鍵（コメント: `mdrfckr`）を速やかに削除します。
    *   システム上の他のユーザーアカウントの`authorized_keys`ファイルもすべて確認し、不正なSSH公開鍵が設置されていないことを徹底的に検査します。
4.  **影響範囲の特定とフォレンジック調査**:
    *   攻撃者が他にどのような活動を行ったか（ファイルの作成/変更/削除、プロセスの実行、ネットワーク通信、データの窃取など）を特定するため、システムの詳細なフォレンジック調査を実施します。
    *   インストールされたマルウェアがないか、バックドアが他に設置されていないかを確認します。

#### 4.2. 恒久対策

1.  **パスワードポリシーの強化と多要素認証 (MFA) の導入**:
    *   すべてのシステムユーザーに対し、強力で推測困難なパスワードポリシー（文字数、文字種、有効期限など）を強制します。
    *   SSHログインを含む重要なサービスには多要素認証 (MFA) を導入し、パスワードが漏洩した場合でも不正ログインを防ぎます。
2.  **SSH設定の強化**:
    *   **パスワード認証の無効化**: SSH鍵認証のみを許可し、パスワード認証を無効にします (`PasswordAuthentication no`)。SSH鍵には強力なパスフレーズを設定し、安全に管理します。
    *   **rootユーザーの直接ログイン禁止**: `root`ユーザーの直接SSHログインを禁止し (`PermitRootLogin no`)、必要に応じて一般ユーザーでログイン後、`sudo`を使用して権限昇格する運用を徹底します。
    *   **SSHポートの変更**: 標準のポート22番以外のポートに変更することで、自動化されたスキャンからのヒット数を減らします（これはセキュリティ・バイ・オブスキュリティであり、根本的な対策ではありませんが、初期の攻撃を抑制する効果があります）。
    *   **アクセス元IPアドレスの制限**: 信頼できるIPアドレス範囲からのみSSHアクセスを許可するよう、ファイアウォールやSSH設定で制限します。
3.  **侵入検知/防御システム (IDS/IPS) の導入・強化**:
    *   SSHブルートフォース攻撃や不正なログイン試行を検知し、自動的にブロックできるIDS/IPSソリューションを導入または設定強化します。
4.  **ログ監視の強化**:
    *   SSHログインログ、認証ログ、コマンド実行ログなどをリアルタイムで監視する体制を構築します。SIEM（Security Information and Event Management）などのツールを活用し、異常なアクセスや活動パターンを自動的に検知してアラートを発報するよう設定します。
5.  **脆弱性管理とパッチ適用**:
    *   オペレーティングシステム、アプリケーション、ミドルウェアなど、すべてのソフトウェアに対して定期的な脆弱性スキャンを実施し、発見された脆弱性には速やかにセキュリティパッチを適用します。
6.  **セキュリティ意識向上トレーニング**:
    *   従業員に対し、フィッシング攻撃、ソーシャルエンジニアリング、強力なパスワードの重要性など、基本的なセキュリティ意識向上トレーニングを定期的に実施します。
