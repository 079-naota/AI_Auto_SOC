## SOCアナリストによる攻撃ログ分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `185.216.119.134` から、対象システムに対してSSHサービスへの集中的なブルートフォース攻撃が確認されました。この攻撃は複数の異なるユーザー名とパスワードの組み合わせを短時間にわたって試行しており、特に `root` ユーザーに対しては、以下の複数のパスワードでログイン成功を収めています。

*   `root` / `123.com.`
*   `root` / `3245gs5662d34`
*   `root` / `nPSpP4PBW0`
*   `root` / `Aa112211.`
*   `root` / `123Qwe!@#`
*   `root` / `Admin12!@#`

ログイン成功後、攻撃者は自身のSSH公開鍵をターゲットシステムの `root` ユーザーの `.ssh/authorized_keys` ファイルに追加しようと試みています。これは、**永続的なアクセス経路（バックドア）を確立し、パスワードなしで将来的にシステムに再アクセスすることを目的としている**と推測されます。また、コマンド実行の前に `chattr -ia .ssh` や `lockr -ia .ssh` といった、`.ssh` ディレクトリの属性変更を試みることで、既存の防御機構を無効化しようとする意図も見て取れます。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃（総当たり攻撃）/ 辞書攻撃**: 広範なユーザー名（`zabbix`, `john`, `satya`, `admin`, `test`, `user` など）とパスワードの組み合わせを自動的に試行しています。特に `root` ユーザーに対するログイン試行が顕著です。
    *   **永続化 (Persistence)**: ログイン成功後、速やかに `rm -rf .ssh && mkdir .ssh && echo "..." >>.ssh/authorized_keys && chmod -R go= ~/.ssh` というコマンドを実行し、自身のSSH公開鍵を登録することで、パスワードなしで再ログインできる状態を確立しようとしています。これはシステムへの長期的なアクセスを確保するための典型的な手法です。
    *   **防御回避**: `chattr -ia .ssh` コマンドでファイル属性を変更しようとするのは、セキュリティ機能（例: immutable属性など）を無効化し、自身の公開鍵を書き込めるようにするための防御回避の試みです。
*   **使用ツール**:
    *   ログの一貫したパターンと自動化されたコマンド実行から、この攻撃は**自動化されたスクリプト、ボット、またはボットネットの一部**によって実行されている可能性が極めて高いです。特定のツール名は特定できませんが、SSHブルートフォースツール（例: Hydra, NmapのNSEスクリプトなど）や、これらを利用するマルウェア（例: Miraiの亜種など）が考えられます。

### 3. 脅威レベルとその理由

*   **脅威レベル**: **緊急 (Critical)**

*   **理由**:
    *   **`root` ユーザーでの複数回のログイン成功**: ログは、システム上で最も高い権限を持つ `root` ユーザーに対して、複数の異なるパスワードでログインが成功したことを示しています。これは、攻撃者がシステムの**完全な制御権**を取得する可能性があったことを意味します。
    *   **永続化メカニズムの積極的な試行**: ログイン成功後、攻撃者はバックドアとしてSSH公開鍵を登録しようと試みており、これはシステムへの永続的なアクセスを確保し、いつでも再侵入する意図があることを明確に示しています。これにより、単なる一時的な侵入ではなく、長期的な悪用を計画していると判断できます。
    *   **自動化された広範な攻撃**: 人間による手動攻撃では考えられないほど短時間で多数のログイン試行と複数のログイン成功、そして定型的なコマンド実行が行われているため、これは自動化されたマルウェアやボットによる攻撃です。このような攻撃は通常、特定のターゲットだけでなく、インターネット上の脆弱なシステムを広くスキャン・攻撃しており、同じ手法で他のシステムも既に侵害されている可能性があります。
    *   **潜在的な影響の甚大さ**: もしこの攻撃が実際のシステムで成功していた場合、攻撃者はシステム内の情報の窃取、改ざん、破壊、他のシステムへの横展開、マルウェアの展開、DDoS攻撃の踏み台化など、あらゆる悪意のある活動を実行できる状態になります。

### 4. 推奨アクション

この深刻な事態に対処するため、以下の緊急対応および恒久対策を速やかに実施することを推奨します。

#### 4.1. 緊急対応（Immediate Actions）

1.  **攻撃元IPのブロック**: ファイアウォール、IPS/IDS、またはネットワークACLにおいて、攻撃元IPアドレス `185.216.119.134` からの全トラフィックを即座にブロックしてください。
2.  **侵害された可能性のある認証情報の無効化/変更**:
    *   ログに記録された `root` ユーザーのパスワード（`123.com.`, `3245gs5662d34`, `nPSpP4PBW0`, `Aa112211.`, `123Qwe!@#`, `Admin12!@#`）が、他のシステムやサービスで使われていないか緊急で確認し、もし使われている場合は速やかに変更してください。
    *   `root` ユーザーでのSSH直接ログインを無効化し、必要に応じて `sudo` を介したアクセスに制限してください。
3.  **バックドアの確認と削除**:
    *   全ユーザー（特に `root`）のホームディレクトリ下の `.ssh/authorized_keys` ファイルを緊急で確認し、ログに示された公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`）が登録されていないか確認し、存在する場合は**即座に削除**してください。
    *   `.ssh` ディレクトリや関連ファイルのパーミッションが不正に変更されていないかも確認してください。
4.  **システム侵害のフォレンジック調査**:
    *   このシステムが実際に侵害された可能性があるため、詳細なフォレンジック調査チームを招集し、システム全体の健全性を評価してください。不審なプロセス、不審なファイル、隠されたバックドア、不審なネットワーク接続、設定変更などを徹底的に調査します。
    *   特に、攻撃者が `chattr` コマンドで永続化を試みたことから、他の永続化メカニズム（例: cronジョブ、サービス、rootkitなど）も確認してください。
5.  **影響範囲の特定**:
    *   同じ認証情報や脆弱性が他のシステムで使用されていないか、組織内の全システムを対象に緊急監査を実施してください。

#### 4.2. 恒久対策（Long-term Measures）

1.  **SSHアクセス制御の強化**:
    *   **公開鍵認証の必須化**: パスワード認証を無効化し、公開鍵認証のみを許可します。
    *   **ポート変更**: SSHサービスのデフォルトポート（22番）を他の非標準ポートに変更し、一般的なスキャンからの可視性を低減します。
    *   **IPアドレス制限**: SSHアクセスを許可するIPアドレスを、信頼できるネットワーク（例: 社内ネットワーク、VPN接続元）に限定します。
    *   **多要素認証 (MFA) の導入**: 公開鍵認証と組み合わせることで、さらにセキュリティを強化します。
2.  **強力なパスワードポリシーの徹底**:
    *   システム内のすべてのアカウントに対し、複雑性、長さ、変更頻度に関する厳格なパスワードポリシーを強制します。
    *   使い回しパスワードの禁止を徹底します。
3.  **アカウント管理の最適化**:
    *   `root` ユーザーでの直接SSHログインを恒久的に禁止します。
    *   不要なユーザーアカウントを削除または無効化します。
    *   一般的なユーザー名（`admin`, `test`, `user`, `zabbix` など）の使用を避けます。
4.  **侵入検知・防御システム (IDS/IPS) の導入・強化**:
    *   SSHブルートフォース攻撃を自動的に検知・ブロックできるIDS/IPSソリューションを導入し、最新の脅威シグネチャに更新します。
    *   異常なログイン試行回数を閾値として、IPアドレスの自動ブロックを設定します。
5.  **ログ監視とSIEM連携の強化**:
    *   SSH認証ログやシステムログをリアルタイムで集中監視し、SIEM (Security Information and Event Management) システムと連携して異常なログイン試行や成功を即座に検知・アラート化できる体制を構築します。
    *   攻撃元IPアドレス `185.216.119.134` を脅威インテリジェンスリストに追加し、監視を強化します。
6.  **脆弱性管理とパッチ適用**:
    *   定期的な脆弱性スキャンを実施し、システムやアプリケーションの既知の脆弱性を速やかに特定・修正します。
7.  **セキュリティ意識向上トレーニング**:
    *   従業員に対し、安全なパスワードの利用、フィッシング詐欺への注意喚起など、セキュリティ意識向上トレーニングを定期的に実施します。
