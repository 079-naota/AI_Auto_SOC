## SOC分析レポート

### 攻撃概要

#### 1. 攻撃の概要と目的

攻撃元IPアドレス `182.70.243.207` から、SSHサービスを標的としたブルートフォース攻撃と、それに続くシステムへの永続的なアクセス確立の試みが確認されました。

攻撃者はまず、一般的なユーザー名（`steam`, `tunnel`, `cloud`, `gitlab`など）やシステムアカウント（`root`）に対して、辞書攻撃またはパスワードリスト攻撃を実行しました。複数回の試行の後、複数のパスワードで `root` ユーザーとしてログインに成功しています。

ログイン成功後、攻撃者は以下の目的で一連のコマンドを実行しました。

*   **永続化の確立**: `~/.ssh/authorized_keys` ファイルに不正なSSH公開鍵（コメント: `mdrfckr`）を追記し、パスワードなしで将来的にシステムにアクセスできるバックドアを設置しようとしました。これにより、アカウントパスワードが変更された場合でもアクセスを維持することを意図しています。
*   **アクセス権限の確保**: `.ssh` ディレクトリの属性を変更しようとするコマンド（`chattr`, `lockr`）を試行しており、これにより `authorized_keys` ファイルへの書き込みを確実にしようとしています。（ただし、`lockr`コマンドは失敗しています）

一連の行動は、ターゲットシステムへの不正な侵入と、その後の継続的なアクセスを確保するための典型的な手口を示しています。

#### 2. 推測される手法・使用ツール

*   **アクセス手法**:
    *   **ブルートフォース攻撃 / 辞書攻撃**: 多数のユーザー名とパスワードの組み合わせを自動的に試行することで、SSHアカウントの認証情報を特定しようとしました。特に `root` ユーザーに対して複数のパスワードで成功していることから、一般的なパスワードリストを使用している可能性が高いです。
    *   **SSHサービスへの接続**: 標準的なSSHクライアント、または自動化されたツールを使用しています。
*   **使用ツール**:
    *   **自動化された攻撃スクリプト**: 連続したログイン試行パターンと、ログイン成功後に決まったコマンド（`cd ~; chattr -ia .ssh; lockr -ia .ssh` および `rm -rf .ssh && mkdir .ssh && echo "..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh`）を実行していることから、特定のマルウェアやハッキングツールに組み込まれた自動スクリプトが使用されていると推測されます。
    *   `lockr` コマンドが失敗していることから、攻撃ツールが対象システムの環境を誤解している、または特定のユーティリティが存在することを前提としている可能性もあります。
    *   公開鍵のコメント `mdrfckr` は、この特定の攻撃キャンペーンや攻撃グループが使用するシグネチャの一部である可能性があります。

### 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:

1.  **rootアカウントへのログイン成功**: 複数回にわたり `root` ユーザーとしてシステムにログインすることに成功しています。root権限の奪取は、システム全体に対する完全な制御を意味し、最も深刻な侵害の一つです。
2.  **永続化の試み**: 攻撃者は `authorized_keys` ファイルに自身の公開鍵を追加することで、システムにバックドアを設置し、パスワード変更後もアクセスを維持しようとしています。これは長期的なシステム侵害の意図を示しており、侵害の深度が高いことを意味します。
3.  **自動化された攻撃**: 複数ユーザー名・パスワードの試行、ログイン後の決まったコマンド実行パターンから、攻撃が自動化されたツールによって行われていることが分かります。これは、特定のターゲットに対する手動攻撃よりも広範な影響を持つ可能性があります。
4.  **複数の成功パスワード**: `root`に対して複数の異なるパスワード (`QAZwsx`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`) でログインが成功しています。これは、実際のシステムであれば、これらのパスワードが容易に推測され得る弱いパスワードであったか、または異なるタイミングで異なる攻撃者/ツールによって脆弱性が悪用された可能性を示唆します。

### 推奨アクション

#### 4. 推奨アクション

このログはCowrieハニーポットのものであるため、実際にシステムが侵害されたわけではありませんが、この種の攻撃に備え、以下の対策を速やかに実施することを強く推奨します。

**A. 緊急対応（即時実施）**

1.  **攻撃元IPのブロック**: 攻撃元IPアドレス `182.70.243.207` からのすべての通信を、ファイアウォール、IDS/IPS、またはセキュリティグループで即座にブロックリストに追加します。
2.  **SSHパスワードの変更**:
    *   ログに記録された `root` ユーザーおよび、ログイン試行のあった全てのユーザーアカウントのパスワードを直ちに、かつ複雑なものに変更します。
    *   特に、ログイン成功に使用されたパスワード (`QAZwsx`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`, `3245gs5662d34`) は、二度と使用されないようにブラックリスト化を検討してください。
3.  **不審なSSH公開鍵の確認と削除**:

4.  **多要素認証 (MFA) の導入**: SSHアクセスにMFAを導入し、認証の強度を高めます。

**B. フォレンジックと詳細調査（潜在的な侵害への対応）**

1.  **システムログの調査**: ログに記録されたログイン成功日時 (`2026-04-26T01:26:29Z`, `01:30:42Z`, `01:31:42Z`, `01:32:43Z`, `01:33:41Z`, `01:34:42Z`) 付近の認証ログ（例: `/var/log/auth.log` や `journalctl`）を詳細に調査し、実際のサーバーに不正ログインが発生した形跡がないか確認します。
2.  **不審なアクティビティの検出**: ログに記録されたコマンド以外の、不審なプロセス、ファイルの作成/変更、ネットワーク接続、異常なユーザーアクティビティがないか広範囲に調査します。
3.  **攻撃ツールの特定**: 攻撃者が使用したと推測される公開鍵 (`mdrfckr`) を脅威インテリジェンスプラットフォームで検索し、既知の攻撃グループやマルウェアキャンペーンに関連するか調査します。
4.  **脆弱性スキャン**: システムに対する脆弱性スキャンを実施し、他の潜在的な侵入経路がないか確認します。

**C. 予防策とセキュリティ強化（将来の攻撃への対策）**

1.  **SSH設定の強化**:
    *   **パスワード認証の無効化**: 可能な限り公開鍵認証のみを許可し、パスワード認証を無効にします。
    *   **SSHポートの変更**: 標準のポート22から別の高ポート番号に変更します（ポートスキャンによる発見は防げませんが、一般的な自動攻撃の対象から外れる可能性が高まります）。
    *   **rootログインの禁止**: `/etc/ssh/sshd_config` で `PermitRootLogin no` を設定し、rootユーザーでの直接ログインを禁止します。
    *   **厳格な認証設定**: `MaxAuthTries` を低く設定し、ログイン試行回数を制限します。
2.  **侵入検知・防止システム (IDPS) の導入**: Fail2banのようなツールを導入し、連続したログイン失敗を自動的に検知・ブロックする仕組みを構築します。
3.  **アカウント管理の強化**:
    *   不要なユーザーアカウントを削除します。
    *   強力なパスワードポリシーを徹底し、定期的なパスワード変更を義務付けます。
    *   最小権限の原則に基づき、ユーザーに必要最小限の権限のみを付与します。
4.  **定期的なセキュリティパッチ適用**: OSやアプリケーションのセキュリティパッチを常に最新の状態に保ち、既知の脆弱性を悪用されるリスクを低減します。
5.  **セキュリティ監視の強化**: ログ監視体制を強化し、不審なログイン試行やシステムアクティビティをリアルタイムで検知できるようなアラート設定を行います。
6.  **ネットワークセグメンテーション**: SSHアクセスを許可するIPアドレスを限定し、必要最小限のネットワークからのアクセスのみを許可するようにします。
