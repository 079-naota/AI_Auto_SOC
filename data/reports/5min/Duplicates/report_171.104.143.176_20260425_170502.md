## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `171.104.143.176` から、当方ハニーポットシステム (Cowrie) に対し、SSHサービスを標的とした攻撃が試みられました。

攻撃は以下の段階で進行しました。

1.  **偵察と認証情報の試行:** 攻撃者はまず、一般的なユーザー名とパスワード (`deploy`/`admin`) を用いた認証を試みましたが、これは失敗しました。
2.  **認証成功と初期侵害:** その後、別の認証情報 (`root`/`1234ASDF` および `root`/`QWEasd123456`) を使用して、rootユーザーとしてログインに成功したと攻撃者は認識しました。
3.  **永続化メカニズムの確立:** ログイン成功後、攻撃者は自身の公開鍵（コメント `mdrfckr`）を `.ssh/authorized_keys` に追加することで、永続的なアクセス経路を確立しようとしました。これにより、今後のアクセスでパスワードなしに接続を試みることが可能となります。また、`chattr` コマンドで`.ssh`ディレクトリの保護属性を変更しようとしています。
4.  **システム偵察と競合排除:** 攻撃者は、システム環境（CPU情報、メモリ、OSバージョン、ディスク使用量、稼働プロセス、cronジョブなど）に関する詳細な情報を収集するため、多数の標準的なLinuxコマンドを実行しました。さらに、`rm -rf /tmp/secure.sh; rm -rf /tmp/auth.sh; pkill -9 secure.sh; pkill -9 auth.sh; echo > /etc/hosts.deny; pkill -9 sleep;` といったコマンドで、他のマルウェアや自身の残骸をクリーンアップし、`/etc/hosts.deny` をクリアすることで、システムの完全な制御を試みる意図が伺えます。
5.  **パスワード変更の試行:** `chpasswd` コマンドにより、rootユーザーのパスワードを複数回変更しようとしています。

これらの行動から、攻撃の主な目的は、root権限でのシステムへの不正侵入とその後の永続的なアクセス経路の確保、そしてシステム情報の収集を通じて、さらなる悪意のある活動（例：マルウェア展開、リソースの悪用、横展開など）のための足がかりを築くことであると推測されます。

### 2. 推測される手法・使用ツール

*   **攻撃手法:**
    *   **SSH ブルートフォース/辞書攻撃 (T1110.001 - Brute Force: Password Guessing):** 複数のユーザー名とパスワードの組み合わせを試行し、認証を突破しようとしています。
    *   **認証情報の盗用/利用 (T1078 - Valid Accounts):** 成功したログイン試行は、有効な認証情報（ハニーポット上での成功）を利用したものです。
    *   **永続化 (T1136.001 - Create Account: Local Account / T1098.004 - Account Manipulation: SSH Authorized Keys):** `.ssh/authorized_keys` ファイルに公開鍵を追加することで、将来的なアクセスを維持しようとしています。
    *   **偵察 (T1592 - Gather Victim Host Information / T1082 - System Information Discovery):** `uname`, `cat /proc/cpuinfo`, `free -m`, `df -h`, `crontab -l`, `top`, `w` などのコマンドを使用して、システムの構成や状態を詳細に把握しようとしています。
    *   **防衛回避 (T1070.004 - Indicator Removal: File Deletion):** `/tmp/secure.sh` などのファイルを削除し、`hosts.deny` をクリアしようとすることで、他の攻撃者による存在や自身の痕跡を消そうとしている可能性があります。
*   **使用ツール/スクリプト:**
    *   **SSHクライアント:** SSH接続のために標準的なSSHクライアントを使用。
    *   **汎用的なシェルスクリプトまたは自動化ツール:** ログイン成功後の一連のコマンド（永続化、偵察、クリーンアップ、パスワード変更）が短時間で多数実行されていることから、攻撃者は事前に準備されたスクリプトや自動化ツール（例：GoLang製SSHボット、Mirai亜種など）を使用している可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

*   **root権限の奪取試行:** 攻撃者はハニーポット上でroot権限でのログインに成功したと認識しており、実システムであれば最高レベルの権限が奪われる状態となります。これはシステムへの完全な制御を意味します。
*   **永続化メカニズムの確立:** SSH公開鍵を`authorized_keys`に追加する試みは、将来的な不正アクセスを容易にし、検知を回避するための典型的な手法です。これにより、恒久的なバックドアが設置されるリスクがあります。
*   **広範な偵察活動:** システムのハードウェア、OS、稼働状況など、多岐にわたる詳細な情報を収集しており、これは攻撃の次の段階（マルウェアの展開、リソースの不正利用、ネットワーク内での横展開など）のための明確な準備行動です。
*   **競合排除の意図:** 他のマルウェアを削除し、システムを独占的に制御しようとする行動は、攻撃者がこのシステムを本格的な活動拠点と見なしていることを強く示唆します。
*   **自動化された攻撃の可能性:** ログのタイムスタンプから、一連の行動が非常に短時間で実行されており、手動ではなく自動化されたスクリプトやボットネットの一部である可能性が高いです。このような攻撃は広範囲に展開され、他のシステムも標的となる可能性があります。

### 4. 推奨アクション

この攻撃はハニーポットに対するものですが、実システムで同様の事態が発生した場合を想定し、以下の推奨アクションを速やかに実行してください。

1.  **攻撃元IPのブロック:**
    *   攻撃元IPアドレス `171.104.143.176` を組織の境界ファイアウォール、IDS/IPS、WAFなどで即座にブロックリストに追加し、今後の接続を拒否します。
2.  **SSH認証の強化:**
    *   **パスワード認証の無効化:** すべての公開 facing のSSHサーバーにおいて、パスワード認証を無効化し、鍵認証のみに限定します。
    *   **多要素認証 (MFA) の導入:** 可能であれば、SSHログインに多要素認証を導入し、セキュリティをさらに強化します。
    *   **デフォルトポートの変更/ポートノッキング:** SSHのデフォルトポート (22) を変更するか、ポートノッキングなどの追加の認証前セキュリティ対策を検討します。
    *   **複雑なパスワードポリシーの強制:** 組織内のすべてのシステムで、推測されにくい複雑なパスワードポリシーを強制し、定期的なパスワード変更を推奨します。特にログに記載されたパスワード（`1234ASDF`, `QWEasd123456`, `dTTVY8OaDRX7`, `UfvraTsg18FW`）が実環境で使われていないか緊急で確認し、変更を促します。
3.  **システム全体の監査と侵害の痕跡調査:**
    *   **`authorized_keys`ファイルの監査:** 全SSHサーバーのすべてのユーザーアカウント（特に`root`）について、`.ssh/authorized_keys`ファイルに身に覚えのない公開鍵が存在しないか確認します。ログに記載された攻撃者の公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) の有無を重点的に確認し、存在する場合は削除します。
    *   **パスワードリセット:** 影響を受けた可能性のあるすべてのユーザー（特に`root`）のパスワードを強制的にリセットします。
    *   **不正なファイルの確認:** `/tmp/secure.sh`, `/tmp/auth.sh` など、ログに示されたファイルの存在をスキャンし、不審なファイルやディレクトリが存在しないか確認します。
    *   **`crontab`の確認:** 全ユーザーの`crontab -l` を確認し、不正な定期実行ジョブが登録されていないか確認します。
    *   **`hosts.deny`の確認:** `/etc/hosts.deny` の内容が不正にクリアされたり、変更されていないか確認します。
    *   **ログの監査:** システムログ（`auth.log`, `secure.log`など）を確認し、SSHログインの失敗/成功履歴、および不明なコマンド実行の痕跡がないか調査します。
4.  **侵入検知システムの強化:**
    *   SSHブルートフォース攻撃、異常なログイン試行、および認証成功後の不審なコマンド実行を検知するIDS/IPSルールやSIEMアラートルールを強化・導入します。
    *   特に`.ssh/authorized_keys`ファイルや`passwd`, `shadow`ファイルなどの重要な設定ファイルの変更を監視するルールを設定します。
5.  **情報共有:**
    *   この攻撃に関する情報を組織内の関連部署や、必要に応じて外部のセキュリティ機関（CSIRT、ISACなど）と共有し、状況に応じた注意喚起と連携を促します。
