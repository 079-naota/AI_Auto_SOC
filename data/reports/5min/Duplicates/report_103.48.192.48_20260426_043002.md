# SOC自動分析レポート: 103.48.192.48

**生成日時:** 2026-04-26 04:32:09

---

## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `103.48.192.48` から、SSHサービスを標的としたブルートフォース攻撃が確認されました。この攻撃者は、複数のユーザー名とパスワードの組み合わせを試行した後、最終的に`root`ユーザーとして複数回ログインに成功しています。ログイン成功後、攻撃者はシステムへの持続的なアクセス経路（バックドア）を確立することを目的とし、以下の悪意ある活動を実行しました。

1.  **rootアカウントへの不正ログイン**: 複数の辞書的なパスワード（例: `Asd123!@#`, `Csdx@13579`, `Pa55word`, `Test123!!`, `Zt123456`）を用いて、`root`ユーザーでのログインに成功しています。
2.  **バックドアの設置**: ログイン成功後、`root`ユーザーのホームディレクトリにある`.ssh`ディレクトリを削除・再作成し、攻撃者自身のSSH公開鍵を`authorized_keys`ファイルに追加しました。これにより、パスワードを知らなくても、この公開鍵を使用することで、いつでもシステムに再ログインできる永続的なアクセス経路が確立されました。追加された鍵のコメントは `mdrfckr` です。
3.  **ファイル属性変更の試み**: バックドア設置に際し、`.ssh`ディレクトリの不可変（immutable）属性や追記専用（append-only）属性を解除しようと`chattr -ia .ssh`コマンドを実行しています。また、`lockr -ia .ssh`という不明なコマンドも試行していますが、これは失敗しています。

攻撃の主な目的は、システムへの最高権限でのアクセスを確保し、その後、将来にわたる持続的なアクセスを可能にするバックドアを設置することであると推測されます。これは、システムの完全な乗っ取り、ボットネットへの組み込み、またはさらなる攻撃のための足がかりとする前段階の行動である可能性が高いです。

### 2. 推測される手法・使用ツール

*   **SSHブルートフォース/辞書攻撃**: ログに記録された多数のログイン失敗と、複数の比較的シンプルな、あるいは一般的なパスワードでのログイン成功から、自動化されたブルートフォースツールや辞書攻撃ツール（例: Hydra, Medusa, Nmapの`ssh-brute`スクリプトなど）が使用されたと推測されます。
*   **OSコマンドインジェクション**: SSHログイン成功後にシェルを介して悪意あるコマンド（`.ssh`ディレクトリの操作、`authorized_keys`への公開鍵書き込み）を実行しており、これは典型的なOSコマンドインジェクションの手法です。
*   **バックドア設置**: 公開鍵認証によるSSHバックドアは、システムへの持続的なアクセスを確立するための一般的な手法です。
*   **自動化されたスクリプト**: ログイン試行からバックドア設置までの一連の行動が迅速かつパターン化されていることから、攻撃者が手動で操作するのではなく、事前に準備されたスクリプトやボットネットの一部として自動的に実行されている可能性が高いです。特に、ログイン成功後に繰り返し`chattr`と`authorized_keys`の変更を試みていることから、これは一般的な攻撃スクリプトに含まれるルーチンであると考えられます。

### 3. 脅威レベルとその理由

**脅威レベル: 非常に高い (Critical)**

**理由:**

1.  **root権限の奪取**: システム上で最も高い権限を持つ`root`アカウントのパスワード認証に成功しているため、攻撃者は理論上、システム上で任意の操作を実行できる状態です。
2.  **持続的アクセス経路の確立**: 攻撃者はSSH公開鍵を`authorized_keys`に設置することで、パスワードを変更されても、あるいはシステムが再起動しても、引き続きシステムへアクセスできるバックドアを確立しようとしています。これは長期的なシステムの乗っ取りを可能にします。
3.  **自動化された攻撃の性質**: 一連の攻撃は自動化されたプロセスによって行われている可能性が高く、これは特定のシステムへの標的型攻撃というよりも、インターネット上の脆弱なSSHサービスを大規模にスキャンし、侵害しようとするボットネット活動の一部である可能性を示唆しています。
4.  **悪意ある意図の明確性**: `.ssh`ディレクトリの操作や公開鍵の追加といった行動は、システムを完全に制御し、悪用するための明確な悪意ある意図を示しています。

### 4. 推奨アクション

この攻撃はハニーポットで捕捉されたものですが、もし実際のシステムで発生していた場合、以下のアクションを速やかに実行する必要があります。

#### 4.1. 緊急対応 (Immediate Action)

1.  **攻撃元IPアドレスのブロック**: ファイアウォール、IDS/IPS、またはSSH設定 (`/etc/hosts.deny`など) を利用して、`103.48.192.48`からの全てのインバウンド接続を直ちにブロックします。
2.  **侵害されたパスワードの変更**: ログに記録されている`root`ユーザーのログイン成功パスワード（`Asd123!@#`, `Csdx@13579`, `Pa55word`, `Test123!!`, `Zt123456`）だけでなく、すべての管理者アカウントおよび重要なユーザーアカウントのパスワードを直ちに複雑で推測困難なものに変更します。
3.  **バックドアの確認と削除**: `root`ユーザーのホームディレクトリ（`/root/.ssh/authorized_keys`）を含む、すべてのユーザーの`~/.ssh/authorized_keys`ファイルを調査し、不明なSSH公開鍵（特にログに記載されている`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`の鍵）があれば削除します。また、`.ssh`ディレクトリやその親ディレクトリのパーミッションが適切であるか確認します。
4.  **システム健全性の確認**: システムに不審なプロセス、不審なファイル、ネットワーク接続がないか、整合性チェックツール（例: AIDE, Tripwire）を使用して調査します。
5.  **システム隔離の検討**: 攻撃が広範囲に及んでいる可能性や、さらなる侵入を阻止するため、侵害されたシステムをネットワークから一時的に隔離することを検討します。

#### 4.2. セキュリティ強化 (Long-term Security Enhancement)

1.  **SSH設定の強化**:
    *   **パスワード認証の無効化**: SSHサービスの設定（`/etc/ssh/sshd_config`）で`PasswordAuthentication no`を設定し、パスワード認証を完全に無効化し、公開鍵認証のみを許可します。
    *   **rootログインの禁止**: `PermitRootLogin no`を設定し、`root`ユーザーの直接ログインを禁止します。sudoを使用して特権コマンドを実行するようにします。
    *   **標準ポートの変更**: SSHポートを標準の22番以外に変更することを検討します（ただし、これは完全な対策ではなく、ブルートフォース攻撃の数を減らす効果がある程度です）。
    *   **多要素認証 (MFA) の導入**: SSHログインにMFAを導入し、セキュリティを強化します。
2.  **ブルートフォース対策ツールの導入**: Fail2banなどのツールを導入し、複数回のログイン失敗があったIPアドレスを自動的にブロックするように設定します。
3.  **強力なパスワードポリシーの適用**: すべてのシステムユーザーに対し、複雑なパスワード（長さ、文字種混合）を強制し、定期的なパスワード変更を義務付けます。
4.  **不必要なアカウントの削除**: 使用されていない、または不必要なユーザーアカウントをすべて削除します。
5.  **システムのパッチ管理**: OSおよびインストールされているすべてのソフトウェアが最新のセキュリティパッチで更新されていることを確認します。
6.  **ログ監視の強化**: SSH認証ログを含むシステムログを集中管理し、異常なログイン試行やコマンド実行パターンをリアルタイムで検知できるSIEM（Security Information and Event Management）システムなどの導入を検討します。
7.  **脅威インテリジェンスの活用**: 攻撃元IPアドレスや使用されたSSH公開鍵のフィンガープリントを脅威インテリジェンスプラットフォームに登録し、他のシステムや組織での同様の攻撃の検知と防御に役立てます。
