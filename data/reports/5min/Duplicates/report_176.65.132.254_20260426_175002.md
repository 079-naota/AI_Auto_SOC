## SOC分析レポート

**件名: SSHサービスに対するブルートフォース攻撃検知および分析**

**報告日:** 2026年04月26日

---

### 1. 攻撃の概要と目的

提供されたCowrieハニーポットログを分析した結果、攻撃元IPアドレス `176.65.132.254` から、当環境のSSHサービスに対する集中的なブルートフォース攻撃が確認されました。

この攻撃は、2026年04月26日00時11分25秒（UTC）から約7分間にわたり実行されています。攻撃者は複数の一般的なユーザー名（`sam`, `odoo`, `hive`, `user`, `elk` など）と弱いパスワードの組み合わせを試行した後、特に `root` ユーザーに対してブルートフォース攻撃を集中させました。ログ上では、以下の7つの異なるパスワードで `root` ユーザーでのログインに成功しています。

*   `root` / `123321`
*   `root` / `abcd@1234`
*   `root` / `12qwaszx`
*   `root` / `root.123`
*   `root` / `A123456a`
*   `root` / `aa123456`
*   `root` / `1234567890`

ログイン成功後、攻撃者は毎回 `uname -s -v -n -r -m` コマンドを実行し、システムの種類、カーネルバージョン、ネットワークノードホスト名、カーネルリリース、マシンハードウェア名といった基本的なシステム情報の収集を試みています。

攻撃の目的は、SSHサービスへの不正アクセスを試み、成功した場合はそのシステムに関する初期偵察を行い、さらなる攻撃（マルウェアの展開、バックドアの設置、他のシステムへの横展開など）のための足がかりを築くことであると推測されます。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **辞書攻撃/ブルートフォース攻撃**: 大量のユーザー名とパスワードの組み合わせを自動的に試行することで、認証情報を突破しようとしました。特に、一般的なユーザー名 (`root` だけでなく、`sam`, `odoo`, `user`, `mysql` など) と、予測しやすいパスワード (`123456`, `password` 系) を組み合わせて試行しています。
    *   **情報収集**: ログイン成功直後に `uname -s -v -n -r -m` コマンドを実行し、ターゲットシステムの基本的なOS情報を収集しています。これは、システムの種類やバージョンに基づいてさらなる脆弱性を悪用するための偵察活動であると考えられます。
*   **使用ツール (推測):**
    *   ログのパターン（短時間での連続したログイン試行、自動化されたコマンド実行）から、攻撃者はHydra, Medusa, NmapのSSHブルートフォーススクリプト、またはPython/Perlなどのスクリプト言語で書かれたカスタムツールといった自動化されたブルートフォースツールを使用している可能性が高いです。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**

1.  **システムへの不正アクセスの試み**: 攻撃者はSSHサービスへの不正ログインを積極的に試みており、実際のシステムが脆弱な認証情報を使用している場合、容易に侵入される可能性があります。
2.  **`root`ユーザーへの複数回ログイン成功**: ログ上ではハニーポットへのログイン成功ですが、実際に運用されているシステムにおいて`root`アカウントで複数の弱いパスワードが使用され、かつ、これらがブルートフォース攻撃によって突破された場合、攻撃者はシステムに対する完全な制御権を獲得し、甚大な被害を引き起こす可能性があります。
3.  **初期偵察の実施**: ログイン成功後に `uname` コマンドを実行していることから、攻撃者は単にアクセスするだけでなく、システムの環境情報を把握し、さらなる悪意ある活動（脆弱性の特定、マルウェアの最適化、権限昇格、永続化など）を計画している可能性が高いです。
4.  **自動化された攻撃**: 短時間で多数のログイン試行が行われていることから、これは自動化されたスクリプトまたはツールによるものであり、広範囲なスキャンや攻撃キャンペーンの一部である可能性が示唆されます。

### 4. 推奨アクション

以下の対策を早急に実施し、類似の攻撃に対する防御体制を強化することを強く推奨します。

1.  **攻撃元IPアドレスのブロック**:
    *   ファイアウォール、IDS/IPS、またはセキュリティグループにて、攻撃元IPアドレス `176.65.132.254` からの全てのインバウンド通信を即座にブロックしてください。
2.  **SSH認証の強化**:
    *   **パスワード認証の無効化**: 可能な限り、SSHサービスではパスワード認証を無効にし、より安全な公開鍵認証のみを許可してください。
    *   **多要素認証 (MFA) の導入**: 重要なシステムへのSSHアクセスには、多要素認証の導入を検討してください。
    *   **パスワードポリシーの強化**: 強固なパスワードポリシー（文字種、長さ、定期的な変更など）を義務付け、デフォルトパスワードや辞書に載っているような弱いパスワードの使用を禁止してください。
    *   **`root`ログインの禁止**: SSHサービスに対する`root`ユーザーの直接ログインを無効にし、通常ユーザーでログイン後、`sudo`を使用して権限昇格する運用を徹底してください。
3.  **ブルートフォース対策ツールの導入**:
    *   Fail2BanやSSHGuardのようなツールを導入し、一定回数以上のログイン失敗が発生したIPアドレスを自動的にブロックする仕組みを構築してください。
4.  **監視体制の強化**:
    *   SSHログインログを常時監視し、不正なログイン試行や成功をリアルタイムで検知・アラートする体制を強化してください。特に、`root`ユーザーへのログイン試行、通常のアクセスパターンと異なるログイン試行に注意を払ってください。
    *   SOCチーム内で、今回のCowrieハニーポットで成功したユーザー名・パスワードの組み合わせを脅威インテリジェンスとして共有し、実際の環境でこれらの認証情報が使用されていないか確認することを推奨します。
5.  **システム設定の確認と脆弱性スキャン**:
    *   全ての公開SSHサービスの設定を確認し、安全な設定になっていることを確認してください。不要なサービスやポートは閉鎖してください。
    *   定期的な脆弱性スキャンを実施し、システムやアプリケーションの脆弱性を早期に特定し、修正してください。
