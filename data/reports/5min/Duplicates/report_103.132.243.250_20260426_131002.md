### 1. 攻撃の概要と目的

提供されたCowrieハニーポットのログによると、攻撃元IPアドレス `103.132.243.250` から、約1時間40分にわたりSSHサービスに対する集中的な攻撃が確認されました。この攻撃は、主にSSHブルートフォース攻撃によって不正アクセスを試み、成功した場合はシステムへの永続的なアクセス経路（バックドア）を確立することを目的としていると分析されます。

具体的には、複数の異なるユーザー名とパスワードを試行した後、特に `root` ユーザーに対して複数のパスワードでログインに成功しています。ログイン成功後、攻撃者は `~/.ssh/authorized_keys` ファイルに自身の公開鍵を追加することで、パスワードなしで再接続できる状態を構築しようとしていました。

### 2. 推測される手法・使用ツール

*   **手法:**
    *   **SSH ブルートフォース/辞書攻撃:** 攻撃者は `oracle`, `mani`, `ftpuser`, そして `root` といった一般的なユーザー名に対し、複数の異なるパスワード（`oracle09!`, `mani`, `135792468`, `abcdefgh`, `qazwsxedc123`, `P@ssw0rd6`, `Admin88888888`, `Gp123456`, `3245gs5662d34` など）を試行しています。特に `root` ユーザーへの攻撃が顕著であり、強力なパスワード推測または辞書攻撃リストを使用している可能性が高いです。
    *   **永続化のためのバックドア設置:** ログイン成功後、攻撃者は直ちに `rm -rf .ssh && mkdir .ssh && echo "ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr">>.ssh/authorized_keys && chmod -R go= ~/.ssh` という一連のコマンドを実行し、自身のSSH公開鍵を`authorized_keys`に追加することで、パスワードなしで再ログインできるバックドアを設置しようとしています。これは、システムへの永続的なアクセスを確立する一般的な手法です。
    *   **属性変更の試行:** `chattr -ia .ssh; lockr -ia .ssh` のコマンド実行も試行されています。`chattr -ia .ssh` は `.ssh` ディレクトリの immutable/append-only 属性を解除し、後の操作を容易にしようとするものです。`lockr` は一般的なLinuxコマンドではないため、何らかの特定のツールやスクリプトの一部であるか、タイポである可能性がありますが、ハニーポット上では失敗しています。
*   **使用ツール:**
    *   大量かつ反復的なログイン試行と、ログイン成功後の定型的なコマンド実行パターンから、攻撃者は **自動化されたスクリプトまたはブルートフォースツール** (例: Hydra, NmapのSSHスクリプトなど) を使用している可能性が非常に高いです。

### 3. 脅威レベルとその理由

*   **脅威レベル: 高 (High)**
*   **理由:**
    1.  **root権限でのログイン成功の可能性:** ログはハニーポットのものですが、もし実際のシステムであった場合、攻撃者は `root` ユーザーとしてシステムに複数回ログインを成功させており、これによりシステムへの完全な制御を奪取できる状態にありました。これは最も深刻な侵害レベルです。
    2.  **永続的なアクセス経路の確立試行:** 攻撃者が `authorized_keys` ファイルに自身の公開鍵を追加しようとした行為は、システムにバックドアを設置し、将来にわたって継続的にアクセスを維持する明確な意図を示しています。これにより、一度侵入が成功すれば、攻撃者は検知されずに長期間システム内に潜伏し、様々な悪意のある活動（情報窃取、マルウェア配布、他のシステムへの踏み台、C2サーバ利用など）を行うことが可能になります。
    3.  **自動化された組織的な攻撃:** 攻撃は断続的かつ自動化されており、特定の標的への執拗な攻撃、または広範なスキャン活動の一環であると考えられます。使用されているパスワードリストは、他のシステムに対しても同様に試行される可能性があります。
    4.  **複数のパスワードの推測成功:** `root`ユーザーに対して複数の異なるパスワードがログイン成功に導いており、これらのパスワードが実環境でも使用されている場合、その脆弱性が深刻であることを示唆しています。

### 4. 推奨アクション

この攻撃がハニーポットに対するものであったとしても、攻撃手法と試行されたクレデンシャルは、実際のシステムにおいても同様の脅威をもたらす可能性があります。以下に推奨アクションを提示します。

#### 4.1. 緊急対応（即時実施）

1.  **攻撃元IP (103.132.243.250) のブロック:** ファイアウォール、IDS/IPS、またはルーターにおいて、このIPアドレスからのすべてのSSH接続試行を直ちにブロックリストに追加し、アクセスを拒否してください。
2.  **SSH公開鍵の確認と削除:** すべてのLinux/Unix系サーバーで、rootユーザーを含む全ユーザーの `~/.ssh/authorized_keys` ファイルを緊急でレビューし、ログに記録された公開鍵 (`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`) と一致するものが存在しないか確認してください。発見された場合は、直ちに該当する行を削除してください。
3.  **侵害された可能性のあるパスワードの特定と変更:** ログに記録されたログイン成功クレデンシャル（例: `root:135792468`, `root:abcdefgh`, `root:qazwsxedc123`, `root:P@ssw0rd6`, `root:Admin88888888`, `root:Gp123456`, `root:3245gs5662d34` および `oracle:oracle09!`, `mani:mani`, `ftpuser:test123` ）が、他の本番環境やテスト環境で利用されていないか緊急で確認してください。該当する場合は、直ちに強力でユニークなパスワードに変更し、パスワードポリシーを再確認してください。

#### 4.2. 予防策と長期対策

1.  **SSHサービス設定の強化:**
    *   **公開鍵認証の強制:** パスワード認証を無効化し、公開鍵認証のみに切り替えることを強く推奨します。
    *   **rootユーザーの直接ログイン禁止:** `PermitRootLogin no` を設定し、一般ユーザーでログイン後、`sudo` や `su` でroot権限に昇格する運用に変更してください。
    *   **SSHポートの変更:** 標準のTCP/22ポートから、他の高位ポートに変更することを検討してください。これにより、一般的なスキャンからの可視性を低減できます。
    *   **IPアドレス制限:** SSHへの接続を、特定の信頼できるIPアドレス範囲に限定する（ホワイトリスト方式）を検討してください。
    *   **多要素認証 (MFA) の導入:** 可能であれば、SSHログインにMFAを導入し、セキュリティをさらに強化してください。
2.  **ログ監視の強化:**
    *   SSHログインの失敗回数、特定のユーザー名でのログイン試行、特権ユーザーでのログインなどの異常を検知できるよう、ログ監視システム（SIEMなど）でのアラート設定を見直してください。
    *   特に、連続するログイン失敗や、短い時間間隔でのログイン試行を検知するルールを設定してください。
3.  **侵入検知/防御システム (IDS/IPS) の導入・強化:**
    *   SSHブルートフォース攻撃や、ログイン後の不正なコマンド実行パターンを自動で検知・防御できるIDS/IPSの導入またはルール強化を行ってください。
4.  **アカウントとパスワードポリシーのレビュー:**
    *   定期的に未使用アカウントの棚卸しを行い、不要なアカウントを削除してください。
    *   強力なパスワードポリシー（文字数、文字種混合、定期的な変更など）を強制し、従業員へのセキュリティ教育を継続的に実施してください。
5.  **システム監査と脆弱性診断:**
    *   定期的にシステム全体のセキュリティ監査および脆弱性診断を実施し、潜在的な弱点を特定・改善してください。
6.  **脅威インテリジェンスの活用:**
    *   攻撃元IPアドレスやSSH鍵のフィンガープリントを脅威インテリジェンスプラットフォームと照合し、他の関連する攻撃情報や攻撃グループの情報がないか確認してください。公開鍵のコメント「mdrfckr」は、攻撃者の目印となる可能性があります。
