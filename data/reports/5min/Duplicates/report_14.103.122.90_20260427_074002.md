### 1. 攻撃の概要と目的

攻撃者はIPアドレス `14.103.122.90` からSSHサービスに対して不正アクセスを試みました。複数の接続試行の後、`root`ユーザーとして非常に脆弱なパスワード `Aa-123456` を用いてログインに成功しています。

ログイン後、攻撃者は以下の目的で一連のコマンドを実行しました。

*   **永続化 (Persistence) の確立**:
    *   自身の公開鍵を `/root/.ssh/authorized_keys` に登録し、鍵認証によるバックドアを設置。これにより、以降はパスワードなしでSSHログインが可能になります。
    *   `root`ユーザーのパスワードを `5zFM8YbtKzlG` に変更しようと試みました。
*   **情報収集 (Discovery)**:
    *   システムのCPU情報、メモリ使用量、ディスク容量、OSバージョン、システムアーキテクチャ、ログインユーザー、cronジョブなど、詳細な環境情報を広範に収集しました。
*   **既存の侵害の競合排除・痕跡削除 (Impact / Defense Evasion)**:
    *   `/tmp/secure.sh` や `/tmp/auth.sh` といった不審なスクリプトを停止・削除し、`/etc/hosts.deny` をクリアするなど、他の攻撃者や自身の以前の活動の痕跡を消去しようとする動きが見られます。これは、このシステムを自身の管理下に置き、独占しようとする意図を示唆しています。

これらの行動から、攻撃の主な目的は、システムの完全な制御権を獲得し、将来にわたるアクセス経路を確保することであると推測されます。

### 2. 推測される手法・使用ツール

MITRE ATT&CKフレームワークに照らし合わせると、以下の手法が推測されます。

*   **初期アクセス (Initial Access)**:
    *   **T1190 - Exploit Public-Facing Application**: SSHサービスへの不正アクセス。
*   **認証情報アクセス (Credential Access)**:
    *   **T1078.004 - Valid Accounts: SSH Keys**: 脆弱なパスワードに対する辞書攻撃またはブルートフォース攻撃により、`root`ユーザーのパスワード `Aa-123456` を特定し、SSHログインに成功しました。
*   **永続化 (Persistence)**:
    *   **T1098.004 - Account Manipulation: SSH Authorized Keys**: 自身の公開鍵（コメント: `mdrfckr`）を `authorized_keys` ファイルに追加し、鍵認証によるアクセス経路を確立しました。
    *   **T1098 - Account Manipulation**: `chpasswd` コマンドを用いて `root` ユーザーのパスワードを `5zFM8YbtKzlG` に変更しようと試みました。
*   **防御回避 (Defense Evasion)**:
    *   **T1070.004 - Indicator Removal: File Deletion**: `/tmp/secure.sh`, `/tmp/auth.sh` の削除や `pkill` によるプロセス停止。
    *   **T1562.001 - Impair Defenses: Disable or Modify System Firewall**: `/etc/hosts.deny` のクリア。
    *   `chattr -ia .ssh` や `lockr -ia .ssh` (失敗しているが) によるファイル属性変更の試行。
*   **情報収集 (Discovery)**:
    *   **T1082 - System Information Discovery**: `uname -a`, `cat /proc/cpuinfo`, `free -m`, `df -h`, `lscpu` などを用いて、OS情報、CPU、メモリ、ディスクといったシステム情報を広範に収集しました。
    *   **T1033 - System Owner/User Discovery**: `w`, `whoami` コマンドにより、ログイン中のユーザーや現在のユーザーを特定しようとしました。
    *   **T1057 - Process Discovery**: `top` コマンドにより、実行中のプロセスやシステムリソースの使用状況を監視しました。
    *   **T1053.003 - Scheduled Task/Job: Cron**: `crontab -l` コマンドにより、既存のスケジュールタスク（cronジョブ）を確認しました。

**使用ツール**:
*   SSHクライアント
*   標準的なLinuxシェルコマンド（`cd`, `rm`, `mkdir`, `echo`, `chmod`, `cat`, `grep`, `wc`, `head`, `awk`, `free`, `which`, `crontab`, `w`, `uname`, `whoami`, `lscpu`, `df`, `top`, `pkill`, `chpasswd`）
*   `chattr` (ファイル属性変更ツール)

### 3. 脅威レベルとその理由

**脅威レベル: 非常に高い (Critical)**

**理由:**

1.  **ルート権限の奪取**: 攻撃者が `root` アカウントでシステムへのログインに成功しています。これはシステムに対する完全な制御権を獲得したことを意味し、機密情報の窃取、システムの改ざん、他のシステムへの攻撃基盤としての悪用、ランサムウェア展開など、あらゆる悪意のある活動が可能となります。
2.  **永続化の確立**: 攻撃者は自身のSSH公開鍵を登録し、鍵認証による再アクセス経路を確保しようとしました。これにより、初期の脆弱なパスワードが変更されたとしても、攻撃者が継続的にシステムにアクセスできる状態が作られます。パスワード変更も試みられており、複数の永続化チャネルを確保しようとする意図が見られます。
3.  **広範な情報収集**: 攻撃者はシステム環境の詳細な情報を広範囲に収集しており、これは攻撃の次の段階（横展開、データの持ち出し、マルウェア配備など）に向けた準備段階である可能性が高いです。
4.  **既存の侵害の排除**: `/tmp/secure.sh` など他の不審なスクリプトの停止・削除や `/etc/hosts.deny` のクリアは、攻撃者が自身の活動を隠蔽し、システムの独占を図ろうとする、高度な攻撃者の行動パターンを示唆しています。

このログはCowrieハニーポットで取得されたものですが、もしこれが実際のプロダクション環境で発生した場合、システムは完全に侵害されており、甚大な被害につながる可能性が極めて高いと判断されます。

### 4. 推奨アクション

この攻撃ログはハニーポット環境で取得されたものですが、もし実環境で同様の事象が発生したと仮定した場合、以下の緊急および恒久的なアクションが推奨されます。

**緊急アクション (Immediate Actions):**

1.  **隔離 (Containment)**: 攻撃元IPアドレス `14.103.122.90` からの全てのインバウンドSSH接続をファイアウォールで即座にブロックします。侵害された可能性のある全てのシステムをネットワークから緊急隔離します。
2.  **パスワードの緊急変更**: 全ての `root` ユーザーおよび特権アカウントのパスワードを直ちに、かつ強力なものに変更します。他のシステムで同じパスワードが使用されていないか確認し、使用されている場合は全て変更します。
3.  **SSH鍵の確認と削除**: `/root/.ssh/authorized_keys` および他のシステムユーザーの `.ssh/authorized_keys` ファイルを徹底的に確認し、身に覚えのない公開鍵を全て削除します。全てのSSH鍵を棚卸しし、不要なものは削除します。
4.  **システムイメージの取得**: 侵害された可能性のあるシステムのメモリイメージおよびディスクイメージを取得し、詳細なフォレンジック調査に備えます。
5.  **不審なプロセスの停止・削除**: `top` や `ps aux` コマンドなどで不審なプロセスが実行されていないか確認し、発見した場合は即座に停止し、関連するファイルも削除します。
6.  **`/etc/hosts.deny` の復元**: `/etc/hosts.deny` ファイルがクリアされている場合、本来の設定に復元するか、適切に保護を強化します。
7.  **ユーザーアカウントの棚卸し**: 不審なユーザーアカウントが作成されていないか確認し、発見した場合は削除します。

**恒久的な対策 (Long-term Actions):**

1.  **脆弱なパスワードの排除**:
    *   パスワードポリシーを強化し、辞書攻撃に強い複雑なパスワードを強制します。
    *   定期的なパスワード変更を義務付けます。
    *   **多要素認証 (MFA) の導入**を最優先で検討し、実装します。
2.  **SSHサービス hardening**:
    *   `root` ユーザーのSSHログインを無効化します (`PermitRootLogin no`)。
    *   パスワード認証を無効化し、**鍵認証のみを許可**します (`PasswordAuthentication no`)。
    *   SSHポートを標準の22番から変更します (ポートノッキングの検討も含む)。
    *   SSH接続元のIPアドレスを厳格に制限します。
    *   SSHログを定期的に監視し、不正なログイン試行をリアルタイムで検知できる体制を構築します。
3.  **IDS/IPSの導入・強化**: 侵入検知システム/侵入防御システムを導入し、SSHブルートフォース攻撃や不審なコマンド実行をリアルタイムで検知・防御できるようにします。
4.  **Endpoint Detection and Response (EDR) の導入**: エンドポイントでの不審なプロセス実行、ファイル変更、ネットワーク接続などを詳細に監視し、脅威を早期に検知・対応できるEDRソリューションを導入します。
5.  **ログ監視の強化**:
    *   SSHを含む全てのシステムログを中央ログ管理システムに集約し、分析・監視を強化します。
    *   不審なログイン試行、特権昇格、ファイル変更、外部への不審な通信などを検知するアラートルールを具体的に設定します。
6.  **定期的な脆弱性診断とペネトレーションテスト**: 定期的にシステムの脆弱性を診断し、潜在的な弱点を特定・修正します。これにより、攻撃者に先んじてセキュリティホールを塞ぎます。
7.  **セキュリティ教育**: 従業員に対し、セキュリティ意識向上トレーニングを定期的に実施し、脆弱なパスワードの使用を避けるよう徹底教育します。
