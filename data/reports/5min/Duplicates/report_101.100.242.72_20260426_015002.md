## SOCアナリストレポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `101.100.242.72` から、システムに対するSSHブルートフォース攻撃が継続的に行われています。攻撃者は複数の一般的なユーザー名（`frappe`, `deployer`, `ubuntu`, `admin`, `tester`, `test`, `oracle`, `user`, `zhaoxj`）とそれらに紐づく可能性のあるパスワードを繰り返し試行しています。

特に注目すべきは、`root` ユーザーに対して複数の異なるパスワード（`qwer1234QWER!@#$`, `Ni123456`, `ubuntu@123`, `Password.123`, `a12348765`）でログインに成功（ハニーポット上での成功）している点です。ログイン成功後には、システムへの永続的なアクセスを確立するため、以下のコマンドが実行されています。

`cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr">>.ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`

このコマンドは、既存の `.ssh` ディレクトリを削除し、新たに作成したディレクトリ内に攻撃者の公開鍵を `authorized_keys` ファイルとして設置することで、将来的にパスワードなしでのSSHアクセスを可能にしようとするものです。攻撃の最終的な目的は、高い権限でのシステム侵入と、その後の永続的なアクセス経路の確保にあると判断されます。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **ブルートフォース攻撃 / 認証情報の推測**: ログからは、多数のユーザー名とパスワードの組み合わせを自動的かつ高速に試行していることが明らかです。これは、一般的なユーザー名リストとパスワード辞書を使用したブルートフォース攻撃、または認証情報推測攻撃と考えられます。
    *   **永続化**: ログイン成功後、システムに攻撃者のSSH公開鍵を登録する行動は、今後も継続的にシステムにアクセスするための永続化手法です。これにより、パスワードが変更されたとしても、鍵認証によって再侵入を試みることが可能になります。
*   **使用ツール**:
    *   SSHクライアントをベースとした自動化スクリプト、またはSSHブルートフォースツール（例: `Hydra`, `Medusa` など）。
    *   ログイン後のコマンド実行パターン（`chattr`, `rm`, `mkdir`, `echo`）から、Linux/Unixベースのターゲットを想定した汎用的な攻撃スクリプトが使用されていると推測されます。`lockr` コマンドは標準的ではなく、ハニーポットがエミュレートしなかったか、タイプミスである可能性があります。

### 3. 脅威レベルとその理由

*   **脅威レベル**: **高 (High)**
*   **理由**:
    *   **執拗な攻撃**: 攻撃元IPからのSSHブルートフォース攻撃は断続的かつ集中的に行われており、システムへの侵入に対する強い意図が見られます。
    *   **rootアカウントへの標的**: 最上位権限である `root` ユーザーに対するログインを複数回試行し、成功していることは、もしこれが実システムであれば、システム全体が乗っ取られる危険性が極めて高いことを示唆します。
    *   **永続的なアクセス経路の確立企図**: 攻撃者が `authorized_keys` に自身の公開鍵を追加しようとすることは、システムに一度侵入した後、継続的にアクセスできるバックドアを設置する意図があるため、非常に危険です。
    *   **自動化された攻撃**: 短時間で多数のログイン試行が行われていることから、手動ではなく自動化されたツールによる攻撃であり、今後も同様の攻撃が継続される可能性が高いです。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPのブロック**:
    *   ファイアウォールまたはIPS/IDSにおいて、攻撃元IPアドレス `101.100.242.72` からの全てのSSH (TCP/22) 接続を即座にブロックしてください。
2.  **既存システムの確認と対策**:
    *   もしこのログがハニーポットではなく実システムのものであった場合、ログに記録された「成功した」`root` ユーザーのパスワード（`qwer1234QWER!@#$`, `Ni123456`, `ubuntu@123`, `Password.123`, `a12348765`）が**現在運用中のシステムで使用されていないか緊急で確認**してください。
    *   これらのパスワードが使用されている場合は、**即座にパスワードを変更**し、他のシステムやサービスで使い回されていないか確認し、あればそこも変更してください。
    *   全てのサーバーで `/root/.ssh/authorized_keys` ファイルの内容を確認し、ログに記載された公開鍵（`ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`）が存在しないことを確認してください。もし存在した場合は、該当する行を削除し、適切なパーミッション（600）に設定し直してください。
    *   実システムで不正なプロセスや活動の兆候がないか、侵入痕跡（IoC）調査を実施してください。

#### 恒久対策 (Long-term Measures):

1.  **SSHサービス設定の強化**:
    *   **パスワード認証の無効化**: 可能な限り、SSHサービスではパスワード認証を無効にし、鍵認証のみの使用を強制してください。
    *   **rootログインの禁止**: `PermitRootLogin no` を設定し、`root` ユーザーによる直接SSHログインを禁止してください。必要に応じて、一般ユーザーでログイン後、`su` または `sudo` で `root` 権限に昇格するように運用してください。
    *   **ブルートフォース対策**: `Fail2ban` などのツールを導入し、一定回数以上のSSH認証失敗があった場合に、そのIPアドレスを自動的にブロックする設定を行ってください。
    *   **SSHポートの変更**: 標準の22番ポートではなく、別のポート番号を使用することも検討してください（これは検出を難しくするもので、根本的なセキュリティ強化ではありません）。
2.  **アカウントとパスワードポリシーの強化**:
    *   **強力なパスワードポリシー**: 複雑さ、長さ、有効期限などを含む強力なパスワードポリシーを全てのシステムアカウントに適用してください。
    *   **多要素認証 (MFA) の導入**: SSHログインに多要素認証を導入し、認証のセキュリティを大幅に向上させてください。
    *   **不要なアカウントの削除**: 使用されていない、または不要なアカウントは削除してください。
3.  **監視とアラート**:
    *   SSHログインログ（`auth.log` など）をSIEMや中央ログ管理システムで集中監視し、ブルートフォース攻撃や不正ログインの試行、異常なログイン成功に対してリアルタイムでアラートが発報されるように設定してください。
4.  **セキュリティ教育**:
    *   従業員に対し、安全なパスワードの利用、フィッシング攻撃への注意、セキュリティベストプラクティスに関する定期的な教育を実施してください。
