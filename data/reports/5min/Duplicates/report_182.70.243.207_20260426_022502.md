## SOCアナリストによる攻撃レポート

### 1. 攻撃の概要と目的

このレポートは、攻撃元IPアドレス `182.70.243.207` からのSSHサービスに対する一連の攻撃活動を分析したものです。ログはCowrieハニーポットで記録されており、攻撃者はこのハニーポットを実際のシステムとして認識して攻撃を試みました。

**攻撃の概要:**
攻撃者は、まず複数のユーザー名とパスワードの組み合わせを用いてSSHサービスへのブルートフォース攻撃を開始しました。特に`root`ユーザーに対する攻撃が執拗に行われ、異なるパスワードで複数回ログインに成功しています（ハニーポットの特性上、複数の弱いパスワードを受け入れたものと推測されます）。ログイン成功後、攻撃者は自身のSSH公開鍵をターゲットシステムの`~/.ssh/authorized_keys`ファイルに追加しようとしました。この行動は、パスワードを知らなくても将来的にSSHアクセスを可能にするバックドアを設置し、システムへの永続的なアクセス経路を確立することを目的としています。

**攻撃の主な目的:**
*   SSHサービスへの不正アクセス権の取得
*   公開鍵認証による永続的なバックドアの設置
*   取得したアクセス権を用いたさらなる侵害活動（マルウェアの展開、データ窃取など）の準備

### 2. 推測される手法・使用ツール

**推測される手法:**
*   **SSHブルートフォース/辞書攻撃:**
    *   `steam`, `tunnel`, `cloud`, `gitlab` など、一般的なユーザー名や推測されやすいユーザー名に対し、辞書ベースのパスワード（`steamsteamsteam`, `123456`, `123`, `gitlab@2025`）を試行しています。
    *   特に`root`ユーザーに対しては、`QAZwsx`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`, `3245gs5662d34`といった複数のパスワードを試行し、複数回成功しています。
    *   `345gs5662d34`という特徴的なユーザー名とパスワードの組み合わせも繰り返し試されています。
*   **永続化メカニズム（バックドア設置）:**
    *   ログイン成功後、`cd ~ && rm -rf .ssh && mkdir .ssh && echo "ssh-rsa AAAAB3NzaC1yc2E..." >> .ssh/authorized_keys && chmod -R go= ~/.ssh && cd ~`というコマンドを実行し、自身の公開鍵を`authorized_keys`に追記することで、パスワードなしでの再アクセス経路を確立しようとしています。鍵のコメントは`mdrfckr`です。
    *   `chattr -ia .ssh; lockr -ia .ssh`というコマンドも実行を試みていますが、`lockr`コマンドは標準的なLinuxコマンドではなく、ハニーポット上で実行に失敗しています。これは、攻撃者が特定の環境（例えば、特定のマルウェアがインストールされている環境）を想定していたか、一般的なセキュリティ対策（ファイルの変更不可属性など）を無効化しようとしたものと推測されます。

**推測される使用ツール:**
*   **自動化されたブルートフォースツール:** HydraやNmapのNSEスクリプト、または攻撃者自身が作成したカスタムスクリプトなど、広範なパスワードリストを用いた自動ログイン試行ツールが使用された可能性が高いです。
*   **シェルスクリプト:** ログイン後の公開鍵設置コマンドは典型的なワンライナーであり、ログインに成功した際に自動的に実行されるシェルスクリプトの一部であると考えられます。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由:**
1.  **ログインの成功:** ハニーポットに対してではありますが、攻撃者は`root`ユーザーで複数回ログインに成功しています。これは現実のシステムであれば、最高権限での侵害を意味し、システムに対する完全な制御を奪われた状態に該当します。
2.  **永続化の試み:** ログイン後、攻撃者は自身のSSH公開鍵を`~/.ssh/authorized_keys`に追加することで、将来的にパスワード認証なしでアクセスできるバックドアを設置しようとしました。これは単なる一時的な偵察や破壊行為ではなく、長期的なシステム掌握と利用を企図していることを明確に示しています。
3.  **攻撃の自動化:** 連続的なログイン試行と、ログイン成功後の定型的なコマンド実行パターンから、攻撃が自動化されたスクリプトまたはツールによって実行されていることが強く推測されます。これにより、広範囲のターゲットに対して効率的に攻撃を展開している可能性があります。
4.  **複数の異なるパスワードでの成功:** 複数の異なるパスワードで`root`ログインに成功していることから、攻撃者が複数種の辞書攻撃を試している、またはハニーポットが複数の弱いパスワードを受け入れる設定であったとしても、攻撃者は複数のパスワードを破る能力を持っていることを示唆します。

これらの理由から、この攻撃は非常に深刻であり、実際のシステムで発生した場合は緊急の対応を要するレベルであると判断されます。

### 4. 推奨アクション

#### 緊急対応 (Immediate Actions):

1.  **攻撃元IPのブロック:** 攻撃元IPアドレス `182.70.243.207` からのすべての通信を、ファイアウォールやIDS/IPSなどのセキュリティデバイスで即座にブロックしてください。
2.  **侵害された認証情報の確認と変更:** ログに記録されたログイン成功パスワード（`QAZwsx`, `Root1234567@@`, `1q2w3e..`, `root123123!!`, `@a123456`, `Root11111111@#`, `3245gs5662d34`）が、組織内の他のシステムやサービスで使われていないか緊急で確認してください。もし使用されている場合は、即座に強力なパスワードに変更してください。特に`root`アカウントや管理者権限を持つアカウントのパスワードは最優先で変更が必要です。
3.  **不正なSSH公開鍵の確認と削除:** 攻撃者が設置しようとした公開鍵（コメント: `mdrfckr`、鍵の内容: `ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr`）が、組織内のどのシステムの`~/.ssh/authorized_keys`ファイルにも存在しないことを確認してください。もし存在した場合は、即座に削除し、そのシステムを侵害されたものとして詳細な調査（フォレンジック）を行ってください。

#### 長期的な対策 (Long-Term Measures):

1.  **SSHセキュリティの強化:**
    *   **パスワード認証の無効化:** SSHサービスではパスワード認証を無効にし、より安全な公開鍵認証のみを許可することを強く推奨します。
    *   **`root`ログインの禁止:** `PermitRootLogin no`を設定し、`root`ユーザーでの直接ログインを禁止してください。管理者作業は一般ユーザーでログイン後、`sudo`を利用して行うべきです。
    *   **SSHポートの変更:** 標準のSSHポート22番から、広く知られていない高番号ポートに変更することで、自動化されたスキャンからのヒット率を下げることができます。
    *   **Fail2banなどの導入:** ブルートフォース攻撃を自動的に検知し、一時的または永続的に攻撃元IPアドレスをブロックするFail2banなどの侵入検知・防止システムを導入してください。
    *   **アクセス元IPアドレスの制限:** VPNやACL（アクセス制御リスト）を利用し、信頼できる特定のIPアドレス範囲からのみSSHアクセスを許可するように制限してください。
    *   **多要素認証 (MFA) の導入:** SSH接続にMFAを導入し、セキュリティをさらに強化してください。
2.  **強力なパスワードポリシーの適用:**
    *   すべてのユーザーアカウントに対して、複雑で推測されにくいパスワードの使用を強制するポリシーを適用してください。定期的なパスワード変更も検討してください。
3.  **ログ監視の強化:**
    *   SSHログイン試行ログ（成功/失敗）、およびSSHセッション内で実行されたコマンドログを継続的に監視する体制を構築してください。SIEM（Security Information and Event Management）システムなどを活用し、異常なログイン試行や不審なコマンド実行をリアルタイムで検知できる仕組みを導入してください。
4.  **脅威インテリジェンスの活用:**
    *   今回の攻撃元IPアドレス`182.70.243.207`を定期的に脅威インテリジェンスデータベースで照会し、既知の悪性IPアドレスリストに載っているかを確認し、必要に応じて防御ルールを更新してください。
5.  **ハニーポットの継続的な運用:**
    *   Cowrieのようなハニーポットは、攻撃者の手法、ツール、意図を理解する上で非常に有用です。継続的に運用し、収集したログをセキュリティ対策の改善に活用してください。
