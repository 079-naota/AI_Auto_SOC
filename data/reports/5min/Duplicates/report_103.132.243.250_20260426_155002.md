## SOC分析レポート

### 1. 攻撃の概要と目的

提供されたCowrieハニーポットのログを分析した結果、攻撃元IPアドレス `103.132.243.250` からSSHサービスに対するブルートフォース攻撃が複数回実施されたことを確認しました。

この攻撃の目的は、SSHアクセス権を奪取すること、およびログイン成功後に永続的なアクセス経路（バックドア）を確立することにあると推測されます。具体的には、`.ssh/authorized_keys` ファイルに攻撃者自身の公開鍵を登録することで、パスワード認証を経由せず、秘密鍵を用いたSSH認証によるアクセスを可能にしようとしていました。

### 2. 推測される手法・使用ツール

*   **手法**:
    *   **SSH ブルートフォース攻撃（パスワードリスト攻撃）**: 複数の異なるユーザー名とパスワードの組み合わせを繰り返し試行し、SSH認証を突破しようとしています。特に`root`ユーザーを標的とし、一般的な、あるいは過去に漏洩したと推測されるパスワード（例: `135792468`, `abcdefgh`, `qazwsxedc123`, `P@ssw0rd6`, `Admin88888888`, `Gp123456`など）を試行しています。
    *   **持続的アクセスの確立**: ログイン成功後、以下の手順で持続的なアクセス経路の確立を試みていました。
        1.  既存の`.ssh`ディレクトリの属性を変更しようと試みる (`chattr -ia .ssh; lockr -ia .ssh` コマンドを使用) 。ただし、`lockr`は標準的なコマンドではないため失敗しています。
        2.  `.ssh`ディレクトリを削除し、再作成 (`rm -rf .ssh && mkdir .ssh`)。
        3.  特定の公開鍵 (`ssh-rsa AAAAB3NzaC1yc2E... mdrfckr`) を`.ssh/authorized_keys`ファイルに追加。
        4.  `.ssh`ディレクトリのパーミッションを適切に設定 (`chmod -R go= ~/.ssh`)。
*   **使用ツール**:
    *   攻撃のパターンから、自動化されたスクリプト、またはSSHブルートフォースツール（例: Hydra, Nmap, Medusa など）が使用されている可能性が高いです。ログイン成功後のコマンド実行も自動化されています。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
1.  **管理者権限の奪取**: ログには`root`ユーザーでのログインが複数回成功した記録があります。これは、もしこれが実際のシステムであった場合、攻撃者がシステムに対する完全な管理者権限を奪取したことを意味します。
2.  **持続的アクセス経路の確立試行**: ログイン成功後、攻撃者は公開鍵認証によるバックドアを設置しようとしています。これは、パスワードが変更されたとしても、攻撃者が秘密鍵を持っていればいつでもシステムに再アクセスできる状態を作り出すため、非常に深刻な脅威となります。
3.  **攻撃の継続性**: 複数の異なるパスワードでログインに成功した後も、同様のバックドア設置コマンドを繰り返し実行しており、攻撃が継続的かつ自動化されていることが示唆されます。
4.  **対象システムの特定**: Cowrieはハニーポットであるため、実際のシステムに対する被害はありませんでしたが、このログは攻撃者がどのようなシステムを狙い、どのような手法で侵入を試みているかを示す重要なインテリジェンスとなります。

### 4. 推奨アクション

1.  **攻撃元IPのブロック**:
    *   攻撃元IPアドレス `103.132.243.250` からの全てのインバウンド接続をファイアウォールやACLで即座にブロックしてください。
2.  **SSHパスワードポリシーの強化**:
    *   SSH認証に使用されるパスワードがブルートフォース攻撃に耐えうる複雑さを持つように、パスワードポリシーを強化してください（例: 最小文字数、大文字・小文字・数字・記号の組み合わせ、定期的なパスワード変更の義務化）。
    *   一般的なパスワードや辞書攻撃で推測されやすいパスワードの使用を禁止するルールを導入してください。
3.  **SSHサービスのセキュリティ強化**:
    *   **公開鍵認証の利用促進**: パスワード認証を無効化し、公開鍵認証のみを許可することを強く推奨します。
    *   **多要素認証 (MFA) の導入**: SSHアクセスに多要素認証を導入し、セキュリティレベルを向上させてください。
    *   **rootログインの禁止**: SSH経由での`root`ユーザーの直接ログインを禁止し、一般ユーザーでログイン後に`su`や`sudo`コマンドを使用する運用に切り替えてください。
    *   **SSHポートの変更**: 可能な場合、デフォルトの22番ポートから別のポートに変更することを検討してください（※ただし、これは一時的な対策であり、根本的なセキュリティ強化にはなりません）。
    *   **Fail2banなどの導入**: 認証失敗の回数を監視し、閾値を超えたIPアドレスを自動的にブロックするツール（例: Fail2ban）を導入してください。
4.  **`authorized_keys`ファイルの監査**:
    *   もしこのハニーポットが実際のシステムであった場合、全てのユーザーの`.ssh/authorized_keys`ファイルに、ログに記録された公開鍵 (`ssh-rsa AAAAB3NzaC1yc2E... mdrfckr`) が含まれていないか、あるいはその他の不正な公開鍵が登録されていないかを確認し、削除してください。
    *   定期的に`authorized_keys`ファイルの内容を監査する仕組みを導入してください。
5.  **脅威インテリジェンスの活用**:
    *   今回のログで得られた攻撃元IP、試行されたユーザー名・パスワード、設置されようとした公開鍵情報を脅威インテリジェンスとして社内共有し、今後のセキュリティ対策に役立ててください。

今回の攻撃はハニーポットによって検知・記録されたものであり、実際のシステムへの被害は発生していませんが、同様の手口が他のシステムに対して試行される可能性を考慮し、上記の推奨アクションを早急に実施することが重要です。
