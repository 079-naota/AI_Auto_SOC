## セキュリティインシデント分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `103.117.56.152` から、当方SSHサービスに対する大規模なブルートフォース攻撃が継続的に行われました。この攻撃は、複数の異なるユーザー名とパスワードの組み合わせを試行し、最終的に`root`ユーザーでのログインに複数回成功しています。

ログイン成功後、攻撃者はシステムへの永続的なアクセス経路を確立することを目的とし、以下の不正なコマンド実行を試みました。

1.  既存のSSH設定ディレクトリ (`~/.ssh`) の属性変更（`chattr -ia .ssh; lockr -ia .ssh`）
2.  既存のSSH設定ディレクトリを削除し、新規作成 (`rm -rf .ssh && mkdir .ssh`)
3.  攻撃者自身のSSH公開鍵（コメント: `mdrfckr`）を `~/.ssh/authorized_keys` に追加
4.  `.ssh` ディレクトリのパーミッション設定 (`chmod -R go= ~/.ssh`)

この一連の行動から、攻撃の目的はSSHサービスの不正アクセスを通じてシステムを乗っ取り、バックドアを仕掛けることで永続的なアクセスを確保することにあると判断されます。

### 2. 推測される手法・使用ツール

*   **攻撃手法**:
    *   **ブルートフォース攻撃 / 辞書攻撃**: 大量のユーザー名とパスワードの組み合わせを試行しており、これは自動化されたブルートフォース攻撃または辞書攻撃の典型的なパターンです。
    *   **認証情報スタッフィング**: 成功したユーザー名とパスワードの組み合わせ (`root` / `Aa@12345`, `root` / `odoo123`, `root` / `nPSpP4PBW0`, `root` / `xxZZ1234`, `root` / `Aa112211.`) が複数回確認されており、攻撃者が保有する既知の認証情報リストを使用している可能性があります。
    *   **永続化**: ログイン成功後、`authorized_keys`ファイルを操作して自身の公開鍵を追加することで、パスワード認証以外の手段（公開鍵認証）で将来的にアクセスできるよう、バックドアを仕掛けることを試みています。

*   **使用ツール**:
    *   **自動化スクリプト / ボット**: 一連のログイン試行とコマンド実行は非常に短時間かつ機械的に行われており、手動操作ではなく、自動化されたスクリプトやボットによって実行されていると強く推測されます。
    *   **標準的なLinuxコマンド**: ログイン成功後に実行されたコマンド (`cd`, `chattr`, `rm`, `mkdir`, `echo`, `chmod`) は、Linux環境で一般的に利用されるコマンドです。`lockr`は標準的なコマンドではないため、ハニーポットの挙動、攻撃スクリプトの誤字、または特定のマルウェアが提供する機能の一部である可能性があります。

### 3. 脅威レベルとその理由

**脅威レベル: 高 (High)**

**理由**:
1.  **管理権限でのログイン成功**: 攻撃者が最も重要な`root`ユーザーでのログインに複数回成功している点は、最も深刻な脅威です。`root`権限はシステムへの完全な制御を意味します。
2.  **永続的なアクセス経路の確立**: ログイン成功後、攻撃者は自身のSSH公開鍵を`authorized_keys`に追加することで、パスワードが変更されたとしても恒久的にアクセスできるバックドアを確立しようとしています。これは長期的なシステム侵害の意図を示唆します。
3.  **自動化された攻撃**: ブルートフォースから永続化の試みまでの一連の行動が自動化されているため、攻撃者は効率的にシステムを侵害しようとしており、標的となったシステム以外にも同様の攻撃を仕掛けている可能性があります。
4.  **広範囲な影響の可能性**: 実際にシステムが侵害された場合、データの窃取、破壊、改ざん、システムの踏み台化、マルウェアの配布元、ランサムウェア感染など、甚大な被害が発生する可能性があります。

### 4. 推奨アクション

この攻撃を受けて、以下の緊急対応および恒久対策を直ちに実施することを推奨します。

#### 緊急対応 (Critical Actions):

1.  **攻撃元IPのブロック**: ファイアウォール、IDS/IPS、WAF等で、攻撃元IPアドレス `103.117.56.152` からの全ての通信を即座にブロックします。
2.  **対象システムの緊急隔離/停止**: 攻撃を受けた可能性のあるシステムをネットワークから緊急隔離し、さらなる侵害や被害の拡大を防ぎます。状況に応じてシステムの一時停止も検討します。
3.  **rootアカウントパスワードの即時変更**: ログに記録された`root`アカウントのパスワード (`Aa@12345`, `odoo123`, `nPSpP4PBW0`, `xxZZ1234`, `Aa112211.`, `3245gs5662d34`) を含め、全てのシステムアカウント（特に特権アカウント）のパスワードを、予測困難で複雑なものに直ちにリセットします。
4.  **SSH公開鍵の確認と削除**: 侵害された可能性のあるシステムにSSH接続し（隔離された環境で）、`root`ユーザーを含む全てのユーザーの `~/.ssh/authorized_keys` ファイルの内容を確認し、攻撃者によって追加された不正な公開鍵（コメント `mdrfckr` の鍵）を削除します。不要な公開鍵も全て削除します。
5.  **ログの徹底的な調査**:
    *   SSH以外のサービスログ（Webサーバー、データベース、その他アプリケーション）に不審なアクセスがないか確認します。
    *   ログイン成功後の期間に、不審なプロセス実行、ファイル作成/変更、ネットワーク通信がないか詳細に調査します。
    *   他のシステムへの横展開の兆候がないか確認します。
6.  **システムのフォレンジック調査**: 侵害の範囲、影響、攻撃者の行動の詳細を特定するため、専門家によるフォレンジック調査を実施します。

#### 恒久対策 (Long-Term Measures):

1.  **SSH設定の厳格化**:
    *   **パスワード認証の無効化**: 公開鍵認証のみを許可し、パスワード認証を無効にします。
    *   **`root`ユーザーによるSSHログインの無効化**: `PermitRootLogin no` を設定し、`root`での直接ログインを禁止します。必要に応じて一般ユーザーでログイン後、`su`または`sudo`で昇格する運用とします。
    *   **多要素認証 (MFA) の導入**: SSHログインにMFAを必須とします。
    *   **SSHポートの変更**: 標準の22番ポートから別の非標準ポートに変更することで、自動化されたスキャンからの露出を減らします（ただし、根本的な対策ではありません）。
    *   **Fail2Banなどの導入**: 不正なログイン試行を検知し、一時的または永続的にIPアドレスをブロックするツールを導入・設定強化します。
2.  **強力なパスワードポリシーの適用**: 全てのシステムアカウントに対し、複雑性、長さ、有効期限などを含む強力なパスワードポリシーを強制し、定期的なパスワード変更を義務付けます。
3.  **システム監視の強化**:
    *   SIEM (Security Information and Event Management) システムを導入し、SSHログイン成功/失敗、特権コマンド実行、ファイル変更などのセキュリティイベントログを一元的に収集・監視します。
    *   異常検知ルールを設定し、リアルタイムでのアラート通知体制を構築します。
4.  **脆弱性管理とパッチ適用**: 定期的にシステムの脆弱性スキャンを実施し、OSやアプリケーションのセキュリティパッチを迅速に適用します。
5.  **インシデントレスポンス計画の見直し**: 今回の事態を教訓に、既存のインシデントレスポンス計画を評価し、必要に応じて改訂します。定期的な訓練を通じて、緊急時の対応能力を向上させます。
6.  **脅威インテリジェンスの活用**: 攻撃元IPや不正なSSH公開鍵の情報を、脅威インテリジェンスプラットフォームに共有・参照し、今後の攻撃検知に役立てます。
