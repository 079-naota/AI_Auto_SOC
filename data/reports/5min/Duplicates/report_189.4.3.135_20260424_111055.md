    *   `/etc/hosts.deny` ファイルをクリアし、IPベースのアクセス制限を無効化しようとしました。

**目的**:
攻撃者の主な目的は、SSHサービスへの不正侵入を足がかりに、システムの完全な制御を奪取し、永続的なアクセス経路を確立することです。収集されたシステム情報は、次の攻撃フェーズ（例えば、特定のマルウェアの導入、データの窃取、ボットネットへの組み込み、他のシステムへの横展開）のための準備と推測されます。

### 2. 推測される手法・使用ツール

**【推測される手法】**:

*   **初期アクセス (Initial Access)**:
    *   **T1110 - Brute Force**: SSHサービスに対するパスワード推測攻撃（ブルートフォース/辞書攻撃）。`root` ユーザーを中心に、複数の認証情報を試行しています。
*   **永続化 (Persistence)**:
    *   **T1098 - Account Manipulation: SSH Authorized Keys**: `~/.ssh/authorized_keys` への攻撃者の公開鍵追加によるバックドアの設置。
    *   **T1098 - Account Manipulation: Password Change**: `root` ユーザーのパスワード変更を試み、永続的なアクセス権の確保を目指しています。
*   **防御回避 (Defense Evasion)**:
    *   **T1070 - Indicator Removal on Host**: マルウェア関連ファイルの削除 (`rm -rf`) やプロセス停止 (`pkill`)、`/etc/hosts.deny` のクリア。
    *   **T1497 - Virtualization/Sandbox Evasion**: `chattr` や `lockr` のようなファイル属性変更の試みは、システム環境の特性（例: ハニーポットかどうか）を検出したり、セキュリティメカニズムを迂回したりする目的がある可能性があります。
*   **偵察 (Discovery)**:
    *   **T1082 - System Information Discovery**: `uname`, `cpuinfo`, `free`, `df -h` などによるOS、ハードウェア、リソース情報の収集。
    *   **T1083 - File and Directory Discovery**: `ls`, `crontab -l` などによるファイルシステムの内容や設定情報の確認。
    *   **T1057 - Process Discovery**: `top` などによる実行中のプロセス情報の確認。
    *   **T1033 - System Owner/User Discovery**: `whoami`, `w` などによる現在ログイン中のユーザーやシステムユーザーの確認。
*   **権限昇格 (Privilege Escalation)**:
    *   `root` ユーザーとしてログイン後、さらに `root` のパスワードを変更しようとしていることから、確実な権限掌握や他の環境での再利用を意図している可能性があります。

### 3. 脅威レベルとその理由

**【脅威レベル】**: **高 (High)**

**【理由】**:

1.  **認証突破の成功**: 攻撃者が `root` ユーザーとして複数回ログインに成功しているという事実は、システムの初期防御が突破されたことを明確に示しています。これは最も重大な脅威です。
2.  **明確な悪意のある意図**: ログイン成功後の行動は、単なる偵察にとどまらず、バックドアの設置、主要アカウントのパスワード変更、広範なシステム偵察、既存マルウェアの痕跡削除および防御回避など、システムを完全に掌握し、悪用しようとする非常に強い意図を示しています。
3.  **永続的なアクセス確立の試み**: `authorized_keys` への公開鍵追加や `root` パスワード変更は、将来的にパスワードなしでシステムへ再侵入するための準備であり、持続的な脅威となる可能性があります。
4.  **広範な偵察活動**: 実行された多数のシステムコマンドは、攻撃者がターゲットシステムを深く理解し、次の攻撃フェーズ（例: 追加のマルウェア導入、データ窃取、ボットネットへの組み込み、C2サーバとの通信確立）を計画していることを示唆しています。
5.  **自動化された攻撃の可能性**: ログイン試行のパターンや、ログイン成功後の定型的なコマンド実行シーケンスから、この攻撃が自動化されたスクリプトまたはボットネットの一部として実行されている可能性が高く、同様の攻撃が他のシステムにも仕掛けられているか、今後も継続する可能性があります。

### 4. 推奨アクション

この攻撃はハニーポットで観測されたものですが、本番環境で同様の事象が発生した場合を想定し、以下の緊急対応と予防措置を推奨します。

**【緊急対応】**:

1.  **攻撃元IPのブロック**: 攻撃元IPアドレス `189.4.3.135` を、ファイアウォール、IDS/IPS、WAF、SSHデーモン設定（例: `/etc/hosts.deny`）などで即座にブロックしてください。
2.  **不正ログイン試行のログ確認**: 本番環境のSSHログ、認証ログ、システムログを直ちに確認し、攻撃元IPからの接続履歴、同様のログイン試行、成功したログイン、および不審なコマンド実行がないか詳細に調査してください。
3.  **アカウントのパスワードリセットと多要素認証の強制**:
    *   ログインに成功した可能性のあるユーザー（特に `root` および `ali`, `test01` など、ログに記載されたユーザー名）のパスワードを直ちにリセットしてください。
    *   推測されにくい複雑なパスワードポリシーを適用し、可能であればすべての特権アカウントに対して多要素認証 (MFA) を必須としてください。
4.  **SSHサービスの設定強化**:

    *   **`root` ユーザーの直接ログイン禁止**: `PermitRootLogin no` を設定し、`root` ユーザーの直接SSHログインを禁止してください。必要な場合は、一般ユーザーでログイン後に `su` や `sudo` を使用するようにします。
    *   **許可ユーザーの限定**: `AllowUsers` または `AllowGroups` を設定し、SSHアクセスを許可するユーザーやグループを最小限に制限してください。
    *   **Fail2Banなどの導入**: ブルートフォース攻撃を自動的に検知・ブロックするツール（例: Fail2Ban）を導入してください。
