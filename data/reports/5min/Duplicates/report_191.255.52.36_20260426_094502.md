# SOC自動分析レポート: 191.255.52.36

**生成日時:** 2026-04-26 09:48:23

## SOC分析レポート

### 1. 攻撃の概要と目的

攻撃元IPアドレス `191.255.52.36` から、SSHサービスに対する継続的なブルートフォース攻撃が確認されました。この攻撃者は複数のユーザー名とパスワードの組み合わせを試行し、最終的に `root` ユーザーとしてシステムへのログインに複数回成功しています。

ログイン成功後、攻撃者は以下の主要な目的を達成しようとしました。

*   **永続的なアクセス経路の確立:** `.ssh/authorized_keys` ファイルに自身の公開鍵を登録することで、パスワードなしでシステムに再アクセスできるバックドアを設置しようとしました。
*   **認証情報の変更:** `root` ユーザーのパスワードを `V7x7C26rMbb4` に変更しようとしました。
*   **システム情報の収集:** CPU情報、メモリ使用量、OSバージョン、実行中のプロセス、cronジョブ、現在のログインユーザーなど、システムの構成や状態に関する詳細な情報を収集しました。これは、その後の攻撃段階（例: マルウェアの展開、リソースの悪用）に向けた偵察活動と考えられます。
*   **防御回避と痕跡のクリーンアップ:** 既存のマルウェアやスクリプト（`secure.sh`, `auth.sh`）を削除し、関連するプロセスを停止させることで、システム上の競合する悪意のある活動を排除し、自身の活動を隠蔽しようとしました。また、`/etc/hosts.deny` をクリアすることで、外部からのアクセス制限を緩和しようとした可能性があります。

### 2. 推測される手法・使用ツール

*   **初期アクセス（Initial Access）:**
    *   **ブルートフォース攻撃:** SSHプロトコルに対するユーザー名とパスワードの総当たり攻撃が行われました。`ubuntu`, `root`, `345gs5662d34`, `wang` などのユーザー名が試行され、複数のパスワードで `root` ユーザーの認証に成功しています。これはHydraやMedusaなどの自動化されたパスワードクラッキングツールによって実行された可能性が高いです。
*   **永続化（Persistence）:**
    *   **SSH公開鍵の追加:** `~/.ssh/authorized_keys` に攻撃者のSSH公開鍵を追加し、パスワードなしでの再ログインを可能にしようとしました。
    *   **アカウント認証情報の変更:** `chpasswd` コマンドを使用して、`root` ユーザーのパスワードを強制的に変更しようとしました。
*   **防御回避（Defense Evasion）:**
    *   **ファイル属性変更の試行:** `chattr -ia .ssh` (ハニーポット環境では失敗) を使用して、`.ssh` ディレクトリの不変属性を解除しようとしました。これは、`.ssh` ディレクトリの保護を無効化し、`authorized_keys` ファイルの操作を容易にする意図があったと考えられます。
    *   **既存マルウェアの排除:** `rm -rf /tmp/secure.sh`, `rm -rf /tmp/auth.sh` および `pkill -9 secure.sh`, `pkill -9 auth.sh`, `pkill -9 sleep` は、他の攻撃者によって設置された可能性のあるマルウェアやバックドア、あるいは自身の活動を妨げる可能性のあるプロセスを削除・停止し、システムの制御を完全に掌握しようとした試みです。
    *   **アクセス制御の緩和:** `echo > /etc/hosts.deny` コマンドで `/etc/hosts.deny` ファイルを空にしようとしました。これにより、システムへのアクセス制御が緩和され、より多くのIPアドレスからの接続を許可してしまうリスクがあります。
*   **偵察（Discovery）:**
    *   **システム情報収集コマンド:** 攻撃者は `cat /proc/cpuinfo`, `free -m`, `uname -a`, `whoami`, `lscpu`, `crontab -l`, `w`, `top`, `which` などの標準的なLinuxコマンドを多数実行し、システムのハードウェア、OS、ユーザー、プロセス、ジョブスケジュールに関する詳細な情報を収集しました。これは、ターゲットシステムの特性を把握し、次の攻撃段階（例: 特定の脆弱性の利用、適切なマルウェアの選択）に備えるためと考えられます。

### 3. 脅威レベルとその理由

**脅威レベル：重大 (Critical)**

**理由:**

1.  **root権限の奪取:** 攻撃者はシステムの最高権限である `root` ユーザーでのログインに成功しています。これにより、システム上のあらゆる操作が可能となり、機密データの窃取、システムの破壊、マルウェアのインストール、他のシステムへの攻撃拠点化など、極めて深刻な被害につながる可能性があります。
2.  **永続的なアクセスの確立:** SSH公開鍵の設置やrootパスワードの変更は、攻撃者が一度侵入した後も長期的にシステムへのアクセスを維持しようとする明確な意図を示しています。これにより、初期の脆弱性が修正されたとしても、攻撃者はシステムにアクセスし続けることができます。
3.  **防御メカニズムの無効化と痕跡のクリーンアップ:** 攻撃者は `/etc/hosts.deny` をクリアしようとし、既存の不正なスクリプトやプロセスを停止・削除することで、自身の活動を隠蔽し、システムの防御策を弱めようとしました。これは、攻撃者が巧妙かつ徹底的にシステムを制御しようとしていることを示唆しています。
4.  **広範な偵察活動:** システムの構成、リソース、稼働状況に関する詳細な情報収集は、攻撃者が次のより高度な攻撃段階へ移行するための準備段階である可能性が高く、将来的なリスクを増大させます。

これらの活動は、ハニーポット環境で検出されたものではありますが、実際のシステムで発生した場合には壊滅的な影響をもたらす可能性があります。

### 4. 推奨アクション

**A. 緊急対応（即時実施）**

1.  **攻撃元IPのブロック:** 攻撃元IPアドレス `191.255.52.36` からのすべての通信を、ファイアウォールまたはIPS/IDSで直ちにブロックします。
2.  **影響範囲の特定と隔離:**
    *   このログが本番環境で発生した場合、影響を受けた可能性のあるサーバーをネットワークから隔離し、全てのユーザーを強制ログアウトさせます。
    *   SSHサービスの停止または一時的なアクセス制限を実施します。
3.  **認証情報の変更と強化:**
    *   `root` ユーザーを含む、全てのシステムユーザーのパスワードを直ちに、かつ強力なものに変更します。二要素認証が可能な場合は、速やかに導入します。
    *   ログに記録されている攻撃者がログインに成功したパスワード (`Hello@123`, `3245gs5662d34`, `odoo123`, `Aa111111!`, `V7x7C26rMbb4`) は、二度と使用されないようにブラックリスト化します。
4.  **SSH公開鍵の確認と削除:**
    *   全てのユーザー（特に `root`）の `~/.ssh/authorized_keys` ファイルを徹底的に確認し、ログに記録された `ssh-rsa AAAAB3NzaC1yc2EAAAABJQAAAQEArDp4cun2lhr4KUhBGE7VvAcwdli2a8dbnrTOrbMz1+5O73fcBOx8NVbUT0bUanUV9tJ2/9p7+vD0EpZ3Tz/+0kX34uAx1RV/75GVOmNx+9EuWOnvNoaJe0QXxziIg9eLBHpgLMuakb5+BgTFB+rKJAw9u9FSTDengvS8hX1kNFS4Mjux0hJOK8rvcEmPecjdySYMb66nylAKGwCEE6WEQHmd1mUPgHwGQ0hWCwsQk13yCGPK5w6hYp5zYkFnvlC8hGmd4Ww+u97k6pfTGTUbJk14ujvcD9iUKQTTWYYjIIu5PmUux5bsZ0R4WFwdIe6+i6rBLAsPKgAySVKPRK+oRw== mdrfckr` を含む、不審な公開鍵を全て削除します。
5.  **システムファイルとディレクトリの整合性チェック:**
    *   `/etc/hosts.deny` ファイルの内容を確認し、元の適切な状態に復元します。
    *   `/tmp` ディレクトリ内の不審なファイル（`secure.sh`, `auth.sh` など）が存在しないか確認し、あれば削除します。
    *   変更された可能性のあるシステムファイル（例: cronジョブ `crontab -l`、設定ファイルなど）を特定し、整合性を確認します。
6.  **フォレンジック調査の実施:** システムのメモリダンプ、ディスクイメージを取得し、攻撃者が残した可能性のあるマルウェア、rootkit、バックドア、およびその他の永続化メカニズムについて詳細なフォレンジック調査を実施します。

**B. 恒久的な対策（再発防止）**

1.  **SSHのセキュリティ強化:**
    *   **パスワード認証の無効化:** SSHではパスワード認証を無効化し、公開鍵認証のみを許可します。
    *   **二要素認証(MFA)の導入:** SSHアクセスにMFAを必須とします。
    *   **ポートの変更:** 標準のSSHポート22番から変更します（セキュリティバイオブスキュリティ）。
    *   **アクセス元IP制限:** 必要なIPアドレス以外からのSSH接続をファイアウォールで厳格に制限します。
    *   **Fail2banなどの導入:** 不正なログイン試行を検出し、一時的にIPアドレスをブロックするツール（Fail2banなど）を導入します。
2.  **強力なパスワードポリシー:** 全てのシステムユーザーに対し、複雑で定期的に変更されるパスワードポリシーを強制します。
3.  **システムログの集中管理と監視:** ログをSIEMなどの集中ログ管理システムに転送し、異常なログイン試行、特権コマンドの実行、ファイル変更などをリアルタイムで監視する体制を強化します。
4.  **定期的な脆弱性診断とパッチ適用:** OSおよびアプリケーションの脆弱性診断を定期的に実施し、発見された脆弱性には迅速にパッチを適用します。
5.  **不審なアクティビティの検知:** プロセス監視、ネットワークトラフィック監視、ファイル整合性監視（Tripwireなど）を導入し、不正な活動を早期に検知できるメカニズムを構築します。
6.  **バックアップと復旧計画:** 定期的なシステムバックアップを確立し、インシデント発生時の迅速な復旧計画を策定・テストします。
7.  **従業員へのセキュリティ意識向上トレーニング:** フィッシング、ソーシャルエンジニアリング、パスワード管理の重要性などについて、従業員への定期的なセキュリティトレーニングを実施します。
